# 面试模拟脚本格式

## 触发条件

当用户明确要求"模拟面试"，或 `route=完整` 时在所有题目文件完成后附加一个综合模拟脚本。

## 格式

```
[面试模拟 - {知识域} - {级别} - 约20分钟]

---

Q1（热身，30秒）：
[概念题，考察基础认知]
→ 预期回答要点：...

Q2（展开，1-2分钟）：
[中等难度，考察理解深度]
→ 追问1：...
→ 追问2：...

Q3（深入，2-3分钟）：
[场景/源码题，考察实战]
→ 追问到具体选型理由
→ 追问底层实现细节

Q4（压力，2-3分钟）：
[开放题/极端场景]
→ 没有标准答案，考察思维过程
→ 面试官关注：是否能识别 trade-off

---

[面试官此时在想：...]
- 如果候选人答到X → 加分，可以问更深
- 如果候选人卡在Y → 给提示，观察反应
- 红旗信号：背诵感过强 / 无法解释 why / 举不出实例
```

## 注意事项

- 问题递进难度，不要一上来就压力测试
- 每个问题标注预期时间，帮助候选人练习节奏
- 追问方向要和实际面试风格一致（不追问冷僻知识）
- 最后给出整体评价维度：基础扎实度 / 深度理解 / 实战经验 / 表达清晰度
