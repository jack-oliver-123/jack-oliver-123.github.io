# 记忆锚点构造策略

## 构造规则

| 题型 | 策略 | 字数上限 | 示例 |
|------|------|----------|------|
| 概念对比 | A...B...对仗句 | ≤15字 | "sync隐式自动，Lock显式手动" |
| 流程/排查 | 动词链(→连接) | ≤30字 | "看→dump→查→改→验" |
| 原理机制 | 关键数字+动作 | ≤20字 | "16容量16扰动，8且64转树，0.75扩" |
| 设计模式 | 角色类比 | ≤15字 | "代理=门卫，装饰=套娃" |
| 场景决策 | 条件分支句(if...则...否则...) | ≤25字 | "数据少用map，数据多用DB，超大用ES" |

## 核心目标

每个语义组块能**展开**为一段回答（触发回答链路），不只是记住关键词。

生成锚点时同时输出 2-3 个**展开触发词**——即从锚点回忆起完整回答的关键中间节点。

## 质量检查

- 锚点中的数字必须可追溯到具体源码常量、配置或文档
- 对仗句两侧必须指向真实的技术差异，不能为对仗牺牲准确性
- 动词链中的每个动词应对应一个可操作的排查/实施步骤
- 条件分支句的条件必须是可判断的（有数量级或场景边界）

## 使用建议

1. 看锚点 → 尝试口述展开 → 对照原文检查遗漏
2. 每个锚点练习 2-3 次后应能在 30 秒内展开为完整回答
3. 将锚点写在卡片/便签上，通勤时快速过一遍

领域专用锚点示例见各 `references/domains/{domain}.md` 文件末尾。
