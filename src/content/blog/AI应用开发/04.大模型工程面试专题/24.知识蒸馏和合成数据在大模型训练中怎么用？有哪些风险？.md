假设公司有一个效果很好但昂贵的大模型。

每天有上千万条“判断工单属于物流、退款还是账户问题”的请求。

让大模型处理每一条当然可以，但成本和延迟都可能不合适。

一种常见思路是：

1. 让强模型为大量代表性输入给出答案或解释；
2. 筛掉错误、重复和不安全样本；
3. 用这些信号训练一个更小的模型；
4. 让小模型承担高频、边界明确的任务；
5. 困难样本仍回退到强模型或人工。

这里同时出现了两个概念：**知识蒸馏**和**合成数据**。

它们经常一起使用，但不是同一件事。

## 本篇目标

学完本篇，你应该能够：

1. 用通俗语言解释教师模型、学生模型和知识蒸馏；
2. 区分硬标签、软标签、Logit 蒸馏和序列蒸馏；
3. 逐个符号解释温度 Softmax 与蒸馏损失；
4. 解释什么是合成指令、合成回答和合成推理轨迹；
5. 设计从种子任务到过滤、训练和评测的数据流水线；
6. 区分蒸馏、微调、量化和压缩；
7. 识别错误放大、模式坍缩、数据污染、隐私和许可证风险；
8. 说明如何避免“模型生成数据，再训练模型”的退化闭环；
9. 在面试中给出可落地的成本、质量与安全取舍。

先记住一句话：

> 蒸馏是“学生如何从教师信号学习”的训练方法；合成数据是“训练样本如何生成”的数据方法。合成数据可以用于蒸馏，也可以用于普通预训练、SFT、偏好训练或评测。

## 60 秒回答

知识蒸馏是让学生模型学习教师模型的行为或分布。白盒场景可以拿到教师 Logit，用带温度的软概率传递类别之间的相似关系；黑盒大模型 API 通常拿不到完整 Logit，就会让教师生成答案、解释、偏好对或推理轨迹，再对学生做 SFT 或偏好训练，这常被称为序列级或响应蒸馏。

合成数据则是由模型、规则、模拟器或程序生成的训练样本。生产流程通常是：从真实需求和种子任务出发，扩展指令与难度，生成多个候选，用规则、执行器、检索证据或人工过滤，做去重和污染检查，再混入高质量真实数据训练。

两者的价值是降低标注成本、扩大长尾覆盖，并把强模型在特定任务上的能力迁移给更小、更便宜的模型。但学生受自身容量限制，教师错误、偏见和措辞习惯也会被复制。反复使用未经控制的模型生成数据，还可能降低数据多样性并放大错误。

工程上我不会把“教师说了”当真值。会记录教师与 Prompt 版本，保留来源和生成链路，对可验证任务使用编译器、测试、规则或外部证据过滤；真实数据、合成数据和评测集严格隔离；按任务切片比较训练前后质量、安全、校准和分布外表现，并设置困难请求回退策略。

## 详细解析

### 一、先用“师傅带徒弟”建立直觉

传统监督学习像这样：

```text
题目：“这条工单是什么类型？”
标准标签：“退款”
学生模型学习：输入 -> 退款
```

学生只知道最终标签，不知道其他选项有多接近。

教师模型可能认为：

```text
退款：0.72
账单：0.23
物流：0.03
账户：0.02
```

这个分布告诉学生：

- 正确答案是退款；
- 账单也有较高相关性；
- 物流和账户明显不相关。

相比单独一个“退款”标签，概率分布包含了类别关系。

Hinton 等人的经典知识蒸馏工作把这种信息称为软目标中的“暗知识”。

### 二、三个角色：教师、学生和训练数据

#### 1. 教师模型

教师通常能力更强，但不必在所有方面都更大。

教师可以是：

- 一个大型通用模型；
- 多个模型组成的集成；
- 经过领域训练的专家模型；
- 规则、搜索和模型组成的系统；
- 能执行代码或检索证据的 Agent。

重要的是教师信号在目标任务上经过验证，而不是只看参数量。

#### 2. 学生模型

学生通常更小、更快或更容易部署。

目标可能是：

- 减少参数和显存；
- 降低延迟与成本；
- 在端侧运行；
- 专门处理一个垂直任务；
- 获得教师的输出风格或推理模式。

#### 3. 训练数据

蒸馏仍然需要输入样本。

如果输入只覆盖简单题，学生很难学会困难和长尾情况。

因此，蒸馏质量取决于三件事：

```text
教师质量 × 输入覆盖 × 过滤与训练方法
```

其中任意一项很差，结果都可能受限。

### 三、硬标签与软标签

#### 1. 硬标签

硬标签只给一个标准类别。

对 $K$ 个类别，标签 $y$ 可以表示为 one-hot 向量：

```text
退款  [1, 0, 0, 0]
```

学生通常最小化交叉熵：

$$
L_{hard}=-\sum_{i=1}^{K}y_i\log p_i
$$

其中：

- $L_{hard}$：硬标签损失；
- $K$：类别数量；
- $y_i$：第 $i$ 类的真实标签，正确类为 1，其余为 0；
- $p_i$：学生预测第 $i$ 类的概率；
- $\log$：自然对数。

因为只有正确类的 $y_i=1$，损失主要推动正确类概率变大。

#### 2. 软标签

软标签保留教师对所有类别的概率。

学生不仅学“哪个对”，还学“其他类别有多像”。

这对类别相似、真实标签稀疏或教师集成的场景可能有帮助。

### 四、温度 Softmax 是什么

模型最后通常先产生 Logit。

Logit 是未归一化分数，可以为任意实数。

带温度的 Softmax 为：

$$
q_i^{(T)}=\frac{\exp(z_i/T)}{\sum_{j=1}^{K}\exp(z_j/T)}
$$

其中：

- $q_i^{(T)}$：温度为 $T$ 时，第 $i$ 类的概率；
- $z_i$：模型对第 $i$ 类给出的 Logit；
- $T$：温度，必须大于 0；
- $K$：类别数；
- $\exp$：指数函数；
- 分母：让所有类别概率之和变成 1。

当 $T=1$ 时，就是普通 Softmax。

当 $T>1$ 时，分布通常更平缓，低概率类别的信息更明显。

例子：

```text
Logit：[8, 4, 1]
```

在较低温度下，第一个类别会占绝大多数概率。

提高温度后，第二和第三类不再接近零，学生能看到教师认为它们的相对关系。

注意，这里的蒸馏温度是训练数学中的温度；不要把它和调用生成 API 时用于调节随机性的 Temperature 混为一谈，虽然二者都作用于概率分布。

### 五、经典蒸馏损失

常见目标把硬标签损失和软目标损失组合：

$$
L=\alpha L_{hard}+(1-\alpha)T^2D_{KL}(q_T^{(T)}\|q_S^{(T)})
$$

逐个符号解释：

- $L$：学生最终优化的总损失；
- $L_{hard}$：学生对真实硬标签的交叉熵；
- $\alpha$：硬标签损失权重，通常在 0 到 1 之间；
- $1-\alpha$：蒸馏损失权重；
- $T$：Softmax 温度；
- $T^2$：常用于补偿高温导致的梯度尺度变化；
- $q_T^{(T)}$：教师在温度 $T$ 下的概率分布；
- $q_S^{(T)}$：学生在同一温度下的概率分布；
- $D_{KL}$：KL 散度，用来衡量两个概率分布的差异。

KL 散度可以写成：

$$
D_{KL}(P\|Q)=\sum_iP_i\log\frac{P_i}{Q_i}
$$

其中：

- $P$：目标分布，这里通常是教师；
- $Q$：要靠近目标的分布，这里通常是学生；
- $P_i$、$Q_i$：两个分布在第 $i$ 类上的概率。

KL 散度不是对称距离：

$$
D_{KL}(P\|Q)\neq D_{KL}(Q\|P)
$$

训练时的方向、温度和权重都应通过验证集确定。

### 六、白盒蒸馏与黑盒蒸馏

#### 1. 白盒蒸馏

白盒表示训练者能访问教师内部信号，例如：

- 完整 Logit；
- 隐藏层表示；
- 注意力分布；
- 中间特征。

可以做：

- 输出分布蒸馏；
- 特征对齐；
- 层间映射；
- 注意力蒸馏。

难点是教师和学生的层数、宽度或 Tokenizer 可能不同，需要设计投影和对齐方式。

#### 2. 黑盒蒸馏

商业 API 通常只返回文本或有限概率，无法获得完整内部状态。

这时常用教师生成：

- 最终回答；
- 解释或推理轨迹；
- 指令—回答对；
- 偏好对；
- 批评与修订；
- 工具调用轨迹。

再用这些序列训练学生。

这类方法常被称为序列级蒸馏、响应蒸馏或数据蒸馏。

它与合成数据高度重叠。

### 七、序列蒸馏最小例子

目标是训练一个小模型识别退款原因。

种子输入：

```text
“同一笔订单昨天扣了两次款。”
```

教师输出：

```json
{
  "category": "duplicate_charge",
  "evidence": "用户明确说同一订单被重复扣款",
  "next_action": "核对支付流水后进入重复扣款流程"
}
```

过滤器检查：

- JSON 是否符合 Schema；
- 类别是否在允许枚举中；
- 证据是否来自输入；
- 动作是否符合业务规则；
- 是否包含个人敏感数据。

通过后，样本进入学生 SFT 数据。

学生学到的是教师输出序列，而不是教师完整概率分布。

### 八、合成数据到底是什么

只要数据主要由程序、规则、模拟器或生成模型创建，而不是直接由真实世界采集和人工标注，就可以称为合成数据。

常见类型包括：

| 类型 | 例子 | 主要用途 |
| --- | --- | --- |
| 合成指令 | 自动扩写任务描述 | 扩大任务覆盖 |
| 合成回答 | 教师为输入生成答案 | SFT 或蒸馏 |
| 合成推理轨迹 | 生成分步解释 | 推理训练 |
| 合成偏好对 | 好答案与差答案 | 偏好训练 |
| 合成对抗样本 | 注入、越权、歧义请求 | 安全训练与评测 |
| 模拟交互轨迹 | 用户—Agent—工具多轮 | Agent 训练 |
| 数据增强 | 改写、翻译、扰动 | 提高鲁棒性 |

“合成”不等于“低质量”。

程序生成的数学题和可执行代码，可能有很可靠的验证器。

反过来，语言流畅的教师回答也可能事实错误。

关键是生成机制和验证机制。

### 九、Self-Instruct 式数据扩展

Self-Instruct 的基本思想是从少量种子任务出发，让模型生成更多指令和答案。

简化流程：

```text
人工种子任务
  -> 生成新指令
  -> 判断是否与已有任务过于相似
  -> 为新指令生成输入和回答
  -> 格式与质量过滤
  -> 加入训练集
```

这降低了纯人工编写指令的成本。

但如果没有领域覆盖控制，模型会偏向生成它熟悉、容易回答、措辞相似的任务。

所以生产系统通常需要明确的任务分类树和目标配额。

### 十、合成推理轨迹为什么可能有用

最终答案只告诉学生“结果是什么”。

解释轨迹还可能展示：

- 如何分解问题；
- 哪些条件重要；
- 如何使用规则；
- 如何检查答案；
- 何时调用工具。

Distilling Step-by-Step 和 Orca 等工作都研究了利用解释信号训练较小模型。

但推理文本存在三类边界：

1. 教师可能答案对、解释错；
2. 教师可能给出听起来合理但并非真实因果过程的解释；
3. 学生可能模仿冗长风格，而不是获得可泛化能力。

因此，应对“最终结果”和“中间步骤”分别验证。

### 十一、一条生产级合成数据流水线

#### 第 1 步：定义能力地图

先列出要覆盖的：

- 任务类型；
- 语言；
- 难度；
- 输入长度；
- 业务实体；
- 风险等级；
- 正常与拒绝场景；
- 工具成功与失败状态。

没有能力地图，数据量再大也可能只是在重复简单模板。

#### 第 2 步：准备种子输入

种子可以来自：

- 脱敏真实请求；
- 领域专家设计；
- 程序模板；
- 文档中的规则与示例；
- 线上失败样本；
- 公开许可数据。

每条种子记录来源和使用权限。

#### 第 3 步：生成候选

可让教师：

- 改写同一意图；
- 增加约束或噪声；
- 生成不同难度；
- 给出多个独立答案；
- 先回答，再批评，再修订；
- 使用工具或检索证据。

不要只用一个固定 Prompt 和一次贪心输出，否则多样性有限。

#### 第 4 步：确定性过滤

优先使用低歧义检查：

- JSON Schema；
- 编译器和单元测试；
- 数学求解器；
- 数据库只读执行；
- 枚举与范围；
- 业务规则；
- 证据引用匹配；
- 敏感信息扫描。

确定性通过只说明满足已检查规则，不代表所有语义都正确。

#### 第 5 步：模型或人工审核

对开放任务，可以使用：

- 第二个模型独立判断；
- 多候选一致性；
- 细化 Rubric 的 Judge；
- 领域专家抽检；
- 高风险样本全量人工审核。

教师和 Judge 最好不要共享完全相同的偏差来源。

#### 第 6 步：去重与多样性控制

需要处理：

- 完全重复；
- 只替换人名数字的模板重复；
- 语义近重复；
- 某一种文风占比过大；
- 某个任务或语言过度代表。

去重不仅节省训练，还能减少模型记忆模板和评测泄漏。

#### 第 7 步：污染检查

检查合成数据是否包含：

- 评测题原文；
- 隐藏测试集改写；
- 生产敏感信息；
- 不允许再分发的内容；
- 教师回答中意外复制的长文本。

训练集、开发集和测试集应按语义近重复检测，而不只是字符串匹配。

#### 第 8 步：混合与加权

合成数据通常不应无条件覆盖真实高质量数据。

可以按来源设置采样权重：

$$
P(d)=\frac{w_d n_d}{\sum_j w_j n_j}
$$

其中：

- $P(d)$：从数据源 $d$ 采样的概率；
- $w_d$：该数据源的质量或业务权重；
- $n_d$：数据源样本量；
- 分母：所有来源的加权样本总量。

权重需要通过下游验证，而不是把来源数量直接当质量。

#### 第 9 步：训练后分层评测

至少比较：

- 蒸馏前学生；
- 蒸馏后学生；
- 教师；
- 当前生产基线。

并按任务、难度、语言、风险和分布外样本切片。

#### 第 10 步：保留回退

学生不需要复制教师的全部能力。

可采用级联系统：

```text
学生处理高频清晰请求
  -> 低置信/验证失败
  -> 教师或人工处理
```

这常比强迫学生覆盖所有边界更务实。

### 十二、蒸馏和普通 SFT 有什么关系

SFT 描述的是训练方式：用输入—目标输出做监督学习。

蒸馏描述的是监督信号来源和目标：让学生模仿教师。

如果目标输出由教师生成，再对学生做 SFT，那么它同时是：

- SFT；
- 序列蒸馏；
- 使用合成数据训练。

如果目标输出由人工标注，则是 SFT，但不一定是蒸馏。

### 十三、蒸馏、量化和剪枝不要混淆

| 方法 | 核心动作 | 是否重新训练 | 主要目标 |
| --- | --- | --- | --- |
| 蒸馏 | 学生模仿教师 | 通常需要 | 把能力迁移到另一模型 |
| 量化 | 降低数值精度 | PTQ 可少量校准，QAT 需训练 | 降显存和计算成本 |
| 剪枝 | 删除权重、通道或结构 | 常需恢复训练 | 减少参数和运算 |
| SFT | 学习输入—输出样本 | 需要 | 对齐任务和格式 |

这些方法可以组合。

例如先蒸馏出小模型，再做 INT4 量化。

组合后的质量损失必须重新评测，不能把各阶段结果简单相加推断。

### 十四、主要风险一：教师错误被复制和放大

教师可能：

- 幻觉事实；
- 误解业务规则；
- 对少数语言表现差；
- 产生过度拒答；
- 给出危险代码；
- 用错误推理碰巧得到正确答案。

如果同一种错误被生成数万次，学生会把它学成稳定模式。

因此，合成规模放大前应先验证错误率和错误相关性。

### 十五、主要风险二：多样性下降与递归训练退化

如果一代模型生成数据训练下一代，再让下一代继续生成下一批数据，分布可能逐渐变窄。

低概率但真实存在的模式会越来越少，高频模式被反复放大。

“The Curse of Recursion”研究了只依赖生成数据可能造成的模型退化现象。

工程缓解方法包括：

- 持续保留真实数据锚点；
- 记录每条数据的代际和来源；
- 限制合成数据占比；
- 对稀有切片设置配额；
- 比较词汇、语义和任务多样性；
- 不让学生未经审查地成为下一轮唯一教师。

### 十六、主要风险三：评测污染

教师可能已经见过公开基准。

让教师生成“类似题目”时，也可能复现题干或答案模式。

如果训练数据包含测试题，评测分数会虚高。

防护包括：

- 测试集严格隔离；
- 生成服务不接触隐藏答案；
- 字符串与语义去重；
- 为公开基准保留污染声明；
- 使用新鲜、私有、可审计的业务集。

### 十七、主要风险四：隐私、版权和合同

合成数据不是天然没有权利风险。

教师输出可能：

- 复现训练语料中的长片段；
- 泄露 Prompt 中的个人信息；
- 包含客户机密；
- 受服务条款限制用于训练其他模型；
- 混入来源不明的代码许可证。

因此要：

- 最小化发送给教师的数据；
- 脱敏和访问控制；
- 记录数据血缘；
- 审核模型服务条款和数据许可证；
- 对长匹配、秘密和个人信息做扫描；
- 设置保留与删除流程。

具体法律判断应由专业人员结合地区和合同完成。

### 十八、主要风险五：学生容量不足

教师可能能做复杂规划，学生只有很小容量。

学生无法无损复制教师所有行为。

典型表现是：

- 常见题接近教师；
- 长尾题明显下降；
- 长上下文能力退化；
- 多步骤工具调用不稳定；
- 校准变差；
- 解释看起来像教师，但结论不可靠。

蒸馏目标应限定在学生可承载、业务真正需要的能力范围。

## 工程实践与边界

### 一、数据血缘必须可追踪

每条样本至少记录：

```json
{
  "sample_id": "refund_00421",
  "source_type": "synthetic_from_seed",
  "seed_id": "seed_0038",
  "teacher_model": "fixed-version-id",
  "prompt_version": "gen-v7",
  "generated_at": "2026-07-15",
  "filters": ["schema-v3", "policy-v5"],
  "review_status": "expert-approved"
}
```

没有血缘，就无法在发现教师错误后定位并删除受影响样本。

### 二、生成和过滤应解耦

生成器负责覆盖和多样性。

过滤器负责正确性、安全和可训练性。

不要让同一个 Prompt 同时扮演出题、答题和最终裁判，而没有独立检查。

### 三、按风险决定审核强度

低风险文本分类可以更多依赖自动验证和抽检。

医疗建议、金融动作或高权限工具轨迹应采用更严格的专家审核和执行限制。

### 四、比较教师上限和学生收益

如果教师在某切片只有 75% 正确，学生很难仅靠模仿在同分布上稳定超过可靠真值上限。

先改善教师、验证器或真实标签，可能比继续扩大合成量更有效。

### 五、用学习曲线判断是否还值得加数据

分别用 1 万、5 万、10 万、50 万条高质量样本训练或续训，观察验证集收益。

如果数据量增长十倍，关键指标几乎不变，应检查：

- 样本是否重复；
- 任务是否偏简单；
- 学生容量是否成为瓶颈；
- 训练超参数是否合适；
- 评测是否能感知真实改进。

### 六、不要用隐藏测试集驱动生成

如果把失败的隐藏题直接交给教师扩写，再继续在同一个隐藏集上报告成绩，隐藏集已经变成训练信号。

应把历史失败移入回归集，并建立新的隔离发布集。

### 七、上线后继续校验

学生模型上线后监控：

- 任务成功率；
- 低置信和回退率；
- 分布外输入；
- 安全违规；
- 不同语言和客户切片；
- 教师复核差异；
- 延迟与成本。

可以对少量流量进行教师影子复核，但要控制费用、数据权限和用户延迟。

## 常见误区

### 误区一：蒸馏就是把模型文件压缩

蒸馏需要学生通过训练学习教师信号，不是普通压缩算法。

### 误区二：合成数据就是蒸馏

合成数据是一类数据来源；蒸馏是一类学习目标。

两者可以组合，也可以分开。

### 误区三：教师参数大，标签就是真值

教师会犯错，也可能在目标领域弱于小型专家系统。

### 误区四：数据越多，学生越好

低质量、重复和偏斜数据可能稀释高质量信号。

### 误区五：推理轨迹越长越有价值

冗长不代表正确。

错误解释还可能教会学生稳定犯错。

### 误区六：合成数据没有隐私和版权风险

输入、输出、服务条款和数据来源都需要审查。

### 误区七：学生可以无损复制教师全部能力

模型容量、架构、Tokenizer、上下文和训练覆盖都会形成边界。

### 误区八：只看学生在训练域的平均分

还要测长尾、分布外、安全、校准和不同人群切片。

### 误区九：用教师做 Judge 就形成独立验证

同一模型可能复制自己的偏差，评分也可能偏爱自己的风格。

### 误区十：蒸馏后不再需要强模型

复杂、高风险和低置信请求通常仍需要可靠回退或人工处理。

## 面试追问

### 追问一：软标签为什么可能比 one-hot 标签信息更多？

它保留教师对非正确类别的相对判断，可以表达类别相似性和不确定性。

### 追问二：温度越高越好吗？

不是。

温度太低时分布过尖，暗知识少；太高时分布过平，差异信号也会变弱。应通过验证集选择。

### 追问三：API 拿不到 Logit，怎么蒸馏？

让教师生成回答、解释、偏好对或工具轨迹，再把经过验证的序列作为监督信号训练学生。

### 追问四：如何验证合成代码数据？

在隔离沙箱中进行解析、编译、静态检查和单元测试；控制网络、文件、时间和资源权限，并检查测试是否真正覆盖需求。

### 延伸问题与回答要点

| 问题 | 回答要点 |
| --- | --- |
| 学生能超过教师吗 | 在特定指标上可能因真实标签、数据组合或归纳偏置超过单次教师输出，但不能把它当普遍保证 |
| 如何选教师 | 看目标任务质量、稳定性、条款、成本、数据权限和输出可验证性，而不只看规模 |
| 合成数据占比多少 | 没有通用比例，应通过来源消融、学习曲线和风险切片决定 |
| 如何防模型坍缩 | 保留真实数据锚点、来源追踪、代际限制、多样性监控和独立评测 |
| 蒸馏后还要量化吗 | 可以组合，但量化会引入新的质量和硬件边界，需要重新测量 |
| 为什么要生成多个候选 | 增加覆盖并让验证器选择，但候选相关性和验证成本会限制收益 |
