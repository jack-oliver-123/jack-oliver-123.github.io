## 题目入口

题目链接：[移除元素](https://leetcode.cn/problems/remove-element/description/)

![image-20260707213641943](https://gitee.com/mu-mo_1_0/images/raw/master/image-20260707213641943.png)

## 一句话读懂题目

给你一个数组 `nums` 和一个目标值 `val`，你要把数组里所有等于 `val` 的元素移除。

最后返回剩下元素的个数 `k`。

比如：

```text
nums = [3, 2, 2, 3]
val = 3
```

数组里要移除的是 `3`，剩下的是：

```text
[2, 2]
```

所以返回：

```text
2
```

注意，题目不是让我们真的把数组长度变短。

因为 Java 的普通数组长度是固定的，不能像 `ArrayList` 那样直接删除元素。

LeetCode 主要检查两件事：

```text
1. 返回的 k 是否等于原数组中不等于 val 的元素个数
2. nums 的前 k 个元素是否正好是这些需要保留的元素
```

至于 `k` 后面的元素是什么，不重要。

题目还说明，最终元素的顺序可以改变。也就是说，前 `k` 个元素只要内容正确即可，不要求和原数组保持完全一样的相对顺序。

## 先用生活例子理解

可以把这道题想象成：你在整理书架。

书架上有一排书，其中有些书是你不想要的，比如所有标着 `val` 的书。

你的目标不是把书架锯短，而是：

> 把想保留的书都挪到书架前面。

后面的空位、旧书、乱序内容，都不用管。

放到数组里，就是：

> 把所有不等于 `val` 的元素放到数组前面。

所以这道题的核心不是“删除”，而是“保留”。

不要盯着要删掉的元素看，而要盯着要留下来的元素看。

## 题目里的“原地修改”是什么意思

题目要求使用 `O(1)` 额外空间。

`O(1)` 额外空间可以简单理解成：不管数组有多长，我们额外使用的变量数量都差不多，比如只用几个指针变量。

所以不能新建一个额外数组来保存答案：

```java
int[] result = new int[nums.length];
```

正确做法是在原数组 `nums` 前面整理出答案。

比如：

```text
nums = [3, 2, 2, 3]
val = 3
```

处理后可以变成：

```text
nums = [2, 2, 2, 3]
k = 2
```

这里后面的 `[2, 3]` 不重要。

因为返回 `k = 2` 后，LeetCode 只检查前两个元素：

```text
[2, 2]
```

## 方法一：快慢指针，保留不是 val 的元素

这道题最稳、最好理解的写法是快慢指针。

准备两个指针：

```text
slow：指向下一个可以放入答案的位置
fast：负责从前往后扫描数组
```

可以把它理解成：

- `fast` 负责检查每个元素；
- `slow` 负责维护数组前面的答案区。

核心判断只有一句：

```text
如果 nums[fast] != val，说明这个元素要保留
```

既然要保留，就把它放到 `nums[slow]`，然后让 `slow` 往后移动一格。

核心逻辑：

```text
如果 nums[fast] != val
    nums[slow] = nums[fast]
    slow++
```

最后 `slow` 就是保留下来的元素个数，也就是答案 `k`。

## 手动走一遍例子

以这个数组为例：

```text
nums = [3, 2, 2, 3]
val = 3
```

刚开始：

```text
slow = 0
fast = 0
答案区：[]
```

### 第一步：fast 看到 3

当前元素是：

```text
nums[fast] = 3
```

它等于 `val`，说明要移除。

所以不放进答案区，直接跳过。

```text
slow 还是 0
答案区：[]
```

### 第二步：fast 看到 2

当前元素是：

```text
nums[fast] = 2
```

它不等于 `val`，说明要保留。

把它放到 `nums[slow]`：

```text
nums[0] = 2
```

然后 `slow++`：

```text
slow = 1
```

此时答案区是：

```text
[2]
```

### 第三步：fast 看到第二个 2

当前元素还是 `2`，不等于 `val`，继续保留。

放到 `nums[slow]`：

```text
nums[1] = 2
```

然后 `slow++`：

```text
slow = 2
```

此时答案区是：

```text
[2, 2]
```

### 第四步：fast 看到最后一个 3

当前元素是 `3`，等于 `val`，跳过。

最后返回：

```text
slow = 2
```

前 `2` 个元素就是答案：

```text
[2, 2]
```

如果把过程压缩成一张表：

| fast 位置 | nums[fast] | 是否等于 val | 操作 | 操作后 slow | 数组前半部分 |
| --- | --- | --- | --- | --- | --- |
| 初始 | - | - | - | `0` | `[]` |
| `0` | `3` | 是 | 跳过 | `0` | `[]` |
| `1` | `2` | 否 | 放入 `nums[0]` | `1` | `[2]` |
| `2` | `2` | 否 | 放入 `nums[1]` | `2` | `[2, 2]` |
| `3` | `3` | 是 | 跳过 | `2` | `[2, 2]` |

这张表里最重要的是最后一列：

> `nums[0...slow-1]` 始终保存着已经处理过的、所有不等于 `val` 的元素。

## Java 代码

先记住一句话：

> 快指针负责筛选，慢指针负责收集。

```java
class Solution {
    public int removeElement(int[] nums, int val) {
        int slow = 0;

        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != val) {
                nums[slow] = nums[fast];
                slow++;
            }
        }

        return slow;
    }
}
```

## 代码逐行拆解

```java
int slow = 0;
```

`slow` 表示下一个可以放入答案的位置。

也可以理解成：当前已经保留下来的元素个数。

```java
for (int fast = 0; fast < nums.length; fast++) {
```

`fast` 从头到尾扫描整个数组。

每个元素都要看一遍，判断它是不是 `val`。

```java
if (nums[fast] != val) {
```

如果当前元素不等于 `val`，说明它应该被保留。

```java
nums[slow] = nums[fast];
```

把当前元素放到答案区的下一个位置。

```java
slow++;
```

答案区多了一个元素，所以 `slow` 往后移动一格。

```java
return slow;
```

最后 `slow` 就是保留下来的元素数量。

注意这里和上一题《删除有序数组中的重复项》不一样：

- 上一题的 `slow` 是答案区最后一个元素的下标，所以返回 `slow + 1`；
- 这一题的 `slow` 是下一个可写位置，也等于答案长度，所以直接返回 `slow`。

## 为什么这题不要求数组有序

上一题《删除有序数组中的重复项》必须依赖数组有序。

因为它要判断“当前数字是不是重复数字”。

但这道题不一样。

这道题只关心一个条件：

```text
当前元素是否等于 val
```

每个元素只要和 `val` 比较就够了，不需要知道它前面或后面有没有重复。

所以数组有序、无序都可以用这套快慢指针。

比如：

```text
nums = [0, 1, 2, 2, 3, 0, 4, 2]
val = 2
```

保留下来的元素是：

```text
[0, 1, 3, 0, 4]
```

返回：

```text
5
```

## 循环不变量

如果想在面试里讲得更稳，可以补一句循环不变量。

这道题的循环不变量是：

> 在每次循环开始前，`nums[0...slow-1]` 始终保存着已经扫描过的元素里，所有不等于 `val` 的元素。

当 `nums[fast] == val` 时，当前元素应该被移除，跳过它不会影响答案区。

当 `nums[fast] != val` 时，当前元素应该被保留，把它写入 `nums[slow]` 后，答案区安全地扩大一格。

这也解释了为什么覆盖是安全的：

> 我们只关心前 `slow` 个有效元素，后面的旧值不属于最终答案。

## 方法二：左右指针，不保留原顺序（选读）

题目说明：元素的顺序可以改变。

所以还有一种写法：遇到 `val` 时，把它和数组末尾的元素交换，然后缩小数组范围。

这种方法的特点是：遇到要删除的 `val` 时，直接用右侧元素覆盖它，不需要把后面所有要保留的元素都依次往前搬。

当 `val` 较多，或者你不需要保留原数组中非 `val` 元素的相对顺序时，可以把它作为进阶写法了解。

思路是：

```text
left 从前往后找 val
right 表示当前有效范围的最后一个位置
如果 nums[left] == val
    用 nums[right] 覆盖 nums[left]
    right--
否则
    left++
```

注意：当 `nums[left] == val` 时，覆盖后不能立刻 `left++`。

因为从右边换过来的新元素还没有检查过。

Java 代码如下：

```java
class Solution {
    public int removeElement(int[] nums, int val) {
        int left = 0;
        int right = nums.length;

        while (left < right) {
            if (nums[left] == val) {
                nums[left] = nums[right - 1];
                right--;
            } else {
                left++;
            }
        }

        return left;
    }
}
```

这份代码返回的也是保留下来的元素个数。

这里的 `right` 表示右边界的后一位，所以有效区间是：

```text
[left, right)
```

因此访问最后一个有效元素时，要写：

```java
nums[right - 1]
```

如果你更习惯让 `right` 表示最后一个有效下标，也可以写成下面这样：

```java
class Solution {
    public int removeElement(int[] nums, int val) {
        int left = 0;
        int right = nums.length - 1;

        while (left <= right) {
            if (nums[left] == val) {
                nums[left] = nums[right];
                right--;
            } else {
                left++;
            }
        }

        return left;
    }
}
```

这两种写法本质一样，只是 `right` 的含义不同：

```text
right = nums.length      -> 有效区间是 [left, right)，用 left < right，访问 nums[right - 1]
right = nums.length - 1  -> 有效区间是 [left, right]，用 left <= right，访问 nums[right]
```

千万不要只改 `right` 的初始值，却不改循环条件和访问位置。

不过它可能会改变元素的相对顺序。

比如：

```text
nums = [3, 2, 2, 3]
val = 3
```

处理后前两个元素可能是：

```text
[2, 2]
```

这没问题。

因为题目只要求前 `k` 个元素包含所有需要保留的元素，不要求顺序和原来一致。

## 两种方法怎么选

如果你是第一次做这道题，推荐优先掌握方法一：快慢指针。

原因很简单：

- 更直观；
- 不容易写错；
- 保留原数组中非 `val` 元素的相对顺序；
- 和很多“原地保留元素”的题目是同一个模板。

方法二适合作为进阶补充。

面试时如果没有特别要求优化写入次数，讲方法一就已经足够。

## 复杂度分析

### 方法一：快慢指针

时间复杂度：`O(n)`。

因为 `fast` 从头到尾扫描了一次数组。

空间复杂度：`O(1)`。

因为只用了 `slow` 和 `fast` 两个变量。

### 方法二：左右指针

时间复杂度：`O(n)`。

因为 `left` 和 `right` 总体都只会朝一个方向移动，最多移动 `n` 次左右。

空间复杂度：`O(1)`。

因为只用了 `left` 和 `right` 两个变量。

## 常见坑

### 1. 返回的是长度，不是数组

题目让你返回的是 `k`，不是返回一个新数组。

LeetCode 会根据你返回的 `k` 检查 `nums` 的前 `k` 个元素。

### 2. `k` 后面的元素不用管

比如：

```text
nums = [2, 2, 2, 3]
k = 2
```

只要前两个元素是 `[2, 2]`，后面是什么都不重要。

### 3. 不要真的删除数组元素

Java 普通数组长度固定，不能从中间删除一个元素后让数组自动变短。

这道题的思路是覆盖：

> 把要保留的元素覆盖到数组前面。

### 4. 方法二里覆盖后不要立刻 left++

在左右指针写法中，如果 `nums[left] == val`，我们会用右边元素覆盖当前位置。

但换过来的这个元素还没检查过。

所以这时只能 `right--`，不能马上 `left++`。

### 5. 注意这一题返回的是 slow，不是 slow + 1

方法一里 `slow` 表示下一个可写位置，也等于答案长度。

所以最后返回：

```java
return slow;
```

不要写成：

```java
return slow + 1;
```

## 面试时怎么讲

### 30 秒版

可以这样表达：

> 我会用快慢指针原地处理数组。`fast` 从前往后扫描每个元素，`slow` 表示下一个可以写入答案的位置。如果 `nums[fast] != val`，说明当前元素要保留，就把它写到 `nums[slow]`，然后 `slow++`。遍历结束后，`nums` 的前 `slow` 个元素就是所有不等于 `val` 的元素，返回 `slow`。时间复杂度是 `O(n)`，空间复杂度是 `O(1)`。

### 面试推荐回答结构

如果面试官让你现场讲，可以按这个顺序说：

```text
1. 先确认题意：原地移除 val，返回保留元素个数 k
2. 定义指针：fast 扫描，slow 指向下一个可写位置
3. 说明规则：遇到 nums[fast] != val 就写入 nums[slow]
4. 说明不变量：nums[0...slow-1] 始终是已经保留的元素
5. 给出复杂度：时间 O(n)，空间 O(1)
6. 补边界：全是 val 返回 0，没有 val 返回 nums.length
```

这样讲比直接背代码更稳。

### 面试官可能追问

**为什么返回 `slow`，不是 `slow + 1`？**

因为这道题里 `slow` 表示下一个可写位置，也等于已经保留下来的元素个数。

比如保留了两个元素后，`slow` 会指向下标 `2`，所以答案长度就是 `2`。

**这道题需要数组有序吗？**

不需要。

因为我们只判断每个元素是否等于 `val`，不需要判断重复关系。

**会不会覆盖还没扫描过的元素？**

方法一不会。

因为 `slow` 永远不会超过 `fast`，写入的位置一定是已经扫描过或当前正在扫描的位置。

**如果所有元素都等于 val 怎么办？**

比如：

```text
nums = [3, 3, 3]
val = 3
```

所有元素都会被跳过，`slow` 一直是 `0`，最后返回 `0`。

**如果没有元素等于 val 怎么办？**

比如：

```text
nums = [1, 2, 3]
val = 4
```

每个元素都会被保留，`slow` 最后变成 `3`，返回 `3`。

**什么时候考虑左右指针写法？**

当题目明确说顺序可以改变，并且你希望遇到 `val` 时直接用末尾元素覆盖，而不是维护稳定顺序时，可以考虑左右指针写法。

但面试中默认讲快慢指针更稳，因为它更清晰，也保留了原来的相对顺序。

**两种写法最大的区别是什么？**

快慢指针会保留非 `val` 元素的相对顺序。

左右指针不保证相对顺序，但可以把需要移除的元素快速换到有效区间外。

## 最后总结

移除元素这道题，表面上是在“删除指定元素”，实际上考的是：

> 如何在原数组前面保留所有符合条件的元素。

本题的保留条件是：

```java
nums[fast] != val
```

快慢指针模板可以这样记：

```text
slow 指向下一个可写位置
fast 从前往后扫描
如果当前元素应该保留
    nums[slow] = nums[fast]
    slow++
最后返回 slow
```

刷题时可以用下面几个边界用例自检：

```text
nums = [], val = 1 -> 返回 0
nums = [3, 3, 3], val = 3 -> 返回 0
nums = [1, 2, 3], val = 4 -> 返回 3
nums = [3, 2, 2, 3], val = 3 -> 返回 2
nums = [0, 1, 2, 2, 3, 0, 4, 2], val = 2 -> 返回 5
```

以后看到这类题目，如果它要求“原地修改数组”，并且本质是在“筛选要保留的元素”，就可以优先想到快慢指针。

最后把这句话记住：

> 不要想着怎么删掉 `val`，要想着怎么把不是 `val` 的元素收集到前面。
