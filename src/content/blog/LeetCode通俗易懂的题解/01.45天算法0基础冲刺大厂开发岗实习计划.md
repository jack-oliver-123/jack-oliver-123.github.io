## 60 秒回答

现在是原计划的第 7 天。前几天已经接触并 AC 了约 16 道数组题，这些投入不需要清零；但从现有记录看，数据结构主课、模板实现、延迟复写和复盘证据还没有真正建立起来，第 6 天也出现了中断。

因此，接下来不把日期重置为第 1 天，而是这样处理：

- 第 8—14 天：追补前 7 天缺失的数据结构基本功；
- 第 15—28 天：建立高频算法模式，同时按课件顺序完成微服务 OJ 主链；
- 第 29—39 天：进入动态规划和综合训练，同时按课件顺序完成 Spring AI Alibaba 项目主链；
- 第 40—45 天：算法模拟、双项目答辩、八股追问和材料收口。

这份计划不再用 AC 数量制造进度感。**AC 只能证明接触过，24—48 小时后闭卷复写、讲清复杂度和边界，才能证明掌握。**

## 详细解析

### 45 天结束时要拿到什么

### 算法结果

- 唯一新题累计 60—75 道，包含前 7 天已经做过的题；
- 完成 80 次以上复写，同一道题可以有第一次和第二次复写；
- 35—45 道达到 A 档，能够延迟后闭卷写出；
- 20 道高频题能在 15 分钟内讲清思路、复杂度和易错点；
- 8—10 个核心模板能够闭卷手写；
- 完成 4 次阶段周测和 3 次综合模拟。

### 项目结果

微服务 OJ 项目必须形成：

- 一条“登录鉴权 → 题目浏览 → 提交 → 判题 → 状态查询或通知”的可演示主链；
- 一张系统架构图、一张关键请求时序图；
- 一份接口、压测或部署证据；
- 10 个能够回答的架构追问。

Spring AI Alibaba 一键应用生成项目必须形成：

- 一条“需求输入 → 代码生成 → 构建预览 → 错误修复 → 截图 → 代码提交”的可演示主链；
- 五个核心 Agent 的职责表和一张 Graph 状态图；
- RAG 或 MCP 至少一个有真实演示证据，另一个能讲清设计取舍；
- 10 个关于安全、可靠性、状态管理和失败恢复的追问答案。

### 八股结果

- 沉淀 80—100 个去重问题；
- 至少 60 个问题能在 60—90 秒内结构化回答；
- 完成 6 次项目追问或模拟面试；
- 最终保留不超过 20 个薄弱问题继续训练。

### 练习与验收：LeetCode

每天的题目分为：

- **锚点题**：当天必做，用于掌握最核心模式；
- **进阶题**：周末或提前完成时再做；
- **到期复写题**：优先级高于任何新题。

如果本地课后作业和当天锚点题属于同一题型，可以直接替换锚点题，不增加总任务量。

### 题目类型、掌握等级与提示等级

### 任务类型

| 标记 | 含义 |
| --- | --- |
| N | 第一次训练的新题 |
| R1 | 第一次延迟复写 |
| R2 | 第二次延迟复写 |
| M | 限时模拟题 |
| E | 面试口述题 |

### 掌握等级

| 等级 | 判定标准 | 后续动作 |
| --- | --- | --- |
| A | 24—48 小时后不看题解，限时独立写出，并能解释复杂度和边界 | 进入 3 天、7 天抽查 |
| B | 在分级提示后完成，次日可以独立复写 | 3 天内再次测试 |
| C | 看懂答案，但还不能独立写出 | 不算掌握，24 小时后闭卷复写 |
| X | 超时、放弃、代码错误或无法解释 | 退回基础例题和知识点，不继续堆难题 |

### AI 提示等级

| 等级 | AI 可以做什么 |
| --- | --- |
| L0 | 复述目标，要求补充输入、输出、约束和已有尝试 |
| L1 | 提示题型或数据结构，不给公式和代码 |
| L2 | 给一个关键观察、反例或边界条件 |
| L3 | 给伪代码骨架或接口签名，关键步骤留空 |
| L4 | 指出当前方案的具体错误，给局部修正方向 |
| L5 | 在已经提交完整尝试后，给最小差异修正和解释 |

使用 L3 以上提示，当天最高只能记 B；看过完整解法，只能记 C。

### Day08—14：用 7 天追补数据结构基本功

#### Day08：集合框架、复杂度和数组复写

- **算法必做**：阅读集合框架与复杂度主课；从已 AC 数组题中随机闭卷复写 4 道；口述 `O(1)`、`O(log n)`、`O(n)`、`O(n log n)`、`O(n²)`。
- **项目任务**：OJ 课件按顺序学习开场、需求、CS/BS 架构和微服务划分，画系统边界草图。
- **八股主题**：为什么需要集合框架；ArrayList、LinkedList、HashMap 分别解决什么问题。
- **验收证据**：4 份复写结果、每题等级、复杂度说明、系统边界图。

#### Day09：泛型、List、ArrayList 与顺序表

- **算法必做**：阅读泛型、List、ArrayList 主课；手写泛型 `MyArrayList<E>` 的扩容、`add`、`get`、`set`、`remove`。
- **锚点题**：[有序数组的平方](https://leetcode.cn/problems/squares-of-a-sorted-array/)。
- **项目任务**：OJ 技术选型和后端工程创建，整理服务、依赖和配置清单。
- **八股主题**：泛型擦除；ArrayList 为什么随机访问快；扩容为什么有成本。
- **验收证据**：顺序表代码、操作复杂度表、依赖清单。

#### Day10：LinkedList 与链表

- **算法必做**：手写无头单向链表的头插、尾插、删除、查找和反转；画 `prev`、`cur`、`next` 的变化。
- **锚点题**：[反转链表](https://leetcode.cn/problems/reverse-linked-list/)。
- **进阶题**：[合并两个有序链表](https://leetcode.cn/problems/merge-two-sorted-lists/)。
- **项目任务**：OJ 按课件学习 MySQL、MyBatis、MyBatis-Plus 和管理员表设计，画最小 ER 图。
- **八股主题**：ArrayList 与 LinkedList 选型；虚拟头节点；MyBatis 与 MyBatis-Plus 的关系。
- **验收证据**：链表代码、指针图、ER 草图。

#### Day11：栈、队列与 Deque

- **算法必做**：手写数组栈和循环队列，能解释判空、判满和环形下标。
- **锚点题**：[有效的括号](https://leetcode.cn/problems/valid-parentheses/)。
- **进阶题**：[用栈实现队列](https://leetcode.cn/problems/implement-queue-using-stacks/)。
- **项目任务**：OJ 按课件学习接口文档、Apifox、加密、全局异常和日志框架，定义一个统一响应和异常示例。
- **八股主题**：为什么推荐 `Deque` 替代 `Stack`；异常分类；日志级别。
- **验收证据**：栈/队列代码、接口契约、异常响应样例。

#### Day12：Map、Set 与哈希

- **算法必做**：理解哈希、冲突和拉链法；手写固定容量的简化哈希桶，只要求 `put`、`get` 和冲突处理。
- **锚点题**：[有效的字母异位词](https://leetcode.cn/problems/valid-anagram/)。
- **进阶题**：[字母异位词分组](https://leetcode.cn/problems/group-anagrams/)。
- **项目任务**：OJ 按课件学习 Nacos 和 Gateway，画“请求进入网关后如何找到服务”的时序图。
- **八股主题**：HashMap 冲突；`equals` 与 `hashCode`；注册中心和网关的职责。
- **验收证据**：简化哈希实现、请求路由时序图、3 个追问答案。

#### Day13：认证、Redis、二叉树与堆

- **算法必做**：递归实现二叉树前序、中序、后序遍历；手写大根堆 `shiftDown`；练习 `PriorityQueue` 比较器。
- **锚点题**：[二叉树的最大深度](https://leetcode.cn/problems/maximum-depth-of-binary-tree/)。
- **进阶题**：[数组中的第 K 个最大元素](https://leetcode.cn/problems/kth-largest-element-in-an-array/)。
- **项目任务**：OJ 按课件学习身份认证和 Redis，画登录、鉴权、缓存读取主链。
- **八股主题**：递归函数返回什么；`Comparable` 与 `Comparator`；Token 和 Redis 如何配合。
- **验收证据**：树遍历、堆调整、鉴权时序图。

#### Day14：排序、对象分层和追补周验收

- **算法必做**：闭卷手写插入排序、快速排序、归并排序；整理时间复杂度、空间复杂度和稳定性。
- **锚点题**：[排序数组](https://leetcode.cn/problems/sort-an-array/)。
- **项目任务**：OJ 按课件学习 Entity、DTO、VO、参数校验、前端开发和分页；前端优先使用半成品，不从零重学 Vue。
- **八股主题**：DTO/VO 为什么分层；参数校验放在哪里；快排与归并如何选择。
- **验收证据**：6 组核心模板抽写、排序对照表、OJ 两分钟架构讲解提纲。

Day14 通过标准：顺序表、单链表、栈/循环队列、树遍历、堆调整、快排/归并这 6 组模板中，至少 4 组可以闭卷写出，其余必须明确标 B/C/X。

## 工程实践与边界

## 常见误区

1. **把 AC 当掌握**：当天看过提示后通过只能记 B 或 C，必须经过延迟复写才能晋级 A。
2. **把《Hello 算法》当第二套主课**：它只负责图解预习和查漏，不能与本地 Java 课程并行赶进度。
3. **按课件顺序等于实现全部功能**：顺序必须保留，但短信、完整 ES/OSS、完整竞赛系统等非主链模块可以暂缓。
4. **项目技术越多越有竞争力**：能解释一致性、幂等、隔离和失败恢复，比罗列中间件更重要。
5. **AI 给出的完整答案也算自己的能力**：AI 只负责诊断、分级提示、审查和追问，不替代独立实现。
6. **落后就用周末补回所有任务**：未完成的可选项应删除，周末只补到期复写和主链阻塞点。

## 面试追问

**问：为什么把原来的题量目标降到 60—75 道？**

**答：** 因为工作日只有 2 小时，还要并行准备项目和八股。新题数量下降后，通过 80 次以上延迟复写、A/B/C/X 分级和综合模拟，把“看过”转化为“能写、能讲”。

**问：两个项目都按课件顺序推进，会不会做不完？**

**答：** 课件顺序用于保证认知链路不倒置，但实现范围按纵向主链裁剪。OJ 只保证鉴权、题目、提交、判题和状态链路；AI 项目只保证需求、生成、构建、修复、截图和提交链路。

**问：为什么 RAG 和 MCP 不要求都完整实现？**

**答：** 两者都容易成为独立大项目。45 天内至少选一个形成真实证据，另一个能够讲清调用边界、权限、失败处理和选型取舍，面试价值高于两个半成品。

**问：AI 陪练怎样避免让我失去独立思考？**

**答：** 使用 L0—L5 分级提示，未提交尝试前禁止完整答案；使用 L3 以上提示会限制当天等级，看过完整解法必须记 C，并在 24 小时后闭卷复写。
