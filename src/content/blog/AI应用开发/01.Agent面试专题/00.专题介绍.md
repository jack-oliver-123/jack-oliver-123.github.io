这是一套面向零基础读者的 Agent 面试教程。你不需要先学会某个 Agent 框架，也不需要一上来就理解 ReAct、幂等或上下文工程。专题会先用生活化例子建立直觉，再解释正式概念，最后给出可以直接用于面试的 60 秒回答和常见追问。

本专题共 26 道题，覆盖 Agent 基础架构、执行与规划、记忆、多 Agent、生产治理、Tool 设计、RAG、协议分层和模型路由。学完后，你不仅能回答“Agent 是什么”，还能够继续解释它怎么执行、为什么会失败、如何保证安全，以及实际项目中怎样选型。

> 学习建议：先理解例子，再背“60 秒回答”；只背术语但说不清例子，面试官继续追问时很容易卡住。

## 开始前先认识 10 个词

| 术语            | 零基础理解                                   |
| ------------- | --------------------------------------- |
| LLM（大语言模型）    | 根据输入生成文本或结构化结果的模型，可以把它理解成 Agent 的“大脑”之一 |
| Agent（智能体）    | 围绕目标持续观察、决策和行动的任务执行系统                   |
| Tool（工具）      | Agent 可以调用的外部能力，例如查订单、搜索资料或发送邮件         |
| Workflow（工作流） | 开发者提前规定好步骤和分支的执行流程                      |
| 状态            | 当前任务已经知道什么、完成了什么、下一步等待什么                |
| 运行时           | 真正维护循环、检查权限并执行工具的外层程序                   |
| Schema        | 对数据字段、类型和格式的明确约定                        |
| 幂等            | 同一个操作重复提交，也不会产生重复副作用                    |
| Trace（追踪）     | 把一次任务中的模型调用、工具调用和状态变化串联起来的记录            |
| PII（个人可识别信息）  | 姓名、手机号、身份证号等能够识别个人的信息                   |

不要求现在就背下这张表。后面的文章每次使用这些概念时，都会结合具体例子解释。

## 使用方法

建议按三遍学习：

1. **第一遍看懂**：先读“详细解析”里的例子，能够用自己的话复述概念。
2. **第二遍会答**：背熟“60 秒回答”，再遮住答案练习“面试追问”。
3. **第三遍会展开**：把“工程实践与边界”与自己的项目经历连接起来，准备一个成功案例和一个失败案例。

面试回答可以使用一个稳定结构：

> 先下定义，再讲工作机制，然后说明工程边界，最后给一个例子。

不要背诵或声称能够查看模型的私有思维链，也不要承诺脱离具体任务的固定成功率或成本降幅。更好的回答是说明指标如何定义、可能怎样失败、系统如何保护，以及最终如何验证。
