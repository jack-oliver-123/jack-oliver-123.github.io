## 栈（Stack）

### 概念

**栈**是一种特殊的线性表，只允许在固定的一端（**栈顶**）进行插入和删除操作。

核心特征：**后进先出**（LIFO, Last In First Out）——最后放入的元素最先被取出。

```
    ┌───┐
    │ C │ ← 栈顶（最后入栈，最先出栈）
    ├───┤
    │ B │
    ├───┤
    │ A │ ← 栈底（最先入栈，最后出栈）
    └───┘
```

两个核心操作：

- **压栈（push）**：将元素放入栈顶
- **出栈（pop）**：将栈顶元素取出

### 生活中的栈

- 一摞盘子——只能从顶部拿和放
- 浏览器的"后退"按钮——最近访问的页面最先回退
- 撤销操作（Ctrl+Z）——最近的操作最先被撤销
- 弹夹——最后压入的子弹最先射出

### Stack 的使用

Java 中的 `Stack` 类继承自 `Vector`（线程安全的动态数组）：

```java
import java.util.Stack;

Stack<Integer> stack = new Stack<>();

// 入栈
stack.push(1);
stack.push(2);
stack.push(3);
System.out.println(stack); // [1, 2, 3]（3 在栈顶）

// 查看栈顶元素（不弹出）
System.out.println(stack.peek()); // 3

// 出栈
System.out.println(stack.pop()); // 3
System.out.println(stack.pop()); // 2

// 判空和大小
System.out.println(stack.empty()); // false
System.out.println(stack.size()); // 1
```

| 方法 | 说明 | 时间复杂度 |
|------|------|-----------|
| `Stack()` | 构造空栈 | — |
| `E push(E e)` | 入栈，返回该元素 | O(1) |
| `E pop()` | 出栈，返回栈顶元素 | O(1) |
| `E peek()` | 查看栈顶元素（不删除） | O(1) |
| `boolean empty()` | 判断是否为空 | O(1) |
| `int size()` | 获取栈中元素个数 | O(1) |

### 为什么不推荐使用 Stack 类

`Stack` 继承自 `Vector`，而 `Vector` 是一个过时的线程安全动态数组——所有方法都加了 `synchronized`，在单线程场景下有不必要的性能开销。

**实际开发中推荐用 `Deque` 接口代替 Stack**：

```java
// ❌ 不推荐：Stack 类
Stack<Integer> stack = new Stack<>();

// ✅ 推荐：用 Deque 接口 + ArrayDeque 实现
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);
stack.push(2);
stack.pop();    // 2
stack.peek();   // 1
```

`ArrayDeque` 底层是循环数组，无锁，性能优于 `Stack`。

### 栈的模拟实现

用数组实现一个简易栈：

```java
public class MyStack<E> {
    private Object[] data;
    private int size;

    public MyStack() {
        data = new Object[10];
        size = 0;
    }

    public void push(E e) {
        ensureCapacity();
        data[size++] = e;
    }

    @SuppressWarnings("unchecked")
    public E pop() {
        if (empty()) throw new RuntimeException("栈为空");
        E val = (E) data[--size];
        data[size] = null; // 帮助 GC
        return val;
    }

    @SuppressWarnings("unchecked")
    public E peek() {
        if (empty()) throw new RuntimeException("栈为空");
        return (E) data[size - 1];
    }

    public boolean empty() {
        return size == 0;
    }

    public int size() {
        return size;
    }

    private void ensureCapacity() {
        if (size == data.length) {
            data = Arrays.copyOf(data, data.length * 2);
        }
    }
}
```

核心观察：栈只在数组尾部操作（`size` 位置），所以 push 和 pop 都是 O(1)——不需要搬移元素。

### 概念辨析：栈 vs 虚拟机栈 vs 栈帧

面试中经常混淆的三个"栈"：

| 概念 | 含义 |
|------|------|
| 数据结构的栈 | LIFO 的数据容器（本文讨论的） |
| JVM 虚拟机栈 | 每个线程私有的内存区域，存放方法调用信息 |
| 栈帧（Stack Frame） | 虚拟机栈中的一个条目，对应一次方法调用（含局部变量表、操作数栈等） |

三者关系：虚拟机栈是一个"数据结构栈"，其中每个元素是一个"栈帧"。

## 栈的经典应用

### 括号匹配

给定只包含 `()`、`{}`、`[]` 的字符串，判断括号是否有效。

**思路**：遇到左括号入栈，遇到右括号弹出栈顶检查是否匹配。

```java
// LeetCode #20 有效的括号
public boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(' || c == '{' || c == '[') {
            stack.push(c);
        } else {
            if (stack.isEmpty()) return false;
            char top = stack.pop();
            if (c == ')' && top != '(') return false;
            if (c == '}' && top != '{') return false;
            if (c == ']' && top != '[') return false;
        }
    }
    return stack.isEmpty(); // 栈必须为空才说明全部匹配
}
```

### 逆波兰表达式求值

逆波兰表达式（后缀表达式）是一种不需要括号的数学表达式表示法。例如 `2 1 + 3 *` 等价于 `(2 + 1) * 3`。

**思路**：遇到数字入栈，遇到运算符弹出两个数计算后结果入栈。

```java
// LeetCode #150 逆波兰表达式求值
public int evalRPN(String[] tokens) {
    Deque<Integer> stack = new ArrayDeque<>();
    for (String token : tokens) {
        if ("+-*/".contains(token)) {
            int b = stack.pop(); // 注意顺序：后弹出的是左操作数
            int a = stack.pop();
            switch (token) {
                case "+": stack.push(a + b); break;
                case "-": stack.push(a - b); break;
                case "*": stack.push(a * b); break;
                case "/": stack.push(a / b); break;
            }
        } else {
            stack.push(Integer.parseInt(token));
        }
    }
    return stack.pop();
}
```

### 最小栈

设计一个支持 O(1) 获取最小值的栈。

**思路**：用两个栈，一个存数据，一个同步维护当前最小值。

```java
// LeetCode #155 最小栈
class MinStack {
    private Deque<Integer> dataStack = new ArrayDeque<>();
    private Deque<Integer> minStack = new ArrayDeque<>();

    public void push(int val) {
        dataStack.push(val);
        if (minStack.isEmpty() || val <= minStack.peek()) {
            minStack.push(val);
        }
    }

    public void pop() {
        int val = dataStack.pop();
        if (val == minStack.peek()) {
            minStack.pop();
        }
    }

    public int top() {
        return dataStack.peek();
    }

    public int getMin() {
        return minStack.peek();
    }
}
```

## 队列（Queue）

### 概念

**队列**是一种特殊的线性表，只允许在一端（**队尾**）插入，在另一端（**队头**）删除。

核心特征：**先进先出**（FIFO, First In First Out）——最先放入的元素最先被取出。

```
队头 ← [A] [B] [C] [D] ← 队尾
出队方向 ←                → 入队方向
```

- **入队（enqueue/offer）**：在队尾插入元素
- **出队（dequeue/poll）**：从队头删除元素

### 生活中的队列

- 排队买奶茶——先来的先买
- 打印机任务队列——先提交的先打印
- 消息队列（MQ）——先发的消息先被消费

### Queue 的使用

Java 中 `Queue` 是一个接口，通常用 `LinkedList` 实例化：

```java
import java.util.Queue;
import java.util.LinkedList;

Queue<Integer> queue = new LinkedList<>();

// 入队
queue.offer(1);
queue.offer(2);
queue.offer(3);
System.out.println(queue); // [1, 2, 3]

// 查看队头（不移除）
System.out.println(queue.peek()); // 1

// 出队
System.out.println(queue.poll()); // 1
System.out.println(queue.poll()); // 2

// 判空和大小
System.out.println(queue.isEmpty()); // false
System.out.println(queue.size());    // 1
```

| 方法 | 说明 | 失败时行为 |
|------|------|-----------|
| `boolean offer(E e)` | 入队 | 返回 false |
| `E poll()` | 出队 | 返回 null |
| `E peek()` | 查看队头 | 返回 null |
| `boolean isEmpty()` | 判空 | — |
| `int size()` | 大小 | — |

> 注意：`Queue` 还有 `add`、`remove`、`element` 方法，功能相同但失败时抛异常。推荐使用 `offer`/`poll`/`peek`——返回特殊值而非抛异常，代码更安全。

### 队列的底层选择

队列用顺序结构（数组）还是链式结构好？

- **链表**：入队在尾部 O(1)，出队在头部 O(1)——天然适合队列
- **数组**：出队需要搬移所有元素 O(n)——不适合（除非用循环数组）

所以 JDK 中 `Queue` 默认用 `LinkedList` 实现。如果想用数组实现，需要引入**循环队列**。

## 循环队列

### 为什么需要循环队列

普通数组实现队列时，出队后前面的空间就浪费了（"假溢出"）。循环队列通过将数组首尾相连，让出队后的空间可以被重新利用。

### 数组下标循环技巧

循环队列的关键是让下标在到达数组末尾后"绕回"到开头：

```java
// 下标往后移动 offset 步（绕回）
index = (index + offset) % array.length;

// 下标往前移动 offset 步（绕回）
index = (index + array.length - offset) % array.length;
```

### 判断空和满

循环队列最大的难题：当 `front == rear` 时，到底是空还是满？

三种解决方案：

| 方案 | 判空条件 | 判满条件 | 代价 |
|------|---------|---------|------|
| 额外 size 计数 | `size == 0` | `size == capacity` | 多一个变量 |
| 浪费一个位置 | `front == rear` | `(rear + 1) % len == front` | 少存一个元素 |
| 标记位 flag | `front == rear && !flag` | `front == rear && flag` | 多一个布尔值 |

### 实现循环队列

以"浪费一个位置"方案为例：

```java
// LeetCode #622 设计循环队列
class MyCircularQueue {
    private int[] data;
    private int front; // 指向队头
    private int rear;  // 指向队尾的下一个位置
    private int capacity;

    public MyCircularQueue(int k) {
        // 多分配一个位置用于判满
        data = new int[k + 1];
        capacity = k + 1;
        front = 0;
        rear = 0;
    }

    public boolean enQueue(int value) {
        if (isFull()) return false;
        data[rear] = value;
        rear = (rear + 1) % capacity;
        return true;
    }

    public boolean deQueue() {
        if (isEmpty()) return false;
        front = (front + 1) % capacity;
        return true;
    }

    public int Front() {
        if (isEmpty()) return -1;
        return data[front];
    }

    public int Rear() {
        if (isEmpty()) return -1;
        // rear 的前一个位置就是队尾元素
        return data[(rear - 1 + capacity) % capacity];
    }

    public boolean isEmpty() {
        return front == rear;
    }

    public boolean isFull() {
        return (rear + 1) % capacity == front;
    }
}
```

## 双端队列（Deque）

### 概念

**双端队列**（Deque, Double Ended Queue）允许两端都可以入队和出队——既可以当栈用，也可以当队列用。

```
     ← 出/入         出/入 →
队头 ← [A] [B] [C] [D] → 队尾
     ← 出/入         出/入 →
```

### Deque 接口

Java 中 `Deque` 是一个接口，常用实现类有两个：

| 实现 | 底层 | 特点 |
|------|------|------|
| `LinkedList` | 双向链表 | 无容量限制 |
| `ArrayDeque` | 循环数组 | 性能更好（缓存友好），不能存 null |

```java
// 当栈使用
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);   // 等价于 addFirst
stack.push(2);
stack.pop();     // 等价于 removeFirst → 2
stack.peek();    // 等价于 peekFirst → 1

// 当队列使用
Deque<Integer> queue = new ArrayDeque<>();
queue.offer(1);  // 等价于 offerLast
queue.offer(2);
queue.poll();    // 等价于 pollFirst → 1
queue.peek();    // 等价于 peekFirst → 2

// 当双端队列使用
Deque<Integer> deque = new ArrayDeque<>();
deque.offerFirst(1);  // 队头入队
deque.offerLast(2);   // 队尾入队
deque.pollFirst();    // 队头出队 → 1
deque.pollLast();     // 队尾出队 → 2
```

### 实际工程建议

在实际开发中，**`Deque` 接口是最通用的选择**：

- 需要栈？用 `Deque<E> stack = new ArrayDeque<>()`，调用 `push`/`pop`
- 需要队列？用 `Deque<E> queue = new ArrayDeque<>()`，调用 `offer`/`poll`
- 需要双端队列？直接用 `Deque` 的全部方法

不要用 `Stack` 类（过时、有锁）。`Queue` 接口在需要限制为 FIFO 语义时使用。

## 经典面试题

### 用栈实现队列

```java
// LeetCode #232
class MyQueue {
    private Deque<Integer> inStack = new ArrayDeque<>();
    private Deque<Integer> outStack = new ArrayDeque<>();

    public void push(int x) {
        inStack.push(x);
    }

    public int pop() {
        if (outStack.isEmpty()) {
            // 将 inStack 全部倒入 outStack
            while (!inStack.isEmpty()) {
                outStack.push(inStack.pop());
            }
        }
        return outStack.pop();
    }

    public int peek() {
        if (outStack.isEmpty()) {
            while (!inStack.isEmpty()) {
                outStack.push(inStack.pop());
            }
        }
        return outStack.peek();
    }

    public boolean empty() {
        return inStack.isEmpty() && outStack.isEmpty();
    }
}
```

**关键思路**：两个栈倒一次就实现了逆序——第一个栈按 LIFO 入，全部倒入第二个栈后顺序变成 FIFO。均摊时间复杂度 O(1)。

### 用队列实现栈

```java
// LeetCode #225
class MyStack {
    private Queue<Integer> queue = new LinkedList<>();

    public void push(int x) {
        queue.offer(x);
        // 把前面的元素全部移到后面，让新元素在队头
        for (int i = 0; i < queue.size() - 1; i++) {
            queue.offer(queue.poll());
        }
    }

    public int pop() {
        return queue.poll();
    }

    public int top() {
        return queue.peek();
    }

    public boolean empty() {
        return queue.isEmpty();
    }
}
```

**关键思路**：每次 push 后，把前面的元素全部出队再入队，让新元素始终在队头。push 是 O(n)，pop 是 O(1)。

## 小结

| 数据结构 | 特征 | Java 推荐实现 | 核心操作 |
|---------|------|-------------|---------|
| 栈 | LIFO 后进先出 | `Deque` + `ArrayDeque` | push / pop / peek |
| 队列 | FIFO 先进先出 | `Queue` + `LinkedList` | offer / poll / peek |
| 双端队列 | 两端都可操作 | `Deque` + `ArrayDeque` | offerFirst/Last, pollFirst/Last |
| 循环队列 | 数组实现的队列 | 手写（面试常考） | 取模实现下标循环 |

**关键认知**：

- 栈和队列不是新的数据结构，而是对线性表**加了限制**——限制带来了确定性，确定性带来了效率
- 实际开发中用 `Deque` 接口统一栈和队列的需求，底层用 `ArrayDeque`
- 不要用 `Stack` 类——它继承了 `Vector` 的历史包袱（线程安全的开销、暴露了不该暴露的 List 方法）
- 循环队列的核心是取模运算 `(index + offset) % length`
