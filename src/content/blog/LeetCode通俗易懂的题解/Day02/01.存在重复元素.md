## 题目入口

题目链接：[存在重复元素](https://leetcode.cn/problems/contains-duplicate/description/)

## 一句话读懂题目

给你一个整数数组 `nums`，判断数组里有没有重复的元素。

如果有任何一个值出现了至少两次，返回 `true`；如果所有元素都不同，返回 `false`。

比如：

```text
nums = [1, 2, 3, 1]
```

`1` 出现了两次，返回：

```text
true
```

再比如：

```text
nums = [1, 2, 3, 4]
```

每个元素都不同，返回：

```text
false
```

## 先用生活例子理解

想象你在活动入口做签到登记。

每个人到了之后报出自己的工号，你在签到表上记一笔。

如果某个工号你之前已经登记过了，说明这个人重复签到了：

> 遇到已经登记过的工号，直接喊"有重复"。

如果所有人都签完了，每个工号都只出现一次，那就"没有重复"。

这里的签到表就是 `HashSet`：

- 来一个工号，先查表里有没有；
- 有就说明重复了；
- 没有就登记进去，继续看下一个。

## 方法一：HashSet（推荐）

这是最直接的做法。

用一个 `HashSet` 记录见过的元素：

```text
遍历数组
    如果当前元素已经在 Set 里 -> 返回 true
    否则把它加入 Set
遍历结束还没返回 -> 返回 false
```

`HashSet` 的查找和插入都是 `O(1)`，所以总时间是 `O(n)`。

nums = [1, 2, 3, 1]
```

set = {}
```

| 步骤 | 当前元素 | set 中是否存在 | 操作 | set 状态 |
| --- | --- | --- | --- | --- |
| 1 | `1` | 否 | 加入 | `{1}` |
| 2 | `2` | 否 | 加入 | `{1, 2}` |
| 3 | `3` | 否 | 加入 | `{1, 2, 3}` |
| 4 | `1` | 是 | 返回 `true` | - |

在第 4 步发现 `1` 已经在 `set` 里了，直接返回 `true`。

再看没有重复的情况：

```text

```

| 步骤 | 当前元素 | set 中是否存在 | 操作 | set 状态 |
| --- | --- | --- | --- | --- |
| 1 | `1` | 否 | 加入 | `{1}` |
| 2 | `2` | 否 | 加入 | `{1, 2}` |
| 3 | `3` | 否 | 加入 | `{1, 2, 3}` |
| 4 | `4` | 否 | 加入 | `{1, 2, 3, 4}` |

遍历结束，没有发现重复，返回 `false`。

## Java 代码

先记住一句话：

> 见过就重复，没见过就记下来。

### 写法一：HashSet（推荐）

```java
class Solution {
    public boolean containsDuplicate(int[] nums) {
        Set<Integer> set = new HashSet<>();

        for (int num : nums) {
            if (!set.add(num)) {
                return true;
            }
        }

        return false;
    }
}
```

这里利用了 `HashSet.add()` 的返回值：

- 如果元素不存在，加入成功，返回 `true`；
- 如果元素已经存在，加入失败，返回 `false`。

所以 `!set.add(num)` 为 `true` 时，说明 `num` 已经存在过了，就是重复。

如果觉得这个写法不太直观，也可以拆开写：

```java
class Solution {
    public boolean containsDuplicate(int[] nums) {
        Set<Integer> set = new HashSet<>();

        for (int num : nums) {
            if (set.contains(num)) {
                return true;
            }
            set.add(num);
        }

        return false;
    }
}
```

两种写法效果一样，第一种更简洁，第二种更清晰。

### 写法二：排序后比较相邻元素

```java
class Solution {
    public boolean containsDuplicate(int[] nums) {
        Arrays.sort(nums);

        for (int i = 1; i < nums.length; i++) {
            if (nums[i] == nums[i - 1]) {
                return true;
            }
        }

        return false;
    }
}
```

## 代码逐行拆解（HashSet 写法）

```java
Set<Integer> set = new HashSet<>();
```

创建一个 `HashSet`，用来存放已经见过的元素。

`HashSet` 不允许重复元素，查找和插入都是 `O(1)`。

```java
for (int num : nums) {
```

遍历数组中的每一个元素。

```java
if (!set.add(num)) {
    return true;
}
```

尝试把当前元素加入 `set`。

如果加入失败（返回 `false`），说明这个元素之前已经存在了，发现重复，直接返回 `true`。

```java
return false;
```

遍历结束都没有发现重复，返回 `false`。

## 方法二：排序后比较相邻元素

另一种思路：先排序，排序后相同的元素一定相邻。

```text
排序数组
遍历数组
    如果 nums[i] == nums[i-1] -> 返回 true
遍历结束 -> 返回 false
```

这种方法不需要额外的 `HashSet`，但会修改原数组，并且时间复杂度是 `O(n log n)`。

## 两种方法怎么选

| | HashSet | 排序 |
| --- | --- | --- |
| 时间复杂度 | `O(n)` | `O(n log n)` |
| 空间复杂度 | `O(n)` | `O(1)` 或 `O(log n)` |
| 是否修改原数组 | 否 | 是 |
| 能否提前退出 | 能（发现重复就返回） | 能（发现相邻相等就返回） |

面试时推荐 HashSet 写法：

- 时间更优；
- 不修改原数组；
- 代码更短。

排序写法适合空间受限的场景，或者面试时作为第二种方法补充。

## 复杂度分析

### 方法一：HashSet

时间复杂度：`O(n)`。

遍历一次数组，每个元素的 `add` 和 `contains` 操作都是 `O(1)`。

空间复杂度：`O(n)`。

最坏情况下，所有元素都不重复，`set` 存了 `n` 个元素。

### 方法二：排序

时间复杂度：`O(n log n)`。

排序的时间复杂度是 `O(n log n)`，遍历是 `O(n)`，总体由排序决定。

空间复杂度：`O(log n)`。

Java 的 `Arrays.sort` 对基本类型使用双轴快排，递归栈深度是 `O(log n)`。

## 常见坑

### 1. 用 int[] 而不是 Integer 放入 HashSet

`HashSet` 是泛型容器，不能直接存基本类型 `int`。

```java
Set<int> set = new HashSet<>();  // ❌ 编译错误
Set<Integer> set = new HashSet<>();  // ✓
```

遍历时 `int num` 会自动装箱成 `Integer`，所以不影响使用。

### 2. 混淆 add 的返回值含义

`set.add(num)` 返回 `true` 表示加入成功（之前不存在）。

返回 `false` 表示加入失败（已经存在）。

所以判断重复是 `!set.add(num)`，不要写反了。

### 3. 忘记空数组或单元素数组

```text
nums = [] -> false（没有元素，不可能重复）
nums = [1] -> false（只有一个元素，不可能重复）
```

代码不需要特判，循环自然处理了这些情况。

### 4. 排序写法从 i = 0 开始比较

排序后比较相邻元素时，要从 `i = 1` 开始，比较 `nums[i]` 和 `nums[i-1]`。

如果从 `i = 0` 开始比较 `nums[i]` 和 `nums[i+1]`，循环条件要写 `i < nums.length - 1`，容易出错。

## 面试时怎么讲

### 30 秒版

> 用一个 HashSet 记录见过的元素。遍历数组，每遇到一个元素就尝试加入 Set，如果加入失败说明之前已经出现过，返回 `true`。遍历结束没发现重复就返回 `false`。时间 `O(n)`，空间 `O(n)`。

### 面试官可能追问

**如果要求 O(1) 空间怎么办？**

先排序，然后比较相邻元素。排序后相同的元素一定挨在一起。时间变成 `O(n log n)`，空间是排序的 `O(log n)`。

如果不允许修改原数组且要求 `O(1)` 空间，这道题就没有好的解法了。

**HashSet 的 O(1) 是怎么做到的？**

`HashSet` 底层是哈希表。每个元素通过哈希函数算出一个桶的位置，直接定位到那个桶。

在没有大量哈希冲突的情况下，查找和插入都是 `O(1)`。最坏情况（所有元素都冲突到同一个桶）是 `O(n)`，但实际几乎不会发生。

**能不能用暴力两层循环？**

可以，但时间是 `O(n²)`，对于大数组会超时。

```java
for (int i = 0; i < nums.length; i++) {
    for (int j = i + 1; j < nums.length; j++) {
        if (nums[i] == nums[j]) return true;
    }
}
return false;
```

面试时可以提一句作为暴力解，然后给出 HashSet 优化方案。

**为什么不用 HashMap？**

`HashMap` 也能做，用 `key` 存元素，`value` 存出现次数。

但这道题只需要判断"有没有出现过"，不需要知道出现了几次，所以 `HashSet` 就够了，更简洁。

## 最后总结

存在重复元素这道题是 HashSet 最经典的入门应用：

> 需要快速判断"之前见没见过"，就用 HashSet。

代码模板可以这样记：

```text
创建 HashSet
遍历数组
    尝试加入 set
    加入失败 -> 说明重复，返回 true
遍历结束 -> 没有重复，返回 false
```

刷题时可以用下面几个边界用例自检：

```text
nums = [1, 2, 3, 1] -> true（有重复）
nums = [1, 2, 3, 4] -> false（无重复）
nums = [1, 1, 1, 3, 3, 4, 3, 2, 4, 2] -> true（多个重复）
nums = [1] -> false（单元素）
nums = [] -> false（空数组）
```

以后看到"判断是否存在"、"是否出现过"、"有没有重复"这类关键词，第一反应就是 HashSet。

最后把这句话记住：

> 判断重不重复，不用记住每个数出现了几次，只需要知道"见过还是没见过"。
