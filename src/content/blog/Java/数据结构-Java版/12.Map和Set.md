```markmap
# Map 和 Set

## 搜索模型
- 纯 Key 模型（Set）
  - 单词是否存在
  - 名字是否在通讯录中
- Key-Value 模型（Map）
  - 单词 → 出现次数
  - 姓名 → 电话号码

## 二叉搜索树（BST）
- 左子树 < 根 < 右子树
- 查找/插入/删除
- 最优 O(log n)，最差 O(n)
- 退化为单支树时性能丧失
- 解决方案：AVL 树、红黑树

## Map 接口
- 存储 Key-Value 键值对
- Key 唯一，Value 可重复
- Map.Entry 内部类
- 实现类
  - TreeMap（红黑树）
  - HashMap（哈希表）

## Set 接口
- 继承 Collection
- 只存 Key，不存 Value
- Key 唯一（去重）
- 实现类
  - TreeSet（红黑树）
  - HashSet（哈希表）

## 哈希表原理
- 哈希函数：Key → 地址
- 哈希冲突不可避免
- 负载因子 = 元素数 / 数组大小
- 冲突解决
  - 闭散列（开放定址）
    - 线性探测
    - 二次探测
  - 开散列（链地址法/哈希桶）
    - JDK8：链表 → 红黑树

## TreeMap vs HashMap
- TreeMap：有序、O(log n)、Key 需 Comparable
- HashMap：无序、O(1)、Key 需 hashCode+equals
```

## 搜索的两种模型

在实际开发中，我们经常需要"搜索"：

**纯 Key 模型**——只需要判断"在不在"：
- 快速查找一个单词是否在词典中
- 判断一个用户名是否已被注册
- 检测一个 IP 是否在黑名单中

**Key-Value 模型**——通过 Key 找到对应的 Value：
- 统计文件中每个单词出现的次数（单词 → 次数）
- 通讯录：姓名 → 电话号码
- 缓存：URL → 网页内容

Java 集合框架中，`Map` 对应 Key-Value 模型，`Set` 对应纯 Key 模型。

## 二叉搜索树（BST）

### 定义

二叉搜索树（Binary Search Tree）又称二叉排序树，满足以下性质：

- 左子树上所有节点的值**小于**根节点
- 右子树上所有节点的值**大于**根节点
- 左右子树本身也是二叉搜索树

```
        5
       / \
      3   7
     / \ / \
    1  4 6  8
```

中序遍历 BST 得到的是**有序序列**：1, 3, 4, 5, 6, 7, 8。

### 查找

```java
public TreeNode search(TreeNode root, int key) {
    if (root == null) return null;
    if (key == root.val) return root;
    if (key < root.val) return search(root.left, key);
    return search(root.right, key);
}
```

### 插入

```java
public TreeNode insert(TreeNode root, int key) {
    if (root == null) return new TreeNode(key);
    if (key < root.val) {
        root.left = insert(root.left, key);
    } else if (key > root.val) {
        root.right = insert(root.right, key);
    }
    // key == root.val 时不插入（不允许重复）
    return root;
}
```

### 删除（难点）

删除节点 `cur` 分三种情况：

1. **cur 没有左孩子**：用右孩子替代 cur
2. **cur 没有右孩子**：用左孩子替代 cur
3. **cur 左右孩子都有**：找到右子树中最小的节点（中序后继），用它的值替换 cur，然后删除那个最小节点

```java
public TreeNode delete(TreeNode root, int key) {
    if (root == null) return null;

    if (key < root.val) {
        root.left = delete(root.left, key);
    } else if (key > root.val) {
        root.right = delete(root.right, key);
    } else {
        // 找到了要删除的节点
        if (root.left == null) return root.right;
        if (root.right == null) return root.left;
        // 左右都有：找右子树的最小值
        TreeNode minNode = root.right;
        while (minNode.left != null) {
            minNode = minNode.left;
        }
        root.val = minNode.val; // 用后继值替换
        root.right = delete(root.right, minNode.val); // 删除后继
    }
    return root;
}
```

### 性能分析

BST 的查找效率取决于树的形状：

- **最优**（接近完全二叉树）：O(log n)
- **最差**（退化为单支链）：O(n)

```
最优 BST：              最差 BST（有序插入 1,2,3,4,5）：
    4                    1
   / \                    \
  2   6                    2
 / \ / \                    \
1  3 5  7                    3
                              \
                               4
                                \
                                 5
```

**解决退化问题**：使用自平衡二叉搜索树——AVL 树或红黑树。Java 中 `TreeMap`/`TreeSet` 的底层就是**红黑树**。

## Map 接口

### 概述

`Map` 是一个接口，存储 Key-Value 键值对。**Key 唯一，Value 可重复**。

`Map` **不继承 `Collection`**——它和 Collection 是并列的两大体系。

### Map.Entry

`Map.Entry<K, V>` 是 Map 内部的接口，代表一个键值对：

| 方法 | 说明 |
|------|------|
| `K getKey()` | 获取 Key |
| `V getValue()` | 获取 Value |
| `V setValue(V value)` | 设置 Value（不能设置 Key） |

### Map 常用方法

| 方法 | 说明 |
|------|------|
| `V put(K key, V value)` | 插入键值对（key 存在则更新 value） |
| `V get(Object key)` | 获取 key 对应的 value |
| `V getOrDefault(Object key, V defaultValue)` | 获取 value，不存在返回默认值 |
| `V remove(Object key)` | 删除 key 对应的映射 |
| `boolean containsKey(Object key)` | 判断是否包含 key |
| `boolean containsValue(Object value)` | 判断是否包含 value |
| `Set<K> keySet()` | 返回所有 key 的 Set |
| `Collection<V> values()` | 返回所有 value 的 Collection |
| `Set<Map.Entry<K, V>> entrySet()` | 返回所有键值对的 Set |
| `int size()` | 键值对数量 |

### TreeMap vs HashMap

| 维度 | TreeMap | HashMap |
|------|---------|---------|
| 底层结构 | 红黑树 | 哈希表 |
| 时间复杂度 | O(log n) | O(1) |
| 是否有序 | ✅ 按 Key 有序 | ❌ 无序 |
| Key 要求 | 必须可比较（Comparable 或 Comparator） | 必须覆写 hashCode + equals |
| null Key | ❌ 不允许 | ✅ 允许一个 null Key |
| 线程安全 | ❌ | ❌ |
| 适用场景 | 需要有序遍历 | 追求最高查找性能 |

### 使用示例

```java
// TreeMap：按 Key 有序
Map<String, Integer> treeMap = new TreeMap<>();
treeMap.put("banana", 3);
treeMap.put("apple", 5);
treeMap.put("cherry", 2);
System.out.println(treeMap); // {apple=5, banana=3, cherry=2}（字典序）

// HashMap：无序，但 O(1) 查找
Map<String, Integer> hashMap = new HashMap<>();
hashMap.put("banana", 3);
hashMap.put("apple", 5);
hashMap.put("cherry", 2);

// 常用操作
System.out.println(hashMap.get("apple"));         // 5
System.out.println(hashMap.getOrDefault("grape", 0)); // 0
hashMap.put("apple", 10); // key 存在则更新 value
System.out.println(hashMap.containsKey("banana")); // true

// 遍历方式
for (Map.Entry<String, Integer> entry : hashMap.entrySet()) {
    System.out.println(entry.getKey() + " = " + entry.getValue());
}
```

### Map 的注意事项

1. **Key 不能直接修改**——如果需要修改 Key，先删除再重新插入
2. **put 相同 Key 会覆盖**——返回旧的 Value
3. **TreeMap 的 Key 不能为 null**——因为需要比较大小
4. **HashMap 的 Key 可以为 null**——但只能有一个

## Set 接口

### 概述

`Set` 继承自 `Collection`，只存储 Key（不存 Value），且 **Key 必须唯一**——天然去重。

### Set 常用方法

| 方法 | 说明 |
|------|------|
| `boolean add(E e)` | 添加元素（重复则不添加，返回 false） |
| `boolean contains(Object o)` | 是否包含 |
| `boolean remove(Object o)` | 删除元素 |
| `int size()` | 元素个数 |
| `boolean isEmpty()` | 是否为空 |
| `Iterator<E> iterator()` | 迭代器 |
| `Object[] toArray()` | 转数组 |

### TreeSet vs HashSet

| 维度 | TreeSet | HashSet |
|------|---------|---------|
| 底层结构 | 红黑树（实际上是 TreeMap） | 哈希表（实际上是 HashMap） |
| 时间复杂度 | O(log n) | O(1) |
| 是否有序 | ✅ 按元素有序 | ❌ 无序 |
| 元素要求 | 必须可比较 | 必须覆写 hashCode + equals |
| null 元素 | ❌ 不允许 | ✅ 允许一个 null |
| 适用场景 | 需要有序去重 | 只需快速去重 |

> `TreeSet` 底层其实是 `TreeMap`，只是 Value 全部用同一个 Object 对象填充。`HashSet` 底层是 `HashMap`，同理。

### 使用示例

```java
// HashSet：快速去重
Set<String> set = new HashSet<>();
set.add("Java");
set.add("Python");
set.add("Java");  // 重复，不会添加
System.out.println(set.size()); // 2
System.out.println(set.contains("Java")); // true

// TreeSet：有序去重
Set<Integer> treeSet = new TreeSet<>();
treeSet.add(5);
treeSet.add(1);
treeSet.add(3);
treeSet.add(1); // 重复
System.out.println(treeSet); // [1, 3, 5]（有序）
```

## 哈希表原理

### 核心思想

之前学的搜索方式：
- 顺序查找：O(n)
- 二分查找：O(log n)，但要求有序
- 二叉搜索树：O(log n)

有没有 **O(1)** 的查找方式？——有，就是**哈希表**。

核心思想：通过**哈希函数**将 Key 直接映射到数组下标，一步到位找到元素。

```
哈希函数：hash(key) = key % capacity

例如 capacity = 10：
key = 14 → hash = 14 % 10 = 4 → 存在数组下标 4
key = 25 → hash = 25 % 10 = 5 → 存在数组下标 5
key = 34 → hash = 34 % 10 = 4 → 💥 冲突！也映射到下标 4
```

### 哈希冲突

不同的 Key 通过哈希函数计算出相同的地址——这就是**哈希冲突**。

下图展示了两个不同的 Key 经过哈希计算后映射到同一桶位，以及链地址法的解决方式：

![哈希冲突解决示意](/images/data-structures/hash-collision.svg)

由于数组容量有限而 Key 的可能值无限，**冲突不可避免**。我们只能尽量减少冲突，并设计冲突的解决方案。

### 哈希函数设计

好的哈希函数应该让 Key 尽量**均匀分布**在数组中。常见方法：

- **除留余数法**：`hash(key) = key % p`（p 取不大于数组大小的质数）
- **直接定址法**：`hash(key) = A * key + B`（适合范围小且连续）
- Java 中 `String.hashCode()`：`s[0]*31^(n-1) + s[1]*31^(n-2) + ... + s[n-1]`

### 负载因子

**负载因子 = 已存元素数 / 数组容量**

负载因子越大，冲突率越高。当负载因子超过阈值时需要**扩容**（rehash）——HashMap 的默认阈值是 **0.75**。

### 冲突解决方案一：闭散列（开放定址法）

发生冲突时，在数组中寻找下一个空位置：

**线性探测**：从冲突位置开始，依次往后找空位（+1, +2, +3...）
- 缺点：冲突的元素容易聚集在一起

**二次探测**：按 +1², -1², +2², -2²... 的步长寻找
- 改善了聚集问题

闭散列的缺陷：空间利用率低——需要保证负载因子 < 0.7。

### 冲突解决方案二：开散列（哈希桶/链地址法）

**Java HashMap 使用的方案**。

数组每个位置存的不是元素本身，而是一个**链表的头节点**。所有映射到同一位置的元素都挂在该位置的链表上：

```
数组下标:  0    1    2    3    4    5    6    7
           |    |    |    |    |    |    |    |
          null null [12] null [4]  [25] null [7]
                     |         |
                    [22]      [14]
                               |
                              [34]
```

查找时：先通过哈希函数定位到数组下标，再在链表中遍历比较。

**JDK 8 的优化**：当链表长度超过 8 且数组容量 ≥ 64 时，链表自动转为**红黑树**——将最坏情况从 O(n) 优化为 O(log n)。

### HashMap 中 hashCode 和 equals 的协作

```
1. 计算 hashCode → 确定数组下标（桶位置）
2. 在桶内遍历，用 equals 比较 Key 是否真正相等
```

这就是为什么**自定义类型作为 HashMap 的 Key 时，必须同时覆写 hashCode 和 equals**：

- 只覆写 equals 不覆写 hashCode → 内容相同的两个对象可能落入不同的桶，永远找不到
- 只覆写 hashCode 不覆写 equals → 不同对象可能落入同一个桶，但 equals 比较为 false，也找不到

**约定**：`equals` 相等的两个对象，`hashCode` 必须相等；反之不要求。

## 实战：统计单词出现次数

```java
public static Map<String, Integer> wordCount(String text) {
    Map<String, Integer> map = new HashMap<>();
    String[] words = text.split("\\s+");
    for (String word : words) {
        // getOrDefault 避免了 null 判断
        map.put(word, map.getOrDefault(word, 0) + 1);
    }
    return map;
}
```

## 实战：找到第一个不重复的字符

```java
// LeetCode #387
public int firstUniqChar(String s) {
    Map<Character, Integer> countMap = new HashMap<>();
    for (char c : s.toCharArray()) {
        countMap.put(c, countMap.getOrDefault(c, 0) + 1);
    }
    for (int i = 0; i < s.length(); i++) {
        if (countMap.get(s.charAt(i)) == 1) {
            return i;
        }
    }
    return -1;
}
```

## 选型决策

```
需要存键值对吗？
├── 是 → 需要按 Key 有序吗？
│         ├── 是 → TreeMap
│         └── 否 → HashMap（绝大多数场景）
└── 否 → 需要去重吗？
          ├── 是 → 需要有序吗？
          │         ├── 是 → TreeSet
          │         └── 否 → HashSet
          └── 否 → 用 List
```

**实际开发建议**：

- 80% 的场景用 **HashMap** / **HashSet**——O(1) 性能，够用
- 需要有序遍历时用 **TreeMap** / **TreeSet**——比如实现范围查询、有序输出
- 需要保持插入顺序时用 **LinkedHashMap** / **LinkedHashSet**

## 小结

| 概念 | 核心要点 |
|------|---------|
| BST | 左 < 根 < 右，中序遍历有序，可能退化 |
| 红黑树 | 自平衡 BST，保证 O(log n)，TreeMap/TreeSet 底层 |
| 哈希表 | 哈希函数 + 冲突解决，O(1) 查找，HashMap/HashSet 底层 |
| 负载因子 | 元素数/容量，HashMap 默认 0.75 时扩容 |
| Map | Key-Value 模型，Key 唯一 |
| Set | 纯 Key 模型，天然去重 |

**关键认知**：

- HashMap 的 O(1) 是均摊的——极端情况下（所有元素都在同一个桶）退化为 O(n)，JDK 8 用红黑树兜底到 O(log n)
- 自定义类型作为 Key：Hash 系列需要 `hashCode` + `equals`，Tree 系列需要 `Comparable` 或 `Comparator`
- HashSet/HashMap 不保证顺序——如果需要插入顺序用 LinkedHashSet/LinkedHashMap
- TreeMap/TreeSet 的"有序"是按 Key 的自然序或比较器序，不是插入顺序
