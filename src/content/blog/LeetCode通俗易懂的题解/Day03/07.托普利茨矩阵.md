## 题目入口

题目链接：[托普利茨矩阵](https://leetcode.cn/problems/toeplitz-matrix/)

## 一句话读懂题目

给你一个 `m × n` 的矩阵，判断它是不是托普利茨矩阵。

托普利茨矩阵的定义：每条从左上到右下的对角线上，所有元素都相同。

比如：

```text
matrix = [[1, 2, 3, 4],
          [5, 1, 2, 3],
          [9, 5, 1, 2]]
```

每条对角线：

```text
对角线 1：[1, 1, 1] ✓
对角线 2：[2, 2, 2] ✓
对角线 3：[3, 3] ✓
对角线 4：[4] ✓
对角线 5：[5, 5] ✓
对角线 6：[9] ✓
```

所有对角线元素都相同，返回 `true`。

再比如：

```text
matrix = [[1, 2],
          [2, 2]]
```

对角线 `[1, 2]`，元素不同，返回 `false`。

## 先用生活例子理解

想象你站在一栋楼梯形的建筑前，每层楼梯的台阶上贴着数字。

托普利茨矩阵就像是：每条斜向下的楼梯线上，所有台阶贴的数字都一样。

你不需要一条条对角线去检查——只需要看每个台阶和它"左上方"那个台阶是不是同一个数字就行了。

如果每个位置都和自己左上角相等，整条对角线自然就全部相等。

## 核心思路

对于同一条对角线上的相邻两个位置 `(i, j)` 和 `(i-1, j-1)`，如果它们相等，那这条对角线上的元素就满足条件。

所以只需要遍历矩阵（从第 1 行第 1 列开始），检查每个元素是否等于它的左上角元素：

```text
如果 matrix[i][j] != matrix[i-1][j-1]，返回 false
```

全部检查通过，返回 `true`。

## 为什么"只比较左上角"就够了

对角线上的元素是：`a₁, a₂, a₃, ..., aₖ`。

如果 `a₁ == a₂`，`a₂ == a₃`，...，`aₖ₋₁ == aₖ`，由等式的传递性，整条对角线的元素都相等。

而"比较左上角"正好覆盖了所有相邻对：`(i, j)` 和 `(i-1, j-1)` 就是对角线上相邻的两个元素。

再看一个反例：

```text

```

检查 `(1, 1)`：值是 `2`，左上角 `(0, 0)` 是 `1`，不相等，返回 `false`。

## Java 代码

先记住一句话：

> 每个格子和它左上角比一比，有不同就不是托普利茨。

```java
class Solution {
    public boolean isToeplitzMatrix(int[][] matrix) {
        int m = matrix.length;
        int n = matrix[0].length;

        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (matrix[i][j] != matrix[i - 1][j - 1]) {
                    return false;
                }
            }
        }

        return true;
    }
}
```

## 复杂度分析

时间复杂度：`O(m × n)`。

每个元素最多检查一次。

空间复杂度：`O(1)`。

只用了几个变量。

## 常见坑

### 1. 从 (0, 0) 开始遍历

如果从 `i = 0` 或 `j = 0` 开始，访问 `matrix[i-1][j-1]` 会数组越界。

必须从 `i = 1, j = 1` 开始。

### 2. 试图一条条对角线去遍历

虽然可以按对角线遍历，但代码会比较复杂（需要处理从第一行出发的对角线和从第一列出发的对角线）。

"比较左上角"的方法一个双层循环就搞定，简洁得多。

### 3. 混淆对角线方向

托普利茨矩阵检查的是"左上到右下"方向的对角线，不是"右上到左下"。

比较的是 `(i-1, j-1)` 不是 `(i-1, j+1)`。

## 面试时怎么讲

### 30 秒版

> 托普利茨矩阵要求每条左上到右下的对角线上元素相同。只需要检查每个元素是否等于它的左上角邻居，利用等式传递性就能覆盖整条对角线。从 `(1,1)` 开始遍历，有不等就返回 `false`。时间 `O(m*n)`，空间 `O(1)`。

### 面试官可能追问

**如果矩阵很大，一行一行从磁盘加载（只能同时持有两行），怎么办？**

只需要同时持有相邻两行。加载第 `i` 行后，和第 `i-1` 行比较，比完后丢弃第 `i-1` 行，加载第 `i+1` 行，以此类推。空间只需要 `O(n)`（两行）。

**如果矩阵按列分块存储呢？**

类似思路，按列加载时只需要同时持有相邻两列。但按列比较左上角就需要前一列的上一个元素，所以需要稍微调整遍历顺序。

**能不能用哈希表按对角线分组？**

可以。同一条对角线上所有元素的 `i - j` 值相同。用 HashMap 以 `i - j` 为 key 存第一个值，后续对比。但空间变成 `O(m + n)`，不如直接比左上角。

## 最后总结

托普利茨矩阵判断是矩阵对角线性质的入门题：

> 整条对角线相等 ⇔ 相邻对角线位置相等 ⇔ 每个格子等于它的左上角。

代码模板：

```text
从 (1, 1) 开始遍历
    如果 matrix[i][j] != matrix[i-1][j-1]
        返回 false
返回 true
```

最后把这句话记住：

> 不需要逐条对角线检查——只需要每个元素看一眼左上角，传递性帮你搞定剩下的。
