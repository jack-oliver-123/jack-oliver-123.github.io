## 题目入口

题目链接：[旋转图像](https://leetcode.cn/problems/rotate-image/)

## 一句话读懂题目

给你一个 `n × n` 的方阵，把它顺时针旋转 90°。

要求原地操作，不能使用额外矩阵。

比如：

```text
matrix = [[1, 2, 3],
          [4, 5, 6],
          [7, 8, 9]]
```

顺时针旋转 90° 后：

```text
[[7, 4, 1],
 [8, 5, 2],
 [9, 6, 3]]
```

## 先用生活例子理解

想象你拿着一张正方形的照片，把它顺时针转 90°。

原来左上角的内容跑到了右上角，原来右上角跑到了右下角……

直接"转"不好在代码里表达，但可以拆成两个简单操作：

1. 先沿对角线翻折（转置）；
2. 再左右镜像翻转。

两步组合起来，效果等同于顺时针转 90°。

## 为什么"转置 + 左右翻转"等于顺时针旋转

假设原坐标是 `(i, j)`，顺时针旋转 90° 后新坐标是 `(j, n-1-i)`。

拆解这个变换：

```text
(i, j) → 转置 → (j, i) → 左右翻转 → (j, n-1-i)
```

正好是旋转后的目标位置。

## 手动走一遍例子

```text
matrix = [[1, 2, 3],
          [4, 5, 6],
          [7, 8, 9]]
```

### 第一步：转置（沿主对角线翻折）

交换 `matrix[i][j]` 和 `matrix[j][i]`（只遍历上三角）：

```text
[[1, 4, 7],
 [2, 5, 8],
 [3, 6, 9]]
```

### 第二步：每行左右翻转

```text
[[7, 4, 1],
 [8, 5, 2],
 [9, 6, 3]]
```

结果正确。

压缩成一张表：

| 步骤 | 操作 | 矩阵状态 |
| --- | --- | --- |
| 初始 | - | `[[1,2,3],[4,5,6],[7,8,9]]` |
| 1 | 转置 | `[[1,4,7],[2,5,8],[3,6,9]]` |
| 2 | 每行左右翻转 | `[[7,4,1],[8,5,2],[9,6,3]]` |

## Java 代码

先记住一句话：

> 先转置，再左右翻，两步搞定旋转。

```java
class Solution {
    public void rotate(int[][] matrix) {
        int n = matrix.length;

        // 第一步：转置
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                int temp = matrix[i][j];
                matrix[i][j] = matrix[j][i];
                matrix[j][i] = temp;
            }
        }

        // 第二步：每行左右翻转
        for (int i = 0; i < n; i++) {
            int left = 0, right = n - 1;
            while (left < right) {
                int temp = matrix[i][left];
                matrix[i][left] = matrix[i][right];
                matrix[i][right] = temp;
                left++;
                right--;
            }
        }
    }
}
```

## 代码逐行拆解

```java
int n = matrix.length;
```

方阵的边长。

```java
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        int temp = matrix[i][j];
        matrix[i][j] = matrix[j][i];
        matrix[j][i] = temp;
    }
}
```

转置：只遍历上三角（`j > i`），交换对称位置的元素。

如果遍历整个矩阵会交换两次，等于没交换。

```java
for (int i = 0; i < n; i++) {
    int left = 0, right = n - 1;
    while (left < right) {
        int temp = matrix[i][left];
        matrix[i][left] = matrix[i][right];
        matrix[i][right] = temp;
        left++;
        right--;
    }
}
```

每行用双指针从两端向中间交换，实现左右翻转。

## 其他旋转方向

如果要逆时针旋转 90°（等价于顺时针旋转 270°）：

- 方法一：转置 + 上下翻转；
- 方法二：上下翻转 + 转置。

如果要旋转 180°：

- 上下翻转 + 左右翻转（或做两次 90° 旋转）。

## 复杂度分析

时间复杂度：`O(n²)`。

转置遍历上三角 `n*(n-1)/2` 次，翻转遍历每行 `n/2` 次，总共 `O(n²)`。

空间复杂度：`O(1)`。

全程原地操作，只用了一个临时变量。

## 常见坑

### 1. 转置时遍历整个矩阵

如果写成 `for j = 0 to n-1`，每对元素会被交换两次，结果还是原矩阵。

必须只遍历上三角：`j` 从 `i + 1` 开始。

### 2. 混淆旋转和转置

转置不等于旋转。转置后还需要翻转一步才是旋转。

```text
转置：(i, j) → (j, i)
顺时针 90°：(i, j) → (j, n-1-i)
```

### 3. 对非方阵使用这个方法

这道题保证是方阵。非方阵无法原地旋转，因为行列数会变。

### 4. 翻转方向搞反

顺时针 90° = 转置 + 左右翻转。

如果做了"转置 + 上下翻转"，得到的是逆时针 90°。

## 面试时怎么讲

### 30 秒版

> 顺时针旋转 90° 可以拆成两步原地操作：先沿主对角线转置，再每行左右翻转。转置把 `(i, j)` 变成 `(j, i)`，翻转再把列从 `i` 变成 `n-1-i`，组合起来就是旋转的目标坐标 `(j, n-1-i)`。时间 `O(n²)`，空间 `O(1)`。

### 面试官可能追问

**能不能一步到位，直接四个位置轮换？**

可以。每次把四个位置 `(i,j)`, `(j,n-1-i)`, `(n-1-i,n-1-j)`, `(n-1-j,i)` 循环赋值。代码更紧凑但更难理解：

```java
for (int i = 0; i < n / 2; i++) {
    for (int j = i; j < n - 1 - i; j++) {
        int temp = matrix[i][j];
        matrix[i][j] = matrix[n-1-j][i];
        matrix[n-1-j][i] = matrix[n-1-i][n-1-j];
        matrix[n-1-i][n-1-j] = matrix[j][n-1-i];
        matrix[j][n-1-i] = temp;
    }
}
```

面试中推荐"转置 + 翻转"，更清晰不容易写错。

## 最后总结

旋转图像这道题，拆解是关键：

> 一个复杂操作（旋转）拆成两个简单操作（转置 + 翻转）。

代码模板：

```text
转置：遍历上三角，交换 mat[i][j] 和 mat[j][i]
翻转：每行双指针左右交换
```

最后把这句话记住：

> 顺时针转 90° = 先转置再左右翻。记住这个拆解，任何方向的旋转都能组合出来。
