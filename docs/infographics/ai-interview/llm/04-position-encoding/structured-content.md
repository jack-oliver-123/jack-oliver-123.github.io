# 4. 大模型的位置编码是干什么用的？sin/cos、RoPE、ALiBi 有什么区别？

## Source Content (Verbatim)

## 60 秒回答

自注意力若不加入位置信息，对输入排列本身不敏感。位置编码让模型区分 Token 的顺序和相对距离。正弦/余弦编码把固定位置向量加到词表示；旋转位置编码（Rotary Position Embedding，RoPE）按位置旋转 Query 和 Key，使点积包含相对位置信息；线性偏置注意力（Attention with Linear Biases，ALiBi）直接给注意力分数加入随距离变化的头级偏置。

三者都不能保证模型在训练长度之外无损工作。长度外推还受训练数据、注意力模式、数值精度和位置缩放方法影响，必须用长文本检索、推理和困惑度等任务实测有效上下文。

## 详细解析

正弦编码为不同频率定义确定性函数：

$$
PE(pos,2i)=\sin(pos/10000^{2i/d}),\quad
PE(pos,2i+1)=\cos(pos/10000^{2i/d})
$$

它不增加可训练位置表，并允许模型从频率组合中学习距离关系，但“公式可计算任意位置”不代表模型学会处理任意长度。

RoPE 将每对特征视作二维向量，并按位置相关角度旋转 $Q$ 与 $K$。两个位置的点积与角度差有关，因此自然表达相对位置。长上下文扩展常调整 RoPE 频率，例如位置插值或 YaRN；这些方法会改变短程分辨率与远程外推的取舍。

ALiBi 不修改 Token 表示，而是在第 $h$ 个头的注意力分数中加入距离偏置：

$$
score_{i,j}^{(h)}=\frac{q_i k_j^\top}{\sqrt{d_k}}+M_{i,j}^{causal}-m_h(i-j),\quad j\le i
$$

不同头使用不同斜率。最后一项是 ALiBi 新增的距离偏置；缩放点积和因果掩码仍然存在。它实现直接且不需要位置向量，但偏置形状也是归纳假设，并不覆盖所有长程任务。

| 方法 | 注入位置 | 额外参数 | 相对距离表达 |
|---|---|---:|---|
| 正弦编码 | 输入隐藏状态 | 0 | 由模型学习 |
| RoPE | Query/Key | 0 | 由旋转点积直接体现 |
| ALiBi | 注意力分数 | 0 | 线性距离偏置 |

## 工程实践与边界

扩展上下文前先确认模型配置中的基频、缩放策略和训练长度。推理引擎必须与模型配置一致，否则即使张量形状合法也会产生质量退化。不要只用“针藏在干草堆”单点命中率评估；还应覆盖多证据聚合、顺序敏感、长文摘要和干扰项。

上下文窗口越长，KV Cache 和 Prefill 成本越高。产品设计还要配合分块检索、摘要或缓存，避免把所有历史无筛选地塞进 Prompt。

## 常见误区

- **认为注意力自带顺序**：没有位置机制时，自注意力本身无法区分排列
- **认为可计算位置等于可泛化位置**：公式定义域与训练后的有效上下文不是同一概念
- **把 RoPE 叫作可学习位置表**：原始 RoPE 使用确定性旋转频率
- **只看窗口声明**：标称长度不能替代任务级有效长度评测

## 面试追问

**问：RoPE 为什么能表达相对位置？**

**答：** 位置旋转后，$q_i$ 与 $k_j$ 的点积包含由 $i-j$ 决定的旋转项，因此相对距离进入注意力分数。

**问：位置插值有什么代价？**

**答：** 它把更长位置范围压到训练过的角度范围，能改善外推，但可能降低相邻位置的分辨率，通常需要继续训练并做短长任务回归。
