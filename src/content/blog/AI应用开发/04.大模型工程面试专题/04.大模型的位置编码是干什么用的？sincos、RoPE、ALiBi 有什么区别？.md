把一句话写在三张卡片上：

```text
小王 / 帮助 / 小李
```

如果把卡片打乱成：

```text
小李 / 帮助 / 小王
```

词还是这些词，意思却可能相反。不带位置机制和方向 Mask 的裸自注意力，只根据卡片内容做两两匹配，不会天然知道哪张在第一位、哪张在第三位。完整模型需要通过位置机制、因果 Mask 或其他结构获得顺序线索。

位置机制就是给 Token 带上“它在什么位置”以及“它和别的位置相隔多远”的线索。

## 本篇目标

读完本篇，你应该能够：

- 解释为什么没有位置机制的自注意力难以区分顺序；
- 区分 Absolute Position（全局位置编号）、相对位置和注意力偏置；
- 看懂原始 Transformer 的 sin/cos 公式；
- 解释 RoPE 为什么把位置写入 Q/K 旋转；
- 解释 ALiBi 为什么直接修改注意力分数；
- 比较三者是否增加参数、作用位置与推理特性；
- 正确理解训练长度、配置长度和有效上下文长度；
- 回答位置插值、YaRN、长度外推等工程追问。

核心结论：

> sin/cos 把全局位置编号对应的向量加到 Token 表示上；RoPE 按位置旋转 Query 和 Key，使点积自然包含相对位置；ALiBi 给注意力分数加入随距离增大的线性惩罚。三者注入位置的地方不同，长上下文效果也取决于训练分布、缩放方法和任务，不能只看“最大长度”配置。

## 60 秒回答

不带位置机制和方向 Mask 的自注意力主要根据 Token 内容做集合式的两两交互，无法可靠区分“甲打乙”和“乙打甲”这类顺序变化。因果 Mask 本身能提供“只能看过去”的方向线索；sin/cos、RoPE、ALiBi 等位置机制则进一步提供全局位置或相对距离归纳偏置，也是当前主流 Transformer 的常见设计。

原始 sin/cos 位置编码为每个全局位置编号生成不同频率的正弦和余弦向量，再与 Token Embedding 相加。它没有可训练位置参数，能提供确定性的多频率位置信号。

RoPE 不把位置向量直接加到隐藏状态，而是按位置对每个注意力头的 Q、K 二维子空间做旋转。两个旋转后向量的点积依赖位置差，因此把相对位置信息写进注意力分数。很多 Decoder LLM 使用 RoPE。

ALiBi 则不生成位置向量，而是在注意力 logits 上加入与 Query-Key 距离成比例的负偏置。距离越远，先验惩罚通常越大；不同头使用不同斜率。

三者都不能单独保证无限外推。超过训练长度后，质量受频率分布、缩放、训练数据、注意力模式和评测任务影响。生产中要通过真实长文本检索、推理和困惑度测试验证有效上下文，而不是只看配置值。

## 详细解析

### 1. 为什么注意力需要位置

先忽略位置编码。自注意力为输入 $X$ 计算：

$$
Q=XW_Q,\quad K=XW_K,\quad V=XW_V
$$

$$
\operatorname{Attention}(X)
=
\operatorname{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
$$

如果用同一个置换矩阵 $P$ 打乱输入位置：

$$
X'=PX
$$

那么自注意力输出也会按同样方式被置换。直观上，它知道“有哪些内容、内容之间多相关”，却没有独立坐标告诉它“谁在前、谁在后”。

这叫置换等变性。它不是说注意力完全无用，而是说顺序必须由位置机制、掩码或其他结构注入。

因果掩码确实提供“只能看过去”的方向信息，一些无显式位置编码的研究模型也能据此学习部分顺序规律。主流架构通常仍加入更细粒度的距离或位置机制，以提供更明确的归纳偏置；其必要性和外推效果应结合具体架构与训练结果讨论。

### 2. 三个层次的位置信息

#### Absolute Position：全局位置编号

回答“这个 Token 在序列第几个位置”。

```text
位置：0, 1, 2, 3, ...
```

学习式位置 Embedding 和原始 sin/cos 都可以提供全局坐标。

#### 相对位置

回答“Query 与 Key 相差多少位置”。

```text
Query 在 10，Key 在 7 → 相对距离 -3
```

语言关系往往与距离相关，例如相邻词组合、远距离指代。

#### 位置偏置

它未必生成一个位置向量，而是直接影响注意力分数，让某些距离更容易或更难被关注。ALiBi 属于这条思路。

### 3. 学习式全局位置 Embedding

先介绍一个对照方案。为每个位置维护一行可训练向量：

$$
H_t=E_{token}(x_t)+E_{pos}(t)
$$

其中：

- $E_{token}(x_t)$：第 $t$ 个 Token 的词向量；
- $E_{pos}(t)$：位置 $t$ 的可训练向量；
- $H_t$：送入 Transformer 的表示。

优点是简单，模型可自行学习每个位置。局限是训练中没更新过的位置没有可靠向量，最大表长也固定。BERT、早期 GPT 等模型使用过学习式全局位置表示。

### 4. sin/cos 位置编码公式

原始 Transformer 定义：

$$
PE(pos,2i)
=
\sin\left(\frac{pos}{10000^{2i/d_{model}}}\right)
$$

$$
PE(pos,2i+1)
=
\cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)
$$

逐个解释：

- $pos$：Token 的全局位置编号，通常从 0 开始；
- $i$：频率索引；
- $2i$：偶数维；
- $2i+1$：与它配对的奇数维；
- $d_{model}$：隐藏向量总维度；
- $10000$：原论文选择的频率基数；
- $PE(pos,\cdot)$：位置 $pos$ 的位置向量。

不同维度具有不同波长：

- 高频维度随位置快速变化，便于分辨近邻；
- 低频维度变化较慢，提供长距离信号。

最后与 Token Embedding 相加：

$$
H_{pos}=E_{token}+PE(pos)
$$

### 5. 为什么 sin 与 cos 成对

利用三角恒等式，位置平移可以表示为原位置 sin/cos 的线性组合：

$$
\sin(a+b)=\sin a\cos b+\cos a\sin b
$$

$$
\cos(a+b)=\cos a\cos b-\sin a\sin b
$$

因此对固定偏移 $k$，$PE(pos+k)$ 与 $PE(pos)$ 存在线性关系。这为模型学习相对偏移提供了有用结构。

这不表示模型会完美学会任意距离，也不表示 sin/cos 自动支持无限上下文。

### 6. sin/cos 的优点与边界

优点：

- 不增加可训练位置参数；
- 任意整数位置都能按公式计算；
- 频率多尺度；
- 实现简单。

边界：

- 位置信息在输入层与 Token 表示相加，后续由网络自行利用；
- 超过训练分布的相位组合可能陌生；
- “公式可算到更远位置”不等于模型在更远位置有效；
- 与现代 Decoder 生态中的 RoPE 优化方案不完全兼容。

### 7. RoPE 的核心直觉：按位置旋转

RoPE 是 Rotary Position Embedding。把向量每两个维度看作一个二维平面。

二维旋转矩阵为：

$$
R(\theta)=
\begin{bmatrix}
\cos\theta & -\sin\theta\\
\sin\theta & \cos\theta
\end{bmatrix}
$$

位置 $m$ 的 Query 子向量 $q$ 变为：

$$
q'_m=R(m\theta)q
$$

位置 $n$ 的 Key 子向量 $k$ 变为：

$$
k'_n=R(n\theta)k
$$

其中：

- $m,n$：Query 和 Key 的位置；
- $\theta$：该二维维度对的基础频率；
- $R(m\theta)$：按位置决定的旋转；
- $q'_m,k'_n$：加入位置后的 Q、K。

### 8. RoPE 为什么表达相对位置

旋转矩阵满足：

$$
R(a)^\top R(b)=R(b-a)
$$

因此旋转后点积：

$$
(q'_m)^\top k'_n
=
q^\top R(m\theta)^\top R(n\theta)k
=
q^\top R((n-m)\theta)k
$$

结果依赖 $n-m$，也就是相对位置差。

这就是 RoPE 最常被问的核心：

> 它把全局位置编号用于旋转 Q/K，但注意力点积自然呈现相对位置关系。

### 9. RoPE 在高维中怎样工作

若头维度为偶数，把它拆成多个二维对。第 $i$ 对使用不同频率，例如常见形式：

$$
\theta_i=base^{-2i/d_h}
$$

其中：

- $base$：频率基数，原始方案常见为 10000，但现代模型可使用其他值；
- $i$：维度对索引；
- $d_h$：每个注意力头的维度。

位置 $m$ 的旋转角是 $m\theta_i$。高低频结合让模型表示不同尺度的相对距离。

概念伪代码：

```python
def rotate_pair(x_even, x_odd, cos, sin):
    y_even = x_even * cos - x_odd * sin
    y_odd = x_even * sin + x_odd * cos
    return y_even, y_odd

q = apply_rope(q, position_ids, frequencies)
k = apply_rope(k, position_ids, frequencies)
```

通常旋转 Q 和 K，不旋转 V，因为位置关系通过注意力权重进入 Value 汇总。

### 10. RoPE 的优点与边界

优点：

- 不需要一张最大位置长度的可训练表；
- 直接影响 Q/K 点积；
- 具有清晰的相对位置性质；
- 容易与因果 Decoder 和 KV Cache 结合；
- 已有丰富的长上下文缩放研究。

边界：

- 远超训练长度时，旋转相位和频率分布发生外推；
- 高频维可能在新长度下变化过快；
- 修改 base 或位置缩放会改变模型看到的位置模式；
- 需要匹配训练配方与缓存中的旋转约定；
- 不同模型对 RoPE 维度范围和缩放实现可能不同。

### 11. ALiBi 的核心直觉：距离越远，先验扣分越多

ALiBi 是 Attention with Linear Biases。它不把位置向量加到隐藏状态，而是在注意力分数上加线性偏置。

对第 $h$ 个头：

$$
s_{ij}^{(h)}
=
\frac{q_i^{(h)}\cdot k_j^{(h)}}{\sqrt{d_h}}
-m_h(i-j)
$$

这里以因果注意力中 $j\le i$ 为例：

- $i$：Query 位置；
- $j$：Key 位置；
- $i-j$：向过去看的距离；
- $m_h>0$：第 $h$ 个头的斜率；
- $-m_h(i-j)$：距离越远越负的偏置；
- $s_{ij}^{(h)}$：加入位置偏置后的注意力 logit。

然后再做 Softmax。

### 12. 为什么不同头用不同斜率

如果所有头都使用同一个斜率，它们会共享同样的距离先验。不同斜率允许：

- 一些头强烈偏好局部 Token；
- 一些头对远距离惩罚较弱；
- 多头共同覆盖不同距离尺度。

斜率通常按预先设计的规则设置，未必需要训练位置参数。

### 13. ALiBi 的优点与边界

优点：

- 不需要位置 Embedding 表；
- 直接进入注意力 logits；
- 实现概念简单；
- 原论文展示了训练短、测试长的外推优势；
- 偏置可按距离动态计算。

边界：

- 线性距离惩罚是一种归纳偏置，不保证适合所有任务；
- 远距离内容可能被先验压低，需要内容分数克服偏置；
- 训练分布和模型规模仍影响外推；
- 现有生态中很多 checkpoint 使用 RoPE，不能无训练替换；
- 双向和非因果场景需要明确距离符号与实现。

### 14. 三种方法放在同一张表里

| 维度 | sin/cos | RoPE | ALiBi |
| --- | --- | --- | --- |
| 注入位置 | 与输入 Embedding 相加 | 旋转 Q、K | 加到注意力 logits |
| 主要表示 | 全局位置，多频率 | 点积中的相对位置 | 相对距离线性偏置 |
| 可训练位置参数 | 无 | 通常无 | 通常无 |
| 是否改变 Q/K | 间接，通过隐藏状态 | 直接 | 不改变向量 |
| 是否改变注意力分数 | 间接 | 通过旋转后点积 | 直接加偏置 |
| 超长位置可计算 | 可以 | 可以 | 可以 |
| 超长能力保证 | 不保证 | 不保证 | 不保证 |

“可计算”只表示代码能生成位置值，不表示模型在该长度上质量可靠。

### 15. 什么是长度外推

假设模型训练时最大长度是 4K，测试时输入 16K。如果它仍能有效利用远距离信息，这通常称为长度外推。

需要区分：

- 配置最大长度：接口允许接收多少 Token；
- 显存可容纳长度：硬件是否放得下；
- 位置机制可计算长度：能否产生位置值；
- 训练长度：训练数据真正覆盖到多长；
- 有效上下文长度：任务中能可靠利用多远信息。

前四个满足，也不自动得到第五个。

### 16. RoPE 位置插值是什么

Position Interpolation 的直觉是把新长序列的位置压回训练过的坐标范围。

若原训练长度为 $L_{train}$，目标长度为 $L_{target}$，可以用缩放位置：

$$
pos'=pos\cdot\frac{L_{train}}{L_{target}}
$$

例如从 4K 扩到 16K，比例为 $1/4$。原位置 12,000 映射到 3,000 附近。

这样避免直接进入完全未见过的超大相位，但相邻位置变得更接近，可能影响局部位置分辨率。通常还要做长上下文继续训练或微调。

### 17. YaRN 等缩放方法解决什么

后续方法尝试对不同 RoPE 频率采取更细致的缩放，并结合训练和推理调整，以兼顾局部分辨率与远距离范围。

面试中不必把所有公式背下来，但应知道：

- 统一线性缩放会改变所有频率；
- 不同频率对外推和局部精度的影响不同；
- 长上下文扩展不是只修改 `max_position_embeddings`；
- 最终必须重新训练或至少适配并评测。

### 18. 位置 ID 与 KV Cache 的关系

在 RoPE 模型中，K 在写入 KV Cache 前通常已经按位置旋转。增量解码必须为新 Token 提供正确位置 ID。

常见错误包括：

- 忽略左侧 Padding 导致位置偏移；
- Prefix Cache 复用时位置对不上；
- 滑动窗口裁剪后错误重置位置；
- 不同并行分片使用不一致的 RoPE 参数；
- 训练和推理使用不同 scaling 配置。

这些问题可能不会立刻报错，却会降低输出质量。

### 19. 一个直观数值例子

假设 ALiBi 某个头斜率为 $m=0.1$，当前 Query 位于 10：

| Key 位置 | 距离 | 位置偏置 |
| ---: | ---: | ---: |
| 10 | 0 | 0.0 |
| 9 | 1 | -0.1 |
| 5 | 5 | -0.5 |
| 0 | 10 | -1.0 |

如果内容点积分数分别为 `[1.0, 1.2, 1.8, 2.1]`，加偏置后变成：

```text
[1.0, 1.1, 1.3, 1.1]
```

远处 Token 并非不能被关注，只是要有更高内容相关性来克服距离惩罚。

## 工程实践与边界

### 1. 不要在推理时任意替换位置机制

位置机制参与了训练。把 RoPE 模型改成 ALiBi，或随意修改 RoPE base，都会改变注意力分布。除非有明确转换方案和继续训练，否则不能当作无损配置开关。

### 2. 长上下文扩展要形成完整闭环

至少包括：

1. 选择位置缩放方案；
2. 准备足够长且高质量的数据；
3. 控制长短样本比例，避免短任务退化；
4. 继续训练或微调；
5. 测困惑度、检索、推理与生成；
6. 测 KV Cache、吞吐、延迟和 OOM；
7. 与原长度能力做回归比较。

### 3. 不要只做 Needle-in-a-Haystack

“大海捞针”可以检查模型能否找回一段明显信息，但不能覆盖：

- 多跳整合；
- 多个相似干扰项；
- 长文档摘要；
- 顺序与时间推理；
- 代码仓库跨文件依赖；
- 长对话指令保持；
- 不同位置和深度的稳定性。

应组合多类评测，并检查不同上下文位置。

### 4. 记录完整位置配置

模型发布和服务日志至少记录：

- 位置编码类型；
- RoPE base；
- scaling 类型和倍率；
- 原训练长度；
- 继续训练长度；
- 推理最大长度；
- 滑动窗口；
- Tokenizer 版本；
- 推理引擎版本。

只记录“支持 128K”不足以复现。

### 5. 评估成本随长度增长

长上下文不仅影响注意力计算，还影响：

- Prefill 延迟；
- KV Cache 容量；
- 可并发请求数；
- 调度和分页；
- 输入 Token 成本；
- 用户等待时间。

业务可能更适合检索、摘要分层或滑动窗口，而不是把所有历史内容全部塞入 Prompt。

### 6. 防止位置相关数据泄漏

长上下文评测材料如果进入训练数据，模型可能靠记忆而不是长距离利用完成任务。需要做好去重、污染检查和私有评测集管理。

### 7. 用 profiler 区分能力与系统瓶颈

模型在 128K 上答错，可能是位置外推失败，也可能是 Prompt 截断、Tokenizer 计数错误、KV Cache 精度、服务引擎 bug 或检索材料放错位置。先核对系统链路，再归因于位置编码。

## 常见误区

### 误区一：自注意力完全不包含任何顺序

裸的内容注意力没有全局坐标，但因果 Mask 已提供方向约束，完整模型还会加入位置机制。回答时要区分组件和系统。

### 误区二：sin/cos 能计算任意位置，所以能无限外推

错误。新位置的数值可计算，不代表模型训练过这些频率组合或能稳定利用远距离内容。

### 误区三：RoPE 是把位置向量加到 Embedding

错误。典型 RoPE 直接旋转 Q 和 K。

### 误区四：ALiBi 会禁止关注远处 Token

错误。它通常施加距离惩罚，不是硬 Mask；足够高的内容相关分数仍可能获得注意力。

### 误区五：RoPE 只表达全局位置编号

不完整。旋转角由全局位置编号决定，但旋转后 Q/K 点积依赖相对位置差，这是它的重要性质。

### 误区六：把最大长度配置改大就完成扩展

错误。还需要位置缩放、数据适配、训练、系统容量和有效上下文评测。

### 误区七：一种位置编码在所有任务上都更好

错误。结论受模型规模、训练长度、数据、架构、任务和系统实现影响。

### 误区八：支持长上下文就应该把所有资料全部输入

错误。更多无关内容可能增加成本和干扰。检索、压缩和分层处理仍然重要。

## 面试追问

### 问：为什么没有位置编码的 Attention 难以区分“我爱你”和“你爱我”

答：内容投影本身对输入位置没有独立坐标，交换 Token 会使输出跟着置换，无法表达同一组词的全局顺序差异。需要位置机制或结构约束。

### 问：RoPE 为什么只旋转 Q/K，不旋转 V

答：位置关系用于决定“从哪里读取”，通过 Q/K 点积进入注意力权重；权重再汇总 V。典型实现无需旋转 V，但研究中可能有其他变体。

### 问：RoPE 的相对位置性质怎样推导

答：利用 $R(a)^TR(b)=R(b-a)$，位置 $m$ 的 Q 与位置 $n$ 的 K 点积可化为包含 $n-m$ 的旋转，因此依赖相对差。

### 问：ALiBi 有可训练参数吗

答：原始方案的头斜率按规则预设，通常不作为可训练位置 Embedding；具体实现可能有变体，回答时应说明所指版本。

### 延伸追问速答

| 追问 | 回答要点 |
| --- | --- |
| 位置插值为什么需要继续训练 | 缩放改变相邻位置和频率分布，需要让模型适应新坐标并学习利用长样本 |
| 训练长度和有效上下文长度有什么区别 | 前者描述训练覆盖；后者描述特定任务和质量标准下可可靠利用的信息距离，需要实验测量 |
| 如何评测位置编码外推 | 在多长度、多位置上测困惑度、检索、多跳推理、摘要和代码依赖，并比较原长度能力 |
| RoPE 与 KV Cache 为何易出现位置 bug | 缓存 K 已带位置旋转；复用前缀、左 Padding 或滑窗时 position ID 不一致会使注意力错位 |
