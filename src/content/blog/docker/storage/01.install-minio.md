## 适用场景

MinIO 是兼容 S3 协议的对象存储服务，适合保存图片、文档、备份文件和其他非结构化数据。本文记录如何使用 Docker Compose 部署单节点 MinIO。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建工作目录：

```bash
# Linux
mkdir -p /opt/minio

# Windows（PowerShell）
mkdir D:/minio-env
```

## 编写 docker-compose

在工作目录下新建 `docker-compose.yml`：

```yaml
services:
  minio:
    image: minio/minio:RELEASE.2025-04-22T22-12-26Z
    container_name: minio-local
    restart: unless-stopped
    ports:
      - "9000:9000"
      - "9001:9001"
    volumes:
      - ./data:/data
      - ./config:/root/.minio
    environment:
      - MINIO_ROOT_USER=your_minio_user
      - MINIO_ROOT_PASSWORD=your_minio_password
      - TZ=Asia/Shanghai
    command: server /data --console-address ':9001'
    healthcheck:
      test: ["CMD-SHELL", "curl -fs http://localhost:9000/minio/health/live || exit 1"]
      interval: 10s
      timeout: 5s
      retries: 3
```

端口说明：

- `9000`：S3 API 访问端口；
- `9001`：Web 控制台端口。

> ⚠️ `MINIO_ROOT_PASSWORD` 必须至少 8 个字符，否则容器启动会失败。

## 启动 MinIO

```bash
docker compose up -d
```

查看日志：

```bash
docker compose logs -f
```

## 验证安装

查看容器状态：

```bash
docker compose ps
```

打开浏览器访问控制台：

```text
http://localhost:9001
```

使用 compose 中配置的 `MINIO_ROOT_USER` 和 `MINIO_ROOT_PASSWORD` 登录。

## 创建 Bucket

登录控制台后，点击 **Create Bucket**，例如创建：

```text
app-files
```

也可以使用 MinIO 命令行客户端 `mc`：

```bash
# 安装 mc（Linux）
curl -O https://dl.min.io/client/mc/release/linux-amd64/mc
chmod +x mc

# 设置别名
./mc alias set local http://localhost:9000 your_minio_user your_minio_password

# 创建 Bucket
./mc mb local/app-files

# 上传文件
./mc cp myfile.txt local/app-files/
```

## 常见问题

### 控制台打不开

检查：

- `9001` 端口是否映射；
- 服务器防火墙或安全组是否放行；
- 容器是否正常运行（`docker compose ps`）。

### API 访问失败

确认客户端连接的是 `9000` 端口（S3 API），而不是控制台 `9001` 端口。

### 密码不满足要求

`MINIO_ROOT_PASSWORD` 必须不少于 8 个字符，否则容器会报错退出。查看日志确认：

```bash
docker compose logs minio
```

## 停止和清理

```bash
# 停止服务（保留数据）
docker compose down

# 停止并删除数据
docker compose down -v
rm -rf ./data
```

## 后续建议

- 生产环境不要使用弱密码。
- 重要数据需要规划备份策略。
- 生产环境建议配置 HTTPS（通过反向代理如 Nginx）。
- 多节点和纠删码模式需要单独规划磁盘和节点数量。
