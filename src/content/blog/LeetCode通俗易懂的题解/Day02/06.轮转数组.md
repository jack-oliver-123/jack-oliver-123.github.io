## 题目入口

题目链接：[轮转数组](https://leetcode.cn/problems/rotate-array/)

## 一句话读懂题目

给你一个数组 `nums`，把它向右轮转 `k` 个位置。

也就是说，把数组末尾的 `k` 个元素搬到开头。

比如：

```text
nums = [1, 2, 3, 4, 5, 6, 7]
k = 3
```

向右轮转 3 步：

```text
[5, 6, 7, 1, 2, 3, 4]
```

末尾的 `[5, 6, 7]` 跑到了前面。

要求原地操作，不能返回新数组。

## 先用生活例子理解

想象你手里有一排扑克牌：`1 2 3 4 5 6 7`。

要把最后 3 张牌挪到最前面，怎么做？

一种巧妙的做法：

```text
第一步：把整副牌翻转 -> 7 6 5 4 3 2 1
第二步：把前 3 张翻转 -> 5 6 7 4 3 2 1
第三步：把后 4 张翻转 -> 5 6 7 1 2 3 4
```

三次翻转，每次都是原地操作，不需要额外空间。

这就是三次翻转法。

## 为什么三次翻转能得到正确结果

假设数组是 `[A | B]`，其中 `A` 是前 `n-k` 个元素，`B` 是后 `k` 个元素。

目标是得到 `[B | A]`。

```text
原数组：      A    B
全部翻转：    B'   A'    （B 和 A 各自的元素顺序反了）
翻转前 k 个：  B    A'    （B 恢复正序）
翻转后 n-k 个：B    A     （A 也恢复正序）
```

三次翻转后就是 `[B | A]`，正好是轮转的结果。

## 一个容易忽略的细节：k 可能大于数组长度

比如：

```text
nums = [1, 2, 3]
k = 5
```

轮转 5 步等价于轮转 `5 % 3 = 2` 步。

因为轮转 `n` 步后数组回到原样。

所以第一步要做：

```text
k = k % nums.length
```

## 手动走一遍例子

```text
nums = [1, 2, 3, 4, 5, 6, 7]
k = 3
```

先取模：`k = 3 % 7 = 3`。

### 第一步：翻转整个数组

```text
[1, 2, 3, 4, 5, 6, 7] -> [7, 6, 5, 4, 3, 2, 1]
```

### 第二步：翻转前 k = 3 个元素

```text
[7, 6, 5, 4, 3, 2, 1] -> [5, 6, 7, 4, 3, 2, 1]
```

### 第三步：翻转后 n - k = 4 个元素

```text
[5, 6, 7, 4, 3, 2, 1] -> [5, 6, 7, 1, 2, 3, 4]
```

结果正确。

压缩成一张表：

| 步骤 | 操作 | 数组状态 |
| --- | --- | --- |
| 初始 | - | `[1, 2, 3, 4, 5, 6, 7]` |
| 1 | 翻转全部 `[0, 6]` | `[7, 6, 5, 4, 3, 2, 1]` |
| 2 | 翻转前 k 个 `[0, 2]` | `[5, 6, 7, 4, 3, 2, 1]` |
| 3 | 翻转后 n-k 个 `[3, 6]` | `[5, 6, 7, 1, 2, 3, 4]` |

## Java 代码

先记住一句话：

> 整体翻转，前半翻转，后半翻转，三步到位。

```java
class Solution {
    public void rotate(int[] nums, int k) {
        int n = nums.length;
        k = k % n;

        reverse(nums, 0, n - 1);
        reverse(nums, 0, k - 1);
        reverse(nums, k, n - 1);
    }

    private void reverse(int[] nums, int left, int right) {
        while (left < right) {
            int temp = nums[left];
            nums[left] = nums[right];
            nums[right] = temp;
            left++;
            right--;
        }
    }
}
```

## 代码逐行拆解

```java
int n = nums.length;
k = k % n;
```

取模处理，防止 `k` 大于数组长度。

如果 `k == 0`，三次翻转后数组不变，不需要特判。

```java
reverse(nums, 0, n - 1);
```

翻转整个数组。

```java
reverse(nums, 0, k - 1);
```

翻转前 `k` 个元素，恢复它们的正序。

```java
reverse(nums, k, n - 1);
```

翻转后 `n - k` 个元素，恢复它们的正序。

```java
private void reverse(int[] nums, int left, int right) {
    while (left < right) {
        int temp = nums[left];
        nums[left] = nums[right];
        nums[right] = temp;
        left++;
        right--;
    }
}
```

翻转辅助方法：用双指针从两端向中间交换。

这个方法是原地操作，不需要额外数组。

## 其他方法

### 方法二：额外数组

最直观的做法：新建一个数组，把每个元素放到轮转后的位置。

```java
class Solution {
    public void rotate(int[] nums, int k) {
        int n = nums.length;
        int[] temp = new int[n];

        for (int i = 0; i < n; i++) {
            temp[(i + k) % n] = nums[i];
        }

        System.arraycopy(temp, 0, nums, 0, n);
    }
}
```

简单易懂，但空间复杂度 `O(n)`。

面试时可以先说这个方法表示理解题意，再给出翻转法作为优化。

## 复杂度分析

### 三次翻转法

时间复杂度：`O(n)`。

每个元素被翻转了两次（一次全局翻转 + 一次局部翻转），总操作量是 `2n`。

空间复杂度：`O(1)`。

只用了一个临时变量做交换。

### 额外数组法

时间复杂度：`O(n)`。

空间复杂度：`O(n)`。

## 常见坑

### 1. 忘记对 k 取模

如果 `k` 大于数组长度，不取模会导致翻转范围越界。

```text
nums = [1, 2], k = 3
k % 2 = 1，实际只需要轮转 1 步
```

### 2. 翻转范围写错

三次翻转的范围是：

```text
第一次：[0, n-1]     整个数组
第二次：[0, k-1]     前 k 个
第三次：[k, n-1]     后 n-k 个
```

如果把第二次写成 `[0, k]` 或第三次写成 `[k+1, n-1]`，结果就错了。

### 3. k = 0 时做了多余操作

`k = 0` 时数组不需要变。三次翻转法天然处理了这种情况（翻转后恢复原样），不需要特判。

但如果你在代码里加了 `if (k == 0) return;` 也没问题，算是小优化。

### 4. 混淆"向右轮转"和"向左轮转"

向右轮转 `k` 步 = 把末尾 `k` 个元素搬到前面。

向左轮转 `k` 步 = 把开头 `k` 个元素搬到后面。

如果搞反了方向，翻转的顺序也得调整。

## 面试时怎么讲

### 30 秒版

> 先对 `k` 取模，然后用三次翻转：先翻转整个数组，再翻转前 `k` 个元素，最后翻转后 `n-k` 个元素。这样原地完成轮转，时间 `O(n)`，空间 `O(1)`。核心原理是：整体翻转把尾部元素带到了前面，但顺序反了，再局部翻转恢复正序。

### 面试官可能追问

**为什么不直接一个一个挪？**

每次把最后一个元素搬到最前面，重复 `k` 次。每次搬需要把所有元素后移一格，时间 `O(n)`，总共 `O(n*k)`，太慢。

**能不能用环形替换法？**

可以。从位置 `0` 开始，把它放到 `(0+k)%n`，再把被覆盖的元素放到它的目标位置，循环往复直到回到起点。如果没遍历完所有元素，从下一个位置重新开始。时间 `O(n)`，空间 `O(1)`，但实现较复杂。

**三次翻转法能推广到其他场景吗？**

能。比如字符串的左旋转（把前 k 个字符搬到末尾）也可以用同样的三次翻转，只是翻转顺序不同。

## 最后总结

轮转数组这道题，三次翻转法是最优雅的原地解法：

> 整体翻转让元素到位，局部翻转让顺序恢复。

代码模板：

```text
k = k % n
翻转 [0, n-1]
翻转 [0, k-1]
翻转 [k, n-1]
```

刷题时可以用下面几个边界用例自检：

```text
nums = [1,2,3,4,5,6,7], k=3 -> [5,6,7,1,2,3,4]
nums = [-1,-100,3,99], k=2 -> [3,99,-1,-100]
nums = [1,2], k=3 -> [2,1]（k > n）
nums = [1], k=0 -> [1]（不需要轮转）
nums = [1], k=1 -> [1]（单元素轮转等于不动）
```

最后把这句话记住：

> 轮转的本质是"尾部搬到头部"。三次翻转就像翻书：先把整本书倒过来，再把前后两段各自翻正。
