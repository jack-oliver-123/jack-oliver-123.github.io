## 适用场景

Prometheus 负责指标采集和存储，Grafana 负责可视化展示。两者配合是当前最流行的开源监控方案，适合监控服务器、容器、中间件和应用程序。本文记录如何使用 Docker Compose 快速搭建这套监控栈。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建工作目录：

```bash
# Linux
mkdir -p /opt/monitoring/prometheus /opt/monitoring/grafana

# Windows（PowerShell）
mkdir D:/monitoring-env/prometheus, D:/monitoring-env/grafana
```

## 编写 Prometheus 配置

在工作目录下新建 `prometheus/prometheus.yml`：

```yaml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  # Prometheus 自身监控
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  # 如果有 Node Exporter（主机监控）
  # - job_name: 'node'
  #   static_configs:
  #     - targets: ['host.docker.internal:9100']

  # 如果有 Spring Boot Actuator
  # - job_name: 'spring-app'
  #   metrics_path: '/actuator/prometheus'
  #   static_configs:
  #     - targets: ['host.docker.internal:8080']
```

## 编写 docker-compose

在工作目录下新建 `docker-compose.yml`：

```yaml
services:
  prometheus:
    image: prom/prometheus:v2.53.0
    container_name: prometheus
    restart: unless-stopped
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
      - ./prometheus/data:/prometheus
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
      - '--storage.tsdb.retention.time=15d'
      - '--web.enable-lifecycle'
    healthcheck:
      test: ["CMD-SHELL", "wget --spider -q http://localhost:9090/-/healthy || exit 1"]
      interval: 10s
      timeout: 5s
      retries: 3

  grafana:
    image: grafana/grafana:11.1.0
    container_name: grafana
    restart: unless-stopped
    ports:
      - "3000:3000"
    volumes:
      - ./grafana/data:/var/lib/grafana
    environment:
      - GF_SECURITY_ADMIN_USER=admin
      - GF_SECURITY_ADMIN_PASSWORD=your_grafana_password
      - GF_USERS_ALLOW_SIGN_UP=false
      - TZ=Asia/Shanghai
    depends_on:
      prometheus:
        condition: service_healthy

networks:
  default:
    name: monitoring
```

关键配置说明：

- `storage.tsdb.retention.time=15d`：指标数据保留 15 天，可按需调整；
- `web.enable-lifecycle`：允许通过 HTTP API 热重载配置；
- `GF_SECURITY_ADMIN_PASSWORD`：Grafana 管理员密码。

## 启动服务

```bash
docker compose up -d
```

查看日志：

```bash
docker compose logs -f
```

> Linux 下如果 Prometheus 数据目录报权限错误：`sudo chown -R 65534:65534 ./prometheus/data`（65534 是容器内 nobody 用户）。
> Grafana 数据目录：`sudo chown -R 472:472 ./grafana/data`。

## 验证安装

### Prometheus

浏览器访问：

```text
http://localhost:9090
```

点击 **Status → Targets**，确认 `prometheus` 这个 job 的状态为 `UP`。

在首页输入查询表达式测试：

```text
up
```

如果返回 `up{job="prometheus"} 1`，说明 Prometheus 正常采集自身指标。

### Grafana

浏览器访问：

```text
http://localhost:3000
```

使用配置的账号登录：

```text
admin / your_grafana_password
```

## 配置 Grafana 数据源

1. 进入 Grafana → **Connections → Data sources → Add data source**；
2. 选择 **Prometheus**；
3. URL 填写：`http://prometheus:9090`（Docker 网络内使用容器名）；
4. 点击 **Save & test**，显示 `Successfully queried the Prometheus API` 即成功。

## 导入监控面板

Grafana 支持从社区导入现成的 Dashboard：

1. 进入 **Dashboards → Import**；
2. 输入 Dashboard ID，常用的有：
   - `1860`：Node Exporter Full（主机监控）
   - `4701`：JVM Dashboard（Java 应用）
   - `12708`：Docker Container Monitoring
3. 选择 Prometheus 数据源后导入。

## 添加监控目标

### 监控宿主机（Node Exporter）

添加一个 Node Exporter 服务到 `docker-compose.yml`：

```yaml
  node-exporter:
    image: prom/node-exporter:v1.8.1
    container_name: node-exporter
    restart: unless-stopped
    ports:
      - "9100:9100"
    volumes:
      - /proc:/host/proc:ro
      - /sys:/host/sys:ro
      - /:/rootfs:ro
    command:
      - '--path.procfs=/host/proc'
      - '--path.sysfs=/host/sys'
      - '--path.rootfs=/rootfs'
```

> Node Exporter 挂载宿主机 `/proc` 和 `/sys`，**仅适用于 Linux**。Windows 下需要用 [windows_exporter](https://github.com/prometheus-community/windows_exporter) 直接安装在宿主机上。

同时在 `prometheus.yml` 中取消注释 node job：

```yaml
  - job_name: 'node'
    static_configs:
      - targets: ['node-exporter:9100']
```

重载 Prometheus 配置：

```bash
curl -X POST http://localhost:9090/-/reload
```

### 监控 Spring Boot 应用

应用添加依赖：

```xml
<dependency>
  <groupId>io.micrometer</groupId>
  <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

`application.yml` 暴露 Prometheus 端点：

```yaml
management:
  endpoints:
    web:
      exposure:
        include: prometheus,health
  metrics:
    export:
      prometheus:
        enabled: true
```

在 `prometheus.yml` 中添加采集配置：

```yaml
  - job_name: 'spring-app'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['host.docker.internal:8080']
```

## 常见问题

### Grafana 登录后报 data source error

确认 Data source URL 使用容器名 `http://prometheus:9090`，而不是 `localhost`（Grafana 容器内 localhost 是它自己）。

### Prometheus Targets 显示 DOWN

检查：
- 目标服务是否已启动；
- 目标端口是否可从 Prometheus 容器内访问（Docker 网络内用容器名，宿主机上的服务用 `host.docker.internal`）；
- 防火墙是否放行。

### 磁盘空间增长过快

调整 Prometheus 保留策略：

```yaml
command:
  - '--storage.tsdb.retention.time=7d'    # 缩短保留时间
  - '--storage.tsdb.retention.size=5GB'   # 限制总大小
```

## 停止和清理

```bash
# 停止服务（保留数据）
docker compose down

# 停止并删除所有监控数据
docker compose down -v
rm -rf ./prometheus/data ./grafana/data
```

## 后续建议

- 生产环境建议配置告警规则（Alertmanager）实现自动报警。
- Grafana 支持钉钉、企业微信、邮件等通知渠道。
- 监控中间件（MySQL、Redis、ES 等）需要对应的 Exporter，可按需添加。
- 大规模场景考虑 Thanos 或 VictoriaMetrics 替代 Prometheus 做长期存储。
