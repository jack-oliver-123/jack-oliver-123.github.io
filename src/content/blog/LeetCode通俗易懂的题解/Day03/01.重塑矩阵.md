## 题目入口

题目链接：[重塑矩阵](https://leetcode.cn/problems/reshape-the-matrix/)

## 一句话读懂题目

给你一个 `m × n` 的矩阵 `mat`，把它重塑成一个 `r × c` 的新矩阵。

新矩阵按行填充原矩阵中的所有元素，顺序不变。

如果无法重塑（元素总数不匹配），返回原矩阵。

比如：

```text
mat = [[1, 2],
       [3, 4]]
r = 1, c = 4
```

重塑后：

```text
[[1, 2, 3, 4]]
```

再比如：

```text
mat = [[1, 2],
       [3, 4]]
r = 2, c = 4
```

2×2 = 4 个元素，但 2×4 = 8 个位置，装不下，返回原矩阵。

## 先用生活例子理解

想象你有一本书，每页有 2 行文字，共 3 页，总共 6 行。

现在要把它重排成每页 3 行的新书。内容顺序不变，只是换了个排版方式。

关键在于：把所有行按顺序拆出来，再按新的每页行数重新装回去。

矩阵重塑也是一样：

1. 把二维矩阵"拉平"成一条线（按行读取）；
2. 再按新的列数"折回"成新矩阵。

## 核心思路：一维索引与二维坐标的转换

对于一个 `m × n` 的矩阵，第 `k` 个元素（从 0 开始）对应的二维坐标是：

```text
行 = k / n
列 = k % n
```

反过来，坐标 `(i, j)` 对应的一维索引是：

```text
k = i * n + j
```

重塑过程：遍历一维索引 `k` 从 `0` 到 `m*n - 1`，用上面的公式分别算出在原矩阵和新矩阵中的坐标。

## 手动走一遍例子

```text
mat = [[1, 2],
       [3, 4]]
r = 1, c = 4
```

原矩阵是 2×2，新矩阵是 1×4。元素总数 4 = 4，可以重塑。

| k | 原矩阵坐标 (k/2, k%2) | 值 | 新矩阵坐标 (k/4, k%4) |
| --- | --- | --- | --- |
| 0 | (0, 0) | `1` | (0, 0) |
| 1 | (0, 1) | `2` | (0, 1) |
| 2 | (1, 0) | `3` | (0, 2) |
| 3 | (1, 1) | `4` | (0, 3) |

新矩阵：`[[1, 2, 3, 4]]`。

## Java 代码

先记住一句话：

> 拉平再折叠，索引除和模就能定位。

```java
class Solution {
    public int[][] matrixReshape(int[][] mat, int r, int c) {
        int m = mat.length;
        int n = mat[0].length;

        if (m * n != r * c) {
            return mat;
        }

        int[][] result = new int[r][c];

        for (int k = 0; k < m * n; k++) {
            result[k / c][k % c] = mat[k / n][k % n];
        }

        return result;
    }
}
```

## 代码逐行拆解

```java
int m = mat.length;
int n = mat[0].length;
```

获取原矩阵的行数和列数。

```java
if (m * n != r * c) {
    return mat;
}
```

元素总数不匹配，无法重塑，直接返回原矩阵。

```java
int[][] result = new int[r][c];
```

创建新矩阵。

```java
for (int k = 0; k < m * n; k++) {
    result[k / c][k % c] = mat[k / n][k % n];
}
```

遍历每个一维索引 `k`：

- `mat[k / n][k % n]`：从原矩阵读取第 `k` 个元素；
- `result[k / c][k % c]`：写入新矩阵的对应位置。

## 复杂度分析

时间复杂度：`O(m × n)`。

每个元素访问一次。

空间复杂度：`O(1)`。

除了结果矩阵外，只用了几个变量（结果矩阵是必须输出的，不算额外空间）。

## 常见坑

### 1. 忘记判断能否重塑

元素总数不匹配时必须返回原矩阵，不能强行重塑。

### 2. 除和模搞混了原矩阵和新矩阵的列数

原矩阵用 `n` 做除和模，新矩阵用 `c` 做除和模，不要写反。

### 3. 用两层循环遍历原矩阵也可以

```java
int index = 0;
for (int i = 0; i < m; i++) {
    for (int j = 0; j < n; j++) {
        result[index / c][index % c] = mat[i][j];
        index++;
    }
}
```

效果一样，只是用 `index` 计算新矩阵的坐标。

## 面试时怎么讲

### 30 秒版

> 先判断元素总数是否匹配。然后用一维索引 `k` 遍历所有元素，通过 `k / 列数` 和 `k % 列数` 分别定位原矩阵和新矩阵的坐标，把元素搬过去。时间 `O(m*n)`，空间 `O(1)`。

### 面试官可能追问

**这个技巧还能用在哪？**

二维矩阵和一维数组之间的转换非常常用。比如用一维数组模拟二维矩阵、螺旋遍历中追踪位置、或者在二分查找中把一维索引映射回二维坐标。

## 最后总结

重塑矩阵的核心是二维坐标和一维索引之间的转换公式：

> `k → (k / cols, k % cols)`，`(i, j) → i * cols + j`。

代码模板：

```text
判断 m*n == r*c
遍历 k 从 0 到 m*n-1
    result[k/c][k%c] = mat[k/n][k%n]
```

最后把这句话记住：

> 矩阵重塑不需要真的"拉平"再"折回"，一个索引公式就能在两个形状之间自由切换。
