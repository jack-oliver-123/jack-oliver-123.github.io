## 题目入口

题目链接：[丢失的数字](https://leetcode.cn/problems/missing-number/description/)

## 一句话读懂题目

给你一个包含 `n` 个数的数组 `nums`，里面的数来自 `[0, n]` 这个范围，而且没有重复。

这个范围里一共有 `n + 1` 个数，但数组只有 `n` 个元素，所以刚好缺了一个。

找出那个缺失的数字。

比如：

```text
nums = [3, 0, 1]
```

范围是 `[0, 3]`，也就是 `0, 1, 2, 3` 四个数。

数组里有 `0, 1, 3`，缺了 `2`，返回：

```text
2
```

再比如：

```text
nums = [0, 1]
```

范围是 `[0, 2]`，数组里有 `0, 1`，缺了 `2`，返回：

```text
2
```

## 先用生活例子理解

想象你是一个班主任，班上应该有编号 `0` 到 `n` 的学生，一共 `n + 1` 人。

今天点名，到了 `n` 个人，刚好少了一个。

怎么快速知道谁没来？

**方法一：算总数**

你知道所有人的编号加起来应该是多少（`0 + 1 + 2 + ... + n`）。

把到场的人的编号加起来，用总数减去到场的和，差值就是缺席的那个人。

**方法二：两两对消**

让每个编号和每个到场的人配对。配上的就消掉，最后剩下来的就是没来的那个。

这就是异或的思路。

**方法三：签到表**

准备一个签到表，把到场的人都打勾，最后看哪个编号没打勾。

这就是 HashSet 的思路。

## 方法一：数学求和（推荐）

最简洁、最高效的做法。

`[0, n]` 的总和用公式直接算出来：

```text
期望总和 = n * (n + 1) / 2
```

然后遍历数组求出实际总和：

```text
实际总和 = nums[0] + nums[1] + ... + nums[n-1]
```

两者的差就是缺失的数字：

```text
缺失数字 = 期望总和 - 实际总和
```

## 手动走一遍例子

```text
nums = [3, 0, 1]
n = 3
```

期望总和：

```text
3 * (3 + 1) / 2 = 6
```

实际总和：

```text
3 + 0 + 1 = 4
```

差值：

```text
6 - 4 = 2
```

所以缺失的数字是 `2`。

再看一个：

```text
nums = [9, 6, 4, 2, 3, 5, 7, 0, 1]
n = 9
```

期望总和：

```text
9 * 10 / 2 = 45
```

实际总和：

```text
9 + 6 + 4 + 2 + 3 + 5 + 7 + 0 + 1 = 37
```

差值：

```text
45 - 37 = 8
```

缺失的数字是 `8`。

## Java 代码

先记住一句话：

> 应该有多少，实际有多少，差多少就是谁丢了。

### 写法一：数学求和（推荐）

```java
class Solution {
    public int missingNumber(int[] nums) {
        int n = nums.length;
        int expectedSum = n * (n + 1) / 2;
        int actualSum = 0;

        for (int num : nums) {
            actualSum += num;
        }

        return expectedSum - actualSum;
    }
}
```

### 写法二：异或

```java
class Solution {
    public int missingNumber(int[] nums) {
        int xor = 0;

        for (int i = 0; i < nums.length; i++) {
            xor ^= i ^ nums[i];
        }

        xor ^= nums.length;

        return xor;
    }
}
```

### 写法三：HashSet

```java
class Solution {
    public int missingNumber(int[] nums) {
        Set<Integer> set = new HashSet<>();

        for (int num : nums) {
            set.add(num);
        }

        for (int i = 0; i <= nums.length; i++) {
            if (!set.contains(i)) {
                return i;
            }
        }

        return -1;
    }
}
```

## 代码逐行拆解（数学求和）

```java
int n = nums.length;
```

数组有 `n` 个元素，数字范围是 `[0, n]`。

```java
int expectedSum = n * (n + 1) / 2;
```

用等差数列求和公式算出 `0 + 1 + 2 + ... + n` 的总和。

这就是"如果没人缺席，所有编号加起来应该是多少"。

```java
int actualSum = 0;

for (int num : nums) {
    actualSum += num;
}
```

遍历数组，算出实际存在的数字的总和。

```java
return expectedSum - actualSum;
```

差值就是缺失的那个数字。

## 方法二详解：异或消除

异或有两个关键性质：

```text
a ^ a = 0    （相同的数异或为 0）
a ^ 0 = a    （任何数和 0 异或等于自己）
```

思路是：把 `[0, n]` 的所有数字和数组里的所有数字全部异或在一起。

出现两次的数字会被消掉，最后剩下的就是只出现一次的——也就是缺失的那个。

比如：

```text
nums = [3, 0, 1]
n = 3
```

全部异或：

```text
(0 ^ 1 ^ 2 ^ 3) ^ (3 ^ 0 ^ 1)
= 0 ^ 0 ^ 1 ^ 1 ^ 2 ^ 3 ^ 3
= 2
```

`0, 1, 3` 各出现两次被消掉，只剩 `2`。

代码中把循环写在一起：

```java
for (int i = 0; i < nums.length; i++) {
    xor ^= i ^ nums[i];
}
xor ^= nums.length;
```

循环里 `i` 从 `0` 到 `n-1`，提供了 `0, 1, 2, ..., n-1`；最后再异或一个 `n`，凑齐 `[0, n]`。

## 三种方法怎么选

| | 数学求和 | 异或 | HashSet |
| --- | --- | --- | --- |
| 时间复杂度 | `O(n)` | `O(n)` | `O(n)` |
| 空间复杂度 | `O(1)` | `O(1)` | `O(n)` |
| 代码量 | 最短 | 较短 | 较长 |
| 理解难度 | 简单 | 需要理解异或 | 简单 |
| 溢出风险 | 理论上有 | 没有 | 没有 |

面试时推荐数学求和：最直观、代码最短、空间最优。

如果面试官追问溢出问题，可以补充异或写法。

## 复杂度分析

### 方法一：数学求和

时间复杂度：`O(n)`。

遍历一次数组求和。

空间复杂度：`O(1)`。

只用了两个变量。

### 方法二：异或

时间复杂度：`O(n)`。

遍历一次数组做异或。

空间复杂度：`O(1)`。

只用了一个变量。

### 方法三：HashSet

时间复杂度：`O(n)`。

遍历一次放入 Set，再遍历一次查找。

空间复杂度：`O(n)`。

Set 最多存 `n` 个元素。

## 常见坑

### 1. 求和公式用错

`0 + 1 + 2 + ... + n` 的和是：

```text
n * (n + 1) / 2
```

不是 `n * (n - 1) / 2`，也不是 `(n + 1) * n / 2 + 1`。

### 2. 搞不清 n 是什么

`n` 是数组的长度，`nums.length`。

数字范围是 `[0, n]`，一共 `n + 1` 个数。

数组里有 `n` 个数，缺了一个。

### 3. 异或时忘了异或 n

循环 `i` 从 `0` 到 `n-1`，少了一个 `n`。

所以循环结束后还要 `xor ^= nums.length`。

### 4. 担心求和溢出

当 `n` 很大时，`n * (n + 1) / 2` 可能超过 `int` 范围吗？

题目约束 `n <= 10^4`，所以 `10^4 * 10^4 / 2 = 5 * 10^7`，远小于 `int` 的最大值 `2.1 * 10^9`，不会溢出。

如果 `n` 更大（比如 `10^5` 级别），也可以在循环中边加边减，避免先算出大数：

```java
int result = nums.length;
for (int i = 0; i < nums.length; i++) {
    result += i - nums[i];
}
return result;
```

### 5. 以为范围是 [1, n]

题目的范围是 `[0, n]`，从 `0` 开始，不是从 `1` 开始。

所以缺失的数字可能是 `0`。

## 面试时怎么讲

### 30 秒版

> 数组有 `n` 个元素，数字范围是 `[0, n]`，缺了一个。我用等差数列公式算出 `0` 到 `n` 的期望总和，再减去数组的实际总和，差值就是缺失的数。时间 `O(n)`，空间 `O(1)`。

### 面试官可能追问

**如果 n 很大，求和会不会溢出？**

在本题约束下不会。如果 `n` 更大，可以改成循环中边加边减的写法，或者用异或方法（异或不涉及加法，完全没有溢出风险）。

**异或为什么能消除重复？**

异或的核心性质是 `a ^ a = 0`。把完整序列和数组的所有数字都异或在一起，出现两次的数字互相抵消为 `0`，只剩下那个只出现一次的——就是缺失的数。

**能不能排序后查找？**

可以。排序后遍历，如果 `nums[i] != i`，说明 `i` 缺失。但排序时间是 `O(n log n)`，不如求和法快。

**如果缺了两个数怎么办？**

一个方程解不出两个未知数。需要加一个等式，比如用"平方和"或者"异或 + 求和"组合来列两个方程，就能解出两个缺失的数。这是 LeetCode 的其他题目了。

**这道题和"只出现一次的数字"有什么关系？**

思路很像。"只出现一次的数字"是所有数异或后剩下的就是答案。这道题是把完整序列和数组放在一起异或，剩下的就是缺失的数。本质都是利用异或的自消除性质。

## 最后总结

丢失的数字这道题，核心思想是：

> 知道"完整"应该是什么样，再和"实际"做对比，差异就是答案。

三种对比方式：

```text
求和法：期望总和 - 实际总和 = 缺失的数
异或法：完整序列 XOR 实际数组 = 缺失的数
哈希法：把实际的存入 Set，逐一检查完整序列
```

刷题时可以用下面几个边界用例自检：

```text
nums = [3, 0, 1] -> 2（中间缺）
nums = [0, 1] -> 2（缺最大的）
nums = [1, 2] -> 0（缺 0）
nums = [0] -> 1
nums = [1] -> 0
nums = [9,6,4,2,3,5,7,0,1] -> 8
```

最后把这句话记住：

> 完整的减去残缺的，缺口就是答案。加法如此，异或也如此。
