## 适用场景

Nginx 常用于静态资源托管、反向代理、负载均衡和 HTTPS 入口。本文记录如何使用 Docker 部署一个可挂载配置和静态页面目录的 Nginx。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建工作目录：

```bash
# Linux
mkdir -p /opt/nginx/conf /opt/nginx/html /opt/nginx/logs

# Windows（PowerShell）
mkdir D:/nginx-env/conf, D:/nginx-env/html, D:/nginx-env/logs
```

## 编写 Nginx 配置

在配置目录下新建 `nginx.conf`：

```nginx
user  nginx;
worker_processes  auto;

error_log  /var/log/nginx/error.log notice;
pid        /var/run/nginx.pid;

events {
    worker_connections  1024;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    sendfile        on;
    keepalive_timeout  65;

    server {
        listen       80;
        server_name  localhost;

        location / {
            root   /usr/share/nginx/html;
            index  index.html index.htm;
            try_files $uri $uri/ /index.html;
        }

        # 反向代理后端服务示例
        # location /api/ {
        #     proxy_pass http://host.docker.internal:8080/;
        #     proxy_set_header Host $host;
        #     proxy_set_header X-Real-IP $remote_addr;
        # }
    }
}
```

> `try_files $uri $uri/ /index.html` 是 SPA 路由回退配置。纯多页静态站点可以去掉。

## 准备测试页面

在 html 目录下新建 `index.html`：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>Nginx 测试页</title>
</head>
<body>
    <h1>Hello Nginx</h1>
</body>
</html>
```

## 编写 docker-compose

新建 `docker-compose.yml`：

```yaml
services:
  nginx:
    image: nginx:1.26-alpine
    container_name: nginx-local
    restart: always
    ports:
      - "80:80"
    volumes:
      - ./conf/nginx.conf:/etc/nginx/nginx.conf
      - ./html:/usr/share/nginx/html
      - ./logs:/var/log/nginx
    environment:
      - TZ=Asia/Shanghai
    healthcheck:
      test: ["CMD-SHELL", "curl -fs http://localhost/ || exit 1"]
      interval: 10s
      timeout: 3s
      retries: 3
```

## 启动 Nginx

```bash
docker compose up -d
```

查看容器状态：

```bash
docker ps
```

## 验证访问

浏览器访问：

```text
http://localhost
```

如果能看到 `Hello Nginx`，说明 Nginx 已正常启动。

## 常用运维命令

检查配置是否合法：

```bash
docker exec nginx-local nginx -t
```

热重载配置（不重启容器）：

```bash
docker exec nginx-local nginx -s reload
```

查看访问日志：

```bash
docker logs -f nginx-local
# 或查看挂载的 logs 目录
```

## 发布静态页面

前端项目构建后的 `dist` 目录内容复制到 html 目录中，刷新浏览器即可生效。

## 常见问题

### 80 端口被占用

修改宿主机端口：

```yaml
ports:
  - "8080:80"
```

然后访问 `http://localhost:8080`。

### 配置文件挂载错误

确保 `nginx.conf` 是文件而非目录，也不要保存成 `nginx.conf.txt`。

### Windows 下权限问题

如果静态文件无法读取，检查 Docker Desktop 的文件共享设置是否包含了工作目录所在磁盘。

## 停止和清理

```bash
docker compose down
```

## 后续建议

- 生产环境建议增加 HTTPS 配置（可用 Let's Encrypt 或自签证书）。
- 反向代理时要明确后端服务地址和跨域策略。
- 配合 `logging` 配置限制日志文件大小。
