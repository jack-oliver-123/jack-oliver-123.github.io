## 适用场景

MongoDB 是面向文档的 NoSQL 数据库，适合存储结构灵活的 JSON 数据，常用于内容管理、用户画像、日志存储和实时分析。本文记录如何使用 Docker Compose 部署单节点 MongoDB。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建工作目录：

```bash
# Linux
mkdir -p /opt/mongodb

# Windows（PowerShell）
mkdir D:/mongodb-env
```

## 编写 docker-compose

在工作目录下新建 `docker-compose.yml`：

```yaml
services:
  mongodb:
    image: mongo:7.0
    container_name: mongodb-local
    restart: unless-stopped
    ports:
      - "27017:27017"
    volumes:
      - ./data:/data/db
      - ./config:/data/configdb
    environment:
      - MONGO_INITDB_ROOT_USERNAME=admin
      - MONGO_INITDB_ROOT_PASSWORD=your_mongo_password
      - TZ=Asia/Shanghai
    healthcheck:
      test: ["CMD", "mongosh", "--eval", "db.adminCommand('ping')"]
      interval: 10s
      timeout: 5s
      retries: 3
```

配置说明：

- `MONGO_INITDB_ROOT_USERNAME`：初始化时创建的管理员用户名；
- `MONGO_INITDB_ROOT_PASSWORD`：管理员密码；
- `./data:/data/db`：数据持久化目录。

> 这两个环境变量仅在数据目录为空时（首次启动）生效。如果已有数据，修改环境变量不会改变密码。

## 启动 MongoDB

```bash
docker compose up -d
```

查看日志：

```bash
docker compose logs -f
```

看到 `Waiting for connections` 说明启动完成。

## 验证连接

使用容器内 `mongosh` 连接：

```bash
docker exec -it mongodb-local mongosh -u admin -p your_mongo_password --authenticationDatabase admin
```

测试命令：

```javascript
db.runCommand({ ping: 1 })
```

返回 `{ ok: 1 }` 说明连接正常。

## 基本操作

```javascript
// 切换/创建数据库
use app_db

// 插入文档
db.users.insertOne({ name: "张三", email: "zhangsan@example.com", age: 28 })

// 查询
db.users.find({ name: "张三" })

// 更新
db.users.updateOne({ name: "张三" }, { $set: { age: 29 } })

// 删除
db.users.deleteOne({ name: "张三" })

// 查看所有集合
show collections
```

## 创建业务用户

生产环境不建议用 admin 账号连接业务库：

```javascript
use app_db
db.createUser({
  user: "app_user",
  pwd: "your_app_password",
  roles: [{ role: "readWrite", db: "app_db" }]
})
```

应用连接时使用：

```text
mongodb://app_user:your_app_password@localhost:27017/app_db?authSource=app_db
```

## 应用接入示例

Spring Boot 项目引入依赖：

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-data-mongodb</artifactId>
</dependency>
```

配置示例（`application.yml`）：

```yaml
spring:
  data:
    mongodb:
      uri: mongodb://app_user:your_app_password@localhost:27017/app_db?authSource=app_db
```

## 常见问题

### 认证失败

确认使用了正确的 `authSource`：

- admin 用户：`authSource=admin`
- 业务用户：`authSource=业务数据库名`

### 数据目录权限问题（Linux）

```bash
sudo chown -R 999:999 ./data
```

`999` 是容器内 mongodb 用户的默认 UID。

### 连接字符串中密码含特殊字符

如果密码包含 `@`、`:`、`/` 等字符，需要 URL 编码（如 `@` → `%40`）。

## 数据备份与恢复

```bash
# 备份所有数据库
docker exec mongodb-local mongodump -u admin -p your_mongo_password --authenticationDatabase admin --out /data/db/backup

# 从宿主机拷出备份
docker cp mongodb-local:/data/db/backup ./backup

# 恢复
docker exec mongodb-local mongorestore -u admin -p your_mongo_password --authenticationDatabase admin /data/db/backup
```

## 停止和清理

```bash
# 停止服务（保留数据）
docker compose down

# 停止并删除数据
docker compose down -v
rm -rf ./data
```

## 后续建议

- 不要在生产环境使用弱密码。
- 建议开启访问日志和慢查询分析。
- 需要高可用时请规划副本集（Replica Set）部署。
- 大数据量场景考虑分片（Sharding）。
- 搭配 [Mongo Express](https://hub.docker.com/_/mongo-express) 可以获得 Web 管理界面。
