## 适用场景

Kafka 是高吞吐量的分布式消息队列，适合日志收集、事件驱动架构、流处理和系统解耦。本文记录如何使用 Docker Compose 部署单节点 Kafka（附带 Zookeeper）。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建工作目录：

```bash
# Linux
mkdir -p /opt/kafka/data /opt/kafka/zk-data

# Windows（PowerShell）
mkdir D:/kafka-env/data, D:/kafka-env/zk-data
```

## 编写 docker-compose

新建 `docker-compose.yml`：

```yaml
services:
  zookeeper:
    image: bitnami/zookeeper:3.9
    container_name: zookeeper
    restart: unless-stopped
    ports:
      - "2181:2181"
    volumes:
      - ./zk-data:/bitnami/zookeeper
    environment:
      - ALLOW_ANONYMOUS_LOGIN=yes
      - TZ=Asia/Shanghai
    healthcheck:
      test: ["CMD-SHELL", "echo ruok | nc localhost 2181 | grep imok"]
      interval: 10s
      timeout: 5s
      retries: 3

  kafka:
    image: bitnami/kafka:3.7
    container_name: kafka
    restart: unless-stopped
    ports:
      - "9092:9092"
    volumes:
      - ./data:/bitnami/kafka
    environment:
      - KAFKA_CFG_ZOOKEEPER_CONNECT=zookeeper:2181
      - KAFKA_CFG_LISTENERS=PLAINTEXT://:9092
      - KAFKA_CFG_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092
      - KAFKA_CFG_AUTO_CREATE_TOPICS_ENABLE=true
      - KAFKA_CFG_LOG_RETENTION_HOURS=168
      - KAFKA_HEAP_OPTS=-Xms256m -Xmx512m
      - ALLOW_PLAINTEXT_LISTENER=yes
      - TZ=Asia/Shanghai
    depends_on:
      zookeeper:
        condition: service_healthy
    healthcheck:
      test: ["CMD-SHELL", "kafka-broker-api-versions.sh --bootstrap-server localhost:9092 || exit 1"]
      interval: 15s
      timeout: 10s
      retries: 5
```

关键配置说明：

- `KAFKA_CFG_ADVERTISED_LISTENERS`：客户端连接 Kafka 时使用的地址。本地开发填 `localhost:9092`，如果客户端在其他机器上，需要改为宿主机 IP；
- `KAFKA_CFG_LOG_RETENTION_HOURS`：消息保留时间，默认 168 小时（7 天）；
- `KAFKA_HEAP_OPTS`：JVM 内存限制，开发环境 256-512MB 够用。

> 如果客户端也在 Docker 网络中，`ADVERTISED_LISTENERS` 改为 `PLAINTEXT://kafka:9092`，并确保客户端和 Kafka 在同一 Docker 网络。

## 启动服务

```bash
docker compose up -d
```

查看日志：

```bash
docker logs -f kafka
```

看到 `Kafka Server started` 说明启动完成。

## 验证安装

### 创建 Topic

```bash
docker exec -it kafka kafka-topics.sh \
  --create \
  --topic test-topic \
  --bootstrap-server localhost:9092 \
  --partitions 3 \
  --replication-factor 1
```

### 查看 Topic 列表

```bash
docker exec -it kafka kafka-topics.sh \
  --list \
  --bootstrap-server localhost:9092
```

### 生产消息

```bash
docker exec -it kafka kafka-console-producer.sh \
  --topic test-topic \
  --bootstrap-server localhost:9092
```

输入几条消息后按 `Ctrl+C` 退出。

### 消费消息

```bash
docker exec -it kafka kafka-console-consumer.sh \
  --topic test-topic \
  --from-beginning \
  --bootstrap-server localhost:9092
```

如果能看到刚才生产的消息，说明 Kafka 正常工作。

## 应用接入示例

Spring Boot 项目引入依赖：

```xml
<dependency>
  <groupId>org.springframework.kafka</groupId>
  <artifactId>spring-kafka</artifactId>
</dependency>
```

配置示例（`application.yml`）：

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: app-group
      auto-offset-reset: earliest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

## 常见问题

### 客户端连接超时

最常见的原因是 `ADVERTISED_LISTENERS` 配置不对。Kafka 会把这个地址返回给客户端用于后续通信。确保客户端能通过这个地址访问到 Kafka：

- 本地开发：`PLAINTEXT://localhost:9092`
- 远程访问：`PLAINTEXT://宿主机IP:9092`
- Docker 网络内：`PLAINTEXT://kafka:9092`

### Zookeeper 启动失败

检查 `./zk-data` 目录权限。Bitnami 镜像内部使用非 root 用户（UID 1001），Linux 下需要：

```bash
sudo chown -R 1001:1001 ./zk-data ./data
```

### 消息积压 / 磁盘空间不足

调整消息保留策略：

```yaml
- KAFKA_CFG_LOG_RETENTION_HOURS=48          # 缩短到 2 天
- KAFKA_CFG_LOG_RETENTION_BYTES=1073741824  # 单分区最大 1GB
```

## 停止和清理

```bash
# 停止服务（保留数据）
docker compose down

# 停止并删除所有数据（Topic 和消息都会丢失）
docker compose down -v
rm -rf ./data ./zk-data
```

## KRaft 模式（无 Zookeeper）

Kafka 3.3+ 支持 KRaft 模式，不再依赖 Zookeeper。如果想尝试：

```yaml
services:
  kafka:
    image: bitnami/kafka:3.7
    container_name: kafka
    restart: unless-stopped
    ports:
      - "9092:9092"
    volumes:
      - ./data:/bitnami/kafka
    environment:
      - KAFKA_CFG_NODE_ID=1
      - KAFKA_CFG_PROCESS_ROLES=broker,controller
      - KAFKA_CFG_CONTROLLER_QUORUM_VOTERS=1@kafka:9093
      - KAFKA_CFG_LISTENERS=PLAINTEXT://:9092,CONTROLLER://:9093
      - KAFKA_CFG_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092
      - KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP=PLAINTEXT:PLAINTEXT,CONTROLLER:PLAINTEXT
      - KAFKA_CFG_CONTROLLER_LISTENER_NAMES=CONTROLLER
      - KAFKA_CFG_AUTO_CREATE_TOPICS_ENABLE=true
      - KAFKA_HEAP_OPTS=-Xms256m -Xmx512m
      - ALLOW_PLAINTEXT_LISTENER=yes
      - TZ=Asia/Shanghai
```

KRaft 模式架构更简单、启动更快，推荐新项目使用。

## 后续建议

- 生产环境至少部署 3 个 Broker 并设置 `replication-factor >= 2`。
- 建议搭配 Kafka UI 工具（如 [kafka-ui](https://github.com/provectus/kafka-ui)）可视化管理 Topic。
- 监控 Kafka 延迟和消费者 lag（可接入 Prometheus + Grafana）。
- 合理规划 Topic 分区数——分区是 Kafka 并行处理的最小单位。
