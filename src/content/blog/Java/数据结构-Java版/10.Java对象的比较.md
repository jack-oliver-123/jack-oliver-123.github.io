# Java 对象的比较

## 基本类型比较
- 直接用 > < == 比较
- 值比较，简单直观

## 引用类型的问题
- == 比较的是引用地址
- > < 编译直接报错
- 默认 equals 也是比地址

## 三种比较方式

### equals 覆写
- 继承自 Object
- 只能判断相等/不相等
- 不能比较大小
- 套路：同引用→null→类型→内容

### Comparable 接口
- 实现 compareTo 方法
- 侵入性强（修改类本身）
- 属于"内部排序"
- 自然顺序（Natural Order）

### Comparator 比较器
- 实现 compare 方法
- 侵入性弱（不修改原类）
- 属于"外部排序"
- 可以有多种比较策略

## 在 PriorityQueue 中的应用
- 默认使用 Comparable
- 构造时传 Comparator 覆盖
- 自定义类型必须二选一

## 三种方式对比
- equals：只能等/不等
- Comparable：类内部定义自然序
- Comparator：外部灵活定义多种序

## 问题引出：PriorityQueue 中插入自定义对象

上一篇中我们往 `PriorityQueue` 里放的都是 `Integer`——因为 `Integer` 已经实现了 `Comparable` 接口，天然可以比较大小。但如果放入自定义类型呢？

```java
class Student {
    String name;
    int age;

    Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

public static void main(String[] args) {
    PriorityQueue<Student> queue = new PriorityQueue<>();
    queue.offer(new Student("Alice", 20));
    queue.offer(new Student("Bob", 18)); // 💥 抛出 ClassCastException！
}
```

**报错**：`java.lang.ClassCastException: Student cannot be cast to java.lang.Comparable`

原因：PriorityQueue 底层是堆，插入元素时需要比较大小来维护堆序。但 `Student` 类没有告诉 JVM "怎么比较两个学生的大小"，所以报错。

这引出了核心问题：**Java 中对象如何比较？**

## 基本类型 vs 引用类型的比较

### 基本类型：直接比较

```java
int a = 10, b = 20;
System.out.println(a > b);   // false
System.out.println(a < b);   // true
System.out.println(a == b);  // false
```

基本类型比较的是**值本身**，简单直接。

### 引用类型：不能直接比较

```java
Student s1 = new Student("Alice", 20);
Student s2 = new Student("Bob", 18);

// s1 > s2;   // ❌ 编译错误！引用类型不支持 > <
// s1 < s2;   // ❌ 编译错误！
s1 == s2;     // ✅ 编译通过，但比较的是引用地址，不是内容
```

- `>` 和 `<`：**编译直接报错**——Java 不知道怎么比较两个对象的"大小"
- `==`：**编译通过但语义错误**——比较的是两个引用是否指向同一个对象（地址），不是内容是否相同

### 默认的 equals 也有问题

所有类默认继承 `Object`，`Object` 提供了 `equals` 方法。但默认实现是：

```java
// Object.equals() 的默认实现
public boolean equals(Object obj) {
    return (this == obj); // 还是比较地址！
}
```

所以不覆写 `equals` 的话，它和 `==` 没有区别。

## 方式一：覆写 equals

### 什么时候用

当你需要判断两个对象的**内容是否相等**时——比如判断两张扑克牌是否是同一张牌（花色和点数都相同）。

### 标准覆写套路

```java
class Card {
    int rank;       // 点数
    String suit;    // 花色

    @Override
    public boolean equals(Object o) {
        // 1. 如果是同一个引用，直接返回 true
        if (this == o) return true;

        // 2. 如果传入 null，返回 false
        if (o == null) return false;

        // 3. 如果类型不同，返回 false
        if (this.getClass() != o.getClass()) return false;

        // 4. 强转后比较内容
        Card other = (Card) o;
        return this.rank == other.rank
            && this.suit.equals(other.suit); // String 用 equals 比较！
    }

    // 覆写 equals 必须同时覆写 hashCode（HashMap/HashSet 依赖）
    @Override
    public int hashCode() {
        return Objects.hash(rank, suit);
    }
}
```

### equals 覆写四步法

1. **同引用检查**：`this == o` → 直接 true（性能优化）
2. **null 检查**：`o == null` → 直接 false
3. **类型检查**：`getClass() != o.getClass()` → false
4. **内容比较**：强转后逐字段比较（引用类型字段也要用 equals）

### equals 的局限

**`equals` 只能判断"相等"或"不相等"，不能判断"大于"或"小于"。**

如果你需要给对象排序（放入 TreeSet、PriorityQueue，或者调用 Collections.sort()），`equals` 就不够用了——你需要 `Comparable` 或 `Comparator`。

## 方式二：实现 Comparable 接口

### 什么时候用

当你的类有一种**自然的、默认的排序方式**时——比如学生按学号排序、字符串按字典序排序。

### 用法

让你的类实现 `Comparable<T>` 接口，覆写 `compareTo` 方法：

```java
class Student implements Comparable<Student> {
    String name;
    int age;

    Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public int compareTo(Student other) {
        // 按年龄升序比较
        // 返回值：负数 = this < other，0 = 相等，正数 = this > other
        return this.age - other.age;
    }
}
```

现在 `Student` 可以放入 PriorityQueue 了：

```java
PriorityQueue<Student> queue = new PriorityQueue<>();
queue.offer(new Student("Alice", 20));
queue.offer(new Student("Bob", 18));
queue.offer(new Student("Charlie", 22));

System.out.println(queue.poll().name); // Bob（18 岁最小，先出队）
System.out.println(queue.poll().name); // Alice（20 岁）
System.out.println(queue.poll().name); // Charlie（22 岁）
```

也可以用 `Collections.sort()` 排序：

```java
List<Student> students = new ArrayList<>();
students.add(new Student("Alice", 20));
students.add(new Student("Bob", 18));
Collections.sort(students); // 使用 compareTo 排序
```

### compareTo 的返回值约定

| 返回值 | 含义 |
|--------|------|
| 负数 | `this` 排在 `other` 前面（`this` 小） |
| 0 | `this` 和 `other` 相等 |
| 正数 | `this` 排在 `other` 后面（`this` 大） |

> ⚠️ **整数溢出陷阱**：`this.age - other.age` 在数值很大或很小时可能溢出！安全写法是 `Integer.compare(this.age, other.age)`。

### Comparable 的特点

- **侵入性强**：需要修改类本身的代码（实现接口）
- **只能定义一种排序方式**：一个类只能有一个 `compareTo`
- **属于"内部排序"**：排序规则是类自己定义的
- `Comparable` 在 `java.lang` 包中，无需额外 import

## 方式三：实现 Comparator 比较器

### 什么时候用

- 你**无法修改**要比较的类（比如第三方库的类）
- 同一个类需要**多种不同的排序方式**（按年龄排、按姓名排、按成绩排）
- 你想把排序逻辑和类本身**解耦**

### 用法

创建一个独立的比较器类，实现 `Comparator<T>` 接口：

```java
import java.util.Comparator;

// 按年龄升序比较器
class AgeComparator implements Comparator<Student> {
    @Override
    public int compare(Student s1, Student s2) {
        return Integer.compare(s1.age, s2.age);
    }
}

// 按姓名字典序比较器
class NameComparator implements Comparator<Student> {
    @Override
    public int compare(Student s1, Student s2) {
        return s1.name.compareTo(s2.name);
    }
}
```

使用比较器：

```java
// PriorityQueue 使用比较器
PriorityQueue<Student> queue = new PriorityQueue<>(new AgeComparator());

// Collections.sort 使用比较器
Collections.sort(students, new NameComparator());

// 也可以用 Lambda 表达式（推荐）
PriorityQueue<Student> queue2 = new PriorityQueue<>(
    (s1, s2) -> Integer.compare(s1.age, s2.age)
);

Collections.sort(students, (s1, s2) -> s1.name.compareTo(s2.name));
```

### Comparator 的特点

- **侵入性弱**：不需要修改原始类
- **可以定义多种排序方式**：不同场景用不同比较器
- **属于"外部排序"**：排序规则由调用者决定
- `Comparator` 在 `java.util` 包中，需要 import

## 三种方式对比

下图直观展示了 Comparable 与 Comparator 的核心区别：

![Comparable vs Comparator 对比](/images/data-structures/comparable-vs-comparator.svg)

| 方式 | 所在包 | 比较能力 | 侵入性 | 使用场景 |
|------|--------|---------|--------|---------|
| `equals` 覆写 | `java.lang`（Object） | 只能判等/不等 | 修改类本身 | HashMap key、HashSet 去重、contains 判断 |
| `Comparable.compareTo` | `java.lang` | 大小比较 | 修改类本身（实现接口） | 定义唯一的自然排序（TreeSet、PriorityQueue 默认） |
| `Comparator.compare` | `java.util` | 大小比较 | 不修改原类 | 灵活定义多种排序、无法修改原类时 |

### 选择决策

```
需要让对象能放入 HashMap/HashSet 吗？
├── 是 → 覆写 equals + hashCode

需要让对象能排序（TreeSet / PriorityQueue / sort）吗？
├── 是 → 这个类有唯一的"自然排序"吗？
│         ├── 是 → 实现 Comparable（类自己定义排序）
│         └── 否 → 使用 Comparator（外部传入排序规则）
└── 否 → 不需要额外处理
```

## 在 PriorityQueue 中的应用

PriorityQueue 支持两种比较方式，优先级如下：

1. **构造时传入了 Comparator** → 使用 Comparator 的 `compare` 方法
2. **没传 Comparator** → 元素必须实现 `Comparable`，使用 `compareTo` 方法
3. **两个都没有** → 运行时抛出 `ClassCastException`

```java
// 方式一：元素实现 Comparable（不传比较器）
class Student implements Comparable<Student> {
    int age;
    @Override
    public int compareTo(Student o) {
        return Integer.compare(this.age, o.age); // 按年龄，小的优先
    }
}
PriorityQueue<Student> pq1 = new PriorityQueue<>(); // 使用 compareTo

// 方式二：传入 Comparator（优先级更高，会覆盖 compareTo）
PriorityQueue<Student> pq2 = new PriorityQueue<>(
    (s1, s2) -> s1.name.compareTo(s2.name) // 按名字字典序
);
```

### 用 Comparator 实现大根堆

默认 PriorityQueue 是小根堆（Comparable 的自然序）。要实现大根堆，传入反序比较器：

```java
// 小根堆（默认）
PriorityQueue<Integer> minHeap = new PriorityQueue<>();

// 大根堆——反转比较逻辑
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(
    (a, b) -> Integer.compare(b, a)
);
// 或者
PriorityQueue<Integer> maxHeap2 = new PriorityQueue<>(
    Comparator.reverseOrder()
);
```

## 常见陷阱

### 陷阱 1：compareTo 中用减法可能溢出

```java
// ❌ 危险：当 a.age 很大而 b.age 是负数时，a.age - b.age 可能溢出
public int compareTo(Student other) {
    return this.age - other.age;
}

// ✅ 安全：使用 Integer.compare
public int compareTo(Student other) {
    return Integer.compare(this.age, other.age);
}
```

### 陷阱 2：覆写 equals 不覆写 hashCode

下图展示了 equals 与 hashCode 之间的契约关系：

![equals 与 hashCode 关系图](/images/data-structures/equals-hashcode.svg)

如果你覆写了 `equals` 但没覆写 `hashCode`，在 `HashMap`/`HashSet` 中会出现"内容相同但找不到"的诡异 bug：

```java
// ❌ 只覆写 equals，没覆写 hashCode
Card c1 = new Card(1, "♠");
Card c2 = new Card(1, "♠"); // 内容相同

Set<Card> set = new HashSet<>();
set.add(c1);
set.contains(c2); // false！因为 hashCode 不同，根本不会调用 equals
```

**规则**：覆写 `equals` 就必须同时覆写 `hashCode`，保证 `equals` 相等的两个对象 `hashCode` 也相等。

### 陷阱 3：Comparable 和 Comparator 不一致

如果一个类同时实现了 `Comparable` 并且使用时又传了 `Comparator`，以 `Comparator` 为准。但如果两者定义的顺序不一致，可能导致 `TreeSet` 和 `equals` 的行为矛盾（两个 `equals` 相等的对象在 TreeSet 中被当作不同元素）。

## 小结

| 概念 | 核心要点 |
|------|---------|
| `==` | 引用类型比较地址，不是内容 |
| `equals` | 默认也是比地址，需要覆写才能比内容 |
| `Comparable` | 类内部定义自然排序，实现 `compareTo` |
| `Comparator` | 外部定义排序策略，实现 `compare`，更灵活 |
| PriorityQueue | 优先用 Comparator，没有则用 Comparable，都没有报错 |

**关键认知**：

- Java 的对象比较不是天然支持的——你必须告诉 JVM "怎么比"
- `Comparable` 适合定义"默认排序"（一个类只有一个），`Comparator` 适合定义"灵活排序"（可以有多个）
- 实际开发中，`Comparator` + Lambda 是最常用的方式——简洁、灵活、不侵入原类
- 覆写 `equals` 必须同时覆写 `hashCode`——这是 HashMap/HashSet 正确工作的前提
