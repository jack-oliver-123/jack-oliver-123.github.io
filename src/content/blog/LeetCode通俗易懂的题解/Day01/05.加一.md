## 题目入口

题目链接：[加一](https://leetcode.cn/problems/plus-one/description/)

## 一句话读懂题目

给你一个用数组表示的非负整数，对它加一，返回结果数组。

数组的每个元素是一位数字，`digits[0]` 是最高位，`digits[digits.length - 1]` 是最低位。

比如：

```text
digits = [1, 2, 3]
```

表示数字 `123`，加一后变成 `124`，返回：

```text
[1, 2, 4]
```

再比如：

```text
digits = [9, 9]
```

表示数字 `99`，加一后变成 `100`，返回：

```text
[1, 0, 0]
```

注意最后这个例子：加一后位数变多了，数组需要变长。

## 先用生活例子理解

想象一个机械计数器，就像老式出租车上的里程表。

当前显示 `0 3 9`，你按一下加一：

- 最右边的 `9` 变成 `0`，同时向左边进一位；
- 中间的 `3` 变成 `4`；
- 进位停止。

结果是 `0 4 0`。

再想象当前显示 `9 9 9`，你按一下加一：

- 最右边 `9` 变 `0`，进位；
- 中间 `9` 变 `0`，继续进位；
- 最左边 `9` 变 `0`，还要进位；
- 但已经没有更高位了，需要多加一位。

结果是 `1 0 0 0`。

这就是这道题的全部逻辑：

> 从右往左加一，遇到进位就继续传播，没有进位就停下来。

## 核心思路：从末位开始处理进位

这道题不需要复杂的算法，关键是处理好进位。

从数组最后一位开始：

```text
把当前位加一
如果当前位变成了 10
    当前位设为 0，继续处理前一位（进位）
如果当前位没有变成 10
    不需要进位，直接返回
```

如果一路进位到最高位都处理完了，还有进位，说明需要扩展数组。

比如 `[9, 9, 9]` 加一后变成 `[1, 0, 0, 0]`。

这种情况有一个规律：

> 所有位都是 `9` 时，加一后一定变成 `1` 后面跟若干个 `0`。

所以只需要新建一个长度加一的数组，把第一位设为 `1`，其余默认就是 `0`。

## 手动走一遍例子

### 例子一：不需要进位

```text
digits = [1, 2, 3]
```

从最后一位开始：

```text
digits[2] = 3，加一变成 4，没有到 10
```

不需要进位，直接返回：

```text
[1, 2, 4]
```

### 例子二：需要进位但不扩容

```text
digits = [1, 2, 9]
```

从最后一位开始：

```text
digits[2] = 9，加一变成 10，设为 0，进位
digits[1] = 2，加一变成 3，没有到 10，停止
```

返回：

```text
[1, 3, 0]
```

### 例子三：需要扩容

```text
digits = [9, 9, 9]
```

从最后一位开始：

```text
digits[2] = 9，加一变成 10，设为 0，进位
digits[1] = 9，加一变成 10，设为 0，进位
digits[0] = 9，加一变成 10，设为 0，进位
```

所有位都处理完了还有进位，需要扩容。

新建数组 `[1, 0, 0, 0]`，返回。

把过程压缩成一张表：

| 步骤 | 当前位 | 原值 | 加一后 | 是否进位 | 数组状态 |
| --- | --- | --- | --- | --- | --- |
| 初始 | - | - | - | - | `[9, 9, 9]` |
| 1 | `2` | `9` | `10` → `0` | 是 | `[9, 9, 0]` |
| 2 | `1` | `9` | `10` → `0` | 是 | `[9, 0, 0]` |
| 3 | `0` | `9` | `10` → `0` | 是 | `[0, 0, 0]` |
| 扩容 | - | - | - | - | `[1, 0, 0, 0]` |

## Java 代码

先记住一句话：

> 从右往左走，不进位就停，全进位就扩。

```java
class Solution {
    public int[] plusOne(int[] digits) {
        for (int i = digits.length - 1; i >= 0; i--) {
            digits[i]++;

            if (digits[i] < 10) {
                return digits;
            }

            digits[i] = 0;
        }

        int[] result = new int[digits.length + 1];
        result[0] = 1;
        return result;
    }
}
```

## 代码逐行拆解

```java
for (int i = digits.length - 1; i >= 0; i--) {
```

从最后一位开始，从右往左遍历数组。

最低位就是 `digits[digits.length - 1]`。

```java
digits[i]++;
```

对当前位加一。

第一轮循环时，这就是题目要求的"加一"操作。

后续循环时，这是处理来自右边的进位。

```java
if (digits[i] < 10) {
    return digits;
}
```

加一后如果没有到 `10`，说明不需要进位了。

直接返回当前数组，后续的位不需要动。

```java
digits[i] = 0;
```

加一后变成了 `10`，说明这一位要变成 `0`，并且向左边进一位。

进位操作不需要额外变量，因为下一轮循环的 `digits[i]++` 就会处理进位。

```java
int[] result = new int[digits.length + 1];
result[0] = 1;
return result;
```

如果循环走完了还没有 `return`，说明每一位加一后都产生了进位。

这意味着原数组每一位都是 `9`，加一后变成 `10...0`。

Java 的 `int[]` 默认值就是 `0`，所以只需要把第一位设为 `1`。

## 为什么不需要额外的 carry 变量

很多人写进位相关的题目习惯用一个 `carry` 变量。

这道题不需要，因为：

```text
每一位的取值范围是 0 到 9
加一后最多变成 10
进位最多是 1
```

所以进位要么是 `0`（不需要进位），要么是 `1`（继续循环处理下一位）。

循环本身就承担了"传播进位"的角色：

- 如果不进位，`return` 提前退出；
- 如果进位，把当前位设为 `0`，循环自然进入下一位。

## 为什么扩容后一定是 1 后面跟 0

能走到循环外面，说明每一位都从 `9` 变成了 `0`。

那原数组一定是全 `9`：

```text
[9]       -> 加一后 -> [1, 0]
[9, 9]    -> 加一后 -> [1, 0, 0]
[9, 9, 9] -> 加一后 -> [1, 0, 0, 0]
```

所以新数组的长度一定是原数组长度加一，且第一位是 `1`，其余是 `0`。

不存在类似 `[8, 9, 9]` 走到循环外面的情况，因为 `8 + 1 = 9 < 10`，在第一位就会提前 `return`。

## 复杂度分析

时间复杂度：`O(n)`。

最坏情况下（全是 `9`），需要遍历整个数组。

平均情况下，很多时候只需要处理最后一位就返回了。

空间复杂度：`O(1)`（不算返回值的空间）。

大多数情况直接修改原数组返回，不需要额外空间。

只有全 `9` 的情况需要新建一个长度加一的数组，这属于返回值本身的空间。

## 常见坑

### 1. 忘记处理全 9 的情况

如果只在原数组上操作，不考虑扩容：

```text
digits = [9, 9] -> 处理后变成 [0, 0]
```

返回 `[0, 0]`，但正确答案是 `[1, 0, 0]`。

### 2. 从左往右遍历

加一是从最低位（最右边）开始的。

如果从左往右遍历，逻辑会变得非常复杂，因为你不知道进位会不会传过来。

### 3. 把数组转成整数再加一

有人想把 `[1, 2, 3]` 转成整数 `123`，加一变成 `124`，再转回数组。

这样做的问题是：

```text
数组可能非常长，超过 int 甚至 long 的范围
```

比如一个 100 位的数字，`long` 只能存 19 位。

所以必须逐位处理，不能转换成数字。

### 4. 进位时忘记把当前位设为 0

`9 + 1 = 10`，要把当前位设为 `0`，然后进位。

如果忘了设为 `0`，当前位会留着 `10`，但每一位只能是一位数。

## 面试时怎么讲

### 30 秒版

> 从数组最后一位开始加一。如果加一后小于 `10`，说明没有进位，直接返回。如果等于 `10`，把当前位设为 `0`，继续处理前一位。如果所有位都进位了，说明原数组全是 `9`，新建一个长度加一的数组，第一位设为 `1` 即可。时间 `O(n)`，空间 `O(1)`。

### 面试官可能追问

**为什么不用 carry 变量？**

因为每一位最大是 `9`，加一最多变成 `10`，进位只可能是 `1`。

循环本身就是进位的传播机制：不进位就 `return`，进位就继续下一轮循环。代码更简洁。

**能不能处理"加 k"而不只是"加一"？**

可以，但逻辑要改：

```text
carry = k
每一位的值变成 (digits[i] + carry)
当前位保留 (digits[i] + carry) % 10
carry 变成 (digits[i] + carry) / 10
```

这时就需要 `carry` 变量了，因为进位可能大于 `1`。

**数组开头会不会有前导零？**

题目保证不会。输入代表的是一个合法的正整数，所以第一位不会是 `0`（除非数组只有一个元素 `[0]`）。

**扩容时为什么不需要把原数组的值复制过去？**

因为能走到扩容这一步，原数组的每一位都已经被设成了 `0`。

而新数组默认值也是 `0`，只需要把第一位改成 `1`。

## 最后总结

加一这道题看起来很简单，但它考察的是对进位传播的理解：

> 关键不在于"加一"这个操作本身，而在于进位会不会向高位传播、传播到哪里停下来。

代码模板可以这样记：

```text
从右往左遍历
    当前位加一
    如果没到 10，直接返回
    如果到了 10，当前位设 0，继续循环
循环结束还没返回，说明全是 9，扩容
```

刷题时可以用下面几个边界用例自检：

```text
digits = [1, 2, 3] -> [1, 2, 4]（不进位）
digits = [1, 2, 9] -> [1, 3, 0]（进一位）
digits = [9, 9, 9] -> [1, 0, 0, 0]（全进位，扩容）
digits = [0] -> [1]（单个零）
digits = [9] -> [1, 0]（单个九）
```

最后把这句话记住：

> 加一不难，难的是知道进位什么时候停。不进位就收手，全进位就扩容。
