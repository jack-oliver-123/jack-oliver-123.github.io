# 面试全流程模块

当 `$ARGUMENTS.type` 为 `mixed` 或用户要求"全流程准备"时加载。
简历模式下默认生成本模块的"自我介绍"和"反问环节"部分。

---

## 自我介绍

### 面试官在听什么

| 时长 | 用途 | 面试官关注点 |
|------|------|-------------|
| 30秒版 | 电话面/群面简述 | 能否一句话定位自己 |
| 1分钟版 | 最常用 | 技术栈 + 亮点项目 + 求职动机 |
| 3分钟版 | 终面/详细了解 | 职业轨迹 + 成长故事 + 价值主张 |

### 1分钟版结构（核心）

```
[一句话定位] + [核心技术栈] + [亮点项目/成果] + [求职动机]
```

**示例（社招3年Java开发）**：
> 我是XX，3年Java后端开发经验，主要技术栈是 Spring Cloud + MySQL + Redis。上一家公司负责订单系统重构，把接口响应从800ms优化到120ms，日处理订单从10万提升到50万。目前希望加入技术挑战更大的团队，在分布式和高可用方向深入发展。

**示例（实习生/应届-软件测试）**：
> 我是XX，XX大学计算机专业大四，之前在XX公司测试组实习了6个月，主要负责接口自动化测试，用 Python + pytest 搭建了自动化框架，覆盖了核心业务80%的接口。这次面试贵司的测试岗，是因为想在自动化测试和性能测试方向继续深入。

### 生成规则（简历模式）

从简历中提取以下要素，生成 1 分钟版自我介绍：
1. 定位：年限 + 方向（从简历推断）
2. 技术栈：取 Top 3-4 核心技术（与目标岗位匹配的优先）
3. 亮点：最近/最相关项目的一句话成果（有数据最佳）
4. 动机：与目标公司/岗位的连接点（如无法推断，用通用表述）

### 常见失败模式

- ❌ 背简历：把简历从头到尾念一遍 → 面试官已经看过，浪费时间
- ❌ 太谦虚："我能力一般，还在学习中" → 不是面试该说的话
- ❌ 太空泛："我热爱编程，乐于学习" → 所有人都这么说
- ❌ 太长：超过1.5分钟面试官就走神了
- ❌ 跟岗位无关：面试测试岗却大篇幅讲开发经历

### 追问预测

面试官听完自我介绍后最可能追问：
- "你提到的XX项目，具体说说？" → 准备项目综合题
- "为什么想离开上一家？" → 准备离职原因
- "你觉得自己最大的优势是？" → 准备一个具体实例佐证

## 反问环节

### 原则

反问 ≠ 随便问问题。反问是你最后一次展示的机会，也是收集信息帮你做 offer 决策的机会。

### 推荐问题库（按面试轮次）

**技术一面（面试官通常是未来同事/leader）**：
- "团队现在主要在做什么方向？我进来后可能负责哪块？" — 了解实际工作内容
- "团队的技术栈是什么？有没有计划迁移或引入新技术？" — 了解技术氛围
- "新人一般多久能独立承担任务？有 mentor 制度吗？" — 了解成长支持

**技术二面/主管面**：
- "团队未来半年的重点方向是什么？" — 展示你关注全局
- "您觉得这个岗位做好的核心能力是什么？" — 暗示你想做好
- "团队的 code review 和技术分享是怎么做的？" — 展示工程素养

**HR面/终面**：
- "这个岗位的晋升路径是怎样的？" — 展示长期意向
- "团队加班强度大概是怎样的？" — 合理关注（正常问）
- "入职后前三个月的期望是什么？" — 展示积极主动

### 禁忌问题

- ❌ "贵公司是做什么的？" → 没做功课
- ❌ "面试我表现怎么样？" → 让面试官为难
- ❌ 一面就问薪资/假期（HR面再问）
- ❌ "没什么想问的" → 显得不关心这个机会

### 生成规则

简历模式下，根据目标岗位 + level 从上述问题库中选取 3-5 个最合适的反问建议，标注适用轮次。

## 薪资谈判

### 核心原则

1. **不要先报价**：尽量让对方先出数字
2. **说范围不说精确值**：给自己留空间
3. **总包思维**：base + 奖金 + 股票/期权 + 福利一起看

### 常见问题应对

**"你的期望薪资是多少？"**

| 策略 | 话术 | 适用场景 |
|------|------|---------|
| 反问 | "想先了解一下贵司这个岗位的薪资范围？" | 初面/不确定市场价时 |
| 范围 | "结合我的经验和市场情况，期望在 X-Y 之间" | 明确知道市场价时 |
| 锚定 | "我目前的 package 是 X，期望有 20-30% 的涨幅" | 有谈判筹码时 |

**"你现在的薪资是多少？"**
- 法律上你没有义务回答（多数地区）
- 安全回答："我目前的总包在XX范围，但更看重贵司这个岗位的发展空间和整体 package"
- 如果对方坚持要数字：可以给总包范围（比精确 base 安全）

**"这个数字超出我们的预算了"**
- "理解。除了 base 之外，是否可以在年终奖/期权/签字费上有调整空间？"
- "我对这个机会很感兴趣，是否可以在试用期后根据表现调薪？"

### 实习生/应届生特殊说明

- 应届生薪资通常是固定 offer，谈判空间有限
- 重点谈：转正标准、转正后薪资、房补/餐补等福利
- 多个 offer 时可以用来议价："我还有XX公司的 offer，薪资是..."

### 生成规则

- 仅在 `route=完整` 或用户明确要求时输出薪资谈判部分
- 根据 level 调整话术推荐（实习生 vs 社招）
- 标注"薪资谈判因地区/行业差异大，以下为通用参考"

## 技术题应对策略（不会/不确定时）

面试中遇到超出经验范围的技术题是常态，尤其对应届生/实习生。以下策略帮助候选人在"不知道"时仍能得分。

### 核心原则

**坦诚 + 展示思维过程 > 乱猜 > 沉默**

### 分级应对策略

| 情况 | 策略 | 话术模板 |
|------|------|---------|
| 完全不知道 | 坦诚 + 关联已知 | "这个我没实际用过，但根据我对XX的理解，我猜测它可能是..." |
| 知道概念不知细节 | 说出已知边界 | "我知道它是用来解决XX问题的，具体实现细节我还没深入研究过" |
| 用过但忘了细节 | 描述使用场景 | "我在XX项目中用过，当时的场景是...，但具体参数/配置我现在记不清了" |
| 没用过但了解替代方案 | 横向对比 | "我没用过X，但我用过Y来解决类似问题，区别可能在于..." |

### 加分技巧

- **展示学习路径**："如果要用这个技术，我会先看官方文档的XX部分，然后搭个demo验证"
- **承认边界后反问**："这块我确实不熟，您能简单介绍一下吗？"（展示好奇心，但只用一次）
- **否定路径有价值**："我没选XX是因为当时考虑到YY问题，但我知道它在ZZ场景下更合适"

### 绝对禁忌

- ❌ 编造答案（面试官追问2层就露馅，且全场信任崩塌）
- ❌ 长时间沉默不说话（超过10秒没任何输出）
- ❌ "这个我没学过"然后停住（没有展示任何思考）
- ❌ 反复说"我不太确定"但不给出任何推测

### 生成规则

简历模式下，为薄弱探测题（模块一中标注[薄弱探测]的题目）自动附加"不会时的应对建议"。

## 输出文件规则

本模块内容输出到 `{序号}-interview-flow.md`，包含：
- 定制版 1 分钟自我介绍（简历模式必输出）
- 反问推荐清单（简历模式必输出）
- 薪资谈判指南（route=完整 或用户要求时输出）

不单独输出文件时（route=速成），在 `00-resume-analysis.md` 末尾附加一段精简版自我介绍建议。
