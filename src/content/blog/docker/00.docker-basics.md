## Docker 是什么

Docker 是一个容器化平台，能把应用和它的依赖打包在一起，在任何安装了 Docker 的机器上以相同方式运行。对开发者来说，最大的好处是：**一次配置，到处运行**——不再纠结于环境差异。

## 核心概念

| 概念 | 说明 |
|------|------|
| **镜像（Image）** | 只读模板，包含运行应用所需的所有文件和配置。类似操作系统的 ISO 镜像 |
| **容器（Container）** | 镜像的运行实例。可以启动、停止、删除。类似一台轻量虚拟机 |
| **数据卷（Volume）** | 持久化存储，容器删除后数据不会丢失 |
| **Docker Hub** | 官方镜像仓库，提供 MySQL、Redis、Nginx 等常用镜像 |

简单关系：`镜像 → 启动 → 容器 → 挂载 → 数据卷`

## 安装 Docker

### Linux（Ubuntu/Debian）

```bash
# 安装依赖
sudo apt update
sudo apt install -y ca-certificates curl gnupg

# 添加 Docker 官方 GPG 密钥和仓库
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker Engine
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 免 sudo 执行 docker（需重新登录生效）
sudo usermod -aG docker $USER
```

### Windows / Mac

下载并安装 [Docker Desktop](https://www.docker.com/products/docker-desktop/)。安装后确认使用 WSL 2 后端（Windows）或 Apple Hypervisor（Mac）。

### 验证安装

```bash
docker --version
docker run hello-world
```

如果看到 `Hello from Docker!`，说明安装成功。

## 常用命令速查

### 镜像管理

```bash
# 搜索镜像
docker search mysql

# 拉取镜像
docker pull mysql:8.0

# 查看本地镜像
docker images

# 删除镜像
docker rmi mysql:8.0
```

### 容器管理

```bash
# 启动容器（后台运行）
docker run -d --name my-mysql -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 mysql:8.0

# 查看运行中的容器
docker ps

# 查看所有容器（包括已停止）
docker ps -a

# 停止容器
docker stop my-mysql

# 启动已停止的容器
docker start my-mysql

# 删除容器（需先停止）
docker rm my-mysql

# 强制删除运行中的容器
docker rm -f my-mysql

# 查看容器日志
docker logs -f my-mysql

# 进入容器内部
docker exec -it my-mysql bash
```

### 数据卷

```bash
# 查看所有数据卷
docker volume ls

# 清理未使用的资源
docker system prune
```

## docker run 参数说明

```bash
docker run -d \
  --name my-app \
  -p 8080:80 \
  -v ./html:/usr/share/nginx/html \
  -e MY_VAR=value \
  --restart unless-stopped \
  nginx:1.26
```

### `-v` 路径写法（重要）

`-v` 参数的格式是 `宿主机路径:容器路径`。**Windows 下路径写错是最常见的坑：**

| 写法 | 效果 |
|------|------|
| `-v ./data:/var/lib/mysql` | ✅ 相对路径，数据在当前目录下的 `data` 文件夹 |
| `-v D:/mysql-env/data:/var/lib/mysql` | ✅ Windows 绝对路径，用正斜杠 |
| `-v /data:/var/lib/mysql` | ❌ 映射到 WSL 2 虚拟机内部，Windows 文件管理器里看不到 |

Linux 下 `/opt/mysql/data` 这种绝对路径是正确的。但 Windows 下如果你写 `/data`，数据会存到 WSL 虚拟机里而不是你的 Windows 磁盘上——容器能跑，但你找不到文件。

**推荐做法**：在工作目录下执行命令，统一用 `./` 相对路径，数据就在你能看到的位置。

常用 `--restart` 策略：

| 策略 | 说明 |
|------|------|
| `no` | 默认，不自动重启 |
| `always` | 容器退出后总是重启 |
| `unless-stopped` | 除非手动停止，否则自动重启 |
| `on-failure:3` | 非正常退出时重启，最多 3 次 |

## 国内镜像加速（可选）

如果拉取镜像速度慢，可以配置镜像加速器。编辑 `/etc/docker/daemon.json`（Linux）或在 Docker Desktop 设置中添加：

```json
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://hub-mirror.c.163.com"
  ]
}
```

配置后重启 Docker：

```bash
sudo systemctl restart docker
```
