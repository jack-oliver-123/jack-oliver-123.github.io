## 适用场景

Sentinel 用于流量控制、熔断降级、系统保护和热点参数防护。本文记录如何使用 Docker 部署 Sentinel Dashboard 控制台。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建工作目录：

```bash
# Linux
mkdir -p /opt/sentinel/logs

# Windows（PowerShell）
mkdir D:/sentinel-env/logs
```

## 编写 docker-compose

新建 `docker-compose.yml`：

```yaml
services:
  sentinel:
    image: bladex/sentinel-dashboard:1.8.6
    container_name: sentinel-local
    restart: always
    ports:
      - "8858:8858"
      - "8719:8719"
    volumes:
      - ./logs:/root/logs
    environment:
      - SERVER_PORT=8858
      - JAVA_OPTS=-Dsentinel.dashboard.auth.username=sentinel -Dsentinel.dashboard.auth.password=your_sentinel_password -Xms128m -Xmx256m
      - TZ=Asia/Shanghai
    deploy:
      resources:
        limits:
          memory: 512M
```

端口说明：

- `8858`：Sentinel Dashboard Web 控制台端口；
- `8719`：客户端与控制台通信的默认端口。

> `bladex/sentinel-dashboard` 是社区维护的 Docker 镜像（[Docker Hub 页面](https://hub.docker.com/r/bladex/sentinel-dashboard)），阿里官方没有提供现成镜像。

## 启动 Sentinel

```bash
docker compose up -d
```

查看日志：

```bash
docker logs -f sentinel-local
```

## 登录控制台

浏览器访问：

```text
http://localhost:8858
```

登录信息：

```text
用户名: sentinel
密码: your_sentinel_password
```

刚登录时左侧菜单可能为空，这是正常现象。只有应用接入 Sentinel 并产生访问流量后，控制台才会显示对应应用。

## 应用接入示例

Spring Cloud Alibaba 项目引入依赖：

```xml
<dependency>
  <groupId>com.alibaba.cloud</groupId>
  <artifactId>spring-cloud-starter-alibaba-sentinel</artifactId>
</dependency>
```

配置示例（`application.yml`）：

```yaml
spring:
  cloud:
    sentinel:
      transport:
        dashboard: localhost:8858
        port: 8719
```

> 如果应用也在 Docker 中运行，`dashboard` 地址需要改为 `sentinel-local:8858`（同一网络下）或 `host.docker.internal:8858`（Windows/Mac）。

### 验证接入

1. 启动应用；
2. 访问几次接入 Sentinel 的接口；
3. 刷新 Sentinel Dashboard；
4. 如果应用出现在左侧菜单，说明接入成功。

## 常见问题

### 控制台没有应用

Sentinel Dashboard 是懒加载机制，只有应用接入并产生流量后才会显示。

### 端口冲突

如果 `8858` 被占用，修改宿主机映射端口：

```yaml
ports:
  - "8868:8858"
```

访问地址改为 `http://localhost:8868`。

### 规则重启后丢失

Sentinel 默认规则存储在内存中，容器或应用重启后规则会丢失。解决方案：

- **Nacos 持久化**：将规则推送到 Nacos，应用启动时自动拉取；
- **文件持久化**：将规则写入本地文件，适合单机场景。

生产环境强烈建议接入 Nacos 做规则持久化。

## 版本兼容性

| Sentinel Dashboard | Spring Cloud Alibaba | Spring Boot |
|---|---|---|
| 1.8.6 | 2021.x / 2022.x | 2.x |
| 1.8.7+ | 2023.x | 3.x |

请根据项目的 Spring Cloud Alibaba 版本选择对应的 Sentinel Dashboard 版本。

## 停止和清理

```bash
docker compose down
```

## 后续建议

- 不要使用默认密码暴露到公网。
- 生产环境务必配置规则持久化（推荐 Nacos 方案）。
- 建议结合 Sentinel 的集群流控功能应对分布式限流场景。
