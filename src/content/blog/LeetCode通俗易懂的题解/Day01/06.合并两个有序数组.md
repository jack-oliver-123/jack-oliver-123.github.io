## 题目入口

题目链接：[合并两个有序数组](https://leetcode.cn/problems/merge-sorted-array/description/)

## 一句话读懂题目

给你两个有序数组 `nums1` 和 `nums2`，把 `nums2` 合并到 `nums1` 里，使合并后的 `nums1` 依然有序。

`nums1` 的长度已经预留了 `nums2` 的空间。具体来说：

```text
nums1 的实际长度是 m + n
前 m 个元素是 nums1 自己的有效元素
后 n 个位置是 0，用来放 nums2 的元素
```

比如：

```text
nums1 = [1, 2, 3, 0, 0, 0], m = 3
nums2 = [2, 5, 6],           n = 3
```

合并后 `nums1` 变成：

```text
[1, 2, 2, 3, 5, 6]
```

注意：不是返回一个新数组，而是直接把结果写进 `nums1`。

## 先用生活例子理解

想象你在超市排队结账。

队伍 A 有 3 个人：身高 `1米6`、`1米7`、`1米8`，已经按身高从矮到高站好了。

队伍 B 也有 3 个人：身高 `1米65`、`1米75`、`1米85`。

现在要把 B 队的人插进 A 队，合并成一个从矮到高的队伍。

如果从前面开始插，每插一个人，后面的人都得往后挪，很麻烦。

但如果从后面开始安排呢？

> 先看两队最高的人，谁高谁站到最后面。

这样每次都是往空位上放人，前面的人完全不用动。

这就是这道题的核心思路：从后往前填充。

## 为什么不能从前往后填充

如果从前往后比较，把较小的元素放到 `nums1[0]`、`nums1[1]`、...

问题是：`nums1` 前面的位置已经有有效元素了。

比如：

```text
nums1 = [1, 2, 3, 0, 0, 0]
nums2 = [2, 5, 6]
```

如果把 `nums2[0] = 2` 放到 `nums1[1]`，原来的 `nums1[1] = 2` 和 `nums1[2] = 3` 就被覆盖了。

要么额外开一个数组保存，要么先把元素往后搬。

两种做法要么浪费空间，要么浪费时间。

但如果从后往前填充，`nums1` 后面的位置本来就是空的（全是 `0`），不怕覆盖。

## 核心思路：逆向双指针

准备三个指针：

```text
p1：指向 nums1 有效部分的最后一个元素，初始值 m - 1
p2：指向 nums2 的最后一个元素，初始值 n - 1
p：指向 nums1 的最后一个位置，初始值 m + n - 1
```

每次比较 `nums1[p1]` 和 `nums2[p2]`：

```text
谁大，就把谁放到 nums1[p]，然后对应的指针和 p 都往前移一格
```

核心逻辑：

```text
如果 nums1[p1] > nums2[p2]
    nums1[p] = nums1[p1]
    p1--
否则
    nums1[p] = nums2[p2]
    p2--
p--
```

当 `p1` 或 `p2` 走完时，循环结束。

如果 `p2` 还没走完，说明 `nums2` 里还有更小的元素没放进来，把它们依次放到 `nums1` 前面。

如果 `p1` 还没走完，不用管，因为它们本来就在 `nums1` 的正确位置上。

## 手动走一遍例子

```text
nums1 = [1, 2, 3, 0, 0, 0], m = 3
nums2 = [2, 5, 6],           n = 3
```

初始状态：

```text
p1 = 2（指向 nums1[2] = 3）
p2 = 2（指向 nums2[2] = 6）
p  = 5（指向 nums1[5]）
```

### 第一步

比较 `nums1[p1] = 3` 和 `nums2[p2] = 6`：

`6` 更大，放到 `nums1[5]`：

```text
nums1 = [1, 2, 3, 0, 0, 6]
p2 = 1, p = 4
```

### 第二步

比较 `nums1[p1] = 3` 和 `nums2[p2] = 5`：

`5` 更大，放到 `nums1[4]`：

```text
nums1 = [1, 2, 3, 0, 5, 6]
p2 = 0, p = 3
```

### 第三步

比较 `nums1[p1] = 3` 和 `nums2[p2] = 2`：

`3` 更大，放到 `nums1[3]`：

```text
nums1 = [1, 2, 3, 3, 5, 6]
p1 = 1, p = 2
```

### 第四步

比较 `nums1[p1] = 2` 和 `nums2[p2] = 2`：

相等，取 `nums2` 的（取哪个都行），放到 `nums1[2]`：

```text
nums1 = [1, 2, 2, 3, 5, 6]
p2 = -1, p = 1
```

### 第五步

`p2 = -1`，`nums2` 已经全部放完，循环结束。

`nums1` 前面剩下的 `[1, 2]` 本来就在正确位置，不用动。

最终结果：

```text
[1, 2, 2, 3, 5, 6]
```

压缩成一张表：

| 步骤 | p1 | p2 | p | 比较 | 操作 | nums1 |
| --- | --- | --- | --- | --- | --- | --- |
| 初始 | `2` | `2` | `5` | - | - | `[1, 2, 3, 0, 0, 0]` |
| 1 | `2` | `2` | `5` | `3 vs 6` | 放 `6` 到 `[5]` | `[1, 2, 3, 0, 0, 6]` |
| 2 | `2` | `1` | `4` | `3 vs 5` | 放 `5` 到 `[4]` | `[1, 2, 3, 0, 5, 6]` |
| 3 | `2` | `0` | `3` | `3 vs 2` | 放 `3` 到 `[3]` | `[1, 2, 3, 3, 5, 6]` |
| 4 | `1` | `0` | `2` | `2 vs 2` | 放 `2` 到 `[2]` | `[1, 2, 2, 3, 5, 6]` |

## Java 代码

先记住一句话：

> 从后往前填，大的先站位，空位不浪费。

```java
class Solution {
    public void merge(int[] nums1, int m, int[] nums2, int n) {
        int p1 = m - 1;
        int p2 = n - 1;
        int p = m + n - 1;

        while (p1 >= 0 && p2 >= 0) {
            if (nums1[p1] > nums2[p2]) {
                nums1[p] = nums1[p1];
                p1--;
            } else {
                nums1[p] = nums2[p2];
                p2--;
            }
            p--;
        }

        while (p2 >= 0) {
            nums1[p] = nums2[p2];
            p2--;
            p--;
        }
    }
}
```

## 代码逐行拆解

```java
int p1 = m - 1;
int p2 = n - 1;
int p = m + n - 1;
```

三个指针：

- `p1` 指向 `nums1` 有效部分的末尾；
- `p2` 指向 `nums2` 的末尾；
- `p` 指向 `nums1` 整个数组的末尾，也就是下一个要填入的位置。

```java
while (p1 >= 0 && p2 >= 0) {
```

只要两个数组都还有元素没处理，就继续比较。

任意一个走完了就退出循环。

```java
if (nums1[p1] > nums2[p2]) {
    nums1[p] = nums1[p1];
    p1--;
}
```

`nums1[p1]` 更大，把它放到当前最后的位置 `nums1[p]`，然后 `p1` 往前移。

```java
} else {
    nums1[p] = nums2[p2];
    p2--;
}
```

`nums2[p2]` 更大或两者相等，把 `nums2[p2]` 放到 `nums1[p]`，然后 `p2` 往前移。

相等时取哪个都行，不影响正确性。

```java
p--;
```

每次填完一个位置，`p` 往前移一格。

```java
while (p2 >= 0) {
    nums1[p] = nums2[p2];
    p2--;
    p--;
}
```

如果 `nums2` 还有剩余元素没放完，把它们依次放到 `nums1` 前面。

为什么不需要处理 `p1` 的剩余？

因为 `nums1` 剩余的元素本来就在 `nums1` 里面，而且它们一定是最小的（已经有序），所以它们天然就在正确位置上。

## 为什么从后往前不会覆盖有效元素

这是很多人第一次看这个解法时的疑问。

关键观察：

```text
p 从 m + n - 1 开始
p1 从 m - 1 开始
```

`p` 一定大于等于 `p1`。

每次操作：

- 如果取的是 `nums1[p1]`，那么 `p` 和 `p1` 同时减一，差距不变；
- 如果取的是 `nums2[p2]`，那么只有 `p` 减一，`p1` 不动，`p` 和 `p1` 的差距缩小了一格。

所以 `p` 永远不会追上 `p1` 之前还没处理的元素。

换句话说：

> 写入位置 `p` 永远在有效元素 `p1` 的后面或同一位置，不可能覆盖还没读过的元素。

## 复杂度分析

时间复杂度：`O(m + n)`。

每个元素最多被处理一次，两个数组加起来一共 `m + n` 个元素。

空间复杂度：`O(1)`。

只用了三个指针变量，没有额外数组。

## 常见坑

### 1. 忘记处理 nums2 的剩余元素

主循环结束后，如果 `p2 >= 0`，说明 `nums2` 还有元素没放进去。

必须把它们放到 `nums1` 前面。

如果漏了这一步：

```text
nums1 = [4, 5, 6, 0, 0, 0], m = 3
nums2 = [1, 2, 3],           n = 3
```

主循环会把 `6, 5, 4` 放到后面，但 `nums2` 的 `[1, 2, 3]` 就丢了。

### 2. 从前往后合并导致覆盖

从前往后填充会覆盖 `nums1` 还没处理的有效元素。

除非先把 `nums1` 的有效部分复制出来，但那就需要 `O(m)` 额外空间。

### 3. 混淆 m 和 nums1.length

`m` 是 `nums1` 的有效元素个数，`nums1.length` 是 `m + n`。

指针初始化时要注意：

```text
p1 = m - 1       （不是 nums1.length - 1）
p  = m + n - 1   （等于 nums1.length - 1）
```

### 4. 边界情况：nums2 为空

如果 `n = 0`，`nums2` 没有元素，不需要任何操作。

代码不需要特判，因为 `p2 = -1`，两个循环都不会进入。

### 5. 边界情况：nums1 有效部分为空

如果 `m = 0`：

```text
nums1 = [0, 0, 0], m = 0
nums2 = [1, 2, 3], n = 3
```

主循环不会进入（`p1 = -1`），第二个循环把 `nums2` 全部复制到 `nums1`。

## 面试时怎么讲

### 30 秒版

> 两个有序数组合并到 `nums1`，从后往前用双指针填充。`p1` 指向 `nums1` 有效末尾，`p2` 指向 `nums2` 末尾，`p` 指向 `nums1` 最后的空位。每次比较 `nums1[p1]` 和 `nums2[p2]`，大的放到 `nums1[p]`。这样写入位置永远在有效元素后面，不会覆盖。最后如果 `nums2` 有剩余就补到前面。时间 `O(m+n)`，空间 `O(1)`。

### 面试官可能追问

**为什么要从后往前而不是从前往后？**

从前往后填充会覆盖 `nums1` 中还没处理的有效元素。从后往前填充利用了 `nums1` 末尾的空位，写入位置永远不会追上未处理的元素。

**为什么 `p1` 剩余时不用处理？**

`nums1` 剩余的元素本身就在 `nums1` 里，并且它们是最小的一批，已经在正确位置上了。

**如果两个数组不是有序的怎么办？**

先排序再合并。但那样时间复杂度至少是 `O((m+n) log(m+n))`。

这道题给定有序条件，就是为了让我们利用有序性做到 `O(m+n)`。

**相等时取 nums1 还是 nums2 的？**

取哪个都可以，不影响正确性。代码里写的是取 `nums2` 的（`else` 分支包含了相等情况）。

如果需要稳定性（相同元素保持原来的相对顺序），相等时应该取 `nums1` 的。但这道题不要求稳定性。

**能不能用 System.arraycopy 把 nums2 直接复制进去再排序？**

可以，但那样时间复杂度是 `O((m+n) log(m+n))`，没有利用数组有序的条件。面试时这种做法只能作为暴力解提一句，不能作为最终答案。

## 最后总结

合并两个有序数组这道题，核心技巧是：

> 利用 `nums1` 末尾的空位，从后往前填充，避免覆盖有效元素。

逆向双指针模板可以这样记：

```text
p1 指向 nums1 有效末尾
p2 指向 nums2 末尾
p 指向 nums1 最后一个空位
每次比较，大的放到 p 位置
最后把 nums2 剩余的补到前面
```

刷题时可以用下面几个边界用例自检：

```text
nums1 = [1,2,3,0,0,0], m=3, nums2 = [2,5,6], n=3 -> [1,2,2,3,5,6]
nums1 = [1], m=1, nums2 = [], n=0 -> [1]（nums2 为空）
nums1 = [0], m=0, nums2 = [1], n=1 -> [1]（nums1 有效部分为空）
nums1 = [4,5,6,0,0,0], m=3, nums2 = [1,2,3], n=3 -> [1,2,3,4,5,6]（nums2 全小）
nums1 = [1,2,3,0,0,0], m=3, nums2 = [4,5,6], n=3 -> [1,2,3,4,5,6]（nums2 全大）
```

最后把这句话记住：

> 有序合并不要抢前面的位置，从后面的空位开始安排，大的先站好，小的自然落位。
