## 适用场景

PostgreSQL 是功能强大的开源关系型数据库，支持 JSON、全文检索、地理空间等高级特性，广泛用于 Web 应用和数据分析场景。本文记录如何使用 Docker 快速部署 PostgreSQL。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建数据目录：

```bash
# Linux
mkdir -p /opt/postgres/data

# Windows（PowerShell）
mkdir D:/postgres-env/data
```

## 编写 docker-compose

新建 `docker-compose.yml`：

```yaml
services:
  postgres:
    image: postgres:16
    container_name: postgres-local
    restart: unless-stopped
    ports:
      - "5432:5432"
    volumes:
      - ./data:/var/lib/postgresql/data
    environment:
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=your_postgres_password
      - POSTGRES_DB=app_db
      - TZ=Asia/Shanghai
      - PGTZ=Asia/Shanghai
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 3s
      retries: 3
```

配置说明：

- `POSTGRES_USER`：超级用户名（默认 `postgres`）；
- `POSTGRES_PASSWORD`：超级用户密码（必填）；
- `POSTGRES_DB`：初始化时自动创建的数据库；
- `PGTZ`：设置数据库时区。

## 启动 PostgreSQL

```bash
docker compose up -d
```

查看日志：

```bash
docker logs -f postgres-local
```

看到 `database system is ready to accept connections` 说明启动完成。

## 验证连接

使用容器内 `psql` 连接：

```bash
docker exec -it postgres-local psql -U postgres -d app_db
```

测试查询：

```sql
SELECT version();
```

外部客户端连接：

```text
Host: 127.0.0.1（或服务器 IP）
Port: 5432
User: postgres
Password: your_postgres_password
Database: app_db
```

## 创建业务用户

```sql
-- 创建用户
CREATE USER app_user WITH PASSWORD 'your_app_password';

-- 授权
GRANT ALL PRIVILEGES ON DATABASE app_db TO app_user;

-- 连接到 app_db 后授权 schema
\c app_db
GRANT ALL ON SCHEMA public TO app_user;
```

## 初始化脚本（可选）

如果需要在容器首次启动时自动执行 SQL，可以挂载初始化脚本目录：

```yaml
volumes:
  - ./data:/var/lib/postgresql/data
  - ./init:/docker-entrypoint-initdb.d
```

把 `.sql` 文件放到 `./init` 目录下，容器首次启动时会按文件名顺序执行。

> 注意：仅在数据目录为空时执行，已有数据的容器不会重复执行初始化脚本。

## 常见问题

### 端口被占用

修改宿主机端口：

```yaml
ports:
  - "5433:5432"
```

连接时端口改为 `5433`。

### 数据目录权限问题（Linux）

如果容器启动失败，提示 `Permission denied`：

```bash
sudo chown -R 999:999 /opt/postgres/data
```

`999` 是容器内 postgres 用户的默认 UID。

### 中文排序不正确

创建数据库时指定 locale：

```sql
CREATE DATABASE app_db
  WITH ENCODING 'UTF8'
  LC_COLLATE = 'zh_CN.UTF-8'
  LC_CTYPE = 'zh_CN.UTF-8'
  TEMPLATE = template0;
```

> 注意：官方镜像默认只包含 `en_US.UTF-8`，中文 locale 需要自定义镜像或使用 ICU 排序规则（PostgreSQL 15+）。

## 数据备份与恢复

```bash
# 备份
docker exec postgres-local pg_dump -U postgres app_db > backup.sql

# 恢复
docker exec -i postgres-local psql -U postgres app_db < backup.sql
```

## 停止和清理

```bash
# 停止服务（保留数据）
docker compose down

# 停止并删除数据
docker compose down -v
rm -rf ./data
```

## 后续建议

- 不要在生产环境使用弱密码。
- 定期备份数据（`pg_dump` 或使用流复制）。
- 生产环境建议调整 `shared_buffers`、`work_mem` 等性能参数（可通过挂载 `postgresql.conf` 实现）。
- 需要高可用时请规划主从复制或使用 Patroni 集群方案。
