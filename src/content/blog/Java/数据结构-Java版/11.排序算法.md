# 排序算法

## 基本概念
- 稳定性：相同元素相对顺序不变
- 内部排序：数据全在内存
- 外部排序：数据在磁盘

## 插入排序
- 直接插入排序
  - O(n²)，稳定
  - 接近有序时最快
- 希尔排序
  - O(n^1.3)，不稳定
  - 分组预排序 + 直接插入

## 选择排序
- 直接选择排序
  - O(n²)，不稳定
  - 每次选最小放前面
- 堆排序
  - O(n log n)，不稳定
  - 升序建大堆

## 交换排序
- 冒泡排序
  - O(n²)，稳定
  - 相邻元素比较交换
- 快速排序
  - O(n log n)，不稳定
  - 分治思想
  - Hoare / 挖坑 / 前后指针
  - 优化：三数取中、小区间插排

## 归并排序
- O(n log n)，稳定
- 分治：先分后合
- 空间 O(n)
- 外部排序首选

## 非比较排序
- 计数排序
  - O(n + k)
  - 适合范围集中的整数

## 复杂度总结
- O(n²)：插入、选择、冒泡
- O(n log n)：堆、快排、归并
- 稳定：插入、冒泡、归并
- 不稳定：选择、堆、希尔、快排

## 排序的概念

**排序**：使一串记录按照某个关键字的大小递增或递减排列的操作。

### 稳定性

如果待排序序列中有两个相同的元素 r[i] 和 r[j]（i < j），排序后 r[i] 仍在 r[j] 前面，则称排序算法是**稳定的**；否则是**不稳定的**。

稳定性在实际开发中很重要——比如先按成绩排序再按班级排序，如果第二次排序不稳定，同班级内的成绩顺序就被打乱了。

### 内部排序 vs 外部排序

- **内部排序**：数据全部在内存中
- **外部排序**：数据太大无法全部加载到内存，需要在内外存之间移动数据

## 插入排序

### 直接插入排序

**核心思想**：像打扑克牌一样，每次将一个新牌插入到已排好序的手牌中正确的位置。

```java
public static void insertionSort(int[] arr) {
    for (int i = 1; i < arr.length; i++) {
        int key = arr[i];  // 待插入的"新牌"
        int j = i - 1;
        // 从后往前找插入位置，边找边后移
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;  // 插入到正确位置
    }
}
```

**特性**：

| 项目 | 值 |
|------|-----|
| 最好时间 | O(n)——已经有序，每个元素比较 1 次 |
| 最坏时间 | O(n²)——逆序，每个元素比较 i 次 |
| 空间 | O(1) |
| 稳定性 | ✅ 稳定 |

**关键结论**：**数据越接近有序，直接插入排序越快**。这一点是希尔排序优化的基础。

### 希尔排序（缩小增量排序）

**核心思想**：先用较大的 gap 将数组分组做预排序，使数组逐渐"接近有序"；然后缩小 gap，最终 gap=1 时做一次直接插入排序——此时数组已经基本有序，插入排序非常快。

```java
public static void shellSort(int[] arr) {
    int gap = arr.length;
    while (gap > 1) {
        gap = gap / 3 + 1;  // Knuth 序列
        // 对每组进行插入排序
        for (int i = gap; i < arr.length; i++) {
            int key = arr[i];
            int j = i - gap;
            while (j >= 0 && arr[j] > key) {
                arr[j + gap] = arr[j];
                j -= gap;
            }
            arr[j + gap] = key;
        }
    }
}
```

**特性**：

| 项目 | 值 |
|------|-----|
| 时间复杂度 | 约 O(n^1.3)（取决于 gap 序列） |
| 空间 | O(1) |
| 稳定性 | ❌ 不稳定（相同元素可能被分到不同组） |

**为什么比直接插入快？** gap > 1 时的预排序让大元素快速移动到数组后部（一次跳 gap 步），而不是像直接插入那样一步步挪。当 gap=1 时，数组已经"基本有序"，直接插入排序的时间接近 O(n)。

## 选择排序

### 直接选择排序

**核心思想**：每次从未排序区间中选出最小的元素，放到已排序区间的末尾。

```java
public static void selectionSort(int[] arr) {
    for (int i = 0; i < arr.length - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < arr.length; j++) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        // 交换
        int temp = arr[i];
        arr[i] = arr[minIdx];
        arr[minIdx] = temp;
    }
}
```

**特性**：

| 项目 | 值 |
|------|-----|
| 时间复杂度 | O(n²)——无论数据是否有序 |
| 空间 | O(1) |
| 稳定性 | ❌ 不稳定 |

直接选择排序思路简单但效率不高，实际中很少使用。

### 堆排序

**核心思想**：利用堆结构选择最值，比直接选择高效得多。

- **升序 → 建大根堆**
- **降序 → 建小根堆**

```java
public static void heapSort(int[] arr) {
    int n = arr.length;
    // 建大根堆
    for (int i = (n - 1 - 1) / 2; i >= 0; i--) {
        shiftDown(arr, i, n);
    }
    // 排序：堆顶与末尾交换，缩减堆，向下调整
    for (int end = n - 1; end > 0; end--) {
        int temp = arr[0];
        arr[0] = arr[end];
        arr[end] = temp;
        shiftDown(arr, 0, end);
    }
}

private static void shiftDown(int[] arr, int parent, int size) {
    int child = 2 * parent + 1;
    while (child < size) {
        if (child + 1 < size && arr[child + 1] > arr[child]) child++;
        if (arr[child] > arr[parent]) {
            int temp = arr[child];
            arr[child] = arr[parent];
            arr[parent] = temp;
            parent = child;
            child = 2 * parent + 1;
        } else {
            break;
        }
    }
}
```

**特性**：

| 项目 | 值 |
|------|-----|
| 时间复杂度 | O(n log n)——建堆 O(n) + n 次调整 O(log n) |
| 空间 | O(1) |
| 稳定性 | ❌ 不稳定 |

## 交换排序

### 冒泡排序

**核心思想**：相邻元素两两比较，大的往后"冒泡"。每一轮将最大的元素放到末尾。

```java
public static void bubbleSort(int[] arr) {
    for (int i = 0; i < arr.length - 1; i++) {
        boolean swapped = false;
        for (int j = 0; j < arr.length - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }
        if (!swapped) break; // 本轮无交换说明已有序
    }
}
```

**特性**：

| 项目 | 值 |
|------|-----|
| 最好时间 | O(n)——已有序，一轮无交换直接退出 |
| 最坏时间 | O(n²)——逆序 |
| 空间 | O(1) |
| 稳定性 | ✅ 稳定 |

### 快速排序

快速排序是实际工程中**使用最广泛的排序算法**——JDK 的 `Arrays.sort()` 对基本类型就使用了双轴快排。

**核心思想**：选一个**基准值（pivot）**，将数组分为"小于 pivot"和"大于 pivot"两部分，然后对两部分递归排序。这和二叉树的前序遍历结构非常相似。

下图展示了一次分区操作的过程——pivot 将数组分为左右两部分：

![快速排序分区过程](/images/data-structures/quicksort-partition.svg)

#### Hoare 法

```java
public static void quickSort(int[] arr, int left, int right) {
    if (left >= right) return;
    int pivotIdx = partition(arr, left, right);
    quickSort(arr, left, pivotIdx - 1);
    quickSort(arr, pivotIdx + 1, right);
}

private static int partition(int[] arr, int left, int right) {
    int pivot = arr[left];
    int i = left, j = right;
    while (i < j) {
        // 右边找小
        while (i < j && arr[j] >= pivot) j--;
        // 左边找大
        while (i < j && arr[i] <= pivot) i++;
        // 交换
        if (i < j) {
            int temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
        }
    }
    // pivot 归位
    arr[left] = arr[i];
    arr[i] = pivot;
    return i;
}
```

#### 前后指针法

```java
private static int partition2(int[] arr, int left, int right) {
    int pivot = arr[left];
    int prev = left;
    int cur = left + 1;
    while (cur <= right) {
        if (arr[cur] < pivot) {
            prev++;
            int temp = arr[prev];
            arr[prev] = arr[cur];
            arr[cur] = temp;
        }
        cur++;
    }
    // pivot 归位
    arr[left] = arr[prev];
    arr[prev] = pivot;
    return prev;
}
```

#### 快速排序优化

**1. 三数取中法选 pivot**

避免在有序/逆序数组上退化为 O(n²)：

```java
private static int medianOfThree(int[] arr, int left, int right) {
    int mid = left + (right - left) / 2;
    if (arr[left] > arr[mid]) swap(arr, left, mid);
    if (arr[left] > arr[right]) swap(arr, left, right);
    if (arr[mid] > arr[right]) swap(arr, mid, right);
    swap(arr, mid, left); // 把中位数放到 left 位置作为 pivot
    return arr[left];
}
```

**2. 小区间使用插入排序**

递归到很小的子数组时，递归开销比排序本身还大，切换到插入排序更高效：

```java
public static void quickSort(int[] arr, int left, int right) {
    if (right - left + 1 <= 16) { // 小于 16 个元素用插入排序
        insertionSort(arr, left, right);
        return;
    }
    int pivotIdx = partition(arr, left, right);
    quickSort(arr, left, pivotIdx - 1);
    quickSort(arr, pivotIdx + 1, right);
}
```

#### 快速排序非递归

用栈模拟递归过程：

```java
public static void quickSortNonRecursive(int[] arr) {
    Deque<Integer> stack = new ArrayDeque<>();
    stack.push(0);
    stack.push(arr.length - 1);

    while (!stack.isEmpty()) {
        int right = stack.pop();
        int left = stack.pop();
        if (left >= right) continue;

        int pivotIdx = partition(arr, left, right);

        // 先右后左入栈（栈后进先出，左边先处理）
        stack.push(pivotIdx + 1);
        stack.push(right);
        stack.push(left);
        stack.push(pivotIdx - 1);
    }
}
```

**快速排序特性**：

| 项目 | 值 |
|------|-----|
| 平均时间 | O(n log n) |
| 最坏时间 | O(n²)——有序数组 + 选端点做 pivot |
| 空间 | O(log n)（递归栈深度） |
| 稳定性 | ❌ 不稳定 |

## 归并排序

**核心思想**：分治法——先将数组递归拆分为两半，分别排序，再将两个有序数组合并为一个有序数组。

下图展示了归并排序的完整分治过程——先拆分到单个元素，再逐层合并为有序数组：

![归并排序分治过程](/images/data-structures/mergesort.svg)

```java
public static void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;
    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);         // 排左半
    mergeSort(arr, mid + 1, right);    // 排右半
    merge(arr, left, mid, right);      // 合并
}

private static void merge(int[] arr, int left, int mid, int right) {
    int[] temp = new int[right - left + 1];
    int i = left, j = mid + 1, k = 0;

    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) {
            temp[k++] = arr[i++];
        } else {
            temp[k++] = arr[j++];
        }
    }
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];

    System.arraycopy(temp, 0, arr, left, temp.length);
}
```

**特性**：

| 项目 | 值 |
|------|-----|
| 时间复杂度 | O(n log n)——永远是 O(n log n)，没有最坏退化 |
| 空间 | O(n)——需要临时数组 |
| 稳定性 | ✅ 稳定 |

**归并排序的独特优势**：它是唯一时间 O(n log n) 且稳定的排序。JDK 中 `Arrays.sort()` 对引用类型（Object[]）使用的就是归并排序的变体（TimSort），因为引用类型排序需要稳定性。

### 外部排序：归并的杀手级应用

当数据量远超内存容量时（比如 100GB 数据只有 1GB 内存），无法一次性加载排序。解决方案：

1. 将 100GB 文件切分为 200 个 512MB 的小文件
2. 每个小文件加载到内存中排序（可用任何内排序）
3. 对 200 个有序文件做**多路归并**——最终得到完整有序文件

归并排序是外部排序的核心算法，这是它存在的最大价值。

## 非比较排序：计数排序

**核心思想**：统计每个元素出现的次数，然后按序输出。

```java
public static void countingSort(int[] arr) {
    int min = arr[0], max = arr[0];
    for (int num : arr) {
        min = Math.min(min, num);
        max = Math.max(max, num);
    }

    int[] count = new int[max - min + 1];
    for (int num : arr) {
        count[num - min]++;
    }

    int idx = 0;
    for (int i = 0; i < count.length; i++) {
        while (count[i] > 0) {
            arr[idx++] = i + min;
            count[i]--;
        }
    }
}
```

**特性**：

| 项目 | 值 |
|------|-----|
| 时间复杂度 | O(n + k)，k 是数据范围 |
| 空间 | O(k) |
| 稳定性 | ✅ 稳定（需要额外处理） |
| 适用条件 | 数据范围集中的整数 |

计数排序的限制：只适用于整数且范围不能太大。如果排序 100 个数但范围是 0~10 亿，需要 10 亿大小的数组——完全不可行。

## 排序算法复杂度与稳定性总结

| 排序算法 | 最好 | 平均 | 最坏 | 空间 | 稳定性 |
|---------|------|------|------|------|--------|
| 直接插入 | O(n) | O(n²) | O(n²) | O(1) | ✅ 稳定 |
| 希尔排序 | O(n) | O(n^1.3) | O(n²) | O(1) | ❌ 不稳定 |
| 直接选择 | O(n²) | O(n²) | O(n²) | O(1) | ❌ 不稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ 不稳定 |
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ 稳定 |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | ❌ 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ 稳定 |

### 如何记忆稳定性

**稳定的排序**：插入、冒泡、归并——口诀"**插冒归**"（插帽龟🐢）

**不稳定的排序**：选择、堆、希尔、快排——口诀"**选堆希快**"（选对媳妇快）

### 排序算法选型

```
数据量小（< 50）？
├── 是 → 直接插入排序（简单高效）
└── 否 → 数据基本有序？
          ├── 是 → 直接插入排序 O(n)
          └── 否 → 需要稳定性？
                    ├── 是 → 归并排序
                    └── 否 → 快速排序（综合性能最好）
                    
特殊场景：
- 内存有限 → 堆排序（O(1) 空间）
- 数据在磁盘 → 归并排序（外部排序）
- 整数范围小 → 计数排序 O(n)
```

## Java 中的排序

JDK 的 `Arrays.sort()` 根据数据类型自动选择排序算法：

- **基本类型（int[] 等）**：双轴快速排序（Dual-Pivot Quicksort）——因为基本类型不需要稳定性
- **引用类型（Object[]）**：TimSort（归并排序的优化版）——因为引用类型需要稳定性

```java
int[] arr = {5, 3, 8, 1, 9};
Arrays.sort(arr); // 双轴快排

String[] strs = {"banana", "apple", "cherry"};
Arrays.sort(strs); // TimSort（稳定）

// 自定义排序：用 Comparator
Arrays.sort(strs, (a, b) -> b.compareTo(a)); // 降序
```

## 小结

| 类别 | 算法 | 核心思想 | 面试频率 |
|------|------|---------|---------|
| 插入类 | 直接插入、希尔 | 逐个插入到有序区间 | ⭐⭐ |
| 选择类 | 直接选择、堆排序 | 每次选最值 | ⭐⭐⭐ |
| 交换类 | 冒泡、快速排序 | 比较后交换 | ⭐⭐⭐⭐⭐ |
| 归并类 | 归并排序 | 分治：先分后合 | ⭐⭐⭐⭐ |

**关键认知**：

- 快速排序是综合性能最好的排序——平均 O(n log n)、O(log n) 空间、缓存友好
- 归并排序是唯一 O(n log n) 且稳定的——外部排序和引用类型排序的首选
- 堆排序是唯一 O(n log n) 且 O(1) 空间的——内存极限场景的选择
- 没有一种排序在所有维度都最优——排序算法的选择本质上是 trade-off
