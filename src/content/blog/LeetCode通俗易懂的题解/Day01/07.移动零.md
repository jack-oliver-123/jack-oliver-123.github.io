## 题目入口

题目链接：[移动零](https://leetcode.cn/problems/move-zeroes/)

## 一句话读懂题目

给你一个数组 `nums`，把所有的 `0` 移到数组末尾，同时保持非零元素的相对顺序。

必须在原数组上操作，不能新建数组。

比如：

```text
nums = [0, 1, 0, 3, 12]
```

处理后：

```text
[1, 3, 12, 0, 0]
```

非零元素 `1, 3, 12` 的相对顺序没变，零全部到了末尾。

## 先用生活例子理解

想象一排座位，有些座位上坐着人，有些是空的（`0` 就是空座）。

现在要求：

> 所有人往前坐，保持原来的先后顺序，空座全部挪到后面。

怎么做最快？

从前往后扫，遇到一个人，就让他坐到最前面的空位上。

扫完之后，前面全是人，后面全是空座。

这就是快慢指针的思路：

- 快指针负责找人（非零元素）；
- 慢指针负责指向下一个空位。

## 和前面题目的关系

这道题和 Day 1 的《移除元素》非常像。

《移除元素》是把所有等于 `val` 的元素移除，保留不等于 `val` 的元素。

这道题是把所有等于 `0` 的元素移到最后，保留非零元素的顺序。

区别在于：

```text
移除元素：后面的位置随便，不用管
移动零：后面的位置必须是 0
```

所以这道题可以分两步理解：

```text
1. 把非零元素按顺序收集到前面（和移除元素一样）
2. 把剩下的位置全部填 0
```

或者用一种更优雅的写法：直接交换，一步到位。

## 方法一：覆盖 + 补零

最直观的写法：

```text
slow 指向下一个可以写入的位置
fast 从前往后扫描
遇到非零元素，放到 nums[slow]，slow++
扫完后，slow 到末尾全部填 0
```

这和《移除元素》的思路完全一样，只是多了最后的补零步骤。

## 方法二：交换（推荐）

更好的写法是用交换代替覆盖：

```text
slow 指向下一个可以写入的位置
fast 从前往后扫描
遇到非零元素，把 nums[fast] 和 nums[slow] 交换，slow++
```

为什么交换更好？

因为交换会自动把 `0` 换到后面去，不需要额外补零。

交换完成后：

- `nums[0...slow-1]` 全是非零元素，保持原顺序；
- `nums[slow...末尾]` 全是 `0`。

## 手动走一遍例子

以交换写法为例：

```text
nums = [0, 1, 0, 3, 12]
```

```text
slow = 0
fast = 0
```

### 第一步：fast = 0

```text
nums[fast] = 0
```

是零，跳过。

### 第二步：fast = 1

```text
nums[fast] = 1，非零
```

交换 `nums[slow]` 和 `nums[fast]`：

```text
交换 nums[0] 和 nums[1]
[0, 1, 0, 3, 12] -> [1, 0, 0, 3, 12]
slow = 1
```

### 第三步：fast = 2

```text
nums[fast] = 0
```

是零，跳过。

### 第四步：fast = 3

```text
nums[fast] = 3，非零
```

交换 `nums[slow]` 和 `nums[fast]`：

```text
交换 nums[1] 和 nums[3]
[1, 0, 0, 3, 12] -> [1, 3, 0, 0, 12]
slow = 2
```

### 第五步：fast = 4

```text
nums[fast] = 12，非零
```

交换 `nums[slow]` 和 `nums[fast]`：

```text
交换 nums[2] 和 nums[4]
[1, 3, 0, 0, 12] -> [1, 3, 12, 0, 0]
```

遍历结束，最终结果：

## Java 代码

先记住一句话：

> 非零元素往前换，零自然被挤到后面。

### 写法一：交换（推荐）

```java
class Solution {
    public void moveZeroes(int[] nums) {
        int slow = 0;

        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != 0) {
                int temp = nums[slow];
                nums[slow] = nums[fast];
                nums[fast] = temp;
                slow++;
            }
        }
    }
}
```

### 写法二：覆盖 + 补零

```java
class Solution {
    public void moveZeroes(int[] nums) {
        int slow = 0;

        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != 0) {
                nums[slow] = nums[fast];
                slow++;
            }
        }

        for (int i = slow; i < nums.length; i++) {
            nums[i] = 0;
        }
    }
}
```

## 代码逐行拆解（交换写法）

```java
int slow = 0;
```

`slow` 表示下一个可以放置非零元素的位置。

也可以理解成：`nums[0...slow-1]` 全是已经排好的非零元素。

```java
for (int fast = 0; fast < nums.length; fast++) {
```

`fast` 从头到尾扫描每个元素。

```java
if (nums[fast] != 0) {
```

当前元素不是零，说明它应该往前放。

```java
int temp = nums[slow];
nums[slow] = nums[fast];
nums[fast] = temp;
```

交换 `nums[slow]` 和 `nums[fast]`。

交换后：

- `nums[slow]` 变成了非零元素（来自 `fast` 位置）；
- `nums[fast]` 变成了原来 `slow` 位置的值（一定是 `0` 或者 `fast == slow` 时就是自己）。

```java
slow++;
```

非零区域扩大一格。

## 为什么交换是安全的

在交换写法中，`slow` 位置的值一定是 `0`（或者 `slow == fast` 时交换自己和自己）。

为什么？

因为 `slow` 只有在遇到非零元素时才会前进。而 `slow` 到 `fast` 之间的元素全是跳过的零。

所以 `nums[slow]` 要么是 `0`（被跳过的），要么等于 `nums[fast]`（`slow == fast` 时）。

交换后，`0` 被换到了 `fast` 位置，而 `fast` 会继续往后走，`0` 自然就被留在后面了。

## 为什么非零元素的相对顺序不变

快指针从前往后扫描，遇到非零元素就依次放到 `slow` 位置。

`fast` 遇到的第一个非零元素放到 `nums[0]`，第二个放到 `nums[1]`，第三个放到 `nums[2]`……

这和原数组中非零元素的出现顺序完全一致。

## 复杂度分析

时间复杂度：`O(n)`。

`fast` 从头到尾扫描一遍数组，每个元素只处理一次。

空间复杂度：`O(1)`。

只用了 `slow`、`fast` 和一个临时变量 `temp`。

## 常见坑

### 1. 只覆盖不补零

如果用覆盖写法，忘了最后的补零循环：

```text
nums = [0, 1, 0, 3, 12]
覆盖后：[1, 3, 12, 3, 12]  ❌（后面还是原来的值）
补零后：[1, 3, 12, 0, 0]   ✓
```

### 2. 交换时 slow 没有自增

如果交换后忘了 `slow++`，`slow` 一直指向 `0`，所有非零元素都往同一个位置放，答案就全乱了。

### 3. 和移除元素搞混

移除元素不需要管后面的位置，所以用覆盖就够了。

移动零要求后面全是 `0`，所以要么补零，要么用交换。

### 4. 以为需要从后往前处理

不需要。从前往后用快慢指针就能一趟搞定。从后往前处理反而会破坏非零元素的相对顺序。

## 面试时怎么讲

### 30 秒版

> 用快慢指针，`fast` 从前往后扫描，`slow` 指向下一个非零位置。遇到非零元素就和 `slow` 位置交换，`slow++`。这样非零元素按原来的顺序排到前面，零自动被换到后面。时间 `O(n)`，空间 `O(1)`。

### 面试官可能追问

**为什么用交换而不是覆盖？**

交换写法一趟完成，不需要额外的补零循环。代码更简洁，操作次数也可能更少（如果数组末尾已经有很多零的话）。

**交换时会不会打乱非零元素的顺序？**

不会。`fast` 按顺序遇到的非零元素，依次放到 `slow = 0, 1, 2, ...` 的位置，顺序和原数组完全一致。

**如果数组没有零怎么办？**

比如：

```text
nums = [1, 2, 3]
```

每次 `fast` 都能进入 `if`，`slow` 和 `fast` 始终相等，交换就是自己和自己换，数组不变。

**如果数组全是零怎么办？**

比如：

```text
nums = [0, 0, 0]
```

`fast` 每次都不进入 `if`，`slow` 始终是 `0`，数组不变。本来就全是零，不需要动。

**这道题和移除元素有什么关系？**

移除元素是"把所有等于 val 的元素移除"，这道题是"把所有等于 0 的元素移到末尾"。

核心逻辑一样：快慢指针保留需要的元素。区别是移动零还要求后面的位置全填 `0`，所以用交换比覆盖更优雅。

**能不能用双指针从两端往中间走？**

不行。题目要求保持非零元素的相对顺序。如果从两端往中间走，可能会把后面的非零元素换到前面，打乱顺序。

## 最后总结

移动零这道题，本质上是快慢指针做原地分区：

> 把数组分成两段——前面是非零元素（保持原序），后面是零。

交换写法的模板：

```text
slow 指向下一个非零位置
fast 从前往后扫描
如果 nums[fast] != 0
    交换 nums[slow] 和 nums[fast]
    slow++
```

刷题时可以用下面几个边界用例自检：

```text
nums = [0, 1, 0, 3, 12] -> [1, 3, 12, 0, 0]
nums = [0] -> [0]（单个零）
nums = [1] -> [1]（单个非零）
nums = [0, 0, 0] -> [0, 0, 0]（全是零）
nums = [1, 2, 3] -> [1, 2, 3]（没有零）
nums = [0, 0, 1] -> [1, 0, 0]（零在前面）
```

最后把这句话记住：

> 不要想着怎么"移动零"，要想着怎么把非零元素往前收集，零自然就被挤到后面了。
