## 题目入口

题目链接：[两数之和](https://leetcode.cn/problems/two-sum/description/)

![image-20260707203736123](https://gitee.com/mu-mo_1_0/images/raw/master/image-20260707203736123.png)

## 一句话读懂题目

给你一个数组 `nums` 和一个目标值 `target`，你要从数组里找出两个数，让它们加起来刚好等于 `target`，最后返回这两个数的下标。

比如：

```text
nums = [2, 7, 11, 15]
target = 9
```

因为 `2 + 7 = 9`，所以答案是：

```text
[0, 1]
```

注意：题目要的是下标，不是数字本身。

题目还给了两个重要前提：

- 同一个元素不能使用两次；
- 每组输入只会对应一个答案。

也就是说，我们不用纠结“会不会有多个答案”，只要找到那一对下标就可以返回。

## 先用生活例子理解

可以把这道题想象成：你手里有一张购物清单，每个数字代表一个商品价格。

现在你有 `target` 元钱，想刚好买两个商品，把钱花完。

你看到一个商品价格是 `x`，那你其实不需要乱猜另一个商品是多少，因为另一个商品必须是：

```text
target - x
```

比如目标是 `9`，你看到 `2`，就会立刻想到：

```text
我还需要一个 7
```

所以这道题的核心不是“两个数怎么凑”，而是：

> 当我看到当前这个数时，之前有没有出现过它需要的那个搭子？

## 方法一：暴力枚举，最直觉但比较慢

最容易想到的方法是两层循环：

- 第一层选第一个数；
- 第二层选它后面的每一个数；
- 只要两个数加起来等于 `target`，就返回它们的下标。

代码如下：

```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        for (int i = 0; i < nums.length; i++) {
            for (int j = i + 1; j < nums.length; j++) {
                if (nums[i] + nums[j] == target) {
                    return new int[]{i, j};
                }
            }
        }
        return new int[0];
    }
}
```

这个方法很好理解，但问题也很明显：如果数组很长，每个数都要和后面的数配一遍，效率不高。

暴力法的时间复杂度是 `O(n^2)`，空间复杂度是 `O(1)`。

`O(n^2)` 可以理解成：如果有 100 个商品，你可能要尝试几千次组合；如果有 10000 个商品，尝试次数就会非常夸张。

## 方法二：哈希表，一边走一边找搭子

我们可以换个思路。

当遍历到当前数字 `nums[i]` 时，假设它是其中一个数，那么另一个数必须是：

```java
target - nums[i]
```

我们只需要快速判断：这个需要的数之前有没有出现过。

这时候就可以用 `HashMap`。

如果你还没学过 `HashMap`，可以先把它理解成一本“超级快的通讯录”：

```text
给它一个名字，它能很快告诉你这个名字对应的电话号码。
```

放到这道题里，就是：

```text
给它一个数字，它能很快告诉你这个数字之前出现在哪个下标。
```

所以 `HashMap` 里存的是：

```text
数字 -> 下标
```

核心流程只有三步：

```text
遍历每个数字 nums[i]
先计算它需要的搭子 need = target - nums[i]
如果 need 之前出现过，返回两个下标；否则记住当前数字和下标
```

比如遍历 `[2, 7, 11, 15]`，目标值是 `9`。

### 第一步：看到 2

当前数是 `2`，需要的搭子是：

```text
9 - 2 = 7
```

但是哈希表里还没有 `7`。

于是把 `2` 存进去：

```text
map = {2 -> 0}
```

### 第二步：看到 7

当前数是 `7`，需要的搭子是：

```text
9 - 7 = 2
```

这次发现哈希表里已经有 `2` 了，它的下标是 `0`。

当前 `7` 的下标是 `1`。

所以直接返回：

```text
[0, 1]
```

## 为什么要先查，再放？

代码里一定要注意顺序：先查找搭子，再把当前数放进哈希表。

原因是题目要求不能重复使用同一个元素。

比如：

```text
nums = [3]
target = 6
```

如果你先把 `3` 放进哈希表，再去查 `6 - 3 = 3`，就可能把同一个 `3` 当成两个数用了。

所以正确顺序是：

1. 先看之前有没有出现过 `target - nums[i]`；
2. 如果有，返回答案；
3. 如果没有，再把当前数字和下标存起来。

## Java 代码

先不用急着记代码，先记住一句话：

> 每来一个数，就问哈希表：你之前见过我的搭子吗？

```java
import java.util.HashMap;
import java.util.Map;

class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();

        for (int i = 0; i < nums.length; i++) {
            int need = target - nums[i];

            if (map.containsKey(need)) {
                return new int[]{map.get(need), i};
            }

            map.put(nums[i], i);
        }

        return new int[0];
    }
}
```

## 代码逐行拆解

```java
Map<Integer, Integer> map = new HashMap<>();
```

创建一个哈希表，用来记录已经见过的数字和它的下标。

```java
int need = target - nums[i];
```

当前数是 `nums[i]`，如果它想和别人凑成 `target`，那么另一个数就必须是 `need`。

```java
if (map.containsKey(need)) {
    return new int[]{map.get(need), i};
}
```

如果之前已经出现过 `need`，说明答案找到了。

`map.get(need)` 是之前那个数的下标，`i` 是当前数的下标。

```java
map.put(nums[i], i);
```

如果没找到搭子，就把当前数字存起来，等后面的数字来找它。

```java
return new int[0];
```

LeetCode 原题说明每组输入只会有一个答案，所以正常情况下不会走到这里。这里保留一个空数组返回，只是为了让 Java 方法在语法上所有路径都有返回值。

## 复杂度分析

### 暴力法

时间复杂度：`O(n^2)`。

因为每个数都要尝试和后面的数配对。

空间复杂度：`O(1)`。

因为没有额外使用会随着数组长度增长的数据结构。

### 哈希表法

时间复杂度：`O(n)`。

因为数组只遍历了一遍，哈希表查找平均是 `O(1)`。

通俗理解就是：暴力法像从头到尾翻购物清单找价格；哈希表像提前做了一本索引，想找哪个价格，直接查索引就行。

空间复杂度：`O(n)`。

最坏情况下，需要把很多数字都存进哈希表里。

这就是典型的“用空间换时间”：多用一个哈希表，换来更快的查找速度。

## 常见坑

### 1. 返回的是下标，不是数值

题目要返回的是：

```text
[0, 1]
```

不是：

```text
[2, 7]
```

### 2. 不能重复使用同一个元素

`nums[i]` 不能自己和自己配对。

所以哈希表写法里推荐先查再存。

### 3. 数组里可能有重复数字

比如：

```text
nums = [3, 3]
target = 6
```

答案是：

```text
[0, 1]
```

哈希表方法可以正确处理，因为第一个 `3` 会先存进去，第二个 `3` 来的时候就能找到它。

### 4. 数字可能是负数或 0

哈希表方法不怕负数，也不怕 `0`。

比如：

```text
nums = [-3, 4, 3, 90]
target = 0
```

`-3 + 3 = 0`，照样可以通过 `target - nums[i]` 找到搭子。

## 面试时怎么讲

### 30 秒版

可以这样表达：

> 我会用哈希表记录已经遍历过的数字和它的下标。遍历数组时，当前数字是 `nums[i]`，它需要的另一个数就是 `target - nums[i]`。如果这个数已经在哈希表中，说明找到了答案；否则把当前数字放入哈希表，继续往后遍历。这样只需要遍历一遍数组，时间复杂度是 `O(n)`，空间复杂度是 `O(n)`。

### 面试官可能追问

**为什么不用两层循环？**

两层循环可以做，但时间复杂度是 `O(n^2)`。使用哈希表后，可以把“查找另一个数”这件事从线性查找优化成平均 `O(1)` 查找。

**为什么哈希表里存下标，而不是只存数字？**

因为题目要求返回的是两个数的下标，所以找到搭子以后，需要立刻拿到它原来的位置。

**如果数组里有重复数字怎么办？**

没有问题。比如 `[3, 3]` 和 `target = 6`，第一个 `3` 会先存入哈希表，第二个 `3` 来的时候就能找到第一个 `3`。

**如果没有答案怎么办？**

LeetCode 原题保证有且只有一个答案。如果是在真实业务里，就需要根据需求返回空数组、`null`，或者抛出异常。

## 最后总结

两数之和是哈希表入门的经典题。

它真正想考的不是加法，而是这个思路：

> 与其拿当前数去和所有数硬凑，不如直接算出它缺谁，再去哈希表里找。

以后看到这类题，可以先提醒自己一句：

> 如果题目在问“某个数是否出现过”“另一个搭子在哪里”“能不能快速找到配对关系”，就优先考虑哈希表。

记住这个开关，后面很多“找另一半”“找是否出现过”“找配对关系”的题，都会变得清晰很多。
