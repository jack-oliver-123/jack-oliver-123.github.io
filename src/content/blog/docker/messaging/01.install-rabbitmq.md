## 适用场景

RabbitMQ 是功能丰富的消息队列中间件，支持多种消息协议，适合异步解耦、流量削峰和事件驱动架构。本文记录如何使用 Docker 部署 RabbitMQ 并启用管理控制台。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建工作目录：

```bash
# Linux
mkdir -p /opt/rabbitmq/data

# Windows（PowerShell）
mkdir D:/rabbitmq-env/data
```

## 编写 docker-compose

新建 `docker-compose.yml`：

```yaml
services:
  rabbitmq:
    image: rabbitmq:3.12-management
    container_name: rabbitmq-local
    restart: always
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      - RABBITMQ_DEFAULT_USER=admin
      - RABBITMQ_DEFAULT_PASS=your_rabbitmq_password
      - TZ=Asia/Shanghai
    volumes:
      - ./data:/var/lib/rabbitmq
    healthcheck:
      test: ["CMD", "rabbitmq-diagnostics", "check_running"]
      interval: 15s
      timeout: 10s
      retries: 3
```

端口说明：

- `5672`：应用连接 RabbitMQ 的 AMQP 端口；
- `15672`：RabbitMQ Web 管理控制台端口。

## 启动 RabbitMQ

```bash
docker compose up -d
```

查看容器状态：

```bash
docker ps
```

查看日志：

```bash
docker logs -f rabbitmq-local
```

> management 插件初始化需要几秒钟，看到 `Server startup complete` 后才能访问控制台。

## 登录管理控制台

浏览器访问：

```text
http://localhost:15672
```

使用 compose 中配置的账号登录：

```text
用户名: admin
密码: your_rabbitmq_password
```

## 应用接入示例

Spring Boot 项目引入依赖：

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-amqp</artifactId>
</dependency>
```

配置示例（`application.yml`）：

```yaml
spring:
  rabbitmq:
    host: 127.0.0.1
    port: 5672
    username: admin
    password: your_rabbitmq_password
    virtual-host: /
```

## 常见问题

### 管理页面打不开

检查：

- 容器是否已完全启动（`docker logs rabbitmq-local` 确认 `Server startup complete`）；
- `15672` 端口映射是否正确；
- 防火墙或安全组是否放行。

### 应用连接失败

排查步骤：

```bash
# 测试端口连通性
# Linux
nc -zv localhost 5672

# Windows（PowerShell）
Test-NetConnection localhost -Port 5672
```

确认用户名、密码、virtual-host 配置是否正确。

### 容器内应用连接

如果应用也在 Docker 中，不能用 `localhost`。应使用容器名 `rabbitmq-local`（同一网络下）或 `host.docker.internal`（Windows/Mac）。

## 停止和清理

```bash
# 停止服务（保留数据）
docker compose down

# 停止并删除数据（队列和用户配置会丢失）
docker compose down -v
```

## 后续建议

- 不要将真实密码提交到代码仓库。
- 生产环境建议设置内存告警阈值（`RABBITMQ_VM_MEMORY_HIGH_WATERMARK`）。
- 需要高可用时请规划集群模式和镜像队列策略。
- 持久化队列（durable）重启后消息不会丢失，非持久化队列则会。
