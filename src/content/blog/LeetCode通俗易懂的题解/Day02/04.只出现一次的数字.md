## 题目入口

题目链接：[只出现一次的数字](https://leetcode.cn/problems/single-number/)

## 一句话读懂题目

给你一个非空整数数组 `nums`，其中除了一个元素只出现一次以外，其余每个元素都出现了两次。

找出那个只出现一次的元素。

要求：时间 `O(n)`，空间 `O(1)`。

比如：

```text
nums = [2, 2, 1]
```

`1` 只出现一次，返回：

```text
1
```

再比如：

```text
nums = [4, 1, 2, 1, 2]
```

`4` 只出现一次，返回：

```text
4
```

## 先用生活例子理解

想象一群人要配对跳舞。

每个人都有一个舞伴（出现两次），只有一个人落单了。

怎么找出落单的那个人？

> 让所有人两两配对，配上的就离场。最后留在场上的就是那个落单的人。

异或运算就是这个"配对离场"的过程：

- 两个相同的数异或，结果是 `0`（配对成功，消失）；
- 任何数和 `0` 异或，结果是它自己（落单的人留下来）。

## 为什么异或能解决这道题

异或（XOR，符号 `^`）有三个关键性质：

```text
a ^ a = 0      相同的数异或为 0（自消除）
a ^ 0 = a      任何数和 0 异或等于自己
a ^ b = b ^ a  交换律，顺序无所谓
```

把数组里所有元素异或在一起：

- 出现两次的元素互相抵消变成 `0`；
- 最后剩下的就是那个只出现一次的元素。

比如：

```text
4 ^ 1 ^ 2 ^ 1 ^ 2
= 4 ^ (1 ^ 1) ^ (2 ^ 2)
= 4 ^ 0 ^ 0
= 4
```

## 手动走一遍例子

```text
nums = [4, 1, 2, 1, 2]
```

从左到右依次异或：

| 步骤 | 当前元素 | 运算 | 结果 |
| --- | --- | --- | --- |
| 初始 | - | - | `0` |
| 1 | `4` | `0 ^ 4` | `4` |
| 2 | `1` | `4 ^ 1` | `5` |
| 3 | `2` | `5 ^ 2` | `7` |
| 4 | `1` | `7 ^ 1` | `6` |
| 5 | `2` | `6 ^ 2` | `4` |

最终结果是 `4`，就是那个只出现一次的数字。

中间的 `5, 7, 6` 看起来没什么规律，但不用在意——异或的顺序不影响最终结果，所有成对的元素最终都会互相消掉。

## Java 代码

先记住一句话：

> 成对的异或为零，落单的留到最后。

```java
class Solution {
    public int singleNumber(int[] nums) {
        int result = 0;

        for (int num : nums) {
            result ^= num;
        }

        return result;
    }
}
```

## 代码逐行拆解

```java
int result = 0;
```

初始值设为 `0`。

因为 `0` 是异或的"零元素"：`0 ^ a = a`，不影响后续计算。

```java
for (int num : nums) {
    result ^= num;
}
```

把数组中所有元素依次异或到 `result` 上。

出现两次的元素会互相抵消为 `0`，只剩下出现一次的那个。

```java
return result;
```

遍历结束，`result` 就是只出现一次的元素。

## 为什么不能用 HashSet 或 HashMap

技术上可以，但空间复杂度是 `O(n)`，不满足题目要求的 `O(1)`。

```java
// HashMap 写法（能通过，但空间 O(n)）
Map<Integer, Integer> map = new HashMap<>();
for (int num : nums) {
    map.put(num, map.getOrDefault(num, 0) + 1);
}
for (int num : nums) {
    if (map.get(num) == 1) return num;
}
```

异或写法只用了一个变量 `result`，空间 `O(1)`，完美满足要求。

## 为什么不能用排序

排序后相同的元素相邻，可以两两比较找出落单的那个。

但排序的时间复杂度是 `O(n log n)`，不满足题目要求的 `O(n)`。

而且排序会修改原数组。

## 异或的更多直觉

如果你对异或还不太熟悉，可以这样理解：

异或就像一个"翻转开关"：

```text
按一次开关，灯亮了（0 ^ 1 = 1）
再按一次同样的开关，灯灭了（1 ^ 1 = 0）
```

数组里每个出现两次的数字就像按了两次同样的开关——效果互相抵消。

只有那个出现一次的数字，只按了一次开关，它的效果保留下来了。

## 复杂度分析

时间复杂度：`O(n)`。

遍历一次数组，每个元素做一次异或运算。

空间复杂度：`O(1)`。

只用了一个变量 `result`。

## 常见坑

### 1. 初始值写成别的数

`result` 初始值必须是 `0`。

如果写成 `1` 或者 `nums[0]`（然后从下标 1 开始也行，但容易写错），最安全的做法就是初始化为 `0`，从头开始异或。

### 2. 以为异或有顺序要求

异或满足交换律和结合律：

```text
a ^ b ^ c = c ^ a ^ b = b ^ c ^ a
```

不管数组里元素的顺序怎样，结果都一样。

### 3. 以为只能处理正数

异或对负数一样有效。

因为异或是按位运算，作用在每一个二进制位上，不区分正负。

### 4. 和"丢失的数字"搞混

两道题都用异或，但场景不同：

- 丢失的数字：把完整序列 `[0, n]` 和数组异或，消掉后剩余的是缺失的数；
- 只出现一次的数字：数组自身异或，成对消掉后剩余的是落单的数。

## 面试时怎么讲

### 30 秒版

> 利用异或的自消除性质：`a ^ a = 0`，`a ^ 0 = a`。把数组所有元素异或在一起，出现两次的互相抵消为 `0`，最后剩下的就是只出现一次的元素。时间 `O(n)`，空间 `O(1)`。

### 面试官可能追问

**如果有两个数字只出现一次呢？**

不能直接全部异或了，因为最终结果是两个落单数的异或值，分不出谁是谁。

需要找到异或结果中为 `1` 的某一位，用它把数组分成两组，每组分别异或就能得到各自的落单数。这是 LeetCode 260 题。

**如果有一个数字出现一次，其他出现三次呢？**

异或不再适用，因为三个相同的数异或不为 `0`（`a ^ a ^ a = a`）。

需要用位计数：统计每一位上 `1` 的个数，对 3 取模，剩下的就是那个出现一次的数。这是 LeetCode 137 题。

**异或的时间复杂度里，每次运算是 O(1) 吗？**

是的。对于固定位宽的整数（Java 的 `int` 是 32 位），一次异或运算就是一条 CPU 指令，`O(1)`。

**能不能用数学方法（求和）解？**

可以。去重后所有数字的和乘以 2，减去原数组的和，就是只出现一次的数。

```text
2 * (a + b + c) - (a + a + b + b + c) = c
```

但这需要用 `HashSet` 去重，空间 `O(n)`，不如异或优雅。

## 最后总结

只出现一次的数字，是异或运算最经典的应用场景：

> 成对出现的数互相消除，剩下的就是唯一的那个。

代码模板简单到只有三行：

```text
result = 0
遍历数组
    result ^= 当前元素
返回 result
```

刷题时可以用下面几个边界用例自检：

```text
nums = [2, 2, 1] -> 1
nums = [4, 1, 2, 1, 2] -> 4
nums = [1] -> 1（单元素）
nums = [-1, -1, -2] -> -2（负数）
nums = [0, 1, 0] -> 1（包含零）
```

以后看到"其他元素都出现两次，找出只出现一次的"，直接写全员异或就行。

最后把这句话记住：

> 异或就是配对消除。出现两次的自动清零，落单的自然浮出水面。
