## 题目入口

题目链接：[买卖股票的最佳时机](https://leetcode.cn/problems/best-time-to-buy-and-sell-stock/)

## 一句话读懂题目

给你一个数组 `prices`，其中 `prices[i]` 是某支股票第 `i` 天的价格。

你只能选择某一天买入，之后的某一天卖出（只能交易一次）。

返回你能获得的最大利润。如果无法盈利，返回 `0`。

比如：

```text
prices = [7, 1, 5, 3, 6, 4]
```

第 2 天买入（价格 `1`），第 5 天卖出（价格 `6`），利润 `6 - 1 = 5`，返回：

```text
5
```

再比如：

```text
prices = [7, 6, 4, 3, 1]
```

价格一路下跌，怎么买都亏钱，返回：

```text
0
```

## 先用生活例子理解

想象你每天早上路过菜市场，会看到今天白菜的价格。

你只能买一次、卖一次，而且必须先买后卖。

你的策略很简单：

> 记住你见过的最低价，然后每天都算一下"如果今天卖，能赚多少"。

如果今天的价格减去历史最低价比之前算过的最大利润还高，就更新最大利润。

走完一遍菜市场，你就知道最佳买卖时机了。

## 核心思路：一次遍历，维护最低价

只需要两个变量：

```text
minPrice：到目前为止见过的最低价格
maxProfit：到目前为止能获得的最大利润
```

遍历数组：

```text
每一天：
    如果今天价格比 minPrice 低 -> 更新 minPrice
    否则算一下今天卖出的利润 -> 和 maxProfit 比较，取更大的
```

为什么这样是对的？

因为对于每一天，我们都在问："如果今天卖，我能从之前的最低点买入获得多少利润？"

遍历完所有天，就比较了所有可能的买卖组合中利润最大的那个。

## 手动走一遍例子

```text
prices = [7, 1, 5, 3, 6, 4]
```

初始状态：

```text
minPrice = 7（第一天的价格）
maxProfit = 0
```

| 天数 | 价格 | minPrice | 今天卖的利润 | maxProfit |
| --- | --- | --- | --- | --- |
| 1 | `7` | `7` | `0` | `0` |
| 2 | `1` | `1` | `0` | `0` |
| 3 | `5` | `1` | `5 - 1 = 4` | `4` |
| 4 | `3` | `1` | `3 - 1 = 2` | `4` |
| 5 | `6` | `1` | `6 - 1 = 5` | `5` |
| 6 | `4` | `1` | `4 - 1 = 3` | `5` |

最终返回 `5`。

关键观察：

- 第 2 天价格跌到 `1`，更新了 `minPrice`；
- 第 5 天价格涨到 `6`，`6 - 1 = 5` 是最大利润。

## Java 代码

先记住一句话：

> 记住最低点，每天都试着卖一下，保留最赚的那次。

```java
class Solution {
    public int maxProfit(int[] prices) {
        int minPrice = prices[0];
        int maxProfit = 0;

        for (int i = 1; i < prices.length; i++) {
            if (prices[i] < minPrice) {
                minPrice = prices[i];
            } else {
                maxProfit = Math.max(maxProfit, prices[i] - minPrice);
            }
        }

        return maxProfit;
    }
}
```

## 为什么不需要两层循环

暴力做法是两层循环：外层选买入日，内层选卖出日，比较所有组合。

```java
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        maxProfit = Math.max(maxProfit, prices[j] - prices[i]);
    }
}
```

时间复杂度 `O(n²)`，大数组会超时。

一次遍历的做法之所以能替代两层循环，是因为：

> 对于每个卖出日 `j`，最优的买入日一定是 `j` 之前的最低价那一天。

所以只要维护一个"到目前为止的最低价"，就能在遍历到每一天时立刻算出最优利润。

## 复杂度分析

时间复杂度：`O(n)`。

遍历一次数组。

空间复杂度：`O(1)`。

只用了两个变量。

## 常见坑

### 1. 忘记处理价格一路下跌的情况

如果价格每天都在跌，任何时候卖出都是亏的。

这时 `maxProfit` 始终是 `0`，正确返回 `0`。

不要返回负数，题目说"无法盈利则返回 0"。

### 2. minPrice 初始值设错

`minPrice` 应该初始化为 `prices[0]`，不能初始化为 `0`。

如果初始化为 `0`，那"今天价格 - 0"永远是正的，会算出错误的利润。

也可以初始化为 `Integer.MAX_VALUE`，效果一样。

### 3. 先买后卖的顺序

题目要求买入一定在卖出之前。

我们从前往后遍历，`minPrice` 记录的是当前位置之前的最低价，所以天然满足"先买后卖"。

### 4. 以为需要记住买入和卖出的具体日期

题目只要求返回最大利润的数值，不需要返回具体哪天买、哪天卖。

## 面试时怎么讲

### 30 秒版

> 一次遍历，维护两个变量：`minPrice` 记录到目前为止的最低价格，`maxProfit` 记录到目前为止的最大利润。每到一天，要么更新最低价，要么用今天的价格减去最低价来更新最大利润。时间 `O(n)`，空间 `O(1)`。

### 面试官可能追问

**如果可以交易多次呢？**

那就是 LeetCode 122。策略变成：只要明天比今天贵，就今天买明天卖（贪心），所有上涨段的利润加起来。

**如果最多交易 k 次呢？**

那是 LeetCode 188，需要动态规划，状态是"第几天、已交易几次、手上有没有股票"。

**这道题能不能用动态规划的思路理解？**

可以。定义 `dp[i]` 为前 `i` 天的最大利润：

```text
dp[i] = max(dp[i-1], prices[i] - 前 i 天的最低价)
```

但因为只依赖前一天的状态和一个最低价，所以压缩成两个变量就行，也就是我们写的代码。

**为什么 else 分支不会漏掉"今天既是新低又能产生利润"的情况？**

不存在这种情况。如果今天是新低（`prices[i] < minPrice`），那今天卖出的利润是负数或零，不可能比之前的 `maxProfit` 更大。所以跳过卖出计算是安全的。

## 最后总结

买卖股票的最佳时机，核心思路是：

> 在每个可能的卖出日，找之前的最低买入价，利润自然最大。

代码模板：

```text
minPrice = prices[0]
maxProfit = 0
从第二天开始遍历
    如果今天更便宜 -> 更新 minPrice
    否则 -> maxProfit = max(maxProfit, 今天价格 - minPrice)
返回 maxProfit
```

刷题时可以用下面几个边界用例自检：

```text
prices = [7, 1, 5, 3, 6, 4] -> 5（低买高卖）
prices = [7, 6, 4, 3, 1] -> 0（一路下跌）
prices = [1, 2] -> 1（最简单的盈利）
prices = [2, 1] -> 0（只跌不涨）
prices = [3, 3, 3] -> 0（价格不变）
```

最后把这句话记住：

> 别想着预测未来，只需要记住过去的最低点，每天问自己"今天卖划不划算"。
