## 前言

在工作或学习中，我们经常会反复修改同一份内容。比如写一篇文档，第一版写完后，对方让你改；改到第三版、第四版时，又突然说第一版更好。

如果只是两三个版本，手动复制文件也许还能应付：

```txt
文档-第一版.docx
文档-第二版.docx
文档-最终版.docx
文档-最终版-真的最终版.docx
```

但如果一个项目改了几十次、几百次，手动保存副本就会变得混乱：

- 不知道每一版改了什么；
- 不知道是谁改的；
- 想回到某个历史版本很麻烦；
- 多个人一起改同一个项目时容易互相覆盖。

Git 就是为了解决这些问题而出现的版本控制工具。

## Git 是什么

Git 是一个分布式版本控制系统。你可以把它理解成一个“项目时间机器”：它会记录项目每一次重要修改，让你随时查看历史、比较差异、回退版本。

对于开发者来说，Git 通常用来管理：

- 项目源码；
- 技术文档；
- 配置文件；
- 博客文章；
- 任何需要长期维护和追踪变化的文本内容。

Git 不只适合代码，只要是需要版本管理的文件，都可以使用 Git 管理。

## Git 能解决什么问题

### 1. 记录每一次修改

Git 可以记录每次提交时的文件状态。你可以知道：

- 什么时候改了文件；
- 改了哪些内容；
- 为什么做这次修改。

### 2. 回到历史版本

如果你发现当前版本有问题，可以回到之前某一次提交时的状态。

### 3. 多人协作

多人可以同时开发同一个项目。Git 可以帮助合并不同人的修改，并在发生冲突时提醒你处理。

### 4. 分支开发

你可以创建一个分支，在分支上尝试新功能。确认没问题后，再合并回主分支。这样不会影响主线代码。

## Git 和 GitHub 的区别

很多初学者会把 Git 和 GitHub 混在一起，它们不是同一个东西。

- **Git**：安装在你电脑上的版本控制工具；
- **GitHub**：一个托管 Git 仓库的网站。

可以这样理解：

```txt
Git = 工具
GitHub = 放代码的网站
```

你可以只使用 Git，不使用 GitHub；也可以把本地 Git 仓库推送到 GitHub，实现远程备份和多人协作。

## 安装 Git

### Windows 安装

1. 打开 Git 官网：<https://git-scm.com/>
2. 下载 Windows 版本安装包。
3. 双击安装包，一路按默认选项安装即可。
4. 安装完成后，打开 PowerShell、CMD 或 Git Bash，输入：

```powershell
git --version
```

如果看到类似下面的输出，说明安装成功：

```txt
git version 2.x.x
```

### macOS 安装

如果你安装了 Homebrew，可以执行：

```bash
brew install git
```

安装完成后检查版本：

```bash
git --version
```

如果没有 Homebrew，也可以直接从 Git 官网下载安装包。

### Linux 安装

Ubuntu / Debian：

```bash
sudo apt update
sudo apt install git -y
```

CentOS / Rocky Linux / AlmaLinux：

```bash
sudo dnf install git -y
```

安装完成后检查：

```bash
git --version
```

## 配置用户名和邮箱

安装 Git 后，建议先配置用户名和邮箱。它们会写入每一次提交记录中，用来标识是谁提交的。

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

查看配置：

```bash
git config --global --list
```

示例输出：

```txt
user.name=你的名字
user.email=你的邮箱@example.com
```

## 第一次使用 Git

可以新建一个测试目录：

```bash
mkdir git-demo
cd git-demo
```

初始化 Git 仓库：

```bash
git init
```

新建一个文件：

```bash
echo "Hello Git" > README.md
```

查看当前状态：

```bash
git status
```

把文件加入暂存区：

```bash
git add README.md
```

提交版本：

```bash
git commit -m "初始化项目"
```

查看提交历史：

```bash
git log --oneline
```

到这里，你已经完成了一次最基础的 Git 使用流程。

## Git 的基础工作流

最常见的 Git 使用流程是：

```txt
修改文件 → git add → git commit → 查看历史或推送远程
```

对应命令是：

```bash
git status
git add 文件名
git commit -m "提交说明"
```

如果要把所有已修改文件加入暂存区，可以执行：

```bash
git add .
```

## 常用命令速查

```bash
# 查看当前仓库状态
git status

# 查看提交历史
git log --oneline

# 查看文件修改差异
git diff

# 添加指定文件到暂存区
git add README.md

# 添加所有修改到暂存区
git add .

# 提交修改
git commit -m "提交说明"

# 查看当前分支
git branch
```

## 常见问题

### git 命令找不到

如果终端提示：

```txt
git: command not found
```

说明 Git 没有安装成功，或者没有加入系统 PATH。可以重新安装 Git，安装完成后重启终端。

### 提交时提示没有配置用户名邮箱

如果提交时提示需要配置身份信息，执行：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

### 提交说明怎么写

提交说明应该简短描述这次修改做了什么，例如：

```txt
新增登录页面
修复文章列表样式
更新 Git 入门教程
```

不要只写：

```txt
update
fix
111
```

这样的说明以后很难看懂。

## 总结

Git 是开发中非常基础也非常重要的工具。它可以帮助我们记录每一次修改、回到历史版本、进行多人协作和分支开发。

刚开始学习 Git 时，不需要一次记住所有命令。先掌握这几个就够了：

```bash
git init
git status
git add .
git commit -m "提交说明"
git log --oneline
```

等你熟悉了基础流程，再继续学习分支、远程仓库、合并冲突和 GitHub 协作。
