## 题目入口

题目链接：[搜索二维矩阵](https://leetcode.cn/problems/search-a-2d-matrix/)

## 一句话读懂题目

给你一个 `m × n` 的矩阵，满足两个条件：

1. 每行的元素从左到右递增；
2. 每行的第一个元素大于前一行的最后一个元素。

判断目标值 `target` 是否在矩阵中。

比如：

```text
matrix = [[1,  3,  5,  7],
          [10, 11, 16, 20],
          [23, 30, 34, 60]]
target = 3
```

`3` 在第一行，返回 `true`。

## 先用生活例子理解

想象一本字典，每一页的词按字母顺序排列，而且上一页的最后一个词一定排在下一页第一个词前面。

你要找一个词，怎么找最快？

1. 先翻到正确的那一页（看每页首词和尾词）；
2. 再在那一页里找到具体位置。

这就是两次二分查找：先定位行，再定位列。

## 核心思路

整个矩阵如果"拉平"成一维数组，就是一个严格递增的有序数组。

### 方法一：把矩阵当一维数组做一次二分

用一维索引 `mid`，通过 `mid / n` 和 `mid % n` 转换成二维坐标。

### 方法二：两次二分

- 第一次二分：找到 `target` 可能在哪一行（该行的第一个元素 ≤ target，最后一个元素 ≥ target）；
- 第二次二分：在那一行内找 `target`。

两种方法本质一样，这里用方法一（代码更简洁）。

## 手动走一遍例子

```text
matrix = [[1,  3,  5,  7],
          [10, 11, 16, 20],
          [23, 30, 34, 60]]
target = 3
m = 3, n = 4, 总共 12 个元素
```

把矩阵看作一维数组 `[1, 3, 5, 7, 10, 11, 16, 20, 23, 30, 34, 60]`。

二分查找 `target = 3`：

| 步骤 | left | right | mid | 对应坐标 | 值 | 比较 |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | 0 | 11 | 5 | (1,1) | `11` | 11 > 3, right = 4 |
| 2 | 0 | 4 | 2 | (0,2) | `5` | 5 > 3, right = 1 |
| 3 | 0 | 1 | 0 | (0,0) | `1` | 1 < 3, left = 1 |
| 4 | 1 | 1 | 1 | (0,1) | `3` | 3 == 3, 找到！ |

返回 `true`。

## Java 代码

先记住一句话：

> 有序矩阵就是一维有序数组，一次二分就够了。

### 写法一：一维二分（推荐）

```java
class Solution {
    public boolean searchMatrix(int[][] matrix, int target) {
        int m = matrix.length;
        int n = matrix[0].length;
        int left = 0, right = m * n - 1;

        while (left <= right) {
            int mid = left + (right - left) / 2;
            int value = matrix[mid / n][mid % n];

            if (value == target) {
                return true;
            } else if (value < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        return false;
    }
}
```

### 写法二：两次二分

```java
class Solution {
    public boolean searchMatrix(int[][] matrix, int target) {
        int m = matrix.length;
        int n = matrix[0].length;

        // 第一次二分：找到 target 所在的行
        int top = 0, bottom = m - 1;
        while (top <= bottom) {
            int mid = top + (bottom - top) / 2;
            if (matrix[mid][0] > target) {
                bottom = mid - 1;
            } else if (matrix[mid][n - 1] < target) {
                top = mid + 1;
            } else {
                // target 在 mid 行的范围内
                // 第二次二分：在 mid 行内查找
                int left = 0, right = n - 1;
                while (left <= right) {
                    int colMid = left + (right - left) / 2;
                    if (matrix[mid][colMid] == target) {
                        return true;
                    } else if (matrix[mid][colMid] < target) {
                        left = colMid + 1;
                    } else {
                        right = colMid - 1;
                    }
                }
                return false;
            }
        }

        return false;
    }
}
```

## 代码逐行拆解（一维二分写法）

```java
int m = matrix.length;
int n = matrix[0].length;
int left = 0, right = m * n - 1;
```

把整个矩阵当成长度为 `m*n` 的一维数组，二分范围是 `[0, m*n-1]`。

```java
int mid = left + (right - left) / 2;
```

取中间索引。用 `left + (right - left) / 2` 而不是 `(left + right) / 2` 防止溢出。

```java
int value = matrix[mid / n][mid % n];
```

一维索引转二维坐标：行 = `mid / n`，列 = `mid % n`。

```java
if (value == target) {
    return true;
} else if (value < target) {
    left = mid + 1;
} else {
    right = mid - 1;
}
```

标准二分查找逻辑。

```java
return false;
```

循环结束没找到，说明不存在。

## 复杂度分析

时间复杂度：`O(log(m × n))`。

等价于在长度为 `m*n` 的有序数组上做一次二分。

空间复杂度：`O(1)`。

只用了几个变量。

## 常见坑

### 1. 索引转坐标时除和模搞混

`mid / n` 是行号，`mid % n` 是列号。这里的 `n` 是列数，不是行数。

### 2. 用 (left + right) / 2 导致溢出

当 `left` 和 `right` 都很大时，`left + right` 可能超过 `int` 上限。用 `left + (right - left) / 2` 更安全。

### 3. 混淆本题和"搜索二维矩阵 II"

本题（LeetCode 74）矩阵是严格行递增 + 行间递增，可以用二分。

"搜索二维矩阵 II"（LeetCode 240）只保证每行递增、每列递增，不能直接拉平成一维数组，需要用"从右上角出发"的搜索法。

### 4. 忘记判断空矩阵

```java
if (matrix.length == 0 || matrix[0].length == 0) return false;
```

虽然题目一般保证矩阵非空，但面试时加一行防御代码不会错。

## 面试时怎么讲

### 30 秒版

> 这个矩阵如果拉平成一维数组就是严格递增的，直接在 `[0, m*n-1]` 上做一次二分查找。每次取 `mid`，通过 `mid/n` 和 `mid%n` 映射回二维坐标取值。时间 `O(log(m*n))`，空间 `O(1)`。

### 面试官可能追问

**和"搜索二维矩阵 II"有什么区别？**

本题矩阵全局有序（可以拉平成一维有序数组），直接二分。

"II"的矩阵只是每行每列各自有序，不能拉平。用"从右上角出发"的方法：当前值大于 target 就左移，小于 target 就下移。时间 `O(m+n)`。

**两次二分和一次二分哪个好？**

时间都是 `O(log(m*n))`（因为 `log m + log n = log(m*n)`）。一次二分代码更简洁；两次二分更直观。面试推荐一次二分。

## 最后总结

搜索二维矩阵的关键洞察是：

> 满足题目条件的矩阵，拉平后就是一个有序数组。

代码模板：

```text
left = 0, right = m*n - 1
while left <= right
    mid = left + (right - left) / 2
    value = matrix[mid/n][mid%n]
    标准二分比较
```

最后把这句话记住：

> 有序矩阵搜索，不要被二维吓住——一维索引加坐标转换，回到最熟悉的二分查找。
