## 题目入口

题目链接：[螺旋矩阵](https://leetcode.cn/problems/spiral-matrix/)

## 一句话读懂题目

给你一个 `m × n` 的矩阵，按顺时针螺旋顺序返回所有元素。

比如：

```text
matrix = [[1,  2,  3],
          [4,  5,  6],
          [7,  8,  9]]
```

螺旋顺序：

```text
[1, 2, 3, 6, 9, 8, 7, 4, 5]
```

走法是：→ → → ↓ ↓ ← ← ← ↑，然后继续向内螺旋。

## 先用生活例子理解

想象你在剥洋葱——一层一层从外往里剥。

每一层的剥法是固定的：先走最上面一行（向右），再走最右边一列（向下），再走最下面一行（向左），最后走最左边一列（向上）。

剥完最外层，四条边界向内收缩，继续剥下一层，直到没有元素可剥。

## 核心思路：四边界收缩

用四个变量维护当前层的边界：

```text
top    = 0          当前最上面一行
bottom = m - 1      当前最下面一行
left   = 0          当前最左边一列
right  = n - 1      当前最右边一列
```

每一轮做四件事：

1. 从左到右遍历 `top` 行，然后 `top++`；
2. 从上到下遍历 `right` 列，然后 `right--`；
3. 从右到左遍历 `bottom` 行（如果 `top <= bottom`），然后 `bottom--`；
4. 从下到上遍历 `left` 列（如果 `left <= right`），然后 `left++`。

重复直到 `top > bottom` 或 `left > right`。

## 手动走一遍例子

```text
matrix = [[1, 2, 3],
          [4, 5, 6],
          [7, 8, 9]]
```

初始：`top=0, bottom=2, left=0, right=2`。

**第一圈：**

| 方向 | 遍历范围 | 元素 | 边界变化 |
| --- | --- | --- | --- |
| → | 第 0 行，列 0→2 | `1, 2, 3` | top → 1 |
| ↓ | 第 2 列，行 1→2 | `6, 9` | right → 1 |
| ← | 第 2 行，列 1→0 | `8, 7` | bottom → 1 |
| ↑ | 第 0 列，行 1→1 | `4` | left → 1 |

**第二圈：**

`top=1, bottom=1, left=1, right=1`。

| 方向 | 遍历范围 | 元素 | 边界变化 |
| --- | --- | --- | --- |
| → | 第 1 行，列 1→1 | `5` | top → 2 |

此时 `top > bottom`，停止。

结果：`[1, 2, 3, 6, 9, 8, 7, 4, 5]`。

## 为什么第 3、4 步需要额外判断

看一个非方阵的例子：

```text
matrix = [[1, 2, 3, 4]]
```

1×4 矩阵。初始 `top=0, bottom=0, left=0, right=3`。

第一步（→）遍历完 `[1,2,3,4]`，`top` 变成 1。

此时 `top > bottom`，如果不判断就继续遍历 `bottom` 行（← 方向），会重复读取元素。

所以第 3 步（←）前要判断 `top <= bottom`，第 4 步（↑）前要判断 `left <= right`。

## Java 代码

先记住一句话：

> 一圈四步，每步收一条边，转完检查边界。

```java
class Solution {
    public List<Integer> spiralOrder(int[][] matrix) {
        List<Integer> result = new ArrayList<>();
        int m = matrix.length;
        int n = matrix[0].length;

        int top = 0, bottom = m - 1;
        int left = 0, right = n - 1;

        while (top <= bottom && left <= right) {
            // 向右
            for (int j = left; j <= right; j++) {
                result.add(matrix[top][j]);
            }
            top++;

            // 向下
            for (int i = top; i <= bottom; i++) {
                result.add(matrix[i][right]);
            }
            right--;

            // 向左
            if (top <= bottom) {
                for (int j = right; j >= left; j--) {
                    result.add(matrix[bottom][j]);
                }
                bottom--;
            }

            // 向上
            if (left <= right) {
                for (int i = bottom; i >= top; i--) {
                    result.add(matrix[i][left]);
                }
                left++;
            }
        }

        return result;
    }
}
```

## 代码逐行拆解

```java
int top = 0, bottom = m - 1;
int left = 0, right = n - 1;
```

四个边界指针，框定当前要遍历的这一圈。

```java
while (top <= bottom && left <= right) {
```

只要还有行和列没被遍历过，就继续。

```java
for (int j = left; j <= right; j++) {
    result.add(matrix[top][j]);
}
top++;
```

向右遍历最上面一行，然后上边界下移。

```java
for (int i = top; i <= bottom; i++) {
    result.add(matrix[i][right]);
}
right--;
```

向下遍历最右边一列，然后右边界左移。

```java
if (top <= bottom) {
    for (int j = right; j >= left; j--) {
        result.add(matrix[bottom][j]);
    }
    bottom--;
}
```

向左遍历最下面一行（只在还有行的情况下），然后下边界上移。

```java
if (left <= right) {
    for (int i = bottom; i >= top; i--) {
        result.add(matrix[i][left]);
    }
    left++;
}
```

向上遍历最左边一列（只在还有列的情况下），然后左边界右移。

## 复杂度分析

时间复杂度：`O(m × n)`。

每个元素恰好访问一次。

空间复杂度：`O(1)`。

除了结果列表外只用了几个变量。

## 常见坑

### 1. 漏掉第 3、4 步的边界判断

如果不检查 `top <= bottom` 和 `left <= right`，单行或单列矩阵会重复遍历。

### 2. 边界收缩后还用旧值

注意 `top++` 后，向下遍历时 `i` 要从新的 `top` 开始，不是旧的。代码中已经正确处理了（`top++` 在向右遍历之后）。

### 3. 搞混方向顺序

必须严格按照 → ↓ ← ↑ 的顺序。顺时针螺旋就是这个方向，写错一步整个结果就乱了。

### 4. while 条件用 || 而不是 &&

必须是 `top <= bottom && left <= right`。用 `||` 会导致在某个方向已经没有元素时还继续遍历。

## 面试时怎么讲

### 30 秒版

> 用四个边界 `top, bottom, left, right` 控制当前层的范围。每轮按"右下左上"四个方向各走一条边，走完收缩对应边界。向左和向上之前额外判断边界是否还合法，防止单行单列重复遍历。时间 `O(m*n)`，空间 `O(1)`。

### 面试官可能追问

**能不能用方向数组 + 已访问标记来做？**

可以。定义方向数组 `dirs = {右,下,左,上}`，用 visited 数组标记已走过的格子，碰到边界或已访问的格子就转向。但这样需要 `O(m*n)` 额外空间，不如边界收缩法。

**螺旋矩阵 II（填充数字到矩阵里）怎么做？**

思路完全一样，只是从"读"变成"写"。用同样的边界收缩法，按顺序填入 `1, 2, 3, ...`。

## 最后总结

螺旋矩阵的核心是模拟遍历路径：

> 一圈一圈剥洋葱，每圈四条边，走完收缩。

代码模板：

```text
while (top <= bottom && left <= right)
    → 遍历 top 行，top++
    ↓ 遍历 right 列，right--
    ← 如果 top <= bottom，遍历 bottom 行，bottom--
    ↑ 如果 left <= right，遍历 left 列，left++
```

最后把这句话记住：

> 螺旋 = 一层一层剥。每层走四条边，记得判断是否只剩一行或一列。
