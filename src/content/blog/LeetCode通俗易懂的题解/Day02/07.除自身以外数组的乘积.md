## 题目入口

题目链接：[除自身以外数组的乘积](https://leetcode.cn/problems/product-of-array-except-self/)

## 一句话读懂题目

给你一个整数数组 `nums`，返回一个数组 `answer`，其中 `answer[i]` 等于 `nums` 中除了 `nums[i]` 以外所有元素的乘积。

要求：不能使用除法，时间 `O(n)`。

比如：

```text
nums = [1, 2, 3, 4]
```

```text
answer[0] = 2 * 3 * 4 = 24
answer[1] = 1 * 3 * 4 = 12
answer[2] = 1 * 2 * 4 = 8
answer[3] = 1 * 2 * 3 = 6
```

返回：

```text
[24, 12, 8, 6]
```

## 先用生活例子理解

想象你站在一排人中间，要计算"除你以外所有人的身高乘积"。

你可以把这个乘积拆成两部分：

```text
你左边所有人的身高乘积 × 你右边所有人的身高乘积
```

左边的乘积，从左往右累乘就能算出来。

右边的乘积，从右往左累乘就能算出来。

两边一乘，就是"除你以外的总乘积"。

这就是前缀积 × 后缀积的思路。

## 为什么不能用除法

最直觉的做法是：先算出所有元素的总乘积，然后每个位置除以自己。

```text
answer[i] = 总乘积 / nums[i]
```

但题目明确说了不能用除法。

而且就算允许除法，如果数组里有 `0`，除法也会出问题（除以零）。

所以需要用"左右累积"的方法来避开除法。

## 核心思路：前缀积 × 后缀积

对于位置 `i`，它的答案等于：

```text
answer[i] = 左边所有元素的乘积 × 右边所有元素的乘积
```

定义：

```text
left[i]：nums[0] × nums[1] × ... × nums[i-1]（i 左边所有元素的乘积）
right[i]：nums[i+1] × nums[i+2] × ... × nums[n-1]（i 右边所有元素的乘积）
```

那么：

```text
answer[i] = left[i] × right[i]
```

`left` 从左往右累乘就能算出来，`right` 从右往左累乘就能算出来。

## 手动走一遍例子

```text
nums = [1, 2, 3, 4]
```

### 第一步：算前缀积（左边的乘积）

```text
left[0] = 1           （左边没有元素，设为 1）
left[1] = 1           （左边只有 nums[0] = 1）
left[2] = 1 × 2 = 2  （左边有 nums[0] × nums[1]）
left[3] = 1 × 2 × 3 = 6
```

### 第二步：算后缀积（右边的乘积）

```text
right[3] = 1           （右边没有元素，设为 1）
right[2] = 4           （右边只有 nums[3] = 4）
right[1] = 4 × 3 = 12 （右边有 nums[3] × nums[2]）
right[0] = 4 × 3 × 2 = 24
```

### 第三步：相乘

```text
answer[0] = left[0] × right[0] = 1 × 24 = 24
answer[1] = left[1] × right[1] = 1 × 12 = 12
answer[2] = left[2] × right[2] = 2 × 4 = 8
answer[3] = left[3] × right[3] = 6 × 1 = 6
```

压缩成一张表：

| i | nums[i] | left[i] | right[i] | answer[i] |
| --- | --- | --- | --- | --- |
| 0 | `1` | `1` | `24` | `24` |
| 1 | `2` | `1` | `12` | `12` |
| 2 | `3` | `2` | `4` | `8` |
| 3 | `4` | `6` | `1` | `6` |

## Java 代码

先记住一句话：

> 左边乘一遍，右边乘一遍，两边乘起来就是答案。

### 写法一：两个数组（好理解）

```java
class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] left = new int[n];
        int[] right = new int[n];
        int[] answer = new int[n];

        left[0] = 1;
        for (int i = 1; i < n; i++) {
            left[i] = left[i - 1] * nums[i - 1];
        }

        right[n - 1] = 1;
        for (int i = n - 2; i >= 0; i--) {
            right[i] = right[i + 1] * nums[i + 1];
        }

        for (int i = 0; i < n; i++) {
            answer[i] = left[i] * right[i];
        }

        return answer;
    }
}
```

### 写法二：空间优化（推荐）

题目的进阶要求是 `O(1)` 额外空间（输出数组不算）。

可以把 `left` 直接存到 `answer` 里，然后用一个变量从右往左累乘代替 `right` 数组：

```java
class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] answer = new int[n];

        answer[0] = 1;
        for (int i = 1; i < n; i++) {
            answer[i] = answer[i - 1] * nums[i - 1];
        }

        int right = 1;
        for (int i = n - 1; i >= 0; i--) {
            answer[i] = answer[i] * right;
            right = right * nums[i];
        }

        return answer;
    }
}
```

## 代码逐行拆解（空间优化写法）

```java
int n = nums.length;
int[] answer = new int[n];
```

创建结果数组。

```java
answer[0] = 1;
for (int i = 1; i < n; i++) {
    answer[i] = answer[i - 1] * nums[i - 1];
}
```

第一次遍历：从左往右，把前缀积存到 `answer` 里。

`answer[i]` 现在保存的是 `nums[0] × nums[1] × ... × nums[i-1]`。

```java
int right = 1;
```

用一个变量 `right` 来维护后缀积，初始为 `1`。

```java
for (int i = n - 1; i >= 0; i--) {
    answer[i] = answer[i] * right;
    right = right * nums[i];
}
```

第二次遍历：从右往左。

- `answer[i] * right`：前缀积 × 后缀积 = 最终答案；
- `right * nums[i]`：把当前元素累乘到后缀积里，供左边的位置使用。

```java
return answer;
```

两次遍历结束，`answer` 就是最终结果。

## 为什么 left[0] 和 right[n-1] 是 1

`left[0]` 表示下标 `0` 左边所有元素的乘积。

但下标 `0` 左边没有元素，所以设为 `1`（乘法单位元）。

同理，`right[n-1]` 表示最后一个元素右边的乘积，右边也没有元素，设为 `1`。

这样公式 `answer[i] = left[i] * right[i]` 对边界位置也成立。

## 复杂度分析

### 写法一：两个数组

时间复杂度：`O(n)`。

三次遍历，每次 `O(n)`。

空间复杂度：`O(n)`。

额外用了 `left` 和 `right` 两个数组。

### 写法二：空间优化

时间复杂度：`O(n)`。

两次遍历。

空间复杂度：`O(1)`。

除了输出数组 `answer` 外，只用了一个变量 `right`。

## 常见坑

### 1. 前缀积的起始值搞错

`left[0]` 必须是 `1`，不能是 `nums[0]`。

因为 `left[i]` 的含义是"i 左边的乘积"，不包括 `nums[i]` 本身。

如果 `left[0] = nums[0]`，那 `answer[0]` 就会多乘一个 `nums[0]`。

### 2. 空间优化时搞错覆盖顺序

第二次从右往左遍历时，要先用 `answer[i] * right` 算出结果，再更新 `right`。

如果先更新 `right`，`right` 就包含了 `nums[i]`，乘到 `answer[i]` 里就错了。

### 3. 以为可以用总乘积除以每个元素

除了题目不允许用除法之外，如果数组里有 `0`，除法就会出问题：

- 有一个 `0`：总乘积是 `0`，除以非零元素结果都是 `0`，但 `0` 位置的答案不是 `0`；
- 有多个 `0`：所有位置的答案都是 `0`。

前缀积 × 后缀积的方法天然处理了含 `0` 的情况。

### 4. 边界情况：数组长度为 2

```text
nums = [3, 5]
answer[0] = 5（右边的乘积）
answer[1] = 3（左边的乘积）
```

代码正常处理，不需要特判。

## 面试时怎么讲

### 30 秒版

> 不能用除法，所以对每个位置 `i`，答案等于它左边所有元素的乘积乘以右边所有元素的乘积。第一遍从左往右算前缀积存到结果数组，第二遍从右往左用一个变量维护后缀积，边算边乘到结果里。两遍遍历，时间 `O(n)`，额外空间 `O(1)`。

### 面试官可能追问

**能不能只遍历一次？**

不行。因为算每个位置的答案需要它两边的信息，至少需要两次遍历（或者两个方向的信息）。

**如果数组里有 0 怎么办？**

前缀积 × 后缀积天然处理了。比如 `nums = [1, 0, 3]`：

```text
left  = [1, 1, 0]
right = [0, 3, 1]
answer = [0, 3, 0]
```

含 `0` 的位置左右乘积为 `0`，`0` 所在位置的答案是除它以外的乘积，都正确。

**这道题和前缀和有什么关系？**

思路完全一样。前缀和是"左边所有元素的加法累积"，前缀积是"左边所有元素的乘法累积"。

把加法换成乘法，把求和换成求积，模板就能复用。

**空间优化的写法为什么合法？**

第一遍把前缀积存到 `answer` 里，第二遍从右往左读取 `answer[i]`（前缀积）并乘上后缀积得到最终结果。每个位置只读一次、写一次，不会产生覆盖冲突。

## 最后总结

除自身以外数组的乘积，核心洞察是：

> 每个位置的答案 = 左边的乘积 × 右边的乘积。

代码模板：

```text
第一遍从左往右：算前缀积，存到 answer
第二遍从右往左：用变量维护后缀积，边算边乘到 answer
```

刷题时可以用下面几个边界用例自检：

```text
nums = [1, 2, 3, 4] -> [24, 12, 8, 6]
nums = [-1, 1, 0, -3, 3] -> [0, 0, 9, 0, 0]（含 0）
nums = [2, 3] -> [3, 2]（最短数组）
nums = [0, 0] -> [0, 0]（多个 0）
nums = [1, 1, 1, 1] -> [1, 1, 1, 1]（全是 1）
```

最后把这句话记住：

> 不能直接算"除自己以外的全部"，就拆成"左边的"和"右边的"分别算，再合起来。
