## 适用场景

Elasticsearch 适合全文检索、日志查询、聚合分析等场景。Kibana 用于可视化管理和查询 ES 数据。本文记录如何使用 Docker Compose 部署 Elasticsearch 7.17、Kibana 和 IK 中文分词器。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建工作目录：

```bash
# Linux
mkdir -p /opt/es/config /opt/es/data /opt/es/plugins

# Windows（PowerShell）
mkdir D:/es-env/config, D:/es-env/data, D:/es-env/plugins
```

### 调整虚拟内存参数

Elasticsearch 需要较大的 `vm.max_map_count`：

```bash
# Linux（永久生效）
echo "vm.max_map_count=262144" >> /etc/sysctl.d/99-elasticsearch.conf
sysctl --system

# Windows（WSL 2，重启后可能失效）
wsl -d docker-desktop -e sysctl -w vm.max_map_count=262144
```

> Windows 重启后这个设置可能失效，如果 ES 启动失败，重新执行一次即可。

### 编写 ES 配置

在配置目录下新建 `elasticsearch.yml`：

```yaml
http.host: 0.0.0.0
```

## 编写 docker-compose

新建 `docker-compose.yml`：

```yaml
services:
  elasticsearch:
    image: elasticsearch:7.17.14
    container_name: elasticsearch
    environment:
      - discovery.type=single-node
      - ES_JAVA_OPTS=-Xms512m -Xmx512m
      - TAKE_FILE_OWNERSHIP=true
    volumes:
      - ./config/elasticsearch.yml:/usr/share/elasticsearch/config/elasticsearch.yml
      - ./data:/usr/share/elasticsearch/data
      - ./plugins:/usr/share/elasticsearch/plugins
    ports:
      - "9200:9200"
    networks:
      - es-net
    healthcheck:
      test: ["CMD-SHELL", "curl -fs http://localhost:9200/_cluster/health || exit 1"]
      interval: 10s
      timeout: 5s
      retries: 5

  kibana:
    image: kibana:7.17.14
    container_name: kibana
    environment:
      - ELASTICSEARCH_HOSTS=http://elasticsearch:9200
      - I18N_LOCALE=zh-CN
    ports:
      - "5601:5601"
    depends_on:
      elasticsearch:
        condition: service_healthy
    networks:
      - es-net

networks:
  es-net:
    driver: bridge
```

> Windows 用户如果遇到数据目录权限问题，`TAKE_FILE_OWNERSHIP=true` 会自动处理。Linux 用户可能需要 `chmod 777 ./data`。

> 建议 Docker Desktop 分配至少 2GB 内存。Windows 下可通过 `%USERPROFILE%/.wslconfig` 中的 `memory` 字段调整 WSL 2 内存上限。

## 启动服务

```bash
docker compose up -d
```

查看容器状态：

```bash
docker ps
```

## 验证安装

访问 Elasticsearch：

```text
http://localhost:9200
```

如果返回包含 `You Know, for Search` 的 JSON，说明 ES 启动成功。

访问 Kibana：

```text
http://localhost:5601
```

首次启动可能需要等待 1-2 分钟（Kibana 会等 ES 健康后才启动）。

## 安装 IK 中文分词器

进入 ES 容器：

```bash
docker exec -it elasticsearch /bin/bash
```

安装与 ES 版本匹配的 IK 插件：

```bash
bin/elasticsearch-plugin install https://get.infini.cloud/elasticsearch/analysis-ik/7.17.14
```

> 备选地址（GitHub Release）：`https://github.com/medcl/elasticsearch-analysis-ik/releases/download/v7.17.14/elasticsearch-analysis-ik-7.17.14.zip`

安装完成后退出容器并重启 ES：

```bash
exit
docker restart elasticsearch
```

## 测试中文分词

打开 Kibana 的 Dev Tools，执行：

```json
POST _analyze
{
  "analyzer": "ik_max_word",
  "text": "精装一室一厅近地铁"
}
```

如果返回结果中能看到中文词语被拆分，说明 IK 插件已经生效。

## 常见问题

### ES 容器启动后立刻退出

先检查 `vm.max_map_count`，再查看日志：

```bash
docker logs elasticsearch
```

### Kibana 访问不到 ES

确认两个服务在同一个 Docker 网络中（`es-net`），并且 `ELASTICSEARCH_HOSTS` 指向 `http://elasticsearch:9200`（容器名作为主机名）。

### 内存不足

ES 默认 JVM 堆为 512MB，加上容器开销大约需要 1GB。如果 Docker Desktop 内存不够，在 `.wslconfig` 中增加 `memory=4GB`。

## 停止和清理

```bash
docker compose down

# 如果需要清除数据重新开始
docker compose down -v
rm -rf ./data/*
```

## 后续建议

- 生产环境需要单独规划安全配置（`xpack.security.enabled: true`）、账号权限和集群部署。
- 插件版本必须和 Elasticsearch 版本严格匹配。
- 本地环境适合开发验证，不建议直接作为生产部署方案。
