## 适用场景

使用 Docker 快速启动 MySQL，适合本地开发、测试环境或轻量级服务部署。本文同时给出 MySQL 5.7 和 8.0 的配置，生产环境请结合备份、监控和安全策略进一步加固。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建工作目录：

```bash
# Linux
mkdir -p /opt/mysql

# Windows（PowerShell），在你喜欢的位置建目录即可
mkdir D:/mysql-env
```

> Windows 用户需确认 Docker Desktop 已启动且使用 WSL 2 后端。

## 编写 docker-compose

在工作目录下新建 `docker-compose.yml`：

### MySQL 8.0（推荐）

```yaml
services:
  mysql:
    image: mysql:8.0
    container_name: mysql80
    restart: unless-stopped
    ports:
      - "3306:3306"
    volumes:
      - ./data:/var/lib/mysql
      - ./conf:/etc/mysql/conf.d
      - ./logs:/var/log/mysql
    environment:
      - MYSQL_ROOT_PASSWORD=your_root_password
      - TZ=Asia/Shanghai
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 3
```

### MySQL 5.7

只需把镜像改为 `mysql:5.7`：

```yaml
services:
  mysql:
    image: mysql:5.7
    container_name: mysql57
    restart: unless-stopped
    ports:
      - "3306:3306"
    volumes:
      - ./data:/var/lib/mysql
      - ./conf:/etc/mysql/conf.d
      - ./logs:/var/log/mysql
    environment:
      - MYSQL_ROOT_PASSWORD=your_root_password
      - TZ=Asia/Shanghai
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 3
```

> ⚠️ 5.7 和 8.0 不能共用同一数据目录。如果需要同时运行两个版本，请放在不同目录下并修改宿主机端口。

### 关于 volume 路径

Compose 中 `./data` 是相对路径，数据直接保存在 `docker-compose.yml` 同级目录下。Linux 和 Windows 通用，不需要额外处理路径。

> ❌ 不要在 Windows 下用 `/data:/var/lib/mysql` 这种写法——它会映射到 WSL 2 虚拟机内部，你在文件管理器里看不到数据。Compose 的 `./data` 相对路径就没有这个问题。

## 启动 MySQL

在 `docker-compose.yml` 所在目录执行：

```bash
docker compose up -d
```

查看日志（首次启动会初始化数据库，需要等十几秒）：

```bash
docker compose logs -f
```

看到 `ready for connections. Version: '8.0.x'` 说明启动完成。

## 验证连接

查看容器状态：

```bash
docker compose ps
```

进入容器并登录：

```bash
docker exec -it mysql80 mysql -u root -p
```

输入 compose 中配置的 `MYSQL_ROOT_PASSWORD`。

外部客户端连接：

```text
Host: 127.0.0.1（或服务器 IP）
Port: 3306
User: root
Password: your_root_password
```

## 创建业务用户

```sql
CREATE DATABASE app_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'app_user'@'%' IDENTIFIED BY 'your_app_password';
GRANT CREATE, DROP, SELECT, INSERT, UPDATE, DELETE, ALTER ON app_db.* TO 'app_user'@'%';
FLUSH PRIVILEGES;
```

## 自定义配置（可选）

在 `./conf/` 目录下新建 `my.cnf`，容器会自动加载：

```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
max_connections=200
```

修改后重启容器生效：

```bash
docker compose restart
```

## 常见问题

### 端口被占用

修改 `docker-compose.yml` 中的宿主机端口：

```yaml
ports:
  - "3307:3306"
```

连接时端口改为 `3307`。

### 数据目录权限异常

如果容器启动失败，先查看日志：

```bash
docker compose logs mysql
```

Linux 下必要时检查目录权限。Windows 下通常不会遇到此问题。

### MySQL 8.0 客户端认证失败

MySQL 8.0 默认使用 `caching_sha2_password` 认证插件，部分旧版客户端（如 Navicat 旧版）可能无法连接。在 `./conf/my.cnf` 中添加：

```ini
[mysqld]
default-authentication-plugin=mysql_native_password
```

然后重启：`docker compose restart`。

## 停止和清理

```bash
# 停止服务（保留数据）
docker compose down

# 停止并删除数据（数据库会丢失！）
docker compose down -v
rm -rf ./data
```

## 数据备份

```bash
docker exec mysql80 mysqldump -u root -p --all-databases > backup.sql
```

## 后续建议

- 不要在生产环境使用弱密码。
- 定期备份数据目录。
- 生产环境建议限制暴露端口，只允许可信来源访问。
- 不要用 `'%'` 通配符授权给生产用户，应限制来源 IP。
