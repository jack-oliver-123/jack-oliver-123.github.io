# 优先级队列与堆

## 优先级队列概念
- 普通队列：先进先出
- 优先级队列：优先级高的先出
- 应用场景
  - 操作系统任务调度
  - 网络数据包优先处理
  - 游戏中来电优先

## 堆（Heap）
- 完全二叉树 + 堆序性
- 大根堆：父 ≥ 子
- 小根堆：父 ≤ 子
- 顺序存储（数组）
- 父子下标关系
  - 左孩子：2i + 1
  - 右孩子：2i + 2
  - 父结点：(i-1) / 2

## 核心操作
- 向下调整（shiftDown）
  - 建堆时使用
  - 删除堆顶时使用
  - O(log n)
- 向上调整（shiftUp）
  - 插入元素时使用
  - O(log n)
- 建堆
  - 从最后一个非叶子开始
  - 自下而上逐个向下调整
  - O(n)

## PriorityQueue 使用
- 默认小根堆
- 大根堆需传 Comparator
- 元素必须可比较
- 不能存 null
- offer / poll / peek
- 扩容机制
  - < 64：2倍
  - ≥ 64：1.5倍

## 应用
- 堆排序
  - 升序建大堆
  - 降序建小堆
  - O(n log n)
- Top-K 问题
  - 求最大 K 个：建小堆
  - 求最小 K 个：建大堆
  - O(n log k)

## 什么是优先级队列

普通队列是**先进先出（FIFO）**——谁先来谁先走。但有些场景下，操作的数据带有优先级，需要优先级高的元素先出队：

- 手机上玩游戏时来电话了——系统优先处理来电
- 操作系统的进程调度——高优先级任务先执行
- 急诊室——病情严重的患者先看病，不是先来的先看

在这种场景下，数据结构需要提供两个基本操作：

1. **添加新对象**（任意优先级）
2. **返回并删除最高优先级对象**

这就是**优先级队列（Priority Queue）**。JDK 中的 `PriorityQueue` 底层使用**堆（Heap）**来实现这两个操作，时间复杂度都是 O(log n)。

## 堆的概念

### 定义

堆是一种特殊的完全二叉树，满足**堆序性**：

- **大根堆（最大堆）**：每个节点的值 ≥ 其所有后代节点的值（根最大）
- **小根堆（最小堆）**：每个节点的值 ≤ 其所有后代节点的值（根最小）

```
大根堆示例：                小根堆示例：
       90                        10
      /  \                      /  \
    80    70                  20    30
   / \   /                   / \   /
  50 60 40                  50 40 80
```

堆的两个性质：

1. **堆中某个节点的值总是不大于（或不小于）其父节点的值**
2. **堆总是一棵完全二叉树**

### 堆的存储——数组

因为堆是完全二叉树，所以可以用数组按层序存储，利用下标关系找到父子节点（无需额外指针）：

| 关系 | 公式（下标从 0 开始） |
|------|---------------------|
| 左孩子 | `2 * i + 1` |
| 右孩子 | `2 * i + 2` |
| 父节点 | `(i - 1) / 2` |
| 最后一个非叶子节点 | `(n - 1 - 1) / 2` = `n/2 - 1` |

例如大根堆 `[90, 80, 70, 50, 60, 40]` 的数组存储：

```
下标:   0   1   2   3   4   5
值:    90  80  70  50  60  40
```

下标 1（值 80）的左孩子 = 2×1+1 = 3（值 50），右孩子 = 2×1+2 = 4（值 60），父节点 = (1-1)/2 = 0（值 90）。全部通过计算得出，不需要存指针。

## 堆的核心操作

### 向下调整（Shift Down）

**场景**：当某个节点比它的孩子小（大根堆）或大（小根堆）时，需要将它"下沉"到正确位置。

**前提**：被调整节点的左右子树已经满足堆的性质。

**算法（以大根堆为例）**：

1. 找到当前节点的较大孩子
2. 如果当前节点 < 较大孩子，交换
3. 继续对交换后的位置重复，直到满足堆序或到达叶子

```java
// 大根堆的向下调整
private void shiftDown(int[] arr, int parent, int size) {
    int child = 2 * parent + 1; // 左孩子

    while (child < size) {
        // 找到左右孩子中较大的
        if (child + 1 < size && arr[child + 1] > arr[child]) {
            child++;
        }
        // 如果孩子比父亲大，交换
        if (arr[child] > arr[parent]) {
            int temp = arr[parent];
            arr[parent] = arr[child];
            arr[child] = temp;
            // 继续向下调整
            parent = child;
            child = 2 * parent + 1;
        } else {
            break; // 已满足堆序，退出
        }
    }
}
```

**时间复杂度**：最坏情况从根一路下沉到叶子，比较次数等于树的高度 = **O(log n)**。

### 向上调整（Shift Up）

**场景**：在堆的末尾插入一个新元素后，需要将它"上浮"到正确位置。

**算法（以大根堆为例）**：

1. 将新元素与其父节点比较
2. 如果新元素 > 父节点，交换
3. 继续向上，直到满足堆序或到达根

```java
// 大根堆的向上调整
private void shiftUp(int[] arr, int child) {
    int parent = (child - 1) / 2;

    while (child > 0) {
        if (arr[child] > arr[parent]) {
            int temp = arr[child];
            arr[child] = arr[parent];
            arr[parent] = temp;
            // 继续向上
            child = parent;
            parent = (child - 1) / 2;
        } else {
            break;
        }
    }
}
```

**时间复杂度**：最坏从叶子上浮到根 = **O(log n)**。

### 建堆

给定一个任意数组，如何将它调整为一个堆？

**思路**：从最后一个非叶子节点开始，自下而上逐个执行向下调整。

```java
// 将数组建成大根堆
public void buildHeap(int[] arr) {
    int n = arr.length;
    // 从最后一个非叶子节点开始，自下而上调整
    for (int i = (n - 1 - 1) / 2; i >= 0; i--) {
        shiftDown(arr, i, n);
    }
}
```

**为什么从下往上？** 因为向下调整的前提是"左右子树已经是堆"。从最底层的非叶子节点开始，保证了调整每个节点时其子树已经满足堆序。

**建堆的时间复杂度是 O(n)**，不是 O(n log n)！

直觉上每个节点都做了 O(log n) 的向下调整，n 个节点应该是 O(n log n)。但实际上大部分节点在底层，它们向下调整的距离很短。精确计算：

- 最后一层（n/2 个节点）：向下调整 0 次（叶子不需要调整）
- 倒数第二层（n/4 个节点）：向下调整最多 1 次
- 倒数第三层（n/8 个节点）：向下调整最多 2 次
- ...

总操作次数 = n/4·1 + n/8·2 + n/16·3 + ... = **O(n)**。

### 堆的插入

1. 将新元素放到数组末尾（`size` 位置）
2. `size++`
3. 对新元素执行**向上调整**

时间复杂度：O(log n)

### 堆的删除

堆的删除**一定是删除堆顶元素**（最大值或最小值）。

1. 将堆顶元素与最后一个元素交换
2. `size--`（逻辑删除原堆顶）
3. 对新的堆顶执行**向下调整**

> 为什么不直接删除堆顶？因为直接删除会破坏完全二叉树结构。和末尾交换后再缩减大小，保持了完全二叉树性质。

时间复杂度：O(log n)

## 手写优先级队列

把上述操作整合成一个完整的优先级队列实现：

```java
public class MyPriorityQueue {
    private int[] data;
    private int size;

    public MyPriorityQueue() {
        data = new int[11]; // 默认容量 11（与 JDK 一致）
        size = 0;
    }

    // 插入元素
    public void offer(int val) {
        if (size == data.length) {
            data = Arrays.copyOf(data, data.length * 2);
        }
        data[size] = val;
        shiftUp(size);
        size++;
    }

    // 取出最小元素（小根堆）
    public int poll() {
        if (size == 0) throw new RuntimeException("队列为空");
        int min = data[0];
        data[0] = data[--size];
        shiftDown(0);
        return min;
    }

    // 查看最小元素
    public int peek() {
        if (size == 0) throw new RuntimeException("队列为空");
        return data[0];
    }

    public boolean isEmpty() { return size == 0; }
    public int size() { return size; }

    // 小根堆：向上调整
    private void shiftUp(int child) {
        int parent = (child - 1) / 2;
        while (child > 0 && data[child] < data[parent]) {
            swap(child, parent);
            child = parent;
            parent = (child - 1) / 2;
        }
    }

    // 小根堆：向下调整
    private void shiftDown(int parent) {
        int child = 2 * parent + 1;
        while (child < size) {
            if (child + 1 < size && data[child + 1] < data[child]) {
                child++;
            }
            if (data[child] < data[parent]) {
                swap(child, parent);
                parent = child;
                child = 2 * parent + 1;
            } else {
                break;
            }
        }
    }

    private void swap(int i, int j) {
        int temp = data[i];
        data[i] = data[j];
        data[j] = temp;
    }
}
```

## PriorityQueue 的使用

### 基本使用

Java 集合框架提供了 `PriorityQueue`，底层就是堆。**默认是小根堆**（每次 poll 取出的是最小值）。

```java
import java.util.PriorityQueue;

// 默认小根堆
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.offer(30);
minHeap.offer(10);
minHeap.offer(20);

System.out.println(minHeap.peek()); // 10（最小值在堆顶）
System.out.println(minHeap.poll()); // 10
System.out.println(minHeap.poll()); // 20
System.out.println(minHeap.poll()); // 30
```

### 创建大根堆

传入自定义的 `Comparator`：

```java
// 大根堆：传入反序比较器
PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
maxHeap.offer(30);
maxHeap.offer(10);
maxHeap.offer(20);

System.out.println(maxHeap.poll()); // 30（最大值在堆顶）
System.out.println(maxHeap.poll()); // 20
System.out.println(maxHeap.poll()); // 10
```

> ⚠️ 注意：`(a, b) -> b - a` 在 a、b 值差很大时可能溢出。更安全的写法是 `(a, b) -> Integer.compare(b, a)` 或 `Comparator.reverseOrder()`。

### 构造方法

| 构造方法 | 说明 |
|---------|------|
| `PriorityQueue()` | 默认容量 11 的小根堆 |
| `PriorityQueue(int initialCapacity)` | 指定初始容量（不能小于 1） |
| `PriorityQueue(Comparator<? super E> comparator)` | 指定比较器 |
| `PriorityQueue(Collection<? extends E> c)` | 用集合初始化（内部会建堆） |

### 常用方法

| 方法 | 说明 | 时间复杂度 |
|------|------|-----------|
| `boolean offer(E e)` | 插入元素（null 会抛 NPE） | O(log n) |
| `E poll()` | 移除并返回堆顶（空返回 null） | O(log n) |
| `E peek()` | 查看堆顶（空返回 null） | O(1) |
| `int size()` | 元素个数 | O(1) |
| `boolean isEmpty()` | 是否为空 | O(1) |
| `void clear()` | 清空 | O(n) |

### 使用注意事项

1. **元素必须可比较**：要么实现 `Comparable`，要么构造时传 `Comparator`，否则抛 `ClassCastException`
2. **不能插入 null**：抛 `NullPointerException`
3. **没有容量限制**：内部自动扩容
4. **不是线程安全的**：多线程用 `PriorityBlockingQueue`

### 扩容机制

JDK 的 `PriorityQueue` 扩容策略：

- 容量 < 64 时：按 **oldCapacity + oldCapacity + 2**（约 2 倍）扩容
- 容量 ≥ 64 时：按 **oldCapacity + oldCapacity >> 1**（约 1.5 倍）扩容
- 超过 `MAX_ARRAY_SIZE` 时按 `Integer.MAX_VALUE` 处理

小容量时翻倍增长，大容量时保守增长——平衡了扩容次数和空间浪费。

## 堆排序

堆排序利用堆的性质实现排序，分两步：

1. **建堆**：将数组建成堆
2. **排序**：反复将堆顶与末尾交换，缩减堆大小，向下调整

关键规则：
- **升序排列 → 建大根堆**
- **降序排列 → 建小根堆**

为什么升序要建大堆？因为大堆的堆顶是最大值，把它换到数组末尾就是最终位置，然后缩减堆大小继续处理剩余元素。

```java
public void heapSort(int[] arr) {
    int n = arr.length;

    // 第一步：建大根堆
    for (int i = (n - 1 - 1) / 2; i >= 0; i--) {
        shiftDown(arr, i, n);
    }

    // 第二步：依次将堆顶换到末尾
    for (int end = n - 1; end > 0; end--) {
        // 堆顶（最大值）和末尾交换
        int temp = arr[0];
        arr[0] = arr[end];
        arr[end] = temp;
        // 对新堆顶向下调整（堆大小缩减为 end）
        shiftDown(arr, 0, end);
    }
}
```

**时间复杂度**：建堆 O(n) + n 次向下调整 O(log n) = **O(n log n)**

**空间复杂度**：O(1)——原地排序

堆排序是不稳定排序（相同元素的相对顺序可能改变），但它是原地的且最坏情况也是 O(n log n)，这是它相对快速排序的优势（快排最坏 O(n²)）。

## Top-K 问题

**问题**：在 N 个数据中找到前 K 个最大（或最小）的元素。N 可能非常大（比如 10 亿），无法全部加载到内存。

### 朴素解法

排序后取前 K 个——O(N log N)，数据量大时不可行。

### 堆的最优解法

**求前 K 个最大元素 → 建大小为 K 的小根堆**

1. 用前 K 个元素建一个大小为 K 的**小根堆**
2. 遍历剩余 N-K 个元素，每个元素与堆顶比较：
   - 如果比堆顶大，替换堆顶，然后向下调整
   - 如果比堆顶小或相等，跳过
3. 遍历完后，堆中的 K 个元素就是前 K 个最大的

**为什么用小根堆而不是大根堆？** 小根堆的堆顶是堆中最小的元素——它充当"门槛"。只有比当前门槛大的元素才能进入堆，最终堆里留下的就是最大的 K 个。

```java
// LeetCode #215 数组中的第 K 个最大元素
public int findKthLargest(int[] nums, int k) {
    // 建大小为 k 的小根堆
    PriorityQueue<Integer> minHeap = new PriorityQueue<>(k);

    for (int num : nums) {
        if (minHeap.size() < k) {
            minHeap.offer(num);
        } else if (num > minHeap.peek()) {
            minHeap.poll();
            minHeap.offer(num);
        }
    }

    return minHeap.peek(); // 堆顶就是第 K 大的元素
}
```

**时间复杂度**：O(N log K)——遍历 N 个元素，每次堆操作 O(log K)

**空间复杂度**：O(K)——只需要大小为 K 的堆

当 K 远小于 N 时（比如从 10 亿个数中找前 100 个），这个方法远优于排序。

### Top-K 口诀

```
求最大的 K 个 → 建小根堆（堆顶是门槛，比门槛大的才能进）
求最小的 K 个 → 建大根堆（堆顶是门槛，比门槛小的才能进）
```

> 这个看起来反直觉——求最大却建小堆。记忆方法：堆顶是**淘汰标准**，小堆的堆顶最小，新来的只要比它大就能替换掉它，最终剩下的都是大的。

## 小结

| 概念 | 核心要点 |
|------|---------|
| 堆 | 完全二叉树 + 堆序性，用数组存储 |
| 向下调整 | 节点下沉，O(log n)，用于建堆和删除 |
| 向上调整 | 节点上浮，O(log n)，用于插入 |
| 建堆 | 从最后一个非叶子节点自下而上调整，O(n) |
| PriorityQueue | JDK 堆实现，默认小根堆，大根堆需传 Comparator |
| 堆排序 | 升序建大堆，O(n log n)，原地不稳定 |
| Top-K | 求最大 K 个建小堆，O(N log K)，K 远小于 N 时极高效 |

**关键认知**：

- 堆不是排序结构——它只保证堆顶是最值，不保证其他元素有序
- `PriorityQueue` 的迭代顺序**不是有序的**！只有 `poll()` 的顺序是有序的
- 建堆是 O(n) 不是 O(n log n)——这是因为大部分节点在底层，调整距离短
- Top-K 用堆只需要 O(K) 空间——适合处理海量数据
