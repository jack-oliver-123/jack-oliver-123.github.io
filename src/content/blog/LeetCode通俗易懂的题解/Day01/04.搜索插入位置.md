## 题目入口

题目链接：[搜索插入位置](https://leetcode.cn/problems/search-insert-position/description/)

## 一句话读懂题目

给你一个排好序的数组 `nums` 和一个目标值 `target`，找到 `target` 在数组中的位置。

如果 `target` 不在数组里，就返回它应该插入的位置，使数组依然有序。

比如：

```text
nums = [1, 3, 5, 6]
target = 5
```

`5` 在数组中，下标是 `2`，所以返回：

```text
2
```

再比如：

```text
nums = [1, 3, 5, 6]
target = 2
```

`2` 不在数组里，但它应该插在 `1` 和 `3` 之间，也就是下标 `1` 的位置，所以返回：

```text
1
```

题目还保证数组中的元素不重复，并且要求时间复杂度为 `O(log n)`。

## 先用生活例子理解

想象你在查一本按拼音排好序的字典。

你要找"m"开头的字。

你不会从第一页一页一页翻到"m"，而是：

> 先翻到字典中间，看看当前是什么字母。

如果中间是"h"，说明"m"在右半边，你就把左半边扔掉。

如果中间是"p"，说明"m"在左半边，你就把右半边扔掉。

每次都把查找范围缩小一半，直到找到或者确定它应该排在哪里。

这就是二分查找。

## 为什么不能用从头到尾遍历

从头到尾扫描一遍数组，当然也能找到答案：

```text
遇到第一个大于等于 target 的元素，返回它的下标
如果没有，返回数组长度
```

但这样时间复杂度是 `O(n)`。

题目明确要求 `O(log n)`，所以必须用二分查找。

二分查找的本质是：

> 利用数组有序的条件，每次排除一半的范围。

## 核心思路：二分查找

准备两个指针：

```text
left：当前查找范围的左边界
right：当前查找范围的右边界
```

每次取中间位置 `mid`：

```text
mid = left + (right - left) / 2
```

然后比较 `nums[mid]` 和 `target`：

```text
如果 nums[mid] == target
    找到了，返回 mid
如果 nums[mid] < target
    说明 target 在右半边，left = mid + 1
如果 nums[mid] > target
    说明 target 在左半边，right = mid - 1
```

循环结束后如果没找到 `target`，此时 `left` 就是它应该插入的位置。

为什么是 `left`？

因为循环结束时，`left` 指向的是第一个大于 `target` 的元素位置，也就是 `target` 应该插入的地方。

## 手动走一遍例子

以这个数组为例：

```text
nums = [1, 3, 5, 6]
target = 2
```

刚开始：

```text
left = 0
right = 3
```

### 第一步：取中间

```text
mid = 0 + (3 - 0) / 2 = 1
nums[mid] = nums[1] = 3
```

`3 > 2`，说明 `target` 在左半边。

```text
right = mid - 1 = 0
```

### 第二步：取中间

```text
mid = 0 + (0 - 0) / 2 = 0
nums[mid] = nums[0] = 1
```

`1 < 2`，说明 `target` 在右半边。

```text
left = mid + 1 = 1
```

### 第三步：循环结束

此时 `left = 1`，`right = 0`，`left > right`，循环结束。

返回 `left = 1`。

验证一下：把 `2` 插到下标 `1`，数组变成 `[1, 2, 3, 5, 6]`，仍然有序，正确。

再看一个找得到的例子：

```text
nums = [1, 3, 5, 6]
target = 5
```

| 步骤 | left | right | mid | nums[mid] | 比较结果 | 操作 |
| --- | --- | --- | --- | --- | --- | --- |
| 初始 | `0` | `3` | - | - | - | - |
| 1 | `0` | `3` | `1` | `3` | `3 < 5` | `left = 2` |
| 2 | `2` | `3` | `2` | `5` | `5 == 5` | 找到，返回 `2` |

再看一个插到末尾的例子：

```text
nums = [1, 3, 5, 6]
target = 7
```

| 步骤 | left | right | mid | nums[mid] | 比较结果 | 操作 |
| --- | --- | --- | --- | --- | --- | --- |
| 初始 | `0` | `3` | - | - | - | - |
| 1 | `0` | `3` | `1` | `3` | `3 < 7` | `left = 2` |
| 2 | `2` | `3` | `2` | `5` | `5 < 7` | `left = 3` |
| 3 | `3` | `3` | `3` | `6` | `6 < 7` | `left = 4` |

`left = 4 > right = 3`，循环结束，返回 `4`。

`4` 正好是数组长度，表示插到最后面。

## Java 代码

先记住一句话：

> 找不到就插到 `left`，因为 `left` 就是第一个比 `target` 大的位置。

```java
class Solution {
    public int searchInsert(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] == target) {
                return mid;
            } else if (nums[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        return left;
    }
}
```

## 代码逐行拆解

```java
int left = 0;
int right = nums.length - 1;
```

初始化查找范围为整个数组。

`left` 和 `right` 都是闭区间的边界，表示当前还有可能包含目标的范围是 `[left, right]`。

```java
while (left <= right) {
```

只要查找范围还有效（左边界不超过右边界），就继续查找。

注意是 `<=`，不是 `<`。

因为我们用的是闭区间 `[left, right]`，当 `left == right` 时，区间里还有一个元素没检查，不能跳过。

```java
int mid = left + (right - left) / 2;
```

取中间位置。

为什么不写 `(left + right) / 2`？

因为当 `left` 和 `right` 都很大时，`left + right` 可能溢出 `int` 范围。

`left + (right - left) / 2` 避免了这个问题，结果一样。

```java
if (nums[mid] == target) {
    return mid;
}
```

如果中间元素刚好等于 `target`，直接返回下标。

```java
} else if (nums[mid] < target) {
    left = mid + 1;
}
```

中间元素比 `target` 小，说明 `target` 在 `mid` 的右边。

把 `left` 移到 `mid + 1`，缩小范围到 `[mid + 1, right]`。

```java
} else {
    right = mid - 1;
}
```

中间元素比 `target` 大，说明 `target` 在 `mid` 的左边。

把 `right` 移到 `mid - 1`，缩小范围到 `[left, mid - 1]`。

```java
return left;
```

循环结束说明没找到 `target`。

此时 `left` 就是 `target` 应该插入的位置。

## 为什么循环结束后返回 left

这是这道题最关键的一点。

循环结束时，一定是 `left > right`，也就是 `left = right + 1`。

这时候 `left` 有什么含义？

```text
nums[right] < target < nums[left]
```

也就是说：

- `right` 指向最后一个小于 `target` 的元素；
- `left` 指向第一个大于 `target` 的元素。

所以 `target` 应该插在 `left` 的位置，把原来 `left` 位置及后面的元素都往后挪一格。

举个例子：

```text
nums = [1, 3, 5, 6]
target = 4
```

循环结束时：

```text
left = 2, right = 1
nums[right] = nums[1] = 3 < 4
nums[left] = nums[2] = 5 > 4
```

所以 `4` 应该插到下标 `2`，数组变成 `[1, 3, 4, 5, 6]`。

## 循环不变量

这道题的循环不变量是：

> 在每次循环开始前，`target` 的正确位置一定在 `[left, right + 1]` 范围内。

- 当 `nums[mid] < target` 时，`mid` 以及 `mid` 左边都不可能是插入位置，所以安全地移动 `left = mid + 1`；
- 当 `nums[mid] > target` 时，`mid` 右边不可能是插入位置，但 `mid` 本身有可能，所以 `right = mid - 1` 后，`mid` 落入了 `right + 1` 的位置，仍在范围内。

循环结束时区间缩小为空，`left == right + 1`，所以 `left` 就是唯一的答案。

## 复杂度分析

时间复杂度：`O(log n)`。

每次循环都把查找范围缩小一半。一个长度为 `n` 的数组，最多需要缩小 `log₂n` 次就会变成空区间。

空间复杂度：`O(1)`。

只用了 `left`、`right`、`mid` 三个变量。

## 常见坑

### 1. 循环条件写成 left < right

如果写成 `left < right`，当 `left == right` 时循环就退出了。

但此时还有一个元素没有检查。

比如：

```text
nums = [1]
target = 1
```

如果用 `left < right`，循环一次都不进，直接返回 `left = 0`，虽然答案碰巧对了。

但如果：

```text
nums = [1]
target = 2
```

用 `left < right` 同样不进循环，返回 `0`，而正确答案是 `1`。

所以必须用 `left <= right`。

### 2. mid 计算溢出

不要写：

```java
int mid = (left + right) / 2;
```

当 `left` 和 `right` 都接近 `Integer.MAX_VALUE` 时，相加会溢出。

正确写法：

```java
int mid = left + (right - left) / 2;
```

### 3. 忘记 left = mid + 1 或 right = mid - 1 中的 +1 / -1

如果写成 `left = mid` 或 `right = mid`，在某些情况下范围不会缩小，导致死循环。

比如：

```text
nums = [1, 3]
target = 2
left = 0, right = 1
mid = 0
nums[0] = 1 < 2
```

如果写 `left = mid`，那 `left` 还是 `0`，下一轮 `mid` 还是 `0`，永远不会结束。

### 4. 混淆"找到"和"没找到"的返回值

找到时在循环内部直接 `return mid`。

没找到时循环结束后 `return left`。

不要把这两个搞混。

## 面试时怎么讲

### 30 秒版

> 数组有序且不重复，要求 `O(log n)`，很明显用二分查找。我用闭区间 `[left, right]` 做二分，每次取 `mid`，如果 `nums[mid] == target` 直接返回；如果 `nums[mid] < target` 说明答案在右半边，`left = mid + 1`；否则在左半边，`right = mid - 1`。循环结束后没找到，`left` 就是插入位置，因为它指向第一个大于 `target` 的位置。时间 `O(log n)`，空间 `O(1)`。

### 面试官可能追问

**为什么循环结束后 `left` 就是插入位置？**

因为循环结束时 `left = right + 1`。此时 `nums[right] < target < nums[left]`，所以 `target` 应该插在 `left` 位置。

**如果 `target` 比数组所有元素都小怎么办？**

比如：

```text
nums = [2, 4, 6]
target = 1
```

每次都是 `nums[mid] > target`，`right` 一直左移，最终 `right = -1`，`left = 0`。

返回 `0`，表示插到最前面，正确。

**如果 `target` 比数组所有元素都大怎么办？**

比如：

```text
nums = [2, 4, 6]
target = 8
```

每次都是 `nums[mid] < target`，`left` 一直右移，最终 `left = 3`，`right = 2`。

返回 `3`，等于数组长度，表示插到最后面，正确。

**能不能用左闭右开区间 `[left, right)` 来写？**

可以。写法是：

```java
int left = 0;
int right = nums.length;

while (left < right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] == target) {
        return mid;
    } else if (nums[mid] < target) {
        left = mid + 1;
    } else {
        right = mid;
    }
}

return left;
```

注意三个区别：

```text
1. right 初始值是 nums.length，不是 nums.length - 1
2. 循环条件是 left < right，不是 left <= right
3. 右边界收缩是 right = mid，不是 right = mid - 1
```

两种写法本质一样，面试时用哪种都行，但要讲清楚你用的是哪种区间定义。

**这道题和普通二分查找有什么区别？**

普通二分查找如果找不到就返回 `-1`。

这道题找不到时不返回 `-1`，而是返回 `target` 应该插入的位置。

而这个位置恰好就是循环结束时 `left` 的值，不需要额外处理。

## 最后总结

搜索插入位置这道题，表面上有两个任务："查找"和"确定插入位置"，但其实只需要一个二分查找就能同时解决：

> 找到了就返回下标，找不到就返回 `left`。

二分查找模板可以这样记：

```text
left = 0, right = nums.length - 1
循环条件：left <= right
取中间：mid = left + (right - left) / 2
比较三种情况：等于返回、小于去右、大于去左
循环结束：返回 left
```

刷题时可以用下面几个边界用例自检：

```text
nums = [1, 3, 5, 6], target = 5 -> 返回 2（找到）
nums = [1, 3, 5, 6], target = 2 -> 返回 1（插中间）
nums = [1, 3, 5, 6], target = 7 -> 返回 4（插末尾）
nums = [1, 3, 5, 6], target = 0 -> 返回 0（插开头）
nums = [1], target = 0 -> 返回 0
nums = [1], target = 2 -> 返回 1
```

以后看到"有序数组 + O(log n)"的关键词，就可以直接想到二分查找。

最后把这句话记住：

> 二分不只是用来"找到"的，它还能告诉你"应该在哪里"。
