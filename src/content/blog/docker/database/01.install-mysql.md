## 适用场景

使用 Docker 快速启动 MySQL，适合本地开发、测试环境或轻量级服务部署。本文同时给出 MySQL 5.7 和 8.0 的部署方式，生产环境请结合备份、监控和安全策略进一步加固。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建数据目录，避免容器删除后数据丢失：

```bash
# Linux
mkdir -p /opt/mysql/data /opt/mysql/conf /opt/mysql/logs

# Windows（PowerShell）
mkdir D:/mysql-env/data, D:/mysql-env/conf, D:/mysql-env/logs
```

> Windows 用户需确认 Docker Desktop 已启动且使用 WSL 2 后端。

## 拉取镜像

```bash
# MySQL 5.7
docker pull mysql:5.7

# MySQL 8.0
docker pull mysql:8.0
```

## 启动 MySQL 5.7

```bash
docker run -d \
  --name mysql57 \
  -p 3306:3306 \
  -e TZ=Asia/Shanghai \
  -e MYSQL_ROOT_PASSWORD=your_root_password \
  -v /opt/mysql/data:/var/lib/mysql \
  -v /opt/mysql/conf:/etc/mysql/conf.d \
  -v /opt/mysql/logs:/var/log/mysql \
  --restart unless-stopped \
  mysql:5.7
```

> **Windows 用户注意 `-v` 路径写法：**
>
> - ✅ `./data:/var/lib/mysql`（相对路径，数据保存在当前目录下的 `data` 文件夹）
> - ✅ `D:/mysql-env/data:/var/lib/mysql`（Windows 绝对路径，用正斜杠）
> - ❌ `/data:/var/lib/mysql`（这会映射到 WSL 2 虚拟机内部，Windows 文件管理器里看不到）
>
> 推荐在工作目录下执行命令，用 `./data` 相对路径，数据就在你能看到的位置。

Windows 示例：

```bash
docker run -d \
  --name mysql57 \
  -p 3306:3306 \
  -e TZ=Asia/Shanghai \
  -e MYSQL_ROOT_PASSWORD=your_root_password \
  -v ./data:/var/lib/mysql \
  -v ./conf:/etc/mysql/conf.d \
  -v ./logs:/var/log/mysql \
  --restart unless-stopped \
  mysql:5.7
```

## 启动 MySQL 8.0

```bash
docker run -d \
  --name mysql80 \
  -p 3306:3306 \
  -e TZ=Asia/Shanghai \
  -e MYSQL_ROOT_PASSWORD=your_root_password \
  -v /opt/mysql/data:/var/lib/mysql \
  -v /opt/mysql/conf:/etc/mysql/conf.d \
  -v /opt/mysql/logs:/var/log/mysql \
  --restart unless-stopped \
  mysql:8.0
```

> ⚠️ 5.7 和 8.0 不能共用同一数据目录。如果需要同时运行，请分开路径（如 `/opt/mysql57/` 和 `/opt/mysql80/`）并修改宿主机端口。

## 验证连接

查看容器状态：

```bash
docker ps
```

查看 MySQL 日志：

```bash
docker logs -f mysql57
```

进入容器并登录：

```bash
docker exec -it mysql57 bash
mysql -u root -p
```

## 创建业务用户

```sql
CREATE DATABASE app_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'app_user'@'%' IDENTIFIED BY 'your_app_password';
GRANT CREATE, DROP, SELECT, INSERT, UPDATE, DELETE, ALTER ON app_db.* TO 'app_user'@'%';
FLUSH PRIVILEGES;
```

## 常见问题

### 端口被占用

如果 `3306` 已被占用，可以换成宿主机其他端口：

```bash
-p 3307:3306
```

连接时端口改为 `3307`。

### 数据目录权限异常

如果容器启动失败并提示权限问题，先查看日志：

```bash
docker logs mysql57
```

Linux 下必要时检查 `/opt/mysql` 目录权限。Windows 下通常不会遇到此问题。

### MySQL 8.0 客户端认证失败

MySQL 8.0 默认使用 `caching_sha2_password` 认证插件，部分旧版客户端（如 Navicat 旧版）可能无法连接。可以在 `/opt/mysql/conf/my.cnf` 中添加：

```ini
[mysqld]
default-authentication-plugin=mysql_native_password
```

然后重启容器：`docker restart mysql80`。

## 后续建议

- 不要在生产环境使用弱密码。
- 定期备份数据目录（`docker exec mysql57 mysqldump -u root -p --all-databases > backup.sql`）。
- 生产环境建议限制暴露端口，只允许可信来源访问。
- 不要用 `'%'` 通配符授权给生产用户，应限制来源 IP。
