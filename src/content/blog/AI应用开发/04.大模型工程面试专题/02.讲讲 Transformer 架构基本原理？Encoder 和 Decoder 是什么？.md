想象你在会议上听到一句话：

> 小王把文件交给小李，因为他明天要汇报。

要理解“他”更可能指谁，不能只看“他”这个字，还要回头联系“小王”“小李”“汇报”等词。人会根据整句话动态关注相关部分。

Transformer 的注意力机制也在做类似的事：序列中每个位置都可以根据当前任务，从其他位置汇总信息。它不是先在固定窗口里一步步传递，而是直接计算位置之间的相关程度。

## 本篇目标

读完本篇，你应该能够：

- 用通俗语言解释 Transformer 的整体结构；
- 理解嵌入、位置、注意力、前馈网络、残差和归一化各自负责什么；
- 从矩阵形状出发推导 Scaled Dot-Product Attention；
- 解释 Query、Key、Value 的含义，而不是只背 Q、K、V；
- 区分 Encoder、Decoder 和 Encoder-Decoder；
- 解释双向注意力、因果掩码和交叉注意力；
- 说明训练可并行与生成仍逐 Token 之间的区别；
- 回答复杂度、Pre-LN/Post-LN 和 Decoder-only 流行原因等追问。

先记住一张最小结构图：

```text
Token ID
  ↓
Token Embedding + 位置信息
  ↓
[注意力 → 残差/归一化 → 前馈网络 → 残差/归一化] × N
  ↓
词表 logits
  ↓
下一个 Token 的概率
```

## 60 秒回答

Transformer 是一种以注意力机制为核心的序列建模架构。输入 Token 先转换成向量并加入位置信息，然后经过多层 Transformer Block。每个 Block 通常包含多头注意力、逐位置前馈网络、残差连接和归一化。

注意力中，输入分别投影成 Query、Key 和 Value。模型用 Query 与所有 Key 的点积衡量相关性，除以 $\sqrt{d_k}$ 稳定数值，再经过 Softmax 得到权重，最后对 Value 加权求和。多头注意力让模型在不同表示子空间学习不同关系。

Encoder 通常允许每个位置同时看到左右上下文，适合理解和表征任务；Decoder 使用因果掩码，只能看当前位置及之前内容，适合自回归生成；Encoder-Decoder 先用 Encoder 编码输入，再让 Decoder 通过交叉注意力读取编码结果，常用于翻译、摘要等输入到输出任务。

Transformer 相比循环网络更容易并行训练，但标准全注意力对序列长度的计算和注意力矩阵内存通常是二次增长。生成式 Decoder 在推理时仍需要逐 Token 解码，工程上依赖 KV Cache、批处理和高效注意力实现加速。

## 详细解析

### 1. Transformer 解决了什么问题

在 Transformer 之前，处理序列常用 RNN、LSTM 或卷积网络。

RNN 按顺序更新隐藏状态：

$$
\mathbf{h}_t=f(\mathbf{h}_{t-1},\mathbf{x}_t)
$$

其中：

- $\mathbf{x}_t$：第 $t$ 个位置的输入；
- $\mathbf{h}_{t-1}$：前一步隐藏状态；
- $f$：循环单元；
- $\mathbf{h}_t$：当前位置的新状态。

这种结构天然表达顺序，但时间步之间存在依赖，训练时很难把所有位置完全并行。长距离信息还要经过多个时间步传递。

Transformer 的核心变化是：

> 让任意位置通过注意力直接读取其他相关位置的信息，并把序列位置同时送入矩阵运算。

这显著提高了训练并行性，也缩短了长距离位置之间的信息路径。

### 2. 从输入到输出的完整旅程

以文本“我喜欢编程”为例：

```text
原始文本
  ↓ Tokenizer
[我, 喜欢, 编程]
  ↓ 词表映射
[102, 850, 7342]
  ↓ Embedding
3 个 d_model 维向量
  ↓ 加入位置信息
保留词义与顺序
  ↓ N 层 Transformer Block
上下文化表示
  ↓ 输出投影
每个位置对应词表大小的 logits
```

设：

- 批大小为 $B$；
- 序列长度为 $L$；
- 隐藏维度为 $d_{model}$；
- 词表大小为 $V$。

那么隐藏状态形状通常是：

$$
X\in\mathbb{R}^{B\times L\times d_{model}}
$$

最后输出 logits 的形状通常是：

$$
Z\in\mathbb{R}^{B\times L\times V}
$$

对 Decoder 训练而言，第 $t$ 个位置的输出用于预测下一个 Token。

### 3. Query、Key、Value 到底是什么

可以用“查资料”来类比：

- Query：我正在寻找什么；
- Key：每份资料用什么标签描述自己；
- Value：这份资料真正携带的内容。

假设当前词是“他”，它的 Query 可能在寻找“能作为代词指代对象的人物”。“小王”和“小李”的 Key 与这个需求有不同匹配程度；匹配后，模型不是复制 Key，而是汇总对应 Value 中的信息。

实际计算中，Q、K、V 都来自可训练线性投影：

$$
Q=XW_Q,\qquad K=XW_K,\qquad V=XW_V
$$

其中：

- $X$：当前层输入；
- $W_Q$：Query 投影矩阵；
- $W_K$：Key 投影矩阵；
- $W_V$：Value 投影矩阵；
- $Q,K,V$：投影后的三个张量。

Q、K、V 并不是人工定义的语义字段，而是通过训练学到的内部表示。上面的类比帮助理解职责，但不能把某一个维度机械解释成“人物标签”。

### 4. Scaled Dot-Product Attention 逐步推导

单头注意力公式是：

$$
\operatorname{Attention}(Q,K,V)
=
\operatorname{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}+M\right)V
$$

逐项解释：

- $Q$：查询矩阵；
- $K$：键矩阵；
- $K^\top$：把 Key 的最后两个维度转置；
- $QK^\top$：每个 Query 与每个 Key 的点积分数；
- $d_k$：每个 Key/Query 的维度；
- $\sqrt{d_k}$：缩放因子；
- $M$：可选掩码，不允许关注的位置加一个很大的负数；
- $\operatorname{softmax}$：把每一行分数变成总和为 1 的权重；
- $V$：值矩阵；
- 最后的矩阵乘法：按注意力权重对 Value 加权求和。

若忽略批次和多头维度，并设：

$$
Q,K\in\mathbb{R}^{L\times d_k},\quad
V\in\mathbb{R}^{L\times d_v}
$$

则：

$$
QK^\top\in\mathbb{R}^{L\times L}
$$

这个 $L\times L$ 矩阵表示每个位置对所有位置的关注分数。

Softmax 后：

$$
A=\operatorname{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}+M\right)
$$

其中 $A_{ij}$ 可以理解为第 $i$ 个位置从第 $j$ 个位置读取信息的权重。最终：

$$
O=AV\in\mathbb{R}^{L\times d_v}
$$

### 5. 为什么要除以 $\sqrt{d_k}$

如果 Query 和 Key 的各维近似独立、均值为 0、方差为 1，那么点积：

$$
q\cdot k=\sum_{i=1}^{d_k}q_i k_i
$$

其方差会随 $d_k$ 增长。维度很大时，点积的数值幅度可能变大，Softmax 容易进入非常尖锐的区域，梯度变小。

除以 $\sqrt{d_k}$ 后，分数尺度更稳定。这里的解释是一种常用统计直觉，不意味着真实训练中每个分量都严格独立同分布。

### 6. Softmax 在做什么

对一行分数 $s_1,\ldots,s_L$：

$$
\operatorname{softmax}(s_i)
=
\frac{e^{s_i}}{\sum_{j=1}^{L}e^{s_j}}
$$

其中：

- $e^{s_i}$：把高分放大并保证结果为正；
- 分母：所有位置指数值之和；
- 输出：每个权重在 0 到 1 之间，总和为 1。

最小数值例子：

```text
原分数：[2.0, 1.0, 0.0]
Softmax：[0.665, 0.245, 0.090]
```

输出不是只选择最高位置，而是混合多个 Value。

### 7. 什么是多头注意力

单个注意力头只有一组投影。多头注意力并行使用 $h$ 组投影：

$$
\operatorname{head}_i
=
\operatorname{Attention}(XW_i^Q,XW_i^K,XW_i^V)
$$

然后拼接：

$$
\operatorname{MHA}(X)
=
\operatorname{Concat}(\operatorname{head}_1,\ldots,\operatorname{head}_h)W_O
$$

其中：

- $h$：注意力头数量；
- $W_i^Q,W_i^K,W_i^V$：第 $i$ 个头的投影；
- $\operatorname{Concat}$：在特征维拼接各头输出；
- $W_O$：输出投影矩阵。

多头机制给模型多个表示子空间。某些头可能更关注局部搭配，另一些可能更关注远距离指代，但不能保证每个头都有稳定、可人工命名的功能。

### 8. 因果掩码为什么必要

训练 Decoder 时，整段目标文本可以并行输入。但位置 $t$ 不能偷看未来正确答案，否则任务会泄漏。

长度为 4 的因果掩码概念上是：

```text
位置 1：可看 [1]
位置 2：可看 [1, 2]
位置 3：可看 [1, 2, 3]
位置 4：可看 [1, 2, 3, 4]
```

矩阵形式：

```text
      K1  K2  K3  K4
Q1     0  -∞  -∞  -∞
Q2     0   0  -∞  -∞
Q3     0   0   0  -∞
Q4     0   0   0   0
```

被遮挡位置在 Softmax 后权重接近 0。

### 9. 前馈网络负责什么

注意力主要在不同 Token 位置之间混合信息；前馈网络对每个位置独立地做非线性变换。

原始 Transformer 使用：

$$
\operatorname{FFN}(x)
=
\max(0,xW_1+b_1)W_2+b_2
$$

其中：

- $W_1$：把隐藏维度扩展到更大的中间维度；
- $b_1$：第一层偏置；
- $\max(0,\cdot)$：ReLU 激活；
- $W_2$：投影回隐藏维度；
- $b_2$：第二层偏置。

现代模型常改用 GELU 或 SwiGLU，具体实现不完全相同，但“逐位置非线性特征变换”这一职责不变。

### 10. 残差连接为什么重要

残差结构可写为：

$$
y=x+F(x)
$$

其中：

- $x$：子层输入；
- $F(x)$：注意力或前馈子层学习到的增量；
- $y$：保留原输入并叠加新信息的输出。

这样网络不需要每一层重新构造完整表示，可以学习“在已有表示上改多少”，有助于深层网络优化和梯度传播。

### 11. Layer Normalization 在做什么

对一个 Token 的隐藏向量 $x\in\mathbb{R}^d$，LayerNorm 常写为：

$$
\operatorname{LN}(x)
=
\gamma\odot\frac{x-\mu}{\sqrt{\sigma^2+\epsilon}}+\beta
$$

其中：

- $\mu$：该隐藏向量各维的均值；
- $\sigma^2$：各维方差；
- $\epsilon$：防止除零的小常数；
- $\gamma$：可训练缩放参数；
- $\beta$：可训练平移参数；
- $\odot$：逐元素乘法。

它帮助控制激活尺度，使深层训练更稳定。

### 12. Post-LN 与 Pre-LN

原始 Transformer 常用 Post-LN：

$$
y=\operatorname{LN}(x+F(x))
$$

许多现代大模型使用 Pre-LN：

$$
y=x+F(\operatorname{LN}(x))
$$

Pre-LN 往往更利于深层模型优化，但两者各有实现和缩放变体。面试中应说“许多现代模型采用”，不要说“所有 LLM 都必须采用 Pre-LN”。

### 13. Encoder 是什么

Encoder 的自注意力通常允许每个非填充位置看到整段输入：

```text
输入：银行 利率 上升

“银行”可以同时读取左侧和右侧信息
```

因此它得到双向上下文化表示，常见用途包括：

- 文本分类；
- 命名实体识别；
- 语义向量；
- 检索编码；
- 抽取式问答。

BERT 是典型 Encoder-only 预训练模型。

“双向”指每个位置可利用左右上下文，不等于数据在时间上倒着处理两遍。

### 14. Decoder 是什么

Decoder 的自注意力使用因果掩码，只看当前及之前 Token：

```text
给定：北京是中国的
预测：首都
```

训练时可以一次计算多个位置，但每个位置的注意力视野受掩码约束。推理生成时，后一个 Token 依赖前一个已生成 Token，所以仍然自回归循环。

GPT 类模型是典型 Decoder-only 架构。

### 15. Encoder-Decoder 是什么

Encoder-Decoder 有两条信息流：

```text
源文本 → Encoder → 源文本表示
                        ↓
目标前缀 → Decoder 自注意力 → 交叉注意力 → 下一个目标 Token
```

Decoder 中通常有：

1. 带因果掩码的自注意力；
2. 读取 Encoder 输出的交叉注意力；
3. 前馈网络。

交叉注意力中：

- Query 来自 Decoder 当前隐藏状态；
- Key 和 Value 来自 Encoder 输出。

T5 是典型 Encoder-Decoder 模型。翻译、摘要、改写等输入输出边界清楚的任务常适合这种结构。

### 16. 三种架构如何比较

| 架构 | 注意力视野 | 典型训练目标 | 常见用途 |
| --- | --- | --- | --- |
| Encoder-only | 双向读取输入 | 掩码 Token、对比学习等 | 分类、抽取、检索表示 |
| Decoder-only | 因果，只看前缀 | 下一 Token 预测 | 对话、续写、代码生成 |
| Encoder-Decoder | Encoder 双向，Decoder 因果并交叉读取 | 条件生成 | 翻译、摘要、结构转换 |

这张表是典型情况，不是适用于所有模型的硬规则。研究和产品中存在前缀 LM、混合掩码、共享参数等变体。

### 17. 为什么很多通用 LLM 使用 Decoder-only

常见原因包括：

- 预训练目标统一，任意文本都可以作为续写数据；
- 输入、输出、示例、工具结果都能串成一个 Token 序列；
- 训练与推理接口简单；
- 规模化规律和工程生态成熟；
- 同一模型可处理开放式条件生成。

但这不证明 Decoder-only 对所有任务都更优。双塔检索、分类和严格输入输出转换仍可能选择 Encoder 或 Encoder-Decoder。

### 18. “Transformer 可并行”究竟指什么

训练阶段，如果完整目标序列已知，可以同时计算所有位置的 Q、K、V，并借助因果掩码避免泄漏。因此比逐时间步 RNN 更适合 GPU 矩阵并行。

生成阶段不一样：

```text
先生成 Token 1
  ↓
才能把 Token 1 加入上下文生成 Token 2
  ↓
再生成 Token 3
```

单条序列的 Decode 仍有顺序依赖。系统通过 KV Cache、连续批处理、推测解码等手段提高效率，但不能把普通自回归生成简单理解为“一次并行生成完整答案”。

### 19. 标准注意力为何是二次复杂度

注意力分数矩阵形状为 $L\times L$。只看 QK 点积的主要计算量约为：

$$
O(L^2d_k)
$$

注意力矩阵元素数量约为：

$$
O(L^2)
$$

序列长度从 4,000 翻倍到 8,000 时，注意力矩阵元素数量约变为 4 倍。实际端到端成本还包含投影、FFN、内核实现、显存带宽和并行策略，不能只用一个大 O 式子精确预测延迟。

### 20. 一个最小自注意力实现

下面是帮助理解形状的 PyTorch 风格伪代码，忽略了批次、多头、Dropout 和数值优化：

```python
import math

def attention(x, w_q, w_k, w_v, causal_mask=None):
    q = x @ w_q                 # [L, d_k]
    k = x @ w_k                 # [L, d_k]
    v = x @ w_v                 # [L, d_v]

    scores = q @ k.T            # [L, L]
    scores = scores / math.sqrt(q.shape[-1])

    if causal_mask is not None:
        scores = scores + causal_mask

    weights = softmax(scores, dim=-1)
    output = weights @ v        # [L, d_v]
    return output
```

生产实现会使用融合内核、低精度计算、并行和避免显式保存完整注意力矩阵的算法。

## 工程实践与边界

### 1. 面试回答先讲数据流，再讲公式

推荐顺序：

1. Token 变向量并加入位置；
2. 注意力跨 Token 汇总信息；
3. FFN 做逐位置非线性变换；
4. 残差和归一化稳定深层训练；
5. 多层堆叠形成上下文化表示；
6. 输出头映射到词表或任务标签。

先说清职责，公式才不会变成死记。

### 2. 根据任务选择架构，不按流行度选择

- 开放式对话和代码生成：优先评估 Decoder-only；
- 低延迟文本向量和分类：优先评估 Encoder；
- 输入输出转换明显、训练数据充分：可评估 Encoder-Decoder；
- 真实选择还受模型供应、部署生态、语言和业务指标影响。

### 3. 注意 Padding Mask 与 Causal Mask 不同

- Padding Mask：不让模型读取补齐位置；
- Causal Mask：不让当前位置读取未来位置。

Decoder 训练中可能同时需要两种掩码。混淆后会造成信息泄漏或错误注意填充。

### 4. 监控数值稳定性

训练深层 Transformer 时关注：

- loss 是否出现 NaN/Inf；
- 梯度范数是否突然爆炸；
- 激活和权重尺度；
- 学习率 warmup；
- 混合精度的 loss scaling；
- 不同并行方式下的数值差异。

### 5. 不要根据注意力权重直接下因果结论

注意力权重可用于观察模型读取模式，但“权重高”不等同于“该 Token 是最终决策的唯一原因”。残差流、多个层和头、FFN 都会影响输出。解释模型行为需要多种方法和对照实验。

### 6. 长上下文瓶颈不只在计算量

还包括：

- KV Cache 显存；
- 显存带宽；
- 位置编码外推；
- 训练数据中的长序列覆盖；
- 模型是否真正使用远距离信息；
- 服务调度和批处理碎片。

支持一个更大的最大长度，不代表在所有位置都能稳定找回信息。

### 7. 从模型配置核对真实实现

“Transformer”不是一份固定配方。生产中应核对：

- 层数与隐藏维度；
- 注意力头和 KV 头数量；
- 激活函数；
- 归一化类型和位置；
- 位置编码；
- 是否使用 Bias；
- 是否采用 MoE；
- 上下文长度和滑动窗口；
- 词表与权重共享。

## 常见误区

### 误区一：Attention 就是 Transformer 的全部

错误。Transformer Block 还依赖 FFN、残差、归一化、位置机制和输出投影。

### 误区二：Q、K、V 是三个不同输入句子

错误。自注意力中它们通常来自同一隐藏状态的不同可训练投影；交叉注意力中 Q 与 K/V 才来自不同侧。

### 误区三：Decoder 训练时只能一个 Token 一个 Token 计算

错误。已知完整目标序列时可以并行计算多个位置，只需用因果掩码防止看到未来；自回归推理才有逐步依赖。

### 误区四：Encoder 完全不能生成文本

说法过度武断。Encoder 本身通常输出表示而非自回归序列，但可以接任务头，或作为 Encoder-Decoder 的组成部分。架构也存在非典型变体。

### 误区五：每个注意力头都有明确的人类可读功能

错误。部分头可能呈现可解释模式，但头之间存在冗余和组合，不能强行逐头命名。

### 误区六：Transformer 完全不知道顺序

不完整。裸注意力对置换不敏感，所以模型需要位置编码或其他位置机制；完整 Transformer 通常已经注入顺序信息。

### 误区七：注意力复杂度始终都是 $O(L^2)$

不严谨。标准全注意力分数矩阵是二次规模，但稀疏、滑动窗口、线性注意力和状态空间等方案具有不同复杂度与能力取舍。

### 误区八：Decoder-only 必定优于 Encoder-Decoder

错误。它在通用生成生态中很常见，但具体任务还要比较质量、推理成本、输入输出长度和已有模型资源。

## 面试追问

### 问：Self-Attention 和 Cross-Attention 有什么区别

答：Self-Attention 的 Q、K、V 都来自同一序列表示；Cross-Attention 的 Q 来自一个序列，K、V 来自另一个序列。Encoder-Decoder 中 Decoder 用交叉注意力读取 Encoder 输出。

### 问：为什么需要位置编码

答：自注意力只根据内容计算两两关系，如果不额外注入位置，交换 Token 顺序会导致表示缺少顺序区分。位置编码、RoPE 或注意力偏置等机制负责提供位置编号或相对距离信息。

### 问：为什么 Attention 要除以 $\sqrt{d_k}$

答：点积尺度会随维度增大，可能让 Softmax 过于尖锐并减小梯度。按 $\sqrt{d_k}$ 缩放可让分数尺度更稳定。

### 问：训练能并行，为什么推理还慢

答：训练时目标序列已知，可以在因果掩码下同时计算多个位置；生成时下一个 Token 依赖刚生成的前一个 Token。Decode 通常是顺序且受显存带宽影响，还要逐步读取 KV Cache。

### 延伸追问速答

| 追问 | 回答要点 |
| --- | --- |
| FFN 为什么经常比隐藏维度宽 | 扩展后做非线性变换再压回，可提高逐位置表示能力；比例由预算、激活和架构决定，不固定为 4 倍 |
| Pre-LN 和 Post-LN 怎么选 | 原始 Transformer 使用 Post-LN；许多深层模型采用 Pre-LN 改善优化，还需结合归一化、残差缩放和初始化实验 |
| Encoder 双向注意力为何不直接做自回归训练 | 若预测位置看到未来正确 Token 会泄漏答案；自回归生成需要因果约束 |
| 标准注意力的主要瓶颈是什么 | 长序列带来 $L\times L$ 计算和中间内存；Decode 还受 KV Cache 容量与带宽影响 |
