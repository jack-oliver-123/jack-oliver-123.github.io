## 适用场景

Redis 常用于缓存、会话管理、消息队列和分布式锁。本文记录如何使用 Docker 部署单节点 Redis，适合本地开发和测试环境。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建工作目录：

```bash
# Linux
mkdir -p /opt/redis/config /opt/redis/data

# Windows（PowerShell）
mkdir D:/redis-env/config, D:/redis-env/data
```

## 编写 Redis 配置

在配置目录下新建 `redis.conf`：

```text
bind 0.0.0.0
protected-mode no
appendonly yes
appendfilename "appendonly.aof"
requirepass your_redis_password
maxmemory 512mb
maxmemory-policy allkeys-lru
```

配置说明：

- `appendonly yes`：开启 AOF 持久化，数据更安全；
- `requirepass`：设置访问密码；
- `maxmemory`：限制最大内存使用；
- `allkeys-lru`：内存达到上限时淘汰最近最少使用的 key。

## 编写 docker-compose

新建 `docker-compose.yml`：

```yaml
services:
  redis:
    image: redis:7.2
    container_name: redis-local
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - ./config/redis.conf:/etc/redis/redis.conf
      - ./data:/data
    command: redis-server /etc/redis/redis.conf
    environment:
      - TZ=Asia/Shanghai
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "your_redis_password", "ping"]
      interval: 10s
      timeout: 3s
      retries: 3
```

## 启动 Redis

```bash
docker compose up -d
```

查看容器状态：

```bash
docker ps
```

查看日志：

```bash
docker logs -f redis-local
```

## 验证连接

使用 redis-cli 连接：

```bash
docker exec -it redis-local redis-cli -a your_redis_password
```

执行测试命令：

```text
127.0.0.1:6379> SET hello world
OK
127.0.0.1:6379> GET hello
"world"
```

外部客户端连接：

```text
Host: 127.0.0.1（或服务器 IP）
Port: 6379
Password: your_redis_password
```

## 验证持久化

重启容器后检查数据是否保留：

```bash
docker compose restart
docker exec -it redis-local redis-cli -a your_redis_password GET hello
```

如果返回 `"world"`，说明 AOF 持久化生效。

## 常见问题

### 连接被拒绝

检查：
- 容器是否正常运行（`docker ps`）；
- 密码是否正确；
- 防火墙或安全组是否放行 `6379`。

### 端口被占用

修改宿主机端口：

```yaml
ports:
  - "6380:6379"
```

连接时端口改为 `6380`。

## 停止和清理

```bash
# 停止服务（保留数据）
docker compose down

# 停止并删除数据卷
docker compose down -v
```

## 后续建议

- 不要使用默认密码暴露到公网。
- 生产环境建议关闭 `protected-mode no` 并配合网络策略限制访问。
- 如果需要 RDB 快照，可以在配置中添加 `save 900 1` 等规则，与 AOF 互补。
- 集群模式和哨兵模式需要单独规划。
