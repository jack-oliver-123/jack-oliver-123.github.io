## 适用场景

Nacos 可以作为服务注册中心和配置中心。本文记录如何使用 Docker 部署 Nacos 单机实例，适合本地开发、测试环境和小型验证场景。

官方网站：<https://nacos.io/>

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建数据目录：

```bash
# Linux
mkdir -p /opt/nacos/logs /opt/nacos/data

# Windows（PowerShell）
mkdir D:/nacos-env/logs, D:/nacos-env/data
```

## 编写 docker-compose

新建 `docker-compose.yml`：

```yaml
services:
  nacos:
    image: nacos/nacos-server:v2.3.0
    container_name: nacos-local
    restart: always
    ports:
      - "8848:8848"
      - "9848:9848"
      - "9849:9849"
    volumes:
      - ./logs:/home/nacos/logs
      - ./data:/home/nacos/data
    environment:
      - MODE=standalone
      - NACOS_AUTH_ENABLE=false
      - JVM_XMS=512m
      - JVM_XMX=512m
      - TZ=Asia/Shanghai
    healthcheck:
      test: ["CMD-SHELL", "curl -fs http://localhost:8848/nacos/actuator/health || exit 1"]
      interval: 15s
      timeout: 5s
      retries: 5
```

端口说明：

- `8848`：Web 控制台和 HTTP API；
- `9848`：Nacos 2.x gRPC 客户端通信端口；
- `9849`：Nacos 2.x gRPC 服务端通信端口。

> ⚠️ 本文为方便开发验证关闭了鉴权。生产环境必须设置 `NACOS_AUTH_ENABLE=true` 并配置 `NACOS_AUTH_TOKEN`、`NACOS_AUTH_IDENTITY_KEY`、`NACOS_AUTH_IDENTITY_VALUE`，同时修改默认密码。

## 连接外部 MySQL（可选）

如果希望配置数据持久保存到 MySQL，需要先创建 Nacos 数据库。

官方建表脚本：<https://github.com/alibaba/nacos/blob/master/distribution/conf/mysql-schema.sql>

在 `docker-compose.yml` 的 `environment` 中追加：

```yaml
    environment:
      - MODE=standalone
      - JVM_XMS=512m
      - JVM_XMX=512m
      - SPRING_DATASOURCE_PLATFORM=mysql
      - MYSQL_SERVICE_HOST=your_mysql_host
      - MYSQL_SERVICE_PORT=3306
      - MYSQL_SERVICE_DB_NAME=nacos_config
      - MYSQL_SERVICE_USER=nacos_user
      - MYSQL_SERVICE_PASSWORD=your_mysql_password
      - "MYSQL_SERVICE_DB_PARAM=characterEncoding=utf8&connectTimeout=1000&socketTimeout=3000&autoReconnect=true&useSSL=false&serverTimezone=Asia/Shanghai"
      - NACOS_AUTH_ENABLE=false
      - TZ=Asia/Shanghai
```

> Linux 下如果 MySQL 跑在宿主机上，`MYSQL_SERVICE_HOST` 填宿主机实际 IP 或使用 `--network host` 模式。Windows/Mac 可以使用 `host.docker.internal`。

## 启动服务

```bash
docker compose up -d
```

查看日志（首次启动可能需要 30-60 秒）：

```bash
docker logs -f nacos-local
```

看到 `Nacos started successfully` 说明启动完成。

## 验证安装

浏览器访问：

```text
http://localhost:8848/nacos
```

默认账号密码：

```text
nacos / nacos
```

## 应用接入示例

Spring Cloud Alibaba 项目引入依赖：

```xml
<dependency>
  <groupId>com.alibaba.cloud</groupId>
  <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
</dependency>
<dependency>
  <groupId>com.alibaba.cloud</groupId>
  <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
</dependency>
```

配置示例（`bootstrap.yml`）：

```yaml
spring:
  application:
    name: demo-service
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848
      config:
        server-addr: 127.0.0.1:8848
        file-extension: yaml
```

> Nacos Config 需要在 `bootstrap.yml`（而非 `application.yml`）中加载，确保在应用启动前获取远程配置。

## 常见问题

### 页面访问 404

确认访问地址带 `/nacos`：`http://localhost:8848/nacos`。

### 微服务无法注册

确认 `9848` 和 `9849` 端口已映射。Nacos 2.x 的 gRPC 通信依赖这两个端口。

### MySQL 连接失败

检查：
- MySQL 地址是否能从容器内访问；
- 数据库和表是否已创建；
- 用户名、密码和权限是否正确；
- MySQL 防火墙和安全组是否放行。

### Docker 内应用连接 Nacos

如果应用也跑在 Docker 中，`server-addr` 不能写 `localhost`，应使用容器名 `nacos-local`（同一网络下）或 `host.docker.internal`（Windows/Mac）。

## 停止和清理

```bash
docker compose down
```

## 后续建议

- 临时验证可以关闭鉴权，正式环境必须开启。
- 生产环境建议接入外部 MySQL 并使用集群模式。
- 不要在公开网络暴露 Nacos 控制台。
