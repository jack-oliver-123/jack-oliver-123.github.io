## 包装类

### 为什么需要包装类

在上一篇文章中我们提到，Java 集合框架只能存储**引用类型**。但日常开发中大量使用 `int`、`double` 等基本类型，它们不继承自 `Object`，无法直接放入集合。

为了解决这个矛盾，Java 为每种基本类型提供了对应的**包装类**（Wrapper Class）——将基本类型"包装"成对象。

### 基本类型与包装类的对应关系

| 基本类型 | 包装类 | 备注 |
|---------|--------|-----|
| `byte` | `Byte` | |
| `short` | `Short` | |
| `int` | `Integer` | ⚠️ 不是 Int |
| `long` | `Long` | |
| `float` | `Float` | |
| `double` | `Double` | |
| `char` | `Character` | ⚠️ 不是 Char |
| `boolean` | `Boolean` | |

> 记忆技巧：除了 `Integer` 和 `Character` 比较特殊，其余 6 种包装类都是基本类型首字母大写。

### 装箱与拆箱

**装箱**（Boxing）：将基本类型转换为对应的包装类对象。

**拆箱**（Unboxing）：将包装类对象中的值取出，放到基本类型变量中。

```java
int i = 10;

// 装箱操作：基本类型 → 包装类型
Integer ii = Integer.valueOf(i);    // 推荐方式
Integer ij = new Integer(i);       // 已废弃（JDK 9+）

// 拆箱操作：包装类型 → 基本类型
int j = ii.intValue();
```

### 自动装箱与自动拆箱

手动调用 `valueOf()` 和 `intValue()` 写起来很繁琐。从 JDK 5 开始，Java 支持**自动装箱/拆箱**，编译器帮你自动插入转换代码：

```java
int i = 10;

Integer ii = i;           // 自动装箱，编译器自动调用 Integer.valueOf(i)
Integer ij = (Integer) i; // 自动装箱（显式写法，效果相同）

int j = ii;               // 自动拆箱，编译器自动调用 ii.intValue()
int k = (int) ii;         // 自动拆箱（显式写法，效果相同）
```

### 面试经典：Integer 缓存池

看下面这段代码，输出什么？

```java
public static void main(String[] args) {
    Integer a = 127;
    Integer b = 127;

    Integer c = 128;
    Integer d = 128;

    System.out.println(a == b); // ?
    System.out.println(c == d); // ?
}
```

**答案**：`true` 和 `false`。

**原因**：`Integer.valueOf()` 内部有一个缓存池，对 **[-128, 127]** 范围内的值会返回同一个对象引用。超出这个范围则每次 `new` 一个新对象。所以 `a == b` 比较的是同一个对象（`true`），而 `c == d` 比较的是两个不同的对象（`false`）。

```java
// Integer.valueOf() 源码简化版
public static Integer valueOf(int i) {
    if (i >= -128 && i <= 127) {
        return IntegerCache.cache[i + 128]; // 返回缓存对象
    }
    return new Integer(i); // 超出范围，创建新对象
}
```

> **实际开发经验**：比较两个 Integer 的值永远用 `.equals()`，不要用 `==`。`==` 比较的是引用地址，只有缓存范围内碰巧相等，这是非常隐蔽的 bug。

## 什么是泛型

### 引出问题

假设你需要实现一个类，其中包含一个数组，可以存放任何类型的数据，也能根据下标返回对应元素：

```java
class MyArray {
    public Object[] array = new Object[10];

    public Object getPos(int pos) {
        return this.array[pos];
    }

    public void setVal(int pos, Object val) {
        this.array[pos] = val;
    }
}

public class TestDemo {
    public static void main(String[] args) {
        MyArray myArray = new MyArray();
        myArray.setVal(0, 10);
        myArray.setVal(1, "hello"); // 字符串也能放进去

        String ret = myArray.getPos(1); // ❌ 编译报错！返回的是 Object
        System.out.println(ret);
    }
}
```

**问题暴露**：

1. 什么类型都能往里塞——缺乏类型约束
2. 取出元素必须强制转换——麻烦且容易出错（`ClassCastException`）

我们真正想要的是：**指定这个容器只能持有某一种类型，让编译器帮我们做类型检查**。

### 泛型的定义

泛型（Generic）是 JDK 1.5 引入的语法，核心思想是**类型参数化**——把类型当作参数传递给类或方法。

一句话概括：**泛型就是让你在定义类/方法时不指定具体类型，使用时再确定类型。**

> 《Java 编程思想》：一般的类和方法，只能使用具体的类型——要么是基本类型，要么是自定义的类。如果要编写可以应用于多种类型的代码，这种刻板的限制对代码的束缚就会很大。

## 泛型类

### 语法

```java
class 类名<类型形参列表> {
    // 这里可以使用类型参数
}

// 示例
class ClassName<T1, T2, ..., Tn> {
}

// 继承时也可以使用
class ClassName<T1, T2> extends ParentClass<T1> {
    // 可以只使用部分类型参数
}
```

### 用泛型改写 MyArray

```java
class MyArray<T> {
    public T[] array = (T[]) new Object[10]; // 注释 1

    public T getPos(int pos) {
        return this.array[pos];
    }

    public void setVal(int pos, T val) {
        this.array[pos] = val;
    }
}

public class TestDemo {
    public static void main(String[] args) {
        MyArray<Integer> myArray = new MyArray<>(); // 注释 2

        myArray.setVal(0, 10);
        myArray.setVal(1, 12);
        int ret = myArray.getPos(1); // 注释 3：不需要强制转换

        myArray.setVal(2, "bit");    // 注释 4：编译报错！类型不匹配
    }
}
```

**代码解析**：

1. **注释 1**：类名后的 `<T>` 是类型占位符，表示当前类是泛型类（不能直接 `new T[]`，原因后面解释）
2. **注释 2**：实例化时用 `<Integer>` 指定类型实参，此后这个对象只能存 `Integer`
3. **注释 3**：取出的元素编译器自动转换为 `Integer`，再自动拆箱为 `int`，无需手动强转
4. **注释 4**：试图存入 `String`，编译器直接报错——**类型安全在编译期保证**

### 类型形参命名规范

类型参数通常使用单个大写字母，常见约定：

| 字母 | 含义 | 典型使用场景 |
|------|------|------------|
| `E` | Element | 集合元素（`List<E>`、`Set<E>`） |
| `K` | Key | 映射的键（`Map<K, V>`） |
| `V` | Value | 映射的值（`Map<K, V>`） |
| `T` | Type | 通用类型 |
| `N` | Number | 数值类型 |
| `S, U, R` | — | 第二、第三个类型参数，或返回值类型 |

### 类型推导（钻石语法）

当编译器能从上下文推断出类型实参时，可以使用空的 `<>`（钻石语法，JDK 7+）：

```java
// 完整写法
MyArray<Integer> list = new MyArray<Integer>();

// 类型推导——编译器根据左边推断右边的类型实参
MyArray<Integer> list = new MyArray<>(); // ✅ 推荐
```

### 裸类型（Raw Type）

裸类型就是使用泛型类但不带类型实参：

```java
MyArray list = new MyArray(); // 裸类型，等价于 MyArray<Object>
```

> ⚠️ **不要自己使用裸类型！** 它只是为了兼容 JDK 5 之前的旧代码而保留的机制。使用裸类型会失去所有泛型的类型检查优势。

### 泛型小结

泛型的三大优点：

1. **类型参数化**：一套代码适用于多种类型
2. **编译期类型检查**：类型错误在编译时就被发现，而不是运行时抛 `ClassCastException`
3. **自动类型转换**：取出元素无需手动强转

**注意：泛型只能接受引用类型，所有基本数据类型必须使用包装类！**

```java
MyArray<int> list = new MyArray<>();     // ❌ 编译错误
MyArray<Integer> list = new MyArray<>(); // ✅ 正确
```

## 泛型的编译——类型擦除

### 擦除机制

泛型到底是怎么编译的？这是一个经典面试题。

通过 `javap -c` 查看字节码，你会发现：**所有的 `T` 都变成了 `Object`**。

```
// 源码
public T getPos(int pos) { return this.array[pos]; }

// 字节码（简化）
public Object getPos(int pos) { return this.array[pos]; }
```

这就是 Java 泛型的**擦除机制**（Type Erasure）：

- 编译过程中，编译器将所有的 `T` 替换为 `Object`（或其上界类型）
- 生成的字节码中**不包含任何泛型类型信息**
- 泛型只在编译期起作用，运行期完全不存在

> 这意味着 Java 的泛型是"伪泛型"——与 C++ 模板在编译时为每种类型生成独立代码不同，Java 泛型只有一份字节码。

### 为什么不能 `new T[]`

既然类型擦除把 `T` 变成 `Object`，那 `T[] ts = new T[5]` 不就等价于 `Object[] ts = new Object[5]` 吗？为什么编译器不允许？

看这个例子就明白了：

```java
class MyArray<T> {
    public T[] array = (T[]) new Object[10];

    public T getPos(int pos) {
        return this.array[pos];
    }

    public void setVal(int pos, T val) {
        this.array[pos] = val;
    }

    public T[] getArray() {
        return array;
    }
}

public static void main(String[] args) {
    MyArray<Integer> myArray = new MyArray<>();
    Integer[] integers = myArray.getArray(); // 运行时报错！
}
```

**运行结果**：

```
Exception in thread "main" java.lang.ClassCastException:
    [Ljava.lang.Object; cannot be cast to [Ljava.lang.Integer;
```

**原因**：类型擦除后，`getArray()` 实际返回的是 `Object[]`。将 `Object[]` 赋值给 `Integer[]` 引用是不安全的——因为 `Object[]` 里可能存着 `String`、`Person` 等任何类型的数据。编译器无法保证运行时的类型安全，所以从根源上禁止 `new T[]`。

### 正确创建泛型数组：使用反射

如果确实需要返回一个具体类型的数组，可以通过反射实现：

```java
class MyArray<T> {
    public T[] array;

    /**
     * 通过反射创建指定类型的数组
     * @param clazz  元素类型的 Class 对象
     * @param capacity 数组容量
     */
    public MyArray(Class<T> clazz, int capacity) {
        array = (T[]) Array.newInstance(clazz, capacity);
    }

    public T getPos(int pos) {
        return this.array[pos];
    }

    public void setVal(int pos, T val) {
        this.array[pos] = val;
    }

    public T[] getArray() {
        return array;
    }
}

public static void main(String[] args) {
    MyArray<Integer> myArray = new MyArray<>(Integer.class, 10);
    Integer[] integers = myArray.getArray(); // ✅ 正常运行
}
```

> **实际开发中**：你几乎不需要自己这样做。JDK 集合类内部已经处理好了这些细节——`ArrayList` 内部就是用 `Object[]` 存储，只在 `get()` 时做强转。了解原理即可，不必纠结。

## 泛型的上界

### 为什么需要上界

有时候你希望泛型不是"什么类型都行"，而是限定在某个范围内。例如，你写了一个数值计算工具类，只希望接受 `Number` 的子类（`Integer`、`Double` 等），不希望接受 `String`。

### 语法

```java
class 类名<类型形参 extends 类型边界> {
    ...
}
```

### 示例：限定为 Number 的子类

```java
public class MyArray<E extends Number> {
    // ...
}

MyArray<Integer> l1;  // ✅ 正常，Integer 是 Number 的子类
MyArray<Double> l2;   // ✅ 正常，Double 是 Number 的子类
MyArray<String> l3;   // ❌ 编译错误！String 不是 Number 的子类
```

编译器报错信息：

```
error: type argument String is not within bounds of type-variable E
    where E is a type-variable:
        E extends Number declared in class MyArray
```

### 示例：限定实现了 Comparable 接口

这是集合框架中最常见的上界约束——要求元素可比较：

```java
public class MyArray<E extends Comparable<E>> {
    // 这里可以安全地调用 e.compareTo()
}
```

`E extends Comparable<E>` 的含义：类型参数 `E` 必须是实现了 `Comparable<E>` 接口的类型。这样你就可以在类内部放心调用 `compareTo()` 方法。

> **为什么用 `extends` 而不是 `implements`？** 在泛型上界中，无论约束的是类还是接口，统一使用 `extends` 关键字。这是语法规定。

### 没有指定上界时

如果定义泛型类时没有写 `extends`，等价于 `<E extends Object>`——即没有任何约束，所有引用类型都可以作为类型实参。

## 泛型方法

除了泛型类，还可以定义**泛型方法**——方法自己声明类型参数，不依赖外层类。

### 语法

```java
方法限定符 <类型形参列表> 返回值类型 方法名称(形参列表) {
    ...
}
```

### 示例：通用交换方法

```java
public class Util {
    // 静态泛型方法：需要在 static 后用 <> 声明类型参数
    public static <E> void swap(E[] array, int i, int j) {
        E t = array[i];
        array[i] = array[j];
        array[j] = t;
    }
}
```

### 使用方式一：类型推导（推荐）

编译器根据实参类型自动推导出 `E` 的具体类型：

```java
Integer[] a = {1, 2, 3, 4, 5};
Util.swap(a, 0, 4); // 编译器推导 E = Integer

String[] b = {"hello", "world"};
Util.swap(b, 0, 1); // 编译器推导 E = String
```

### 使用方式二：显式指定类型

也可以手动指定类型实参（通常在推导不出来时使用）：

```java
Integer[] a = {1, 2, 3, 4, 5};
Util.<Integer>swap(a, 0, 4);

String[] b = {"hello", "world"};
Util.<String>swap(b, 0, 1);
```

### 泛型方法 vs 泛型类中的方法

```java
// 这是泛型类中的普通方法——T 来自类的类型参数
class MyArray<T> {
    public T getPos(int pos) { ... }
}

// 这是泛型方法——<E> 是方法自己声明的类型参数，与类无关
class Util {
    public static <E> void swap(E[] array, int i, int j) { ... }
}
```

关键区别：**泛型方法可以是 `static` 的**（因为它有自己的类型参数），而泛型类中使用类的类型参数的方法不能是 `static`（因为 `T` 属于实例级别）。

## 小结

| 概念 | 核心要点 |
|------|---------|
| 包装类 | 基本类型 → 引用类型的桥梁，集合只能存包装类型 |
| 自动装箱/拆箱 | 编译器自动插入 `valueOf()` 和 `xxxValue()` 调用 |
| 泛型 | 类型参数化，编译期检查类型安全，避免强转 |
| 类型擦除 | 编译后 T → Object，运行时无泛型信息 |
| 泛型上界 | `extends` 限制类型参数范围，可以是类也可以是接口 |
| 泛型方法 | 方法级别声明类型参数，可以是 static |

**重要认知**：

- **泛型是编译期机制**——它让你在编译期就能发现类型错误，而不是运行时才炸一个 `ClassCastException`。
- **泛型不创造新类型**——`MyArray<Integer>` 和 `MyArray<String>` 在运行时是同一个类（都是 `MyArray`）。
- **泛型只能用引用类型**——基本类型必须用对应的包装类。
