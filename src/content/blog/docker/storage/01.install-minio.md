## 适用场景

MinIO 是兼容 S3 协议的对象存储服务，适合保存图片、文档、备份文件和其他非结构化数据。本文记录如何使用 Docker 部署单节点 MinIO。

## 准备工作

确认 Docker 可用：

```bash
docker --version
```

创建数据和配置目录：

```bash
# Linux
mkdir -p /opt/minio/data /opt/minio/config

# Windows（PowerShell）
mkdir D:/minio-env/data, D:/minio-env/config
```

## 拉取镜像

```bash
docker pull minio/minio:RELEASE.2025-04-22T22-12-26Z
```

## 启动 MinIO

```bash
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=your_minio_user \
  -e MINIO_ROOT_PASSWORD=your_minio_password \
  -v /opt/minio/data:/data \
  -v /opt/minio/config:/root/.minio \
  --restart unless-stopped \
  minio/minio:RELEASE.2025-04-22T22-12-26Z \
  server /data --console-address ':9001'
```

> Windows 用户把 `/opt/minio/` 替换为 `D:/minio-env/` 即可。

> ⚠️ `MINIO_ROOT_PASSWORD` 必须至少 8 个字符，否则容器启动会失败。

端口说明：

- `9000`：S3 API 访问端口；
- `9001`：Web 控制台端口。

## 验证安装

查看容器状态：

```bash
docker ps
```

查看日志：

```bash
docker logs -f minio
```

打开浏览器访问控制台：

```text
http://服务器IP:9001
```

使用启动命令中配置的 `MINIO_ROOT_USER` 和 `MINIO_ROOT_PASSWORD` 登录。

## 创建 Bucket

登录控制台后，点击 **Create Bucket**，例如创建：

```text
app-files
```

也可以使用 MinIO 命令行客户端 `mc`：

```bash
# 安装 mc（Linux）
curl -O https://dl.min.io/client/mc/release/linux-amd64/mc
chmod +x mc

# 设置别名
./mc alias set local http://localhost:9000 your_minio_user your_minio_password

# 创建 Bucket
./mc mb local/app-files

# 上传文件
./mc cp myfile.txt local/app-files/
```

## 常见问题

### 控制台打不开

检查：

- `9001` 是否映射；
- 服务器防火墙或安全组是否放行；
- 容器是否正常运行。

### API 访问失败

确认客户端连接的是 `9000` 端口（S3 API），而不是控制台 `9001` 端口。

### 密码不满足要求

`MINIO_ROOT_PASSWORD` 必须不少于 8 个字符，否则容器会报错退出。

## 停止和清理

```bash
docker stop minio
docker rm minio

# 数据保留在宿主机目录中，重新启动挂载同一目录即可恢复
```

## 后续建议

- 生产环境不要使用弱密码。
- 重要数据需要规划备份策略。
- 生产环境建议配置 HTTPS（通过反向代理如 Nginx）。
- 多节点和纠删码模式需要单独规划磁盘和节点数量。
