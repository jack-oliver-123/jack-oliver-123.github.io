如果你第一次看到下面这些词：

> Transformer、RoPE、LoRA、DPO、KV Cache、Prefill、FP8、FSDP……

很可能会有一种感觉：

**每个词都听过，但放在一起完全不知道从哪里开始。**

这不是你的问题。

大模型工程本来就横跨了很多领域：

- 自然语言处理；
- 深度学习；
- 分布式训练；
- GPU 计算；
- 在线服务；
- 评测与安全；
- 数据工程。

很多资料默认读者已经学过其中一半，于是直接从公式、论文或框架参数开始讲。

本专题采用相反的顺序：

1. 先用具体场景解释“为什么会有这个技术”；
2. 再把术语翻译成普通话；
3. 然后讲原理、公式和最小例子；
4. 最后讨论生产环境中的取舍和面试追问。

你不需要先把高等数学、CUDA 或分布式系统全部学完。

遇到公式时，文章会解释每个符号在表示什么；遇到英文缩写时，也会先给出完整名称和用途。

## 这个专题适合谁

本专题主要面向：

- 第一次系统学习大模型的开发者；
- 从 Java、前端、后端、测试或运维转向 AI 应用开发的人；
- 会调用模型 API，但不理解模型内部和推理部署的人；
- 准备大模型应用、算法工程、推理工程或 AI 平台岗位面试的人；
- 学过零散概念，希望建立完整知识地图的人。

如果你已经在训练或部署大模型，也可以把每篇的“工程实践与边界”“常见误区”和“面试追问”当作复习清单。

## 学完以后应该达到什么程度

这 33 道题不是为了让你只会背定义。

学完后，你应该能够完成四件事。

### 1. 用普通话解释概念

例如，不只说：

> “KV Cache 是缓存 Key 和 Value。”

而是能继续解释：

- 为什么自回归生成会重复计算；
- 缓存了什么张量；
- 它为什么随上下文和并发增长；
- GQA、分页缓存和缓存量化会改变什么；
- 它对吞吐和显存有什么影响。

### 2. 说清技术之间的关系

例如，你应该能区分：

- 预训练、SFT、偏好优化和推理时计算；
- DPO、PPO、GRPO、RLHF 和拒绝采样；
- MHA、MQA、GQA 和 FlashAttention；
- 权重量化、KV Cache 量化和低精度训练；
- KV Cache、Prefix Cache 和 Semantic Cache；
- 数据并行、张量并行、流水线并行和参数分片。

### 3. 给出有条件的工程选择

真实面试很少只问“它是什么”。

面试官通常会继续问：

> “那你们项目为什么选它？”

你需要根据以下约束回答：

- 效果；
- 延迟；
- 吞吐；
- 显存；
- 硬件；
- 数据；
- 成本；
- 隐私；
- 可维护性；
- 团队能力。

### 4. 知道结论的边界

大模型领域变化很快。

“某个框架在所有负载下都最快”“某个参数在所有任务上都最好”“模型越大效果就会持续变好”都不是可靠的工程结论。

本专题会尽量把以下内容分开：

- 数学定义；
- 论文在特定实验中的结论；
- 常见工程经验；
- 需要按模型和版本重新验证的事实。

## 先建立一张大模型生命周期地图

你可以把一个大模型从诞生到上线理解为下面这条链路：

```text
原始数据
   ↓ 清洗、去重、配比、切分
Tokenizer
   ↓ 文本变成 Token ID
预训练
   ↓ 学习预测 Token 的基础能力
监督微调 SFT
   ↓ 学习指令、任务和回答格式
偏好/强化学习后训练
   ↓ 调整有用性、安全、推理与风格
离线评测与安全测试
   ↓ 决定是否达到发布门槛
量化与推理优化
   ↓ 降低延迟、显存和成本
在线部署
   ↓ 调度、缓存、扩缩容、监控
业务反馈与回归评测
   └────────────→ 返回数据和训练环节
```

这 33 道题分别位于这条链路的不同位置。

如果一开始不知道某个概念属于哪里，先回到这张图定位。

## 零基础学习前需要知道的 12 个词

### 1. Token

模型通常不直接按完整汉字或单词计算。

Tokenizer 会把文本切成 Token，再转换成整数 ID。

一个 Token 可能是一个字、词的一部分、标点或代码片段。

### 2. Parameter

Parameter 就是模型在训练中学习到的数值。

“7B 模型”通常表示大约 70 亿个参数，但具体统计口径要看模型说明。

### 3. Tensor

Tensor 可以先理解为多维数字数组。

向量、矩阵和更高维数组都可以是 Tensor。

大模型的参数、输入、激活和缓存都以 Tensor 形式计算。

### 4. Forward

Forward，也就是前向传播，是把输入送进模型并得到输出的过程。

### 5. Loss

Loss 是一个表示“模型当前错得有多严重”的训练目标。

训练会计算 Loss 对参数的梯度，再调整参数。

### 6. Gradient

Gradient，也就是梯度，描述参数向哪个方向改变会让 Loss 变化。

反向传播负责计算梯度。

### 7. Batch

Batch 是一次共同处理的一组训练样本或推理请求。

Batch 变大可能提高硬件利用率，但通常也会增加显存和等待时间。

### 8. Epoch 与 Step

Epoch 通常表示完整遍历一次训练数据。

Step 通常表示完成一次参数更新。

大规模预训练更常按训练 Token 数和 Step 描述进度。

### 9. Context Window

Context Window 是模型一次能够接收和处理的 Token 范围上限。

标称窗口很长，不代表模型在全部长度上都能有效利用信息。

### 10. Inference

Inference，也就是推理，是模型训练完成后，根据输入生成或计算输出的过程。

### 11. Latency 与 Throughput

Latency 是一次请求要等多久。

Throughput 是单位时间能处理多少请求或 Token。

提高吞吐的批处理有时会增加单个用户等待时间。

### 12. GPU Memory

GPU Memory，也就是显存，不只存模型权重。

训练还需要梯度、优化器状态和激活；推理还可能需要 KV Cache 和临时工作区。

## 建议学习路线

### 第一阶段：模型为什么能够生成文本

先读 01～07。

你会建立以下基础：

- LLM 与传统 NLP 的关系；
- Transformer 的整体结构；
- 注意力如何计算；
- 模型怎样知道 Token 的位置；
- 文本怎样变成 Token；
- 预训练在做什么；
- 参数、数据和算力扩展有什么规律。

这一阶段不要急着背所有公式。

先做到能画出：

```text
文本 → Token → Embedding → 多层 Transformer → 下一个 Token 概率
```

### 第二阶段：模型怎样适配任务和人类偏好

阅读 08～11。

重点理解：

- 全量微调与参数高效微调的区别；
- LoRA 为什么用低秩矩阵；
- SFT、奖励模型、PPO、DPO、GRPO 的层级；
- 离线偏好优化和在线策略优化的差异。

这一阶段最容易犯的错误，是把所有 Post-Training 方法当作互斥选项。

真实训练配方可能把它们按阶段组合。

### 第三阶段：模型怎样生成、为什么会出错

阅读 12～18。

你会学习：

- 贪心、Beam Search 和采样；
- Temperature、Top-p 和 Top-k；
- KV Cache 和 Prompt Caching；
- 权重量化；
- Prompt 工程；
- Chain of Thought；
- 幻觉的来源、评测和缓解。

完成后，应能解释“模型为什么每次回答不同”和“为什么流畅不等于真实”。

### 第四阶段：架构、部署与评测

阅读 19～22。

重点是：

- MoE 为什么只激活部分专家；
- vLLM、SGLang、llama.cpp 和 TGI 的选型维度；
- 如何设计离线与在线评测；
- 为什么不能直接用公开排行榜替代业务选型。

### 第五阶段：现代推理工程

阅读 23～31。

这一阶段覆盖：

- Test-Time Compute；
- 蒸馏和合成数据；
- Speculative Decoding；
- 三类缓存；
- Prefill/Decode 分离；
- FP8、FP4 和 INT4；
- 长上下文；
- 多模态对齐；
- 约束解码和结构化生成。

这些内容经常出现在中高级岗位追问中。

### 第六阶段：分布式训练与显存

最后阅读 32～33。

你会把前面的参数、激活、KV Cache、训练和部署知识串起来：

- 模型放不下时到底该切什么；
- DP、TP、PP、ZeRO、FSDP 有什么区别；
- 全局 Batch Size 如何计算；
- 训练和推理显存分别由什么组成；
- 为什么会在某个瞬间 OOM；
- 怎样根据测量结果优化，而不是盲目调参数。

## 一套通用的面试回答框架

面对任何“什么是 X”或“X 和 Y 怎么选”的问题，可以按四层回答。

### 第一层：一句话定义

先说它解决什么问题。

例如：

> “量化是用更低位宽表示模型权重或运行数据，以降低显存、带宽和部分计算成本。”

### 第二层：核心机制

讲清它怎么做到。

例如继续解释：

- FP16 到 INT4 发生了什么；
- Scale 和分组有什么作用；
- 权重是离线量化还是训练感知量化；
- 计算时是否需要反量化。

### 第三层：收益和代价

不要只讲优点。

继续说明：

- 节省多少依赖哪些条件；
- 可能损失什么能力；
- 哪些算子或硬件真正支持；
- 延迟是否真的会下降；
- 如何验证质量回退。

### 第四层：项目选择和验证

最后落到工程：

- 业务目标是什么；
- 对比过哪些候选；
- 测了什么指标；
- 为什么选择当前方案；
- 上线后如何监控和回滚。

这四层可以记成：

```text
定义 → 原理 → 取舍 → 项目验证
```

## 公式应该怎么学

不需要把所有公式当作考试题死记。

看到一个公式时，按四步理解。

### 第一步：它在计算什么

例如注意力公式是在计算“每个 Query 应该从哪些 Key 对应的 Value 中取多少信息”。

### 第二步：每个符号表示什么

不要跳过 Tensor 形状。

知道 $Q$、$K$、$V$ 的维度，很多矩阵乘法会自然变清楚。

### 第三步：哪个量增长会变慢或变大

例如 KV Cache 公式能告诉你：

- 上下文越长，占用越大；
- 并发越高，占用越大；
- KV Head 越少，占用越小。

### 第四步：公式忽略了什么

理论式可能没有包含：

- 对齐；
- 元数据；
- 临时缓冲；
- 分布式通信；
- 框架实现；
- 硬件利用率。

能说出公式边界，通常比只背公式更有工程价值。

录音后检查：

- 是否先给结论；
- 是否堆了很多缩写却没解释；
- 是否只讲优点；
- 是否把论文结论说成普遍定律；
- 是否能落到项目指标。

### 第四遍：做最小实验

如果条件允许，可以选择几个低成本实验：

- 改变 Temperature、Top-p，比较输出；
- 打印 Tokenizer 的切分结果；
- 比较开启和关闭 KV Cache 的速度；
- 测不同上下文、Batch 下的显存；
- 用小模型做 LoRA；
- 用推理框架压测 TTFT 和 TPOT；
- 用约束解码生成 JSON；
- 用两张卡观察 DDP 梯度同步。

实验不需要很大。

重要的是把概念和可观察现象对应起来。

## 面试前的快速复习清单

### 基础概念

- Token、Embedding、Parameter、Logits 分别是什么；
- Encoder-only、Decoder-only、Encoder-Decoder 的区别；
- Attention 为什么要除以 $\sqrt{d_k}$；
- 位置编码为什么必要；
- 自回归预训练目标是什么。

### 训练与后训练

- 预训练、SFT、偏好优化分别使用什么数据；
- LoRA 训练哪些参数；
- PPO、DPO、GRPO 的信号和在线/离线差异；
- Reward Hacking 为什么发生；
- 数据质量、去重和污染为什么重要。

### 生成与可靠性

- Temperature、Top-k、Top-p 的处理顺序和作用；
- KV Cache 的形状和增长因素；
- 幻觉不是一个单一根因；
- Prompt 不能替代权限、校验和业务规则；
- CoT 效果与局限如何评测。

### 推理与部署

- TTFT、TPOT、吞吐和端到端延迟；
- Continuous Batching、PagedAttention、Prefix Cache；
- Prefill 与 Decode 的资源特征；
- 量化格式、硬件内核和质量回退；
- 推理引擎选型需要锁版本压测。

### 分布式与显存

- DP、TP、PP 分别切什么；
- ZeRO 三个阶段分片什么；
- 全局 Batch Size 怎么算；
- 参数、梯度、优化器、激活、KV Cache 各占什么；
- OOM 发生阶段如何帮助定位根因。

## 关于“背题”的正确方式

这套文章可以帮助你覆盖大部分常见面试知识点，但不建议逐字背诵。

逐字背诵有三个问题：

1. 面试官换一种问法就容易断线；
2. 追问项目条件时，很难从固定答案中跳出来；
3. 某些框架和模型事实会随版本变化。

更可靠的背法是记住每题的骨架：

```text
它解决什么问题？
核心机制是什么？
最重要的收益是什么？
代价和失败条件是什么？
项目中如何选择、测量和回滚？
```

只要这个骨架稳定，面试题换一种说法也能重新组织答案。

## 资料与版本说明

本专题优先使用原始论文和官方文档。

但仍要注意：

- 论文结论依赖实验设置；
- 官方文档可能对应特定版本；
- 模型技术报告不一定披露全部训练细节；
- API 兼容不代表参数语义完全相同；
- 公开 Benchmark 不等于你的业务效果；
- 本地硬件和生产负载可能改变性能排序。

如果文章中的版本事实和你实际使用的文档不一致，以你锁定版本的官方说明和实验结果为准。
